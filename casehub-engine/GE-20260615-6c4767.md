## Symptom

`UnsatisfiedResolutionException: Unsatisfied dependency for type io.quarkmind.agent.StrategyTrustRouter and qualifiers [@Default]`

A concrete CDI bean that previously injected fine suddenly fails with "unsatisfied dependency for qualifiers [@Default]" after a custom qualifier annotation was added to the bean class.

## Context

`StrategyTrustRouter` is `@ApplicationScoped`. `StrategyTrustObserver` injects it:

```java
@Inject StrategyTrustRouter trustRouter;  // unqualified — expects @Default
```

Adding `@CaseType("starcraft-game")` (a `@jakarta.inject.Qualifier`-annotated annotation) to `StrategyTrustRouter` causes the deployment failure at CDI validation.

## Root Cause

CDI rule: when a bean carries **any** `@Qualifier`-annotated annotation (beyond `@Any`), Arc removes `@Default` from that bean's qualifier set. The bean is now reachable only via `@Any` or the explicit qualifier — not via unqualified `@Inject`.

The error message names the **concrete type** and `[@Default]`, which looks like the bean doesn't exist — the qualifier connection is not visible.

## Fix

**Option A (Phase 1):** Remove the custom qualifier from the bean and defer it to Phase 2, when the annotation will be promoted to plain metadata (not a CDI qualifier).

**Option B (immediate):** Keep the qualifier and update all injection points that inject the concrete type to also include the qualifier:

```java
@Inject @CaseType("starcraft-game") StrategyTrustRouter trustRouter;
```

Or use `@Any` to accept any qualifier match:

```java
@Inject @Any StrategyTrustRouter trustRouter;
```

## Why This Is Non-Obvious

The error names the concrete type and `[@Default]` — it reads as "the bean doesn't exist" when the bean is clearly declared. The CDI rule (any custom qualifier removes @Default) is documented in the spec but rarely encountered in application code where beans either have no qualifier or are only ever injected via that qualifier. The case where a bean has a qualifier AND is injected without it is uncommon enough that the rule is easily forgotten.
