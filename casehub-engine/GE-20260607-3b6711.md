### What exists

`casehub-engine` runtime contains `ChainedReactiveActionRiskClassifier` (internal class in `io.casehub.engine.internal.worker`). It implements `ReactiveActionRiskClassifier` and uses:

```java
@Inject @RiskClassifier Instance<ActionRiskClassifier> classifiers;
```

It collects **all beans annotated with `@RiskClassifier`** from the CDI container and reduces them via `mostRestrictive()`. The result: multiple `ActionRiskClassifier` implementations coexist and compose automatically — no registration step, no explicit wiring.

### Behaviour

- Zero `@RiskClassifier` beans → `Autonomous` (safe default, existing behaviour preserved)
- One bean → that bean's decision
- Multiple beans → `mostRestrictive()` across all decisions: `GateRequired` beats `Autonomous`; between two `GateRequired`, `narrower()` is applied (see companion entry on `candidateGroups` semantics)
- Bean throws → engine applies fail-safe: `GateRequired("Classifier error — manual review required before proceeding", reversible=true, null groups, null expiry, null scope)`

### Usage pattern

```java
// Just annotate with @RiskClassifier — CDI discovery does the rest
@ApplicationScoped
@RiskClassifier
public class LifeActionRiskClassifier implements ActionRiskClassifier {
    @Override
    public RiskDecision classify(PlannedAction action) { ... }
}
```

No registration in application.properties or any config file.

### Discovery path

Only discoverable by decompiling the engine runtime jar:
```bash
jar tf casehub-engine-0.2-SNAPSHOT.jar | grep ChainedReactive
# → io/casehub/engine/internal/worker/ChainedReactiveActionRiskClassifier.class
```

Not documented in `casehub-engine-api`, the `ActionRiskClassifier` interface Javadoc, or the engine#402 issue description.

### Why this is non-obvious

`ActionRiskClassifier` is a simple one-method interface. Nothing in its surface suggests: (a) multiple implementations can coexist, (b) `@RiskClassifier` is the CDI discovery mechanism, or (c) composition and fail-safe handling are provided by the engine automatically.
