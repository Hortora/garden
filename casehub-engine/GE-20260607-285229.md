### Breaking changes in engine#402

Three simultaneous breaking changes landed without consuming-repo migration notes.

---

#### 1. `function()` now returns `WorkerResult` instead of `Map<String, Object>`

FuncDSL workers in `YamlCaseHub` subclasses (and any code using `Worker.builder().function(lambda)`) must return `WorkerResult`, not `Map<String, Object>`.

**Symptom:**
```
error: no suitable method found for function(...)
  method Worker.Builder.function(Function<Map<String,Object>,WorkerResult>) is not applicable
    (argument mismatch; bad return type in lambda expression
        no instance(s) of type variable(s) K,V exist so that Map<K,V> conforms to WorkerResult)
```

**Migration:**
```java
// Before
.function((Map<String, Object> input) -> {
    return Map.of("confirmed", true, "appointmentId", "APT-123");
})

// After — wrap with WorkerResult.of()
.function((Map<String, Object> input) -> {
    return WorkerResult.of(Map.of("confirmed", true, "appointmentId", "APT-123"));
})
```

`WorkerResult` is at `io.casehub.api.model.WorkerResult`. Static factories:
- `WorkerResult.of(Map<String, Object> output)` — no planned action
- `WorkerResult.of(Map<String, Object> output, PlannedAction action)` — worker declares a planned action for the risk classifier

---

#### 2. `TrustRoutingPolicy` record gained `bootstrapEscalationRequired` boolean (6th field)

The 5-arg canonical constructor no longer exists.

**Symptom:**
```
error: constructor TrustRoutingPolicy in record TrustRoutingPolicy cannot be applied to given types;
  required: double,int,double,double,Map,boolean
  found:    double,int,double,double,Map
```

**Migration:**
```java
// Before
return new TrustRoutingPolicy(threshold, minObs, margin, blendFactor, qualityFloors);

// After — add false as 6th arg (existing behaviour: no bootstrap escalation)
return new TrustRoutingPolicy(threshold, minObs, margin, blendFactor, qualityFloors, false);
```

---

#### 3. `TrustGateService.dimensionScores()` renamed to `allDimensionScores()`

**Symptom:**
```
error: cannot find symbol
  symbol:   method dimensionScores(String)
```

**Migration:**
```java
// Before
Map<String, Double> dims = trustGateService.dimensionScores(actorId);

// After
Map<String, Double> dims = trustGateService.allDimensionScores(actorId);
```

### Why this is non-obvious

All three changes arrive silently in a SNAPSHOT update — no consuming-repo migration note in the engine#402 issue or CHANGELOG. Compile errors surface together across all files that touch the engine API, making them look like a single deep breakage rather than three orthogonal fixes.
