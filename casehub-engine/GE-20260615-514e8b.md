## Symptom

Adding `casehub-engine-blackboard` as a test-scope Maven dependency causes 34 CDI deployment failures when running `@QuarkusTest`:

```
Found 34 deployment problems:
[1] Unsatisfied dependency for type io.casehub.engine.common.spi.EventLogRepository
[2] Unsatisfied dependency for type io.casehub.engine.common.spi.scheduler.JobScheduler
[3] Unsatisfied dependency for type io.casehub.engine.common.spi.CaseInstanceRepository
[4] Unsatisfied dependency for type io.casehub.engine.common.spi.SubCaseGroupRepository
...
```

## Context

The blackboard module was added as a test dependency to access `CaseContextImpl(Map<String,Object>)` for constructing test CaseContext instances without requiring the full engine runtime.

## Root Cause

`casehub-engine-blackboard` is NOT a standalone API module. It contains CDI `@ApplicationScoped` beans (plan management infrastructure) that inject engine persistence SPIs — `EventLogRepository`, `JobScheduler`, `CaseInstanceRepository`, `SubCaseGroupRepository`. These SPIs have no `@DefaultBean` implementations on the classpath without the full engine runtime (or `casehub-engine-persistence-memory`). Even as test scope, these beans are discovered by Quarkus Arc at augmentation time.

The pattern from `casehub-workers` applies: use `casehub-engine-api` (pure interfaces, no CDI beans) rather than `casehub-engine-blackboard` (runtime CDI beans).

## Fix

Remove `casehub-engine-blackboard` from test scope. For `CaseContext` construction in tests, use an alternative that doesn't pull in engine runtime beans:

- **Option A:** Wrap a poc `CaseFile` with `CaseFileContext` (a `CaseContext` adapter already in the app classpath for Phase 1 bridges):
```java
var cf = new InMemoryCaseFileRepository()
    .create("test", Map.of(KEY, value), PropagationContext.createRoot());
CaseContext ctx = new CaseFileContext(cf);
```

- **Option B:** Add `casehub-engine-persistence-memory` alongside the blackboard dep and configure `quarkus.arc.selected-alternatives` for `MemorySubCaseGroupRepository` and `MemoryPlanItemStore` (more invasive — see GE-20260531-1e51d4).

## Why This Is Non-Obvious

`casehub-engine-blackboard` sounds like a pure API/model module (it contains `DefaultCasePlanModel`, `PlanItem`, etc.) but it ships CDI runtime beans alongside the model. The 34 error messages all name internal engine SPIs — there is nothing in the error output connecting the failures to the new test dependency.
