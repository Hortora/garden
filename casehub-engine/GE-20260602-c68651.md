### Symptom
`TrustWeightedAgentStrategy` is not activated for plugin selection despite `casehub-ledger` being on the classpath. Trust routing falls back to default availability routing silently — no error, no warning.

### Context
Wiring trust-weighted plugin routing in a CaseHub harness app (e.g. QuarkMind L6, devtown C5). `casehub-ledger` added as a dependency for audit trail (L4); trust routing expected to activate as a result.

### Root cause
`TrustWeightedAgentStrategy` lives in `casehub-engine-ledger` — a separate optional module — not in `casehub-ledger` itself. The strategy activates by classpath presence of `casehub-engine-ledger`. Adding `casehub-ledger` alone does not activate it.

### Fix
Add `casehub-engine-ledger` as a dependency alongside `casehub-ledger`:
```xml
<dependency>
  <groupId>io.casehub</groupId>
  <artifactId>casehub-engine-ledger</artifactId>
</dependency>
```

### Why this is non-obvious
The `casehubio/ledger` repo description states "immutable audit ledger with hash chain, attestations, and EigenTrust reputation" — implying EigenTrust is fully contained in `casehub-ledger`. Nothing in the repo README, module list, or API surface indicates that EigenTrust activation requires a separate `casehub-engine-ledger` module and is classpath-conditional. The split is only visible in the casehub-engine documentation's optional module table.

### Once activated — which scores drive routing?

`TrustWeightedAgentStrategy` routes by **CAPABILITY scores** (Bayesian Beta per-capability
ratios), not GLOBAL scores (EigenTrust graph output). `casehub-engine-ledger` bundles both
`TrustWeightedAgentStrategy` and `TrustScoreJob` (which computes EigenTrust GLOBAL scores
nightly), but they are separable concerns: the routing strategy reads CAPABILITY scores for
plugin selection; GLOBAL scores are computed independently and not consumed for capability
dispatch.

In a single-attestor deployment (one harness observing all plugins, no peer attestation),
EigenTrust GLOBAL computation is structurally inert — the power-iteration result converges
to the same values as the direct observations already in CAPABILITY scores. EigenTrust
adds routing information only when a genuine peer attestation graph exists.
