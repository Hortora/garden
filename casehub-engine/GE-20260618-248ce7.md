## Symptom

`@InjectMock LifeOpenClawChatModelProvider openClaw` is set up in `@BeforeEach` for an `AppointmentCycleCaseHub` integration test. The stub returns a specific `ChatModel`. But when `agent.execute()` is called, the ChatModel from a completely different test class (or the real provider) is used instead — the mock is silently ignored.

## Root Cause

`AgentBuilder.build()` calls `chatModelProvider.get()` exactly **once** and stores the result in the `Agent` object:

```java
ChatModel resolvedModel = model;
if (resolvedModel == null && chatModelProvider != null) {
    resolvedModel = chatModelProvider.get();  // called ONCE at build time
}
return new Agent(... resolvedModel ...);
```

When `AppointmentCycleCaseHub.getDefinition()` uses a double-checked lock:

```java
private volatile CaseDefinition augmentedDefinition;

@Override
public CaseDefinition getDefinition() {
    if (augmentedDefinition == null) {
        synchronized (this) {
            if (augmentedDefinition == null) {
                augmentedDefinition = augment(super.getDefinition());
            }
        }
    }
    return augmentedDefinition;
}
```

`augment()` calls `Agent.build()` which calls `provider.get()` — only on the **first** `getDefinition()` call per JVM lifetime. The baked `ChatModel` is stored permanently in `augmentedDefinition`.

Quarkus's `@InjectMock` reset between test methods restores the CDI delegate of the provider bean — but the `Agent` already holds a direct reference to the `ChatModel` returned by the first `get()`. No subsequent `@InjectMock` setup can change what `agent.execute()` uses.

## Fix

Use a test-only `@Alternative @Priority(10) @ApplicationScoped` CDI bean (registered in `quarkus.arc.selected-alternatives` in test config) that returns a request-aware `ChatModel`. Whichever test class triggers `getDefinition()` first bakes the request-aware model — all subsequent test classes benefit.

```java
@Alternative
@Priority(10)
@ApplicationScoped
public class TestLifeOpenClawChatModelProvider extends LifeOpenClawChatModelProvider {
    @Override
    public ChatModel get() {
        return new ChatModel() {
            @Override
            public ChatResponse doChat(ChatRequest request) {
                // request-aware: examine message text to route responses
                boolean decline = extractUserText(request).toLowerCase().contains("unavailable");
                return decline ? declineResponse() : successResponse();
            }
        };
    }
}
```

**Never use `@InjectMock`** on `ChatModelProvider` implementations when the provider's `get()` is called during lazy-init of an `@ApplicationScoped` singleton. The baked result cannot be replaced after the first call.

## Why this is non-obvious

The CDI proxy for `LifeOpenClawChatModelProvider` IS replaced by `@InjectMock` — every call to `openClaw.type()` or `openClaw.get()` would go through the mock if called directly. But `Agent` stores the `ChatModel` value returned by `get()`, not a reference to the provider. Once stored, the CDI proxy is bypassed entirely.

Nothing in the `@InjectMock` documentation, LangChain4J API, or Quarkus test framework warns that `ChatModel` references baked into eager-building beans cannot be replaced.
