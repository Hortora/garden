### Symptom
Subclassing `WorkItemService` with null dependencies for a test stub throws
`NullPointerException` at construction time — before any test method runs.

### Context
Unit testing a class that depends on `WorkItemService.findById()`. The natural
approach: subclass with nulls for unused constructor parameters, override
`findById()`. The constructor has 9 parameters and most tests only need one method.

### Root cause
`WorkItemService` constructor body calls `config.sla()` to initialize
`ClaimSlaPolicy`. Passing `null` for the `WorkItemsConfig` parameter causes an
NPE at line 93 of the constructor — before any overridden method is reachable.

### Fix
Don't subclass `WorkItemService`. Instead, inject the specific method you need
via a `Function` parameter:

```java
// Production CDI constructor delegates to package-visible constructor
@Inject
public MyRecorder(WorkItemService workItemService, ...) {
    this(workItemService::findById, ...);
}

// Package-visible constructor for testing — takes the lookup function directly
MyRecorder(Function<UUID, Optional<WorkItem>> workItemLookup, ...) {
    this.workItemLookup = workItemLookup;
}
```

Test stub becomes trivial:
```java
var recorder = new MyRecorder(
    id -> workItem.id.equals(id) ? Optional.of(workItem) : Optional.empty(),
    ...);
```

### Why this is non-obvious
The constructor parameter list looks like standard CDI injection — nothing
suggests that passing null for an unused parameter will fail. The
`config.sla()` call in the constructor body is invisible without reading
the `WorkItemService` source. A developer's first instinct is to subclass
and override, not to inspect the superclass constructor body.

### Alternatives considered
- **Passing a real `WorkItemsConfig`**: requires constructing a valid config
  object with SLA policies — heavyweight for a test that only calls `findById()`.
- **Using `WorkItemStore` directly**: avoids the service layer but the
  `WorkItemStore` interface has fewer methods, making stubs simpler. However,
  this couples the consumer to the repository layer.
