## What It Is

`io.casehub.api.model.evaluator.ExpressionEvaluator` in casehub-engine's `api` module is a
marker-only interface. It declares only `String type()` — no `evaluate()` or `test()` method.

## How to Use It

To evaluate an `ExpressionEvaluator` against a `CaseContext`, you must dispatch by concrete type:

```java
private boolean evaluateCondition(ExpressionEvaluator evaluator, CaseContext context) {
    if (evaluator == null) return true; // null = always-met convention
    if (evaluator instanceof LambdaExpressionEvaluator lambda) {
        return lambda.test(context);    // direct evaluation
    }
    // For JQ and other types: delegate to ExpressionEngineRegistry or Instance<ExpressionEngine>
    return expressionEngineRegistry.evaluate(evaluator, context);
}
```

`LambdaExpressionEvaluator` extends `ExpressionEvaluator` and implements
`java.util.function.Predicate<CaseContext>` — its `test(CaseContext)` method does the
actual work.

## Why Not Documented

The `ExpressionEvaluator` interface in `api/` is intentionally minimal — it's a discriminated
union type. The evaluation mechanism is in the SPI: `ExpressionEngine.evaluate(evaluator, context)`.
This design is in the source but not in any README, Javadoc, or design doc entry. The natural
assumption (call `.evaluate()` on the interface) leads to a compilation error with no helpful
message.

## Notes

- For `@QuarkusTest` classes that only use lambda conditions, injecting `Instance<ExpressionEngine>`
  is unnecessary — `LambdaExpressionEvaluator` can be dispatched directly without CDI
- For JQ expressions, you must go through the `ExpressionEngineRegistry` or CDI `Instance`
