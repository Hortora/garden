## Behaviour

After PR #28 refactoring, `CaseHubRuntime.startCase(CaseHubDefinition)` no longer
registers the definition at runtime. It only looks up pre-registered definitions from
`CaseDefinitionRegistry`. If the definition was not registered at startup, `startCase()`
throws `RuntimeException: CaseMetaModel not found for caseDefinition: ...`.

## Context

Case definitions are registered at application startup by `CaseDefinitionRegistry`
which observes `@StartupEvent` and iterates all CDI `CaseHub` beans via
`Instance<CaseHub>`. There is no runtime registration path for arbitrary definitions.

## Impact on testing

Integration tests that try to call `runtime.startCase(arbitraryDefinition)` to test
behaviour (e.g. validation of bad expressions) will get a `CaseMetaModel not found`
exception, not the expected validation failure.

## Workaround

Call `CaseDefinitionRegistry.registerCaseDefinition(definition)` directly. Make the
method package-private (it is `private` by default) to allow test access:

```java
// In CaseDefinitionRegistry.java
Uni<CaseMetaModel> registerCaseDefinition(CaseHubDefinition model) { ... }
// (remove 'private' modifier — package-private allows same-package test access)
```

Then in `@QuarkusTest`:
```java
@Inject CaseDefinitionRegistry registry;

registry.registerCaseDefinition(definition)
    .await().atMost(Duration.ofSeconds(5));
```

Note: this requires a Vert.x context for the DB path. Invalid-expression paths
(which fail before hitting the DB) work without context.

## Where to look

`engine/src/main/java/io/casehub/engine/internal/engine/CaseHubReactor.java` —
`startCase()` method calls `getCaseInstance(definition, context)` which calls
`getCaseMetaModel(definition)` — no registration.
