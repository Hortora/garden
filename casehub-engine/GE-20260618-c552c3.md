## Symptom

All `@QuarkusTest` classes fail with:

```
Caused by: java.lang.RuntimeException: Failed to start quarkus
Caused by: java.lang.IllegalStateException: Already a default codec registered
  for class class io.casehub.blackboard.event.StageActivatedEvent
  at io.casehub.blackboard.event.BlackboardEventCodecRegistrar.onStart(...)
```

The error message points to a codec registration conflict — but nothing in the diff touched codecs. The real cause is an `@InjectMock` annotation added to a test class.

## Root Cause

`@InjectMock` changes the CDI bean configuration for the test class. When switching from a test class without `@InjectMock` to one with it, Quarkus computes a different CDI profile hash and **restarts the application** to use the new configuration.

During the restart, `BlackboardEventCodecRegistrar.onStart()` (annotated `@Observes StartupEvent`) fires and tries to register Vert.x default codecs for blackboard event types. But the codecs were already registered during the previous Quarkus start. Vert.x's `CodecManager.registerDefaultCodec()` throws if a codec is registered twice.

Result: Quarkus fails to start for the mocked class AND for all subsequent classes that also share the restart.

Additionally, Surefire's `rerunFailingTestsCount=2` makes this catastrophic: on retry, Quarkus tries to restart again in the same JVM → same double-registration → all remaining test classes skip.

## Fix

**Never use `@InjectMock`** on beans whose CDI profile change would force a Quarkus restart when interleaved with non-mocked test classes.

Instead, use an `@Alternative @Priority(10) @ApplicationScoped` CDI test bean registered in `quarkus.arc.selected-alternatives` in test `application.properties`. This is part of the base CDI profile — no restart occurs between test classes.

```properties
# test/resources/application.properties
quarkus.arc.selected-alternatives=\
  ...,\
  io.example.TestOpenClawChatModelProvider
```

```java
@Alternative
@Priority(10)
@ApplicationScoped
public class TestOpenClawChatModelProvider implements ChatModelProvider {
    // request-aware implementation that serves all test classes
}
```

**Root fix** (in casehub-engine-blackboard): make `BlackboardEventCodecRegistrar.onStart()` idempotent:
```java
private <T> void registerIfAbsent(Class<T> type, MessageCodec<T, T> codec) {
    try {
        eventBus.registerDefaultCodec(type, codec);
    } catch (IllegalStateException ignored) {
        // Already registered — idempotent in environments where augmentation
        // phase starts the application before test execution
    }
}
```

## Why this is non-obvious

1. The error message ("Already a default codec registered") points to the codec, not to `@InjectMock`. A developer removing codecs or checking codec registration logic will find nothing wrong.

2. `@InjectMock` appears to work correctly for the test class that uses it — only subsequent classes fail.

3. Surefire's retry (`rerunFailingTestsCount`) compounds the damage: the codec error appears on the FIRST run of a test that never used mocks, making the `@InjectMock` connection even less obvious.

4. The fix (CDI alternative bean) is non-obvious: most developers reach for `@InjectMock` as the standard Quarkus test double mechanism.
