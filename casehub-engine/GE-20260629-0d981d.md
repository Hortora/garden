### Symptom
Tests or specs assume Layer 9 investigations produce SAR outcomes (`sar-filed`, `gate-rejected`) identical to Layer 6. Integration tests polling for `outcome.type` never find a value — outcome is always null.

### Context
Both `AmlLayer6Resource` and `AmlLayer9Resource` inject `AmlInvestigationOutcomeService` and call `resolveInvestigation()`. The resources look structurally identical. Layer 6 uses `AmlEngineCoordinator`; Layer 9 uses `AmlOversightCoordinator` — but both return `InvestigationResolution` with the same `InvestigationOutcome` type.

### What was tried (didn't work)
- Writing integration tests that poll `GET /api/layer9/investigations/{id}` for `outcome.type == "sar-filed"` after officer review — times out, outcome never appears
- Assuming the `AmlWorkItemLifecycleObserver` fires for gate WorkItems — it doesn't

### Root cause
Layer 9's YAML case definition (`aml-oversight-investigation.yml`) contains only `entity-resolution → entity-link-proposal (PlannedAction) → investigation-summary`. There is no `sar-drafting` or `compliance-review` capability. Therefore:

1. No SAR drafting worker runs → no compliance review WorkItem is created
2. `AmlWorkItemLifecycleObserver` only fires for WorkItems with `callerRef = "aml:investigation:<UUID>"` — gate WorkItems use `callerRef = "case:<UUID>/gate:<name>"` and are invisible to the observer
3. No `AmlSarOfficerReviewedLedgerEntry` is written → `resolveOutcome()` finds nothing → outcome is null

### Fix
Accept that Layer 9 outcome is null by design. Write tests that explicitly assert `outcome` is null and document the architectural difference:

```java
// Layer 9 investigations do NOT produce SAR outcomes — only oversight gates exist
given().when().get("/api/layer9/investigations/" + caseIdStr)
        .then().statusCode(200)
        .body("outcome", nullValue());
```

### Why this is non-obvious
The two resources are structurally identical — same service injection, same `resolveInvestigation()` call, same `InvestigationResolution` return type. The difference is entirely in the YAML case definition (which capability tags are wired) and the callerRef pattern (which determines whether the observer fires). Neither of these is visible from the resource or service code. Both the plan author and an independent adversarial design reviewer assumed Layer 9 produced SAR outcomes — the assumption survived four rounds of review before implementation exposed it.
