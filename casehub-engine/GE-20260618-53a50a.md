### What's surprising

`TaskDefinitionRegistry` is annotated `@ApplicationScoped` but can be safely instantiated with `new TaskDefinitionRegistry()` in unit tests. Its `register()` and `getForCaseType()` methods use only `ConcurrentHashMap` internally. Despite having `@Inject PoisonPillDetector poisonPillDetector` as a field, `PoisonPillDetector` is only used in `ListenerEvaluator.evaluateAndCreatePlanItems()` — not in the registry itself.

### How to use

```java
TaskDefinitionRegistry registry = new TaskDefinitionRegistry();
registry.register(myStubPlugin, Set.of("starcraft-game"));
List<TaskDefinition> plugins = registry.getForCaseType("starcraft-game");
// Works without a Quarkus container
```

`register()` runs a cycle-detection DFS over `entryCriteria()` and `producedKeys()` — ensure stub plugins return disjoint key sets to avoid `CircularDependencyException`. Stub plugins implementing both `io.casehub.core.TaskDefinition` and the app-specific `TaskDefinition` interface can be registered directly.

### Root cause

`PoisonPillDetector` was added as an `@Inject` field to `TaskDefinitionRegistry` for quarantine support, but the quarantine check (`isQuarantined()`) is invoked only in `ListenerEvaluator`, not in the registry's own methods. The registry is effectively a thread-safe typed map with cycle detection.

### Why this is non-obvious

`@ApplicationScoped` CDI beans are conventionally assumed to require a container. The presence of `@Inject PoisonPillDetector` reinforces this assumption — nothing in the API or documentation indicates that the injected field is not used in the primary query methods. Developers attempting to test `TaskDefinitionRegistry`-aware code typically reach for `@QuarkusTest` (significant boot overhead) when a plain `new` instantiation suffices.
