## Symptom

In a test using `CasehubEnabledProfile`, goals show as satisfied in engine logs (`Goal X REACHED`)
but `caseInstanceRepository.findByUuid(caseId).getState()` returns `CaseStatus.RUNNING`
indefinitely. The case never transitions to `COMPLETED`.

## Context

`CaseEngineRoundTripTest.CasehubEnabledProfile` explicitly excludes `CaseStatusChangedHandler`
in its `quarkus.arc.exclude-types` override:

```java
+ "io.casehub.engine.internal.engine.handler.CaseStatusChangedHandler,"
```

This handler is responsible for updating the `CaseInstance.state` field when the engine
determines a case should complete. Without it, goals can be reached and `GoalReachedEventHandler`
fires, but the case state is never written.

## Root Cause

`CaseStatusChangedHandler` listens for the engine's internal status change event and persists
the state transition. `CasehubEnabledProfile` excludes it to avoid side effects
(status change notifications, external hooks) that are out of scope for round-trip tests.
This is intentional for the existing `CaseEngineRoundTripTest` but becomes a trap when you
need to test full case lifecycle including completion.

## Fix

Build a dedicated test profile (e.g. `ResearcherCaseCasehubProfile`) derived from
`CasehubEnabledProfile` that removes `CaseStatusChangedHandler` from the exclude list:

```java
// CasehubEnabledProfile exclude-types includes CaseStatusChangedHandler — remove it
// Also remove SignalReceivedEventHandler + DefaultWorkerExecutionRecoveryService if signal-driven completion is needed
"quarkus.arc.exclude-types",
"io.casehub.ledger.repository.CaseLedgerEntryRepository,"
// + ... other exclusions ...
// CaseStatusChangedHandler NOT in this list → active → case can reach COMPLETED
```

## Why This Is Non-Obvious

Goal satisfaction is logged (`Goal X REACHED`) which implies the engine is processing the
case lifecycle correctly. The absence of `CaseStatusChangedHandler` doesn't produce an error —
the state just silently stays `RUNNING`. There's nothing in `CasehubEnabledProfile` that hints
at the exclusion; you must read the exclusion list carefully.
