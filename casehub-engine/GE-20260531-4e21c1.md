### Symptom

Quarkus startup fails with:

```
java.lang.NoSuchMethodError: 'io.smallrye.mutiny.Uni io.casehub.engine.common.spi.CaseMetaModelRepository.findByKey(java.lang.String, java.lang.String, java.lang.String)'
```

Or workers fail at runtime with `NoSuchMethodError` on `PlanExecutionContext` constructor.

### Context

The engine source added `tenancyId` to `CaseLifecycleEvent` (now 8 params, was 7), `tenancyId` to `CaseMetaModelRepository.findByKey()` (now 4 params, was 3), and `outputMappingExpression` to `PlanItemSaveRequest` (now 8 fields, was 7). The installed SNAPSHOT jars have the old signatures. Rebuilding from source fails because the engine itself has unfixed compilation errors (engine#408).

### What was tried (didn't work)

- `mvn install -DskipTests -f ../engine/pom.xml` — test compilation errors in PlanItemStoreContractTest
- `mvn install -Dmaven.test.skip=true -f ../engine/pom.xml` — production compilation error in QuartzWorkerExecutionJobListener (stale CaseLifecycleEvent constructor call)

### Root cause

Three API changes in engine source were not propagated to all call sites within the engine itself:
1. `CaseLifecycleEvent` gained `tenancyId` — `QuartzWorkerExecutionJobListener` still calls 7-param constructor
2. `CaseMetaModelRepository.findByKey()` gained `tenancyId` — installed jar's `DefaultCaseDefinitionRegistry` calls 3-param version
3. `PlanItemSaveRequest` gained `outputMappingExpression` — test call sites pass 7 fields

### Fix

Wait for engine#408 to be resolved. The engine maintainer needs to update all call sites, then consumers rebuild. Workaround: mark `@QuarkusTest` integration tests as `@Disabled("engine#408")` or use `assumeTrue` — pure unit/DSL tests still pass.

### Why this is non-obvious

The `NoSuchMethodError` names the 3-param `findByKey` — but the method exists in the source with 4 params. The error looks like a missing dependency, not a signature mismatch. You need to compare the installed jar's bytecode against the current source to diagnose. The engine's own compilation errors prevent a local rebuild, creating a circular blocker.
