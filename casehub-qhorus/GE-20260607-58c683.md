## Problem

A CDI-free unit test (no Quarkus container) needs to inject two test stubs that share the same backing state: stub A writes (via `save()`), stub B reads (via `findByMessageId()` or similar). The default instinct is to wire them independently, leaving each with its own internal list — so B never sees what A saved.

## Technique

Inject the shared backing list by constructor into both stubs at test setup:

```java
@BeforeEach
void setUp() {
    List<LedgerEntry> shared = new ArrayList<>();
    var ledger   = new StubLedgerEntryJpaRepository(shared);   // writes go here
    var messages = new StubMessageLedgerEntryRepository(shared);// reads come from same list

    service.ledger      = ledger;
    service.messageRepo = messages;
}
```

Each stub class accepts the shared list:

```java
public class StubLedgerEntryJpaRepository implements LedgerEntryRepository {
    public final List<LedgerEntry> entries;
    public StubLedgerEntryJpaRepository(List<LedgerEntry> entries) {
        this.entries = entries;
    }
    @Override
    public LedgerEntry save(LedgerEntry e) {
        if (e.id == null) e.id = UUID.randomUUID(); // simulate @PrePersist
        entries.add(e);
        return e;
    }
    // ...
}

public class StubMessageLedgerEntryRepository extends MessageLedgerEntryRepository {
    final List<LedgerEntry> entries;
    public StubMessageLedgerEntryRepository(List<LedgerEntry> entries) {
        this.entries = entries;
    }
    @Override
    public Optional<MessageLedgerEntry> findByMessageId(Long id) {
        return entries.stream()
            .filter(e -> e instanceof MessageLedgerEntry m && id.equals(m.messageId))
            .map(e -> (MessageLedgerEntry) e)
            .findFirst();
    }
}
```

## Why this is non-obvious

The natural approach is independent stubs with internal lists, relying on Mockito or post-construction field assignment to wire them. Constructor injection makes the sharing explicit and permanent — the stubs cannot diverge. It also avoids the Mockito overhead in tests that are otherwise CDI-free. Most developers reach for Mockito when they need cross-bean state sharing; this pattern sidesteps it entirely.

### Note on inheritance

If a stub extends a concrete class that has `@Inject EntityManager em`, that field will be `null` in CDI-free tests. Add a warning comment in the stub class and override any non-implemented method with `throw new UnsupportedOperationException()` rather than inheriting the NPE-prone parent implementation.
