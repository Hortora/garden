### Symptom

Channel registration fails at startup with a validation error when a channel name constant contains a dot (e.g. `"scouting.intel"`). The error surfaces during CDI bean initialisation — the application boots but the channel is never registered, leaving `MessageObserver` beans silently unsubscribed.

### Context

Occurs when defining a `CHANNEL_NAME` constant for a qhorus observe channel and following a dotted-namespace naming convention (e.g. `"scouting.intel"`, `"tactics.commands"`). The dot seems natural for hierarchical names but is not a valid slug character.

### Root cause

`ChannelSlugValidator` enforces a slug pattern per path segment. For slash-separated paths (e.g. `life/actor/ext-uuid`), each segment must match:

```
[a-z][a-z0-9]*(-[a-z0-9]+)*
```

Two rules that trip developers:
1. **First character must be a letter** (`[a-z]`) — digits are not allowed at the start
2. **Dots are not in the allowed set** — only lowercase letters, digits, and hyphens

The validator fires when the `MessageChannel` entity is created or registered, not when `MessageDispatch` is called. No exception propagates to the caller in some code paths — the channel simply does not exist when consumers try to subscribe.

### Fix

Use hyphen-separated slugs for all channel name constants:

```java
// Bad — rejected by ChannelSlugValidator
static final String CHANNEL_NAME = "scouting.intel";

// Good
static final String CHANNEL_NAME = "scouting-intel";
```

Hyphens are the standard separator for qhorus channel slugs. Apply this convention to all `CHANNEL_NAME` constants throughout the codebase.

**UUID segments:** never use a raw UUID directly as a channel name segment — 10/16 UUID hex prefixes (0-9) start with a digit. Prefix with a meaningful label:

```java
// Bad — fails for ~62% of UUIDs (those starting with 0-9)
"life/actor/" + externalActorId

// Good — always starts with a letter
"life/actor/ext-" + externalActorId
```

This manifests as a random test failure (passes when UUID starts with a-f, fails otherwise) that can take days to reproduce deterministically.

### Why this is non-obvious

Dotted namespacing is a common Java convention for identifiers (package names, config keys, JNDI names). The validator error may not surface prominently if the channel registration failure is swallowed during CDI initialisation. The `MessageObserver` bean appears to register successfully; only at runtime when a dispatch occurs with no subscribers does the absence of the channel become apparent. There is no compile-time or early-boot signal.

### Constraints

Applies to all channel name constants passed to `MessageChannel` creation. Does not affect `sender` or `actorType` fields in `MessageDispatch`.

### Invalidation triggers

Revisit if `ChannelSlugValidator` slug pattern is updated to permit dots, or if a separate validator is introduced for qualified channel names.
