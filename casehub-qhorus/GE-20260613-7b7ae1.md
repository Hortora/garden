### Symptom

Compile error on `ChannelService.create()` call:

```
no suitable method found for create(String, String, ChannelSemantic, <null>, <null>, <null>, <null>, <null>, String)
    method ChannelService.create(String, String, ChannelSemantic, String) is not applicable
      (actual and formal argument lists differ in length)
```

No runtime error — it fails at compile time.

### Context

Calling `channelService.create()` with a trailing `String allowedTypes` parameter (comma-separated message types like `"QUERY,COMMAND,DONE,DECLINE,EVENT"`) to configure which message types a channel accepts.

### Root cause

A qhorus SNAPSHOT removed the 9-arg overload that accepted `String allowedTypes` as the last parameter. The `allowedTypes` parameter moved to `ChannelCreateRequest` as `Set<MessageType>` instead of `String`.

### Fix

Replace the 9-arg `create()` call with `ChannelCreateRequest`:

```java
// Old (removed):
channelService.create(name, description, ChannelSemantic.APPEND,
    null, null, null, null, null, "QUERY,COMMAND,DONE,DECLINE,EVENT");

// New:
private static final Set<MessageType> ALLOWED_TYPES = Set.of(
    MessageType.QUERY, MessageType.COMMAND, MessageType.DONE,
    MessageType.DECLINE, MessageType.EVENT);

channelService.create(new ChannelCreateRequest(
    name, description, ChannelSemantic.APPEND,
    null, null, null, null, null,
    ALLOWED_TYPES,   // Set<MessageType> — was String
    null, null, null, null, null
));
```

`ChannelCreateRequest` constructor signature (positional):
`(name, description, semantic, barrierContributors, allowedWriters, adminInstances, rateLimitPerChannel, rateLimitPerInstance, allowedTypes, deniedTypes, inboundConnectorId, externalKey, outboundConnectorId, outboundDestination)`

### Why this is non-obvious

Projects using Maven local cached stubs (or GitHub Packages cached artifacts) may still compile successfully against the old overload until they pull a fresh SNAPSHOT. The failure only surfaces when building against a newer artifact, often in CI. The field name `allowedTypes` is the same — only the type changed from `String` to `Set<MessageType>`.
