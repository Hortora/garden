## Symptom

A test class named `FooIT` (or any `*IT.java`) compiles successfully, appears in the `target/test-classes/` directory, but never executes during `mvn test`. No error, no skip message — it is simply invisible.

## Context

Naming an integration test class with the `IT` suffix (e.g. `SemanticRoutingIT`) to signal its nature, then expecting `mvn test` to run it.

## What was tried

Adding `@QuarkusTest` annotation, ensuring imports are correct, checking for compile errors — all fine.

## Root cause

Maven Surefire's default include pattern covers `**/*Test.java`, `**/Test*.java`, and `**/*Tests.java`. The pattern `**/*IT.java` is explicitly excluded by default and handled by the `maven-failsafe-plugin` which runs during the `integration-test` lifecycle phase (`mvn verify`), not `mvn test`.

## Fix

Rename the class to use the `*Test` suffix instead:

```
SemanticRoutingIT.java  →  SemanticRoutingTest.java
```

Or, if the class genuinely requires the integration-test lifecycle, configure Surefire to include it:

```xml
<plugin>
  <groupId>org.apache.maven.plugins</groupId>
  <artifactId>maven-surefire-plugin</artifactId>
  <configuration>
    <includes>
      <include>**/*Test.java</include>
      <include>**/*IT.java</include>
    </includes>
  </configuration>
</plugin>
```

## Why this is non-obvious

The build succeeds, the class compiles, and no warning is printed. The class is silently skipped. If the test name signals integration-test intent (`IT`) but it runs fine in the unit-test lifecycle, renaming to `*Test` is the correct fix.
