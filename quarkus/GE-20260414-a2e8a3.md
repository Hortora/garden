## @WithDefault("") in @ConfigMapping throws ConfigValidationException — empty string treated as null

**ID:** GE-20260414-a2e8a3
**Stack:** Quarkus 3.32.2, SmallRye Config, quarkus-extension-processor
**Symptom:** Application fails to start with:
```
ConfigValidationException: Configuration validation failed:
  java.util.NoSuchElementException: SRCFG00040: The config property
  quarkus.myext.some-url is defined as the empty String ("") which the
  following Converter considered to be null: io.smallrye.config.Converters$BuiltInConverter
```
**Context:** `@ConfigMapping` interface with a `String` return method annotated `@WithDefault("")` to provide an empty-string fallback for an optional URL or string field that has no sensible default.

### What was tried (didn't work)

- `@WithDefault("")` on a `String` return type — throws at startup
- `@WithDefault(" ")` (space) — works but is semantically wrong
- Omitting `@WithDefault` entirely — throws `NoSuchElementException` if property is absent from config

### Root cause

SmallRye Config's converter chain processes the empty string `""` and treats it as equivalent to `null` / absent for `String` type. The validation then fails because the property appears defined (it has a `@WithDefault`) but its value converts to null — which is rejected for a non-optional type.

This behaviour applies specifically to `String` return types. The same `@WithDefault("")` on an `int` or `boolean` would fail for different reasons (parse error), but for `String` the silent null-conversion is the trap.

### Fix

Use `Optional<String>` as the return type. SmallRye Config correctly handles absent values for `Optional` — the method returns `Optional.empty()` when the property is not set:

```java
@ConfigMapping(prefix = "quarkus.myext")
public interface MyExtConfig {

    /** Group for agent card config. */
    AgentCard agentCard();

    interface AgentCard {
        /** Public URL. Should be set per deployment; absent when not configured. */
        Optional<String> url();   // ← NOT @WithDefault("") String url()

        /** Name with a real default. */
        @WithDefault("My Extension")
        String name();
    }
}
```

At the call site:
```java
String url = config.agentCard().url().orElse("");  // or orElseThrow() if required
```

Remember: `@ConfigMapping` in a Quarkus extension requires Javadoc on every method, including `Optional<String>` methods, because `quarkus-extension-processor` generates config documentation from them.

### Why non-obvious

`@WithDefault("")` looks like it should mean "default to empty string if not set". The SmallRye Config null-conversion behaviour is not documented in the `@WithDefault` Javadoc or the Quarkus Config guide. The error message names the Converter class but gives no hint that the fix involves changing the return type.
