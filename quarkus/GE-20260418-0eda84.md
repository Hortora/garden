## What happened

Configuring Hibernate schema generation for test profile based on documentation that uses JPA-standard terminology:

```properties
%test.quarkus.hibernate-orm.database.generation=create-drop
```

Quarkus failed at startup with a config validation error:

```
Cannot convert create-drop to enum class HibernateOrmRuntimeConfigPersistenceUnit$HibernateGenerationStrategy,
allowed values: drop,drop-and-create,create,update,none,validate
```

## Root cause

Quarkus uses its own enum (`HibernateGenerationStrategy`) rather than the JPA-standard `create-drop` value. The Quarkus-specific name is `drop-and-create`. This mismatch is not caught at compile time and only fails at startup.

## Fix

```properties
%test.quarkus.hibernate-orm.database.generation=drop-and-create
```

Allowed values: `drop`, `drop-and-create`, `create`, `update`, `none`, `validate`.

## Why this is non-obvious

JPA standard and Hibernate native both use `create-drop`. External tooling, third-party docs, and AI assistants often suggest `create-drop` because it is the standard JPA term. Quarkus silently diverges without aliasing the standard value.
