## Quarkus bakes JDBC driver class into Agroal at augmentation time — switching db-kind at runtime silently fails

### Symptom

```
WARN  Datasource '<default>': Driver does not support the provided URL: jdbc:postgresql://localhost:45387/test
java.sql.SQLException: Driver does not support the provided URL: jdbc:postgresql://localhost:45387/test
```

Or alternatively:

```
quarkus.datasource.db-kind is set to 'postgresql' but it is build time fixed to 'h2'.
Did you change the property quarkus.datasource.db-kind after building the application?
```

Tests use `@TestProfile` or `QuarkusTestResource` to inject a PostgreSQL JDBC URL, but Quarkus
still tries to connect using the H2 driver. The URL is injected correctly — the driver is wrong.

### Context

A module's `application.properties` sets `quarkus.datasource.db-kind=h2` and
`quarkus.datasource.jdbc.url=jdbc:h2:mem:...` for its standard `@QuarkusTest` suite.
A second test class annotated with `@TestProfile` or
`@QuarkusTestResource` attempts to switch to PostgreSQL by overriding `db-kind` and providing
a PostgreSQL JDBC URL. The switch fails regardless of how the URL is provided.

### Root cause

`quarkus.datasource.db-kind` is a **build-time property**. During augmentation, Quarkus
determines which JDBC driver class to use for Agroal's connection factory and bakes that
class name into the augmented artifact (`target/quarkus-app/`). At runtime, Agroal
instantiates the baked-in driver class and calls `driver.acceptsURL(url)`. If the URL is
for a different database than the baked-in driver, `acceptsURL` returns false and the
connection fails.

`@TestProfile.getConfigOverrides()` and `QuarkusTestResourceLifecycleManager.start()` are
applied at **runtime**, after the augmentation cache check has already decided which driver
to use. They can change the URL, username, and password — but not the driver class.

### Fix

Set `quarkus.datasource.db-kind=postgresql` as a **Surefire system property** so it is
visible to the augmentation process before the cache hash is computed:

```xml
<execution>
  <id>postgres-tests</id>
  <phase>test</phase>
  <goals><goal>test</goal></goals>
  <configuration>
    <reuseForks>false</reuseForks>
    <systemPropertyVariables>
      <quarkus.datasource.db-kind>postgresql</quarkus.datasource.db-kind>
      <quarkus.datasource.devservices.enabled>false</quarkus.datasource.devservices.enabled>
    </systemPropertyVariables>
    <includes>
      <include>**/PostgresTest*.java</include>
    </includes>
  </configuration>
</execution>
```

Then use `QuarkusTestResourceLifecycleManager` to start the container and inject the actual
JDBC URL at runtime (before Quarkus starts):

```java
public class PostgresTestResource implements QuarkusTestResourceLifecycleManager {
    static final PostgreSQLContainer<?> PG = new PostgreSQLContainer<>("postgres:16-alpine");

    @Override
    public Map<String, String> start() {
        PG.start();
        return Map.of(
            "quarkus.datasource.jdbc.url", PG.getJdbcUrl(),
            "quarkus.datasource.username", PG.getUsername(),
            "quarkus.datasource.password", PG.getPassword(),
            "quarkus.flyway.migrate-at-start", "false",
            "quarkus.hibernate-orm.database.generation", "drop-and-create");
    }

    @Override
    public void stop() { if (PG.isRunning()) PG.stop(); }
}
```

### Why this is non-obvious

The error message mentions the URL but the actual problem is the driver class. Developers
naturally assume injecting the correct PostgreSQL URL is sufficient. The warning
"build time fixed to 'h2'" only appears with newer Quarkus versions and doesn't say what to
do. Nothing in the `@TestProfile` or `QuarkusTestResource` documentation warns that
`db-kind` is immutable after augmentation.

### Alternatives considered

- `@TestProfile.getConfigOverrides()` — does not work; overrides applied after augmentation cache check
- `QuarkusTestResource` config map with `db-kind=postgresql` — same limitation
- Separate Maven module with PostgreSQL `application.properties` — works but adds a module
