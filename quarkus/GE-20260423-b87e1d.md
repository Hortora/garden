## Symptom

Running `mvn test` a second time in quick succession fails with:

```
Port already bound: 8081: Address already in use
```

The process that holds port 8081 is not a Quarkus server — it is a Maven Surefire JVM from the previous test run that did not fully exit before the next run started.

## Context

Occurs when running `mvn test` repeatedly without waiting for the JVM to fully exit. Quarkus `@QuarkusTest` starts an HTTP server on the default test port (8081). When Surefire forks the JVM, the server can remain bound briefly after the test completes.

## Root Cause

Quarkus uses port 8081 as the default test port. Without explicit configuration, every test run competes for the same port. The Surefire forked JVM may not release the port before the next run starts.

## Fix

Add `quarkus.http.test-port=0` to `src/test/resources/application.properties`. Port 0 tells the OS to assign a random available port for each test run, eliminating contention.

```properties
# src/test/resources/application.properties
quarkus.http.test-port=0
```

## Why This Is Non-Obvious

The error looks like another process (a dev server, a previous run that crashed) is holding the port. The actual cause — the Surefire JVM from the immediately preceding test run — is not visible in the error message. `lsof -i :8081` reveals the process is a `java` invocation with a Surefire classpath, not a Quarkus server.
