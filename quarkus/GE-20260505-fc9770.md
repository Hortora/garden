### What and why
When building a CDI interceptor that needs to pass context to downstream code (e.g. an
`@EntityListener` or enricher running at `@PrePersist`), the natural choice is
`@RequestScoped` for the context holder bean. But `@RequestScoped` requires an active HTTP
request scope — it fails in `@Scheduled` jobs, async workers, and `@QuarkusTest` without HTTP.

Use `@ApplicationScoped` with a `static ThreadLocal<Deque<T>>` instead. Thread-local storage
is inherently scoped to the current execution thread (effectively request-scoped for most
workloads) without depending on any CDI scope activation.

### Pattern

```java
@ApplicationScoped
public class ProvenanceContext {

    record SourceState(String entityType, String entityId, String entitySystem) {}

    // Static so it's per-thread regardless of CDI proxy boundary
    private static final ThreadLocal<Deque<SourceState>> STACK =
            ThreadLocal.withInitial(ArrayDeque::new);

    public void push(String type, String id, String system) {
        STACK.get().push(new SourceState(type, id, system));
    }

    public void pop() {
        Deque<SourceState> stack = STACK.get();
        if (!stack.isEmpty()) stack.pop();
        if (stack.isEmpty()) STACK.remove();   // prevent classloader leak on hot reload
    }

    public boolean isActive() { return !STACK.get().isEmpty(); }
    public SourceState current()  { return STACK.get().peek(); }
}
```

The interceptor always calls `pop()` in a `finally` block — exception safety is guaranteed.
`STACK.remove()` when the deque empties prevents classloader leaks on Quarkus hot reload.

### Why non-obvious
Most developers reach for `@RequestScoped` for request-bounded context. `@RequestScoped`
fails silently outside HTTP — `ContextNotActiveException` at runtime. The `ThreadLocal` +
`Deque` pattern handles nesting (inner interceptor wins via `peek()`), cleans up reliably,
and works uniformly in scheduled jobs, `@QuarkusTest`, and production HTTP paths.

A `static` `ThreadLocal` on an `@ApplicationScoped` bean also avoids the CDI proxy
field-access trap: because `STACK` is static, both the proxy and the bean instance see
the same field.

### When to use
CDI interceptor binding that needs to pass context to code running at `@PrePersist`, inside
a repository call, or any point that may not have an active CDI scope. Suitable for
workflow provenance, audit decoration, tenant context, or trace enrichment.
