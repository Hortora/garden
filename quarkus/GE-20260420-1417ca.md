### The technique

When a `@QuarkusTest` integration test creates database state via HTTP (e.g. calling a REST endpoint that persists data), `@TestTransaction` cannot roll it back ‚Äî the HTTP request runs in its own transaction on the server side. The test transaction and the HTTP transaction are independent.

To clean up HTTP-created DB state in `@AfterEach`, inject `UserTransaction` and wrap Panache deletes manually:

```java
@QuarkusTest
@TestSecurity(user = "test", roles = "user")
class MyIntegrationTest {

    @Inject QhorusMcpTools tools;   // creates DB state
    @Inject UserTransaction ut;

    private String channelName;

    @BeforeEach
    void setup() {
        channelName = "test-" + System.nanoTime();
        tools.createChannel(channelName, "desc", "APPEND", null);  // committed immediately
    }

    @AfterEach
    void cleanup() throws Exception {
        ut.begin();
        Channel ch = Channel.<Channel>find("name = ?1", channelName).firstResult();
        if (ch != null) {
            Message.delete("channelId = ?1", ch.id);  // delete dependents first
            ch.delete();
        }
        ut.commit();
    }
}
```

### Why this is non-obvious

`@TestTransaction` is the standard Quarkus test isolation mechanism ‚Äî but it only wraps the test method body. CDI beans with `@Transactional(REQUIRED)` called from the test body join the test's transaction and roll back with it. However, REST endpoint calls go through a new HTTP request with its own server-side transaction ‚Äî that transaction commits independently, leaving the state in the DB.

The distinction only matters when: (1) you're testing HTTP endpoints that persist data, (2) that data would cause false failures in other tests relying on an empty table (e.g. `body("$", hasSize(0))` assertions). Setup with bare `@Inject` CDI calls (not via HTTP) is also affected since `@Transactional(REQUIRED)` CDI beans commit their own transaction when called directly.

Delete dependents (child rows by FK) before the parent row, or use a JPQL DELETE query rather than `entity.delete()` if no Panache cascade is defined.
