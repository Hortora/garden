## What It Is

casehub-engine has a `persistence-memory` Maven profile in `engine/pom.xml` that activates the in-memory persistence layer for tests. Without this profile, engine integration tests attempt to use Testcontainers (Docker) and fail with class-loader errors when Docker is not available.

## How to Use It

```bash
# Engine tests without Docker (correct command):
TESTCONTAINERS_RYUK_DISABLED=true mvn test -pl engine \
  -Ppersistence-memory \
  -Dexcludes="**/SignalTest.java"
```

The profile sets `quarkus.test.profile=memory` via `engine.test.extra.jvm.args` and adds `casehub-persistence-memory` as a test dependency.

## Why Not Documented

The profile was added in PR #86 ("test(engine): add Maven profiles for dual-persistence testing") with no update to CLAUDE.md or README. The profile name and its requirement for Docker-free development was only discoverable by reading `engine/pom.xml`.

## Notes

- Without `-Ppersistence-memory`, tests `SignalDedupExtendedTest` and `WorkerScheduleDedupTest` fail at JUnit class discovery (not at test execution) when Docker is unavailable
- `SignalTest` has a separate pre-existing class-loader issue unrelated to Docker — always exclude it
- CLAUDE.md has since been updated to document this requirement
