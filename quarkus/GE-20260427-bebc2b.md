### Symptom
```
error: cannot find symbol
  symbol: class DefaultBean
  location: package jakarta.enterprise.inject
```
Trying to use `@DefaultBean` to create a CDI producer that yields to user-provided beans.

### Context
Looking for a CDI way to say "use this bean unless the application provides its own." CDI 4.x has no such annotation. You find references to `@DefaultBean` in Quarkus documentation and examples but the package is not obvious.

### Root cause
`@DefaultBean` is a Quarkus ArC extension, not part of the CDI specification. It lives in:
```java
import io.quarkus.arc.DefaultBean;
```
Not `jakarta.enterprise.inject`, not `jakarta.enterprise.inject.spi`.

### Fix
```java
import io.quarkus.arc.DefaultBean;

@Produces
@DefaultBean
@ApplicationScoped
public HolidayCalendar produceDefault() {
    return new ConfigHolidayCalendar(config);
}
```
Any `@ApplicationScoped` bean implementing `HolidayCalendar` provided by the application wins automatically — no `@Alternative` needed.

### Why this is non-obvious
CDI documentation doesn't mention it. Quarkus docs mention it in guides but don't always show the import. The class name sounds like it would be in the CDI spec (`jakarta.enterprise.inject`) by analogy with `@Default`, which is a CDI standard qualifier. `io.quarkus.arc` is not an obvious first guess.
