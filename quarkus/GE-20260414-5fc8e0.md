## What It Does

The CNCF Serverless Workflow SDK provides a `CallableTaskBuilder` SPI for registering custom callable task types. The SPI is registered via Java ServiceLoader and controls how `call:` tasks in workflow YAML are executed.

## Exact Interface Signatures

```java
// io.serverlessworkflow.impl.executors.CallableTaskBuilder
public interface CallableTaskBuilder<T extends TaskBase> extends ServicePriority {
    boolean accept(Class<? extends TaskBase> clazz);
    void init(T task, WorkflowDefinition definition, WorkflowMutablePosition position);
    CallableTask build();
}

// io.serverlessworkflow.impl.executors.CallableTask
@FunctionalInterface
public interface CallableTask {
    CompletableFuture<WorkflowModel> apply(
        WorkflowContext workflowContext,
        TaskContext taskContext,
        WorkflowModel input);
}
```

**Registration:** `META-INF/services/io.serverlessworkflow.impl.executors.CallableTaskBuilder`

**CDI access inside CallableTask:** `Arc.container().instance(MyBean.class).get()` works inside the `apply()` lambda — the Arc container is available at execution time.

**Suspension:** Return an incomplete `CompletableFuture<WorkflowModel>` to suspend the workflow. Complete it later with the output model to resume.

**WorkflowModel creation:** `workflowContext.definition().application().modelFactory().fromMap(Map<String, Object>)` or `.from(String json)`.

## Why This Is Undocumented

The Quarkus Flow documentation describes the YAML task types but does not document the `CallableTaskBuilder` SPI for custom extensions. The interface signatures are only discoverable by reading the `serverlessworkflow-impl-core` sources JAR from the local Maven repository.
