## Zero-duplication reactive test doubles: wrap InMemory*Store in a delegation shell returning Uni

**ID:** GE-20260420-c1d394
**Stack:** Quarkus 3.32.2, Hibernate Reactive, Mutiny

**Problem:** A dual-stack extension (blocking + reactive) needs in-memory test implementations for both `ChannelStore` (returns `T`) and `ReactiveChannelStore` (returns `Uni<T>`). Duplicating all the in-memory state and logic doubles the maintenance surface.

### Technique
Create `InMemoryReactiveChannelStore` as a thin delegation shell — it holds an `InMemoryChannelStore` instance and wraps every method call in `Uni.createFrom().item(...)`. All state and business logic stay in the blocking in-memory store:

```java
@Alternative @Priority(1) @ApplicationScoped
public class InMemoryReactiveChannelStore implements ReactiveChannelStore {

    private final InMemoryChannelStore delegate = new InMemoryChannelStore();

    @Override
    public Uni<Optional<Channel>> findByName(String name) {
        return Uni.createFrom().item(() -> delegate.findByName(name));
    }

    @Override
    public Uni<Channel> save(Channel channel) {
        return Uni.createFrom().item(() -> delegate.save(channel));
    }

    @Override
    public void clear() { delegate.clear(); }
}
```

The `@BeforeEach clear()` call resets the single shared delegate — no separate state management needed in the reactive wrapper.

### Why non-obvious
The instinct is to create a fully separate `InMemoryReactiveChannelStore` with its own `Map` fields. The delegation approach only works cleanly because the blocking in-memory store is already stateless-beyond-its-fields — no `EntityManager` or Panache plumbing to skip. The pattern also reveals that `clear()` should be on the interface, not just the impl, so tests can reset via the reactive interface.
