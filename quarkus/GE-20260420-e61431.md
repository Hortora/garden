### What happened

`QhorusMcpTools.sendMessage()` throws `IllegalArgumentException("Channel not found: name")` for unknown channels. In a JAX-RS resource, catching `IllegalArgumentException` directly missed this — the test for 404 returned 409 instead.

### Root cause

`@WrapBusinessError` on the `@Tool`-annotated CDI bean intercepts `IllegalArgumentException` and `IllegalStateException` before they exit the CDI proxy boundary and re-wraps them as `ToolCallException(Throwable cause)`. The JAX-RS resource sees only `ToolCallException` — the original exception is `e.getCause()`.

`e.getMessage()` on the wrapped exception returns the cause's `toString()` (not just the message), so message-string matching is fragile.

### Fix

Catch `ToolCallException` and inspect the cause type:

```java
} catch (ToolCallException e) {
    Throwable cause = e.getCause();
    if (cause instanceof IllegalArgumentException) {
        return Response.status(404).entity(cause.getMessage()).build();
    }
    String msg = cause != null ? cause.getMessage() : e.getMessage();
    return Response.status(409).entity(msg).build();
}
```

Also keep a bare `catch (IllegalArgumentException e)` above as a guard for the (unlikely) case where `@WrapBusinessError` is not present.

### Why this is non-obvious

The `@WrapBusinessError` annotation is on the `@Tool` method, not on the CDI bean class. Its effect applies at the CDI proxy boundary — callers outside the proxy never see the original exception type, only `ToolCallException`. The interceptor is implemented in `quarkus-mcp-server-http` and is not documented as a caller-visible behaviour.

Methods that map all `ToolCallException`s identically (e.g., `return List.of()`) never expose this wrapping. Only code that needs to distinguish exception subtypes hits the problem.

### Update — 2026-05-05: `assertThrows` in `@QuarkusTest` has the same trap

The same wrapping causes test assertion failures. `assertThrows(IllegalArgumentException.class, ...)` fails because the test sees `ToolCallException`:

```
Unexpected exception type thrown, expected: <java.lang.IllegalArgumentException>
but was: <io.quarkiverse.mcp.server.ToolCallException>
```

Fix: assert on `ToolCallException.class` and optionally inspect the cause:

```java
ToolCallException ex = assertThrows(ToolCallException.class,
        () -> tools.registerBackend("ch", "bad-id", "human_observer"));
assertInstanceOf(IllegalArgumentException.class, ex.getCause());
```
