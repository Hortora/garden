## Behaviour

`@io.quarkus.hibernate.orm.PersistenceUnit("name")` is both a CDI qualifier and an injection point on its own. The correct form for named persistence unit `EntityManager` injection in Quarkus is:

```java
// Correct
@PersistenceUnit("mypu")
EntityManager em;

// Wrong — adding @Inject breaks CDI resolution
@Inject
@PersistenceUnit("mypu")
EntityManager em;
```

Adding `@Inject` alongside `@PersistenceUnit` causes CDI to fail to resolve the injection point, typically with an `UnsatisfiedResolutionException` or similar at startup.

## Context

Only relevant when using named (non-default) Hibernate persistence units — i.e. when `quarkus.hibernate-orm."name".*` config is used. The default persistence unit's `EntityManager` is injected with `@Inject EntityManager em` (no qualifier needed).

## Why Undocumented

The Quarkus named datasource and multiple persistence unit documentation shows the `@PersistenceUnit` annotation but does not explicitly state that `@Inject` must be omitted. Developers familiar with standard Jakarta EE `@PersistenceContext` (which does use `@Inject` in CDI contexts) apply the same pattern and encounter a confusing startup failure.

The Jakarta EE `@PersistenceContext` annotation is entirely different from Quarkus's `@PersistenceUnit` — the Quarkus annotation is a custom CDI producer-based qualifier, not a standard JPA injection mechanism.
