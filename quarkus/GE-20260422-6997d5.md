## @Scheduled bean testability via package-private Clock+Duration constructor

### The technique

Give the `@ApplicationScoped` `@Scheduled` bean two constructors: one `@Inject` for production (CDI), one package-private for tests (takes `Clock` and `Duration` explicitly). Tests instantiate the bean directly with a fake clock and call the scheduled method synchronously — zero sleeps, fully deterministic.

```java
@ApplicationScoped
public class CaseTimeoutEnforcer {

    private final Clock clock;
    private final Duration limit;
    private final CaseInstanceCache cache;

    @Inject
    public CaseTimeoutEnforcer(CaseInstanceCache cache) {
        this(cache, Clock.systemUTC(), Duration.ofMinutes(5));
    }

    // Package-private: used by unit tests only
    CaseTimeoutEnforcer(CaseInstanceCache cache, Clock clock, Duration limit) {
        this.cache = cache;
        this.clock = clock;
        this.limit = limit;
    }

    @Scheduled(every = "${casehub.resilience.timeout.check-interval:1s}")
    public void scanForTimeouts() {
        Instant now = clock.instant();
        // ... logic using now instead of Instant.now()
    }
}
```

Test using the package-private constructor:

```java
class CaseTimeoutEnforcerTest {

    private CaseTimeoutEnforcer enforcer;

    @BeforeEach
    void setUp() {
        CaseInstanceCache cache = new CaseInstanceCache();
        // Set clock 10 minutes in the future relative to when cases "started"
        Clock fakeClock = Clock.fixed(Instant.now().plus(Duration.ofMinutes(10)), ZoneOffset.UTC);
        enforcer = new CaseTimeoutEnforcer(cache, fakeClock, Duration.ofMinutes(5));
    }

    @Test
    void running_case_with_exhausted_budget_is_faulted() {
        // add a RUNNING case, call scanForTimeouts() directly — no Awaitility, no sleep
        enforcer.scanForTimeouts();
        assertThat(instance.getState()).isEqualTo(CaseStatus.FAULTED);
    }
}
```

### Why this is non-obvious

Most developers either (a) use `@QuarkusTest` and Awaitility with real time waits (introduces sleeps and CI flakiness) or (b) mock `Instant.now()` via PowerMock-style static mocking (fragile, deprecated in modern Java). The package-private constructor pattern avoids both: no container needed, no static mocking, no time waits. CDI ignores the package-private constructor because it doesn't have `@Inject`, so production behaviour is unaffected.
