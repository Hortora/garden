## quarkus-flow TaskExecutorFactory SPI — undocumented extension point for custom task execution

**ID:** GE-20260414-14d244
**Stack:** quarkus-flow, CNCF Serverless Workflow SDK 7.17.0.Final

### What it is

`io.serverlessworkflow.impl.executors.TaskExecutorFactory` is a Java SPI interface that allows external implementations to handle custom task types in quarkus-flow workflows. It is registered via `META-INF/services/io.serverlessworkflow.impl.executors.TaskExecutorFactory` and discovered at runtime through Java's `ServiceLoader` mechanism.

Not mentioned in any quarkus-flow or CNCF Serverless Workflow official documentation. Discovered by decompiling the `quarkus-mcp-server-core` JAR and examining `FlowNativeProcessor.java` which registers it for native image compilation.

### Discovery method

```bash
# Find the JAR
find ~/.m2/repository/io/quarkiverse/flow -name "*.jar" | grep -v sources

# Decompile the class
cd /tmp
jar xf ~/.m2/repository/.../quarkus-flow-core-*.jar \
    io/quarkiverse/flow/deployment/FlowNativeProcessor.class
javap io/quarkiverse/flow/deployment/FlowNativeProcessor.class

# Inspect the interface
jar xf ~/.m2/.../serverlessworkflow-impl-*.jar \
    io/serverlessworkflow/impl/executors/TaskExecutorFactory.class
javap io/serverlessworkflow/impl/executors/TaskExecutorFactory.class
```

### Interface signature

```java
public interface TaskExecutorFactory extends ServicePriority {
    TaskExecutorBuilder<? extends TaskBase> getTaskExecutor(
        WorkflowContext position,
        Task task,
        WorkflowDefinition definition);
}
```

`ServicePriority` provides ordering — higher priority wins when multiple factories match the same task type.

### How to register a custom implementation

```java
// 1. Implement the interface
public class HumanTaskExecutorFactory implements TaskExecutorFactory {
    @Override
    public TaskExecutorBuilder<? extends TaskBase> getTaskExecutor(
            WorkflowContext ctx, Task task, WorkflowDefinition def) {
        if (/* task matches your custom type */) {
            return new HumanTaskExecutorBuilder(task, def);
        }
        return null; // not handled by this factory
    }
}

// 2. Register in META-INF/services/io.serverlessworkflow.impl.executors.TaskExecutorFactory
//    (one fully-qualified class name per line)

// 3. For native image: register in deployment @BuildStep
@BuildStep
void registerForReflection(BuildProducer<ReflectiveClassBuildItem> producer) {
    producer.produce(ReflectiveClassBuildItem
        .builder(HumanTaskExecutorFactory.class).build());
}
```

### Why it matters

This SPI is the correct integration point for projects that need to add custom task types to quarkus-flow workflows — for example, a human task inbox (Quarkus Tarkus) that suspends a workflow until a human approves/rejects a work item. Without this SPI, the only option would be forking quarkus-flow or using a workaround via `listen()` + CloudEvents.

The `quarkus-tarkus-flow` integration module uses exactly this SPI to intercept `humanTask` workflow steps.
