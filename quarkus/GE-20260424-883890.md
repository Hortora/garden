### Symptom
`@QuarkusTest` augmentation fails with:

```
jakarta.enterprise.inject.spi.DeploymentException:
POST /examples/spawn is declared by:
  io.quarkiverse.work.examples.spawn.SpawnScenario#run consumes *, produces application/json
  io.quarkiverse.work.examples.spawn.SpawnScenarioResource#run consumes *, produces application/json
```

Two classes declare the same JAX-RS path and method with identical media types. No obvious reason for the duplicate — you only wrote one.

### Context
An agent running in a git worktree was interrupted mid-task after creating `SpawnScenarioResource.java`. When the main session resumed and created `SpawnScenario.java` at the same `@Path("/examples/spawn")`, both files were on disk. Quarkus discovered both during augmentation.

### Root cause
Interrupted agents (worktree agents, subagents, or cancelled tasks) can leave uncommitted or committed files in the working tree that the next session doesn't know about. If a new file is created at the same JAX-RS path as an existing file from a prior interrupted task, Quarkus reports a duplicate endpoint at augmentation time — not at compile time.

### Fix
Search for all `@Path` declarations that match the conflicting path:

```bash
grep -rn '"/examples/spawn"' --include="*.java" src/
```

Identify the stale file (usually the one from the interrupted session) and delete it:

```bash
git rm quarkus-work-examples/src/main/java/.../SpawnScenarioResource.java
```

### Why this is non-obvious
The error message names both classes clearly, but when working with agents and worktrees it's non-obvious that a prior interrupted session left a file behind. The fix is trivial once identified; the confusion is in not knowing a second file exists.
