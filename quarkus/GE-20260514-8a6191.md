### Symptom
JUnit test discovery fails with:
```
ClassSelector [className = 'io.example.MyTest'] resolution failed
Cause: PreconditionViolationException: Could not load class with name: io.example.MyTest
```
The class exists and compiled successfully. Zero compilation errors.

### Context
Running `@QuarkusTest` tests that need a datasource backed by Dev Services (H2 or PostgreSQL via Testcontainers). Docker or Podman daemon is not running.

### Root cause
Quarkus augmentation fails during test startup because Dev Services cannot launch a container for the datasource. The Quarkus JUnit extension never completes augmentation, so its classloader never initialises — JUnit then reports the test class as unloadable. The "could not load class" error is an augmentation failure masquerading as a classloading problem.

### Fix
Start Docker or Podman before running the tests:
```bash
# Podman
podman machine start

# Docker
open -a Docker   # macOS
```
Then re-run the tests normally.

### Why this is non-obvious
The error message says `Could not load class` and names a specific test class — implying a compilation, classpath, or bytecode problem. The actual cause (no container runtime) appears nowhere in the error output. The class exists on disk and was compiled successfully; the failure is several layers removed in the Quarkus augmentation pipeline.
