## Symptom

An SSE endpoint (or any endpoint) that serialises `SessionResponse` objects silently returns `[]` (empty array) instead of the actual workers. No exception is visible to the caller ‚Äî the exception is caught internally and the fallback value is returned. The server logs may show a `com.fasterxml.jackson.databind.exc.InvalidDefinitionException` at DEBUG level.

## Context

`SessionResource` in Claudony had:
```java
private static final ObjectMapper MAPPER = new ObjectMapper();
```

`SessionResponse` contains `Instant createdAt` and `Instant lastActive`. Serializing a `List<SessionResponse>` through this `MAPPER` threw:
```
com.fasterxml.jackson.databind.exc.InvalidDefinitionException:
Java 8 date/time type `java.time.Instant` not supported by default:
add Module "com.fasterxml.jackson.datatype:jackson-datatype-jsr310"
```

The exception was caught in the `buildCaseSnapshot` helper and the fallback `"data: []"` was returned silently.

## Root Cause

`new ObjectMapper()` creates a bare instance with no modules registered. Quarkus configures its CDI-managed `ObjectMapper` bean with `JavaTimeModule` (and other modules like `Jdk8Module`) automatically. A static field bypasses this configuration.

## Fix

Replace the static field with CDI injection:

```java
// WRONG ‚Äî no JavaTimeModule
private static final ObjectMapper MAPPER = new ObjectMapper();

// CORRECT ‚Äî uses Quarkus-configured instance with all modules
@Inject
ObjectMapper MAPPER;
```

## Why This Is Non-Obvious

The class had a `static final` `ObjectMapper` that worked fine for other uses (simple POJOs without time fields). The failure only surfaced when the snapshot function started serialising `SessionResponse` with `Instant` fields. Because the exception was caught with a fallback, the endpoint returned valid JSON (`[]`) ‚Äî not an error ‚Äî making it very hard to notice.
