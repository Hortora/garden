## What

Jackson 2.12+ natively deserializes Java records using constructor parameter names — without any `@JsonProperty` annotations — when the code is compiled with the `-parameters` flag. The JSON key names must match the record component names exactly.

```java
// No @JsonProperty needed — Jackson binds by constructor parameter name
public record EnemyStrategy(
    List<EnemyBuildStep> buildOrder,
    boolean loop,
    int mineralsPerTick,
    EnemyAttackConfig attackConfig
) {}
```

This works for both serialization and deserialization, including nested records.

## How It Works

The `-parameters` compiler flag (JEP 118 / Java 8+) preserves method and constructor parameter names in the `.class` file. Without it, parameter names are erased to `arg0`, `arg1`, etc. Jackson's `RecordNamingStrategyPatchModule` (integrated in Jackson 2.15+ into the core, available via `RecordDeserializer` from 2.12+) reads the preserved parameter names and uses them as JSON keys.

Quarkus sets `<parameters>true</parameters>` in `maven-compiler-plugin` configuration by default to support CDI injection point name resolution. This same flag enables Jackson record deserialization for free.

## Verification

Confirmed by:
1. Adding `@JsonProperty` annotations to three domain records
2. Code reviewer flagged annotations as redundant vs other un-annotated records in the same codebase
3. Removing all `@JsonProperty` annotations
4. JSON round-trip tests passing without any changes to test code

## Why Not Documented

The Jackson documentation focuses on `@JsonCreator` + `@JsonProperty` as the standard approach for immutable types. The record-specific support via `-parameters` is mentioned in Jackson release notes (2.12) but not in the main object-binding documentation or the Java Records guide. Developers annotate records by habit before discovering the annotations are unnecessary.
