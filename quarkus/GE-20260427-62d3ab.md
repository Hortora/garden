## Use @Alternative @Priority(1) inner beans in @QuarkusTest to spy on SPI call sites without Mockito

### Stack
Quarkus 3.x, CDI 4.0, JUnit 5

### Context
You've wired an SPI into the engine and want to verify it's called at the right lifecycle
points. Mockito `@InjectMock` replaces the bean only for its own test class and requires
adding a test dependency. The CDI alternative pattern avoids both.

### Technique

Define a recording implementation as a `static` inner class in the test with
`@Alternative @Priority(1) @ApplicationScoped`. CDI 4.0 activates it globally across
the entire test suite — `@Priority(1)` beats the default no-op (priority 0) automatically.
Use `static` fields for the recorded state so all injected references share it, and reset
in `@BeforeEach`:

```java
@QuarkusTest
class SpiWiringTest {

  @Inject RecordingWorkerStatusListener listener;

  @BeforeEach
  void reset() {
    RecordingWorkerStatusListener.reset();
  }

  @Test
  void onWorkerStartedCalledWhenWorkerBegins() {
    // ... start case ...
    await().atMost(15, SECONDS).untilAsserted(() ->
        assertThat(RecordingWorkerStatusListener.startedWorkerIds).isNotEmpty());
  }

  @Alternative
  @Priority(1)
  @ApplicationScoped
  public static class RecordingWorkerStatusListener implements WorkerStatusListener {

    static final List<String> startedWorkerIds = new CopyOnWriteArrayList<>();
    static volatile WorkResult lastCompletedResult;

    static void reset() {
      startedWorkerIds.clear();
      lastCompletedResult = null;
    }

    @Override
    public void onWorkerStarted(String workerId, Map<String, String> sessionMeta) {
      startedWorkerIds.add(workerId);
    }
    // ... other methods ...
  }
}
```

### Why this is non-obvious
Most developers reach for Mockito `@InjectMock` or `@QuarkusTestProfile` to swap
implementations per test. The CDI alternative approach requires no extra dependency and
integrates cleanly with the full container — but the global activation scope of
`@Priority(1)` (documented as a gotcha elsewhere: GE-20260415-884e48) becomes an
advantage here: the recording bean is active for all tests, and static reset in
`@BeforeEach` provides isolation. The key insight is that recording state that doesn't
change observable behaviour is safe to activate globally.
