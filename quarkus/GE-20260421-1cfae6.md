## @Produces @DefaultBean @ApplicationScoped on producer methods enables consumer-replaceable CDI defaults

**ID:** GE-20260421-1cfae6
**Stack:** Quarkus Arc 3.32.2

### The pattern

Annotate a producer method with `@Produces @DefaultBean @ApplicationScoped` to provide a built-in default that consumers can silently replace by declaring their own CDI bean of the same type — no `@Alternative`, no `beans.xml`, no config required.

```java
@ApplicationScoped
public class LedgerPrivacyProducer {

    @Inject LedgerConfig config;
    @Inject EntityManager em;  // CDI proxy — resolves to current transaction's session

    @Produces
    @DefaultBean
    @ApplicationScoped
    public ActorIdentityProvider actorIdentityProvider() {
        if (config.identity().tokenisation().enabled()) {
            return new InternalActorIdentityProvider(em);  // plain Java object
        }
        return new PassThroughActorIdentityProvider();
    }
}
```

A consumer overrides by declaring their own bean — the producer's output is silently ignored:

```java
@ApplicationScoped
public class MyCustomActorIdentityProvider implements ActorIdentityProvider {
    // replaces the producer's default without any configuration
}
```

### Key property: EntityManager CDI proxy in a non-CDI object

The `em` field in the producer is a CDI proxy. Passing it to a plain Java constructor (`new InternalActorIdentityProvider(em)`) is safe — the proxy is thread-safe and resolves to the current persistence context when any method is called. As long as callers operate within `@Transactional`, the EntityManager works correctly without the constructed object itself being a CDI bean.

### Why this is non-obvious

Quarkus docs show `@DefaultBean` on class-level `@ApplicationScoped` beans only. Applying it to a `@Produces` method is undocumented but works correctly in Quarkus Arc. The producer method form is more powerful: it can return different implementations based on runtime config, construct plain Java objects receiving CDI proxies, and avoid the ambiguity of multiple `@ApplicationScoped` implementations of the same type.

Most developers who want runtime-switchable CDI defaults reach for `@Alternative` + `@Priority` or build-time `@IfBuildProperty`. This pattern achieves the same at runtime with zero consumer-side changes.
