## quarkus.datasource.reactive=false suppresses Hibernate Reactive boot when extension is on classpath but no reactive pool exists

**ID:** GE-20260420-45d53b
**Stack:** Quarkus 3.32.2

**What it does:** Setting `quarkus.datasource.reactive=false` in `application.properties` prevents Hibernate Reactive from attempting to acquire a reactive (Vert.x) connection pool at startup, even when `quarkus-hibernate-reactive-panache` is on the classpath and has registered reactive persistence units at build time.

**Where it works:** Test `application.properties`, per-profile properties, and main `application.properties`.

```properties
# Prevents FastBootHibernateReactivePersistenceProvider from failing
# when no reactive datasource is configured
quarkus.datasource.reactive=false
```

### Why non-obvious
Nothing in the Quarkus extension developer docs mentions this property as a reactive-boot suppressor. The reactive extension activates at build time based on classpath scanning; suppressing it at runtime via a datasource flag is counter-intuitive. The natural search path (Hibernate Reactive docs, `@Alternative`, `quarkus.hibernate-reactive.*` properties) doesn't lead here. Found by trial and error.
