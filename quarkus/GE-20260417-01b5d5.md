## What exists but isn't documented

`WrapBusinessErrorInterceptor` checks whether the intercepted method is annotated with `@Tool` before wrapping any exception. This guard is in the interceptor source:

```java
private Throwable wrapIfNecessary(Throwable t, InvocationContext context) {
    if (context.getMethod().isAnnotationPresent(Tool.class) && matches(t, context)) {
        return new ToolCallException(t);
    }
    return t;
}
```

This means placing `@WrapBusinessError` at the class level only wraps `@Tool`-annotated methods — not every method on the bean. A class with a mix of `@Tool` methods and ordinary helper methods will only wrap the `@Tool` ones.

## Why this matters

The `@WrapBusinessError` javadoc says: "Wraps a matching exception thrown from a 'feature' method." What counts as a "feature" method is not defined externally. The source code is the only way to discover this guard.

Without knowing about this guard, you might avoid class-level placement entirely and add the annotation per `@Tool` method (39 annotations instead of 1) to avoid accidentally wrapping helper methods.
