## Symptom

Panel or EventSource client shows empty data or silently fails to parse. The server emits events (observable via `curl`), but `JSON.parse(event.data)` in the browser returns an error or the data is wrong.

## Context

Building a `GET /api/sessions/{id}/case-events` SSE endpoint in Quarkus using:
```java
@GET
@Path("/{id}/case-events")
@Produces("text/event-stream")
public Multi<String> caseEvents(@PathParam("id") String id) { ... }
```

The `buildCaseSnapshot` helper was returning pre-formatted SSE frames:
```java
return "data: " + MAPPER.writeValueAsString(workers) + "\n\n";
```

## Root Cause

Quarkus REST (via `quarkus-rest` or `quarkus-resteasy-reactive`) automatically adds SSE framing when the return type is `Multi<String>` with `@Produces("text/event-stream")`. Each item emitted from the `Multi` is wrapped as:
```
data: <item>\n\n
```

So returning `"data: " + json + "\n\n"` produces the following on the wire:
```
data: data: [{"id":"..."}]\n\n\n\n
```

The browser's `EventSource.onmessage` delivers `e.data = "data: [...]"` — the raw string including the second `data:` prefix. `JSON.parse` then fails (or silently catches and returns `[]` if the exception is swallowed).

## Fix

Return plain JSON strings from the snapshot function — no SSE framing:

```java
// WRONG — Quarkus adds "data:" automatically
return "data: " + MAPPER.writeValueAsString(workers) + "\n\n";

// CORRECT — return plain content
return MAPPER.writeValueAsString(workers);
```

The wire format becomes `data: [{"id":"..."}]\n\n` and `JSON.parse(event.data)` works correctly.

## Why This Is Non-Obvious

The existing `MeshResource.events()` in the same codebase uses `"data: " + json + "\n\n"` and works — because that endpoint is consumed by a JavaScript polling client doing a raw `fetch()` and reading the response body as a string, not by `EventSource.onmessage`. The two consumption patterns require opposite approaches. It's natural to follow the existing pattern when adding a new SSE endpoint, and the failure is silent (exception caught, empty array returned).
