## @ConfigMapping in a Quarkus extension requires Javadoc on every method, including group accessors

**ID:** GE-20260414-c87a14
**Stack:** Quarkus 3.32.2, `quarkus-extension-processor` on `annotationProcessorPaths`, SmallRye Config
**Symptom:** Compilation fails with: `Fatal error compiling: java.lang.IllegalStateException: Unable to scan config mapping without config root: io.example.MyConfig: Unable to find javadoc for config item io.example.MyConfig groupName()`.
**Context:** Defining a `@ConfigMapping` interface in a Quarkus extension runtime module where `quarkus-extension-processor` is on the annotation processor path. The group accessor method (e.g. `Cleanup cleanup()`) had no Javadoc even though the leaf methods inside the group did.

### What was tried (didn't work)
- Adding Javadoc to the leaf methods inside the nested group interface (`staleInstanceSeconds()`, `dataRetentionDays()`) — error persisted pointing at the group accessor
- Checking SmallRye Config docs for `@ConfigMapping` — no mention of Javadoc requirement
- Checking Quarkus config extension docs — no mention either

### Root cause

When `quarkus-extension-processor` is present as an annotation processor, it processes `@ConfigMapping` interfaces to generate config documentation metadata (for Dev UI, docs site, and Quarkus config source tooling). This processor requires Javadoc on **every method** in the interface — both the leaf config methods (`@WithDefault("120") int staleInstanceSeconds()`) and the group accessor methods that return nested interfaces (`Cleanup cleanup()`).

Standard Quarkus **application** code does not have this restriction — the extension processor is only active in extension modules. The requirement applies to all levels of nesting.

### Fix

Add a Javadoc comment to every method, including group accessors:

```java
@ConfigMapping(prefix = "quarkus.myext")
public interface MyExtConfig {

    /** Cleanup and retention settings. */   // ← required, even for group accessors
    Cleanup cleanup();

    interface Cleanup {
        /** Seconds before an idle instance is considered stale. */
        @WithDefault("120")
        int staleInstanceSeconds();

        /** Days to retain old data before purging. */
        @WithDefault("7")
        int dataRetentionDays();
    }
}
```

### Why non-obvious

Standard Quarkus apps use `@ConfigMapping` without this restriction. The requirement only applies when `quarkus-extension-processor` is on the annotation processor path — a configuration step that doesn't exist in app development. The error message names the class and method but gives no indication that Javadoc is the missing element. Nothing in the Quarkiverse extension guide or SmallRye Config docs documents this requirement.
