## Use Optional<String> in @ConfigMapping for optional string properties that can be legitimately absent

**ID:** GE-20260414-614675
**Stack:** Quarkus 3.32.2, SmallRye Config, quarkus-extension-processor

### What the technique does

In a `@ConfigMapping` interface, return `Optional<String>` (instead of `String` with a workaround default) for properties that are legitimately optional ŌĆö where the consuming app may not set them, and absence is meaningful rather than an error.

### The problem it solves

Two common approaches fail:

1. `@WithDefault("") String url()` ŌĆö SmallRye Config treats `""` as null and throws `ConfigValidationException` at startup (see related gotcha GE-20260414-a2e8a3).
2. `String url()` with no default ŌĆö throws `NoSuchElementException` at startup if the property is absent from config.

Both leave the developer with no clean way to express "this property is optional and has no sensible universal default."

### How to apply it

```java
@ConfigMapping(prefix = "quarkus.myext")
public interface MyExtConfig {

    /** Agent card settings. */
    AgentCard agentCard();

    interface AgentCard {
        /** Display name. Has a sensible default. */
        @WithDefault("My Extension")
        String name();

        /**
         * Public URL of this deployment.
         * Should be set per-deployment; returns empty when not configured.
         */
        Optional<String> url();   // ŌåÉ no @WithDefault needed

        /**
         * Version string.
         */
        @WithDefault("1.0.0")
        String version();
    }
}
```

At the call site:
```java
// Absent ŌåÆ empty string (safe fallback)
String url = config.agentCard().url().orElse("");

// Absent ŌåÆ error (required field in practice)
String url = config.agentCard().url()
    .orElseThrow(() -> new IllegalStateException(
        "quarkus.myext.agent-card.url must be configured for production"));

// Absent ŌåÆ custom value
String url = config.agentCard().url().orElse("http://localhost:8080");
```

### Quarkus extension processor note

When using this pattern in a Quarkus extension runtime module (with `quarkus-extension-processor` on the annotation processor path), every method in the `@ConfigMapping` interface ŌĆö including `Optional<String>` methods ŌĆö **must have Javadoc**. The processor generates config documentation from the Javadoc and fails compilation if any method is missing it. Even group accessor methods like `agentCard()` need a `/** ... */` comment.

### Why developers wouldn't naturally reach for this

The Java `Optional<String>` return type on a config interface method is not an obvious choice ŌĆö it looks like it might require special framework support. But SmallRye Config handles `Optional<T>` natively: if the property is absent from config, the method returns `Optional.empty()`. The pattern is documented in the SmallRye Config guide but not prominently in the Quarkus config extension documentation.
