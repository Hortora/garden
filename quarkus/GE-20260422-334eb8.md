## What exists but isn't documented

When using multiple Hibernate ORM persistence units in Quarkus, injecting a specific PU's `EntityManager` requires a Quarkus-specific CDI qualifier — **not** the standard JPA annotation.

**Wrong** (standard JPA — doesn't work as a CDI qualifier):
```java
@PersistenceContext(unitName = "qhorus")
EntityManager em;
```

**Wrong** (another natural attempt):
```java
@Inject
@Named("qhorus")
EntityManager em;
```

**Correct** (Quarkus CDI qualifier):
```java
import io.quarkus.hibernate.orm.PersistenceUnit;

@Inject
@PersistenceUnit("qhorus")
EntityManager em;
```

## Context

Applies to any class that directly injects `EntityManager` when multiple Hibernate ORM persistence units are configured via `quarkus.hibernate-orm.<name>.*`. Panache-based stores (`PanacheEntityBase`, `PanacheRepositoryBase`) don't need this — they automatically route to the correct PU based on the `quarkus.hibernate-orm.<name>.packages` config covering the entity's package.

## Why non-obvious

The JPA specification defines `@PersistenceContext`/`@PersistenceUnit` for named PU injection. Quarkus replaces this mechanism with its own CDI qualifier `io.quarkus.hibernate.orm.PersistenceUnit`. The Quarkus multiple datasources documentation mentions named PUs but doesn't clearly state that CDI-style `@Inject + @PersistenceUnit` is the required pattern rather than the standard JPA annotations.
