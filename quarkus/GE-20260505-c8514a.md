## Context

A `@QuarkusTest` calls `gateway.receiveHumanMessage(ref, raw)` which internally calls `messageService.send()` — a `@Transactional` method that persists a message. Without transaction rollback, that message persists across tests and appears in subsequent tests' `checkMessages()` / full-text search results.

## The Problem

```java
// Test A (no @TestTransaction):
gateway.receiveHumanMessage(ref,
    new InboundHumanMessage("user-42", "Can you stop the analysis?", ...));
// Message committed to DB — visible to ALL subsequent tests

// Test B (unrelated):
var results = tools.checkMessages("some-channel", "observer", ...);
// Sees "Can you stop the analysis?" — unexpected match in artefact search for "analysis"
```

This is subtle because the leaking test looks like a read-only test (it only calls `receiveHumanMessage`) but actually triggers a persistent write via the gateway inbound path.

## Fix

Annotate gateway inbound tests with `@TestTransaction` to roll back after each test:

```java
@Test
@TestTransaction
void receiveHumanMessage_createsMessageWithHumanSender() {
    var ch = tools.listChannels().stream()
            .filter(c -> "test-ch".equals(c.name())).findFirst().orElseThrow();
    gateway.receiveHumanMessage(new ChannelRef(ch.channelId(), "test-ch"),
            new InboundHumanMessage("user-42", "Can you stop?", Instant.now(), Map.of()));

    var messages = tools.checkMessages("test-ch", "observer", ...);
    assertTrue(messages.messages().stream()
            .anyMatch(m -> "human:user-42".equals(m.sender())));
    // Rolled back after test — no leakage
}
```

## Why This Is Non-Obvious

Tests that only invoke gateway methods look like they don't write to the DB. The persistence happens inside `MessageService.send()` which is called internally by the gateway — invisible to someone reading the test. Standard `@QuarkusTest` isolation doesn't roll back CDI-managed transactions unless `@TestTransaction` is explicitly present. The failure mode is a distant test failure with no obvious connection to the inbound test.
