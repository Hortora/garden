## What Most Developers Would Do

Inject the blocking `InMemory*Store` into the reactive wrapper via `@Inject`, tying the reactive test double to CDI and requiring Quarkus to boot for every unit test.

## The Technique

Give `InMemoryReactive*Store` a `private final InMemory*Store delegate = new InMemory*Store()` field — no CDI injection. The class becomes directly instantiable in plain JUnit without Quarkus:

```java
@Alternative
@Priority(1)
@ApplicationScoped
public class InMemoryReactiveChannelStore implements ReactiveChannelStore {

    private final InMemoryChannelStore delegate = new InMemoryChannelStore();

    @Override
    public Uni<Channel> put(Channel channel) {
        return Uni.createFrom().item(() -> delegate.put(channel));  // lazy supplier
    }

    @Override
    public Uni<Void> delete(UUID id) {
        return Uni.createFrom().voidItem().invoke(() -> delegate.delete(id));
    }

    public void clear() { delegate.clear(); }
}
```

Unit tests instantiate directly and unwrap with `.await().indefinitely()`:

```java
class InMemoryReactiveChannelStoreTest {
    private final InMemoryReactiveChannelStore store = new InMemoryReactiveChannelStore();

    @BeforeEach void reset() { store.clear(); }

    @Test void put_assignsId() {
        Channel saved = store.put(ch).await().indefinitely();
        assertNotNull(saved.id);
    }
}
```

The same class also activates cleanly in `@QuarkusTest` contexts when selected via `quarkus.arc.selected-alternatives`.

## Key Details

- Use `Uni.createFrom().item(() -> delegate.method())` — the **supplier** form (lazy), not `item(value)` (eager)
- Void returns: `Uni.createFrom().voidItem().invoke(() -> delegate.method())`
- The internal delegate has its own state — not shared with any other CDI bean (fine because tests use one stack at a time)
- `clear()` must delegate to the inner store for `@BeforeEach` isolation to work

## Why This Is Non-Obvious

The obvious CDI approach (`@Inject InMemoryChannelStore delegate`) works in `@QuarkusTest` but breaks direct instantiation in unit tests. The internal delegation approach sacrifices shared state (between blocking and reactive beans) but gains test simplicity and zero bootstrap overhead.
