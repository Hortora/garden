### What it is
`quarkus-ledger` 0.2-SNAPSHOT ships `LedgerEntityManagerProducer` which produces a `@LedgerPersistenceUnit EntityManager`. By default it selects the `@Default` EntityManager. Setting `quarkus.ledger.datasource=<name>` routes it to a named persistence unit instead.

### How to use it
```properties
# In application.properties — route ledger to a named datasource:
quarkus.ledger.datasource=qhorus
```

`LedgerEntityManagerProducer.produce()` calls `instance.select(new PersistenceUnitLiteral("qhorus")).get()` instead of `instance.select(Default.Literal.INSTANCE).get()`.

### Why it matters
Apps with no default datasource (only named datasources) get `Unsatisfied dependency for type EntityManager` at Quarkus augmentation without this property. Setting it routes ledger's EntityManager to the correct named PU with no default datasource required.

### How to discover it
Only by reading `LedgerEntityManagerProducer.java` and `LedgerConfig.java` in the quarkus-ledger source. Not in the extension README or any public documentation as of 2026-04-24.
