### Context
Cleaning up JPA state between `@QuarkusTest` test methods — e.g. deleting all rows from relevant tables before each test to prevent state leaking between tests.

### Technique
Annotate the `@BeforeEach` method with `@Transactional` directly on the test class:

```java
@QuarkusTest
class MyServiceTest {

    @BeforeEach
    @Transactional
    void cleanup() {
        AuditEntry.deleteAll();
        WorkItem.deleteAll();
        WorkItemTemplate.deleteAll();
    }
}
```

This works because Quarkus registers `@QuarkusTest` instances as CDI beans and hands JUnit the CDI **proxy**, not the raw instance. JUnit calls `@BeforeEach` on the proxy — so `@Transactional` interception fires normally, exactly as it does for regular CDI method calls.

### Why this is non-obvious
The well-known CDI proxy bypass gotcha says `this.method()` skips the proxy. Since `@BeforeEach` is "called on the test instance," it seems like `@Transactional` on a lifecycle method won't be intercepted. It is intercepted — because Quarkus gives JUnit the proxy, not `this`. The bypass only applies when calling *from within* the same class instance, not when JUnit calls *into* it from outside.
