## Downstream @QuarkusTest modules fail with 'qrtz_triggers does not exist' after Flyway moves to a separate module

**ID:** GE-20260417-e71f46
**Stack:** Quarkus 3.17.x, Quartz, Flyway, Maven multi-module
**Symptom:** `IllegalStateException: Unable to create Scheduler` / `ERROR: relation "qrtz_triggers" does not exist` in `@QuarkusTest` modules that depend on the engine module.
**Context:** Multi-module Maven project where the engine module configures `quarkus.quartz.store-type=jdbc-cmt` in production. Flyway was moved from the engine to a separate `persistence-hibernate` module. Other modules (e.g. `blackboard`, `resilience`) depend on the engine but not on `persistence-hibernate`.

### What was tried (didn't work)

- The error message pointed to Quartz, not Flyway — chased Quartz configuration instead of the schema setup
- Checked if `quartz.store-type` was set in the failing module — it wasn't (inherited from engine via dependency classpath)

### Root cause

The engine's `application.properties` (`quarkus.quartz.store-type=jdbc-cmt`) is on the classpath of all downstream test modules that depend on it. When those modules run `@QuarkusTest`, Quartz starts with JDBC store and immediately queries for `qrtz_triggers`. Without Flyway or the `persistence-hibernate` module on the classpath, the Quartz tables don't exist.

### Fix

Add `%test.quarkus.quartz.store-type=ram` to each downstream module's `src/test/resources/application.properties`:

```properties
# Flyway DDL for Quartz tables lives in casehub-persistence-hibernate,
# which is not a dependency here. Use RAM Quartz for tests.
%test.quarkus.quartz.store-type=ram
```

Also add `%test.quarkus.hibernate-orm.schema-management.strategy=create` if the module also needs Hibernate tables but doesn't have Flyway.

### Why this is non-obvious

The error says Quartz can't find its tables — it looks like a Quartz configuration problem, not a Flyway scope/module problem. Nothing in the error points to the fact that Flyway was moved to a different module that isn't on this module's classpath.
