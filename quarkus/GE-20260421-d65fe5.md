## @IfBuildProperty and @UnlessBuildProperty are evaluated at build time — QuarkusTestProfile overrides have no effect

### Symptom

A `QuarkusTestProfile.getConfigOverrides()` sets `quarkus.qhorus.reactive.enabled=true`. The test runs. Beans annotated `@IfBuildProperty(name = "quarkus.qhorus.reactive.enabled", stringValue = "true")` are NOT registered — the test behaves as if the property was never set.

### Context

`@IfBuildProperty` and `@UnlessBuildProperty` are processed by Quarkus's build-time CDI processor during compilation. The annotation instructs the Arc container to include or exclude the bean when the deployment is built. `QuarkusTestProfile.getConfigOverrides()` sets runtime config overrides — these run after the build phase and have no effect on which beans are compiled into the deployment.

### What was tried

Setting `quarkus.qhorus.reactive.enabled=true` in `ReactiveTestProfile.getConfigOverrides()` and expecting `@IfBuildProperty(name = "quarkus.qhorus.reactive.enabled", stringValue = "true")` beans to be registered in `@QuarkusTest`.

### Root cause

`@IfBuildProperty` is a build-time annotation. The Quarkus deployment (including which CDI beans are registered) is fixed at compile/build time. Runtime config overrides cannot change which beans are included in an already-built deployment.

### Fix

To activate `@IfBuildProperty`-conditioned beans in tests, the property must be set at build time — not via a test profile. Options:
- Set it as a system property during the Maven build (`-Dquarkus.qhorus.reactive.enabled=true`)
- Use a separate Maven profile that sets the build property
- Prefer CDI `@Alternative` beans (without `@IfBuildProperty`) and activate them via `quarkus.arc.selected-alternatives` in the test profile — this IS a runtime-switchable mechanism

### Why this is non-obvious

The Quarkus docs show `@IfBuildProperty` and `QuarkusTestProfile` as separate features without clarifying their incompatibility. A developer naturally assumes that if you can set a config property, you can toggle build-time conditions with it.
