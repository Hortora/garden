## Use `session.getReference()` to set a `@ManyToOne` FK target in Panache `persist()` without `TransientPropertyValueException`

**ID:** GE-20260416-e10d09
**Stack:** Quarkus 3.31.1, Hibernate Reactive, Panache
**Labels:** `#hibernate` `#reactive` `#jpa` `#panache`
**What it achieves:** Persists an entity with a `@ManyToOne` FK reference to an already-persisted entity using only the referenced entity's ID — without loading the full entity from the database and without triggering `TransientPropertyValueException`.
**Context:** JPA repository implementations where the caller provides a domain object with a referenced entity ID set (e.g. `CaseInstance.getCaseMetaModel().getId()`) but you don't want a database round-trip to load the full referenced entity just to satisfy Hibernate's session association requirement.

### The technique

Inside a `Panache.withTransaction()` block, use `session.getReference(EntityClass.class, id)` to create a proxy for the referenced entity. This proxy is associated with the current session (so Hibernate doesn't treat it as transient) but does not execute a SELECT:

```java
@Override
public Uni<CaseInstance> save(CaseInstance instance) {
    return Panache.withTransaction(
        () -> Panache.getSession()
            .chain(session -> {
                CaseInstanceEntity entity = new CaseInstanceEntity();
                entity.uuid = instance.getUuid();
                entity.state = instance.getState();
                // Set FK without loading CaseMetaModelEntity from DB:
                if (instance.getCaseMetaModel() != null) {
                    entity.caseMetaModel = session.getReference(
                        CaseMetaModelEntity.class,
                        instance.getCaseMetaModel().getId()
                    );
                }
                return entity.persist()
                    .map(v -> {
                        instance.id = entity.id;
                        return instance;
                    });
            }));
}
```

The proxy satisfies Hibernate's session-association requirement. On INSERT, Hibernate uses the proxy's ID as the FK column value — no SELECT is issued.

### Alternatives considered
- **Assign a plain `new CaseMetaModelEntity()` with `id` set manually** — throws `TransientPropertyValueException: object references an unsaved transient instance`; Hibernate sees a new object with no session association and refuses to use it as a FK target
- **Load the full entity first with `findById()`** — works but executes an unnecessary SELECT; the FK column only needs the ID value

### Why this is non-obvious
The error `TransientPropertyValueException` implies the referenced entity hasn't been persisted — but it has. The actual problem is that the new entity object isn't *session-associated*, even though the row exists in the database. `getReference()` is a standard JPA/Hibernate API but is almost never shown in basic Panache examples, which typically model the full entity-loading pattern.

### When to use it
When all of these hold:
- The referenced entity is already persisted (has an ID in the DB)
- You only have the referenced entity's ID (not a session-managed instance)
- You're inside a `Panache.withTransaction()` or `Panache.withSession()` block
- You want to avoid a round-trip SELECT
