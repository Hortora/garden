## Context

When implementing a CDI observer for `WorkItemLifecycleEvent` from the quarkus-work extension, the issue documentation and natural API assumptions suggest calling `event.workItem()` to access the `WorkItem` entity (and from it, `callerRef`, `resolution`, etc.).

## Symptom

```
cannot find symbol
  symbol:   method workItem()
  location: variable event of type io.quarkiverse.work.runtime.event.WorkItemLifecycleEvent
```

`workItem` is a private field on the class — no public accessor method exists.

## Root Cause

`WorkItemLifecycleEvent` stores the `WorkItem` as a private field `workItem` with no corresponding public getter. The class extends `WorkLifecycleEvent` which declares an abstract `source()` returning `Object`. The implementation of `source()` returns the `workItem` field.

Discoverable only by decompiling the class:
```bash
javap -p -c io/quarkiverse/work/runtime/event/WorkItemLifecycleEvent.class
# shows: public java.lang.Object source() { return this.workItem; }
```

## Fix

Cast `event.source()` to `WorkItem`:

```java
void onWorkItemLifecycle(@ObservesAsync WorkItemLifecycleEvent event) {
    if (!(event.source() instanceof WorkItem workItem)) return;
    String callerRef = workItem.callerRef;
    String resolution = workItem.resolution;
    // ...
}
```

## Why This Is Non-Obvious

The official description of the event contract says to access `workItem().callerRef` — the field exists, the method does not. There is no public API documentation listing which methods are available. The `source()` method is declared on the abstract parent class with return type `Object`, giving no indication it returns a `WorkItem`.
