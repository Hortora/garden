## Behaviour

In a `@ConfigMapping` interface, annotating a `String` method with `@WithDefault("")` causes Quarkus to throw a deployment exception at startup:

```
SRCFG00040: The config property quarkus.qhorus.agent-card.url is required but it could not be resolved from any config source
```

SmallRye Config interprets `""` (empty string) as equivalent to `null` for non-Optional fields. Since the field is not Optional and the "default" resolves to null, it fails the required-field validation.

## Context

Encountered when trying to make a URL config property optional with an empty default — the intent was "absent by default, set per deployment". Using `@WithDefault("")` seemed natural.

## Fix

Use `Optional<String>` without `@WithDefault`:

```java
// WRONG — "" treated as null, fails required-field validation at startup
@WithDefault("")
String url();

// RIGHT — genuinely optional, absent by default
Optional<String> url();
```

The `Optional<String>` approach correctly models "this value may not be present" and SmallRye Config handles it without a default annotation.

## Why this is undocumented

The SmallRye Config documentation explains `@WithDefault` as setting a default value, with no mention that `""` is a special case treated as null. The error message (`"required but could not be resolved"`) doesn't indicate that the problem is the empty-string default — it looks like a missing config key.
