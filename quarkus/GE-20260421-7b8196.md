## What to do

Declare a `@Singleton` static nested class inside a `@QuarkusTest` class. Quarkus's CDI bean discovery picks up nested static classes with CDI annotations. Use `@Observes` on a method to capture CDI events. Inject it back into the outer test class with `@Inject`.

```java
@QuarkusTest
class SessionIdleSchedulerTest {

    @Inject EventCaptor eventCaptor;

    @BeforeEach
    void setUp() {
        eventCaptor.events.clear();
    }

    @Test
    void expiryEventFiredForExpiredSession() {
        // ... trigger expiry ...
        assertEquals(1, eventCaptor.events.size());
    }

    @Singleton
    static class EventCaptor {
        final List<MyEvent> events = new ArrayList<>();
        void observe(@Observes MyEvent e) { events.add(e); }
    }
}
```

Note: CDI events fired via `Event.fire()` are synchronous — the observer runs on the same thread as the test, so no synchronisation needed. Use `CopyOnWriteArrayList` if the event could be fired from a different thread.

## Why this is non-obvious

The standard approach is a separate `@ApplicationScoped` or `@Singleton` class in the test source tree — two files, more boilerplate. Developers familiar with JUnit 5 extension patterns wouldn't expect CDI to scan nested static classes inside the test class itself. The Quarkus documentation doesn't call this out.
