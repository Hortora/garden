## Symptom

`@QuarkusTest` fails at boot with `AmbiguousResolutionException: Ambiguous dependencies for type X`. The production code compiles and the beans look correct.

## Context

Defining multiple `@ApplicationScoped` implementations of the same SPI interface (e.g. three `SkillProfileProvider` implementations) so users can choose which to activate. All three are unqualified, so CDI cannot resolve which one to inject at a single injection point.

## What was tried

Adding all three as `@ApplicationScoped` beans, assuming CDI would pick the first one or throw a clear build-time error.

## Root cause

CDI requires exactly one unambiguous bean per injection type when no qualifier is used. Multiple unqualified `@ApplicationScoped` beans implementing the same interface all compete ŌĆö CDI cannot resolve the injection point and throws at container startup, not at compile time.

## Fix

Designate exactly one implementation as the default (leave it unqualified `@ApplicationScoped`). Mark all others with `@Alternative`:

```java
// Default ŌĆö CDI injects this when no override is present
@ApplicationScoped
public class WorkerProfileSkillProfileProvider implements SkillProfileProvider { ... }

// Not active by default ŌĆö user activates via @Alternative @Priority(1)
@ApplicationScoped
@Alternative
public class CapabilitiesSkillProfileProvider implements SkillProfileProvider { ... }

@ApplicationScoped
@Alternative
public class ResolutionHistorySkillProfileProvider implements SkillProfileProvider { ... }
```

Users override the default at their own application level:

```java
@ApplicationScoped
@Alternative
@Priority(1)
public class MyCustomProvider implements SkillProfileProvider { ... }
```

## Why this is non-obvious

The error only appears at `@QuarkusTest` boot ŌĆö compile succeeds silently. The fix (marking non-default implementations `@Alternative`) is easy once known, but the CDI resolution rules around `@Alternative` vs unqualified beans are non-obvious. The natural instinct is to let the user pick via config, not via CDI qualifiers.
