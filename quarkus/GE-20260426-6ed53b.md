## Symptom

After packaging a Quarkus fat jar with `mvn package` and running with `-Dquarkus.profile=replay`, endpoints backed by `@IfBuildProfile("replay")` beans return 404. The profile appears active in logs ("Profile replay activated") but the beans do not exist.

## Context

Quarkus CDI augmentation runs at build time. `@IfBuildProfile("replay")` instructs the augmentation to include the annotated bean **only when the build-time profile matches**. If `mvn package` is run without `-Dquarkus.profile=replay`, those beans are excluded from the generated bytecode entirely.

Setting `-Dquarkus.profile=replay` at JVM launch (`java -Dquarkus.profile=replay -jar app.jar`) changes configuration resolution but cannot conjure beans that do not exist in the jar.

## What was tried

Running the jar with `-Dquarkus.profile=replay` as a JVM flag. The profile is activated (logs confirm), Quarkus reads profile-specific config correctly, but CDI beans annotated with `@IfBuildProfile("replay")` are absent.

## Root cause

Quarkus CDI augmentation is build-time. `@IfBuildProfile` / `@UnlessBuildProfile` are build-time annotations evaluated during `quarkus:build` / `mvn package`. They are not runtime switches.

## Fix

Package with the target profile:

```bash
mvn package -DskipTests -Dquarkus.profile=replay -q
```

Then run with:

```bash
java -Dquarkus.profile=replay -jar target/quarkus-app/quarkus-run.jar
```

## Why this is non-obvious

Spring Boot profiles are entirely runtime â€” you can build once and switch profiles via `--spring.profiles.active`. Quarkus profiles serve two overlapping roles: build-time CDI inclusion *and* runtime config resolution. The same `-Dquarkus.profile` flag controls both, masking the distinction. Logs show the profile as active, which suggests beans should be present.

### Why this fix

Building per-target-profile is the only option for `@IfBuildProfile`. Alternative: remove `@IfBuildProfile` and use runtime guards (`if (profile.equals("replay"))`) inside beans that are always included â€” but this loses the CDI exclusion benefit and complicates injection.
