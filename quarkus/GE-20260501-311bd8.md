## `quarkus.mcp.server.tools.page-size` controls tools/list pagination — undocumented default of 50 silently truncates large tool sets

### Stack
quarkus-mcp-server 1.11.1 (Quarkiverse)

### What it is
`quarkus.mcp.server.tools.page-size` is a runtime configuration property in quarkus-mcp-server that controls how many tools are returned per `tools/list` MCP response. It is defined in `McpServerRuntimeConfig.java` as a nested interface:

```java
public interface Tools {
    @WithDefault("50")
    int pageSize();
    // ...
}
```

### How to use it

**Disable pagination** (return all tools in one response):
```properties
quarkus.mcp.server.tools.page-size=0
```

**Set a custom page size:**
```properties
quarkus.mcp.server.tools.page-size=100
```

Setting to 0 or any negative number disables pagination entirely — all registered tools are returned in a single `tools/list` response regardless of count.

### Where it's not documented
As of quarkus-mcp-server 1.11.1, this property does not appear in the main Quarkiverse documentation for the extension. It is only discoverable by:
- Reading the `McpServerRuntimeConfig.java` source
- Noticing that `tools/list` returns fewer tools than expected (no warning is emitted)
- Decompiling the deployment JAR

### Discovery path
Found by: diagnosing silent tool truncation (49 Qhorus + 8 Claudony = 57 total tools, response showing exactly 50 alphabetically-sorted tools). Traced through `ToolMessageHandler` → `fetchPage()` → `serverConfig.tools().pageSize()` → `@WithDefault("50")` in `McpServerRuntimeConfig`.

### Related config
The same nested config likely controls pagination for other MCP feature types (prompts, resources) via similar `pageSize()` methods. The naming convention is `quarkus.mcp.server.<feature-type>.page-size`.
