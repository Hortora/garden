### Symptom
A code fix appears to have no effect even after confirming the source file is correct.
The running `mvn quarkus:dev` server still exhibits pre-fix behaviour.

### Context
Developer edits source, then runs `mvn test` to verify the fix. Tests pass. Developer
opens the browser against the already-running `mvn quarkus:dev` server expecting to see
the fix. Bug is still present.

### What was tried (didn't work)
- Verified source file contains the fix (it did)
- Confirmed `mvn test` compiled and passed (it did, at timestamp T+1)
- Assumed Quarkus hot reload had picked up the changes

### Root cause
`mvn quarkus:dev` compiles from source at startup time, not from the `target/classes/`
compiled by `mvn test`. If the server started at T and edits were saved at T+1,
the running server has the pre-edit classes regardless of what `mvn test` compiled.

Quarkus hot reload watches source files and triggers a soft restart on the next HTTP
request — but only if the server is already running when the file changes. If the
server started before the edits landed, it compiled the pre-edit sources and will not
automatically re-augment `@IfBuildProfile` beans.

### Diagnosis
Compare timestamps: source file mtime vs server startup time in the log:
```
# Source file modified at 14:56
ls -la src/main/java/.../EmulatedEngine.java

# Server started at 14:41 (pre-edit)
grep "started in" /tmp/quarkmind-emulated.log
```
If server start < source mtime → server is running pre-edit code.

### Fix
Kill and restart `mvn quarkus:dev` after making edits. A fresh start always compiles
from current sources:
```bash
pkill -f "quarkus:dev"
lsof -ti:8080 | xargs kill -9 2>/dev/null
mvn quarkus:dev -Dquarkus.profile=emulated
```

### Why this is non-obvious
`mvn test` passing creates a false sense that the fix is compiled and deployed.
The two compilation outputs (`mvn test` → `target/classes/`, `mvn quarkus:dev` → own
augmented classes) are independent. Hot reload works for edits made *while* the server
is running — not for edits made before the server started.
