## Symptom

Flyway migration fails at startup with:

```
org.h2.jdbc.JdbcSQLSyntaxErrorException: Syntax error in SQL statement
"CREATE INDEX idx_name ON table (column) [*]WHERE condition"
```

The same SQL works fine against PostgreSQL production.

## Context

PostgreSQL supports partial indexes — indexes with a WHERE clause that only index rows matching the predicate. H2 (used as the in-memory test database in Quarkus projects) does not support the WHERE clause on `CREATE INDEX`.

Common example: trying to create an index on an active-records-only filter:

```sql
CREATE INDEX idx_schedule_next_fire ON work_item_schedule (next_fire_at)
WHERE active = TRUE;  -- ❌ fails in H2
```

## Root cause

H2 does not implement the SQL standard extension for partial/filtered indexes. The `WHERE` clause on `CREATE INDEX` is a PostgreSQL (and SQL:2003) extension. H2 parses the statement and fails at the `WHERE` keyword with a syntax error.

## Fix

Two options:

**Option 1 (preferred for Flyway migrations):** Remove the partial index predicate and replace with a separate regular index on the boolean column:

```sql
-- ✅ Works in both H2 and PostgreSQL
CREATE INDEX idx_schedule_next_fire ON work_item_schedule (next_fire_at);
CREATE INDEX idx_schedule_active ON work_item_schedule (active);
```

The query planner in PostgreSQL will use both indexes when needed. Performance is slightly worse than a true partial index but correctness is maintained in all databases.

**Option 2 (H2-specific Flyway workaround):** Use Flyway's vendor-specific migration naming:
- `V1__create_table.sql` — shared
- `V1.1__postgres_partial_index.sql` — PostgreSQL only (name with `postgres` in the filename + configure `flyway.locations`)

## Why this is non-obvious

The SQL looks valid to any developer familiar with PostgreSQL. The `WHERE` clause on an index is a common optimisation. H2's error message says "Syntax error" at the `WHERE` keyword, which might send developers looking at Flyway syntax or the migration version number rather than at database compatibility.
