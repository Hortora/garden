## What to do

Add a `String name()` method to a strategy interface. All implementations are `@ApplicationScoped` beans. A registry bean takes `@Any Instance<T>` and builds `Map<String, T>` at startup — zero explicit registration, O(1) runtime lookup.

```java
public interface ExpiryPolicy {
    String name();
    boolean isExpired(Session session, Duration timeout);
}

@ApplicationScoped
public class UserInteractionExpiryPolicy implements ExpiryPolicy {
    @Override public String name() { return "user-interaction"; }
    // ...
}

@ApplicationScoped
public class ExpiryPolicyRegistry {
    private final Map<String, ExpiryPolicy> policies;
    private final String defaultName;

    @Inject
    ExpiryPolicyRegistry(@Any Instance<ExpiryPolicy> all, MyConfig config) {
        this.defaultName = config.defaultPolicy();
        this.policies = StreamSupport.stream(all.spliterator(), false)
                .collect(Collectors.toMap(ExpiryPolicy::name, p -> p));
        if (!policies.containsKey(defaultName)) {
            LOG.warnf("Configured default policy '%s' not found. Available: %s",
                    defaultName, policies.keySet());
        }
    }

    public ExpiryPolicy resolve(String name) {
        if (name != null && policies.containsKey(name)) return policies.get(name);
        var fallback = policies.get(defaultName);
        if (fallback != null) return fallback;
        if (policies.isEmpty()) throw new IllegalStateException("No ExpiryPolicy beans registered");
        LOG.warnf("Default policy '%s' not found; using first available", defaultName);
        return policies.values().iterator().next();
    }
}
```

New strategies: add a new `@ApplicationScoped` class — no registration code needed. Strategy is available at next boot.

## Why this is non-obvious

The standard approach is `@Named("user-interaction")` + `Instance.select(new NamedLiteral("user-interaction"))` — which requires a `NamedLiteral` helper class and CDI `@Named` semantics that don't map cleanly to runtime string dispatch. Or a manual registration map where each implementation registers itself. The `@Any Instance<T>` + `name()` approach is cleaner: the map is built once, lookup is O(1), and startup validation catches misconfiguration immediately rather than at first call.
