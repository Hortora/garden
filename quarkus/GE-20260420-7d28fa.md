## Symptom

At runtime, calling `repo.listAll()` (or other Panache repository methods) throws:

```
java.lang.IllegalStateException: This method is normally automatically overridden in subclasses:
did you forget to annotate your entity with @Entity?
    at io.quarkus.hibernate.orm.panache.common.runtime.AbstractJpaOperations.implementationInjectionMissing
    at io.quarkus.hibernate.orm.panache.PanacheRepositoryBase.listAll
```

The entity does have `@Entity`. The error message is completely misleading.

## Context

Occurs when a repository class implements `PanacheRepository<Entity, Id>` (or `PanacheRepositoryBase<Entity, Id>`) but the entity is a plain JPA `@Entity` POJO that does **not** extend `PanacheEntityBase` or `PanacheEntity`.

## Root Cause

Quarkus Panache's bytecode enhancement for repository methods (the automatic method override mentioned in the error) is only applied to repositories whose entity type went through Panache's own build-time processor. That processor only recognises entities that extend `PanacheEntityBase`. A plain `@Entity` is never processed by Panache's build step, so the repository methods are never enhanced — the stubs remain and throw at runtime.

## Fix

Do **not** use `PanacheRepository<PlainEntity, Id>` when the entity is a plain JPA POJO. Instead, inject `EntityManager` directly and use JPQL:

```java
@ApplicationScoped
@Alternative
public class JpaLedgerEntryRepository implements LedgerEntryRepository {

    @Inject
    EntityManager em;

    @Override
    public List<LedgerEntry> findBySubjectId(UUID subjectId) {
        return em.createQuery(
            "SELECT e FROM LedgerEntry e WHERE e.subjectId = :id ORDER BY e.sequenceNumber ASC",
            LedgerEntry.class)
            .setParameter("id", subjectId)
            .getResultList();
    }
}
```

## Why This Is Non-Obvious

The Quarkus docs describe `PanacheRepository<Entity, Id>` as a "repository pattern" suitable for any JPA entity — distinct from the active-record pattern that requires `PanacheEntityBase`. The docs don't mention that the entity must still be processed by Panache's build step. The error message says "did you forget to annotate with @Entity" which actively misleads — the entity has `@Entity` and that's not the problem.
