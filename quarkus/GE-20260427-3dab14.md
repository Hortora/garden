## Symptom

After writing a new `.java` file (e.g. via a code generation tool or IDE), running `mvn test -Dno-format` immediately fails with:

```
[ERROR] Failed to execute goal net.revelc.code.formatter:formatter-maven-plugin:2.29.0:validate
on project quarkus-qhorus: File '/path/to/NewFile.java' has not been previously formatted.
Please format file and commit before running validation!
```

or:

```
[ERROR] Failed to execute goal net.revelc.code.formatter:impsort-maven-plugin:1.13.0:check
on project quarkus-qhorus: Imports are not sorted in /path/to/NewFile.java
```

## Root cause

The Quarkiverse parent POM enforces two validation plugins: `formatter-maven-plugin` (checks code style) and `impsort-maven-plugin` (checks import ordering). These run during the `validate` lifecycle phase, before compilation. Externally written files that don't match the Quarkiverse style or import order fail immediately.

The `-Dno-format` flag skips the formatter from *applying* changes but does NOT skip the *validation* check. Files must be pre-formatted before validation runs.

## Fix

Run `mvn process-sources` (without `-Dno-format`) after writing any new file and before running tests. This applies the formatter and sorts imports:

```bash
JAVA_HOME=$(/usr/libexec/java_home -v 26) mvn process-sources -pl runtime -q
# Then run tests normally:
JAVA_HOME=$(/usr/libexec/java_home -v 26) mvn test -pl runtime -Dno-format -q
```

## Why this is non-obvious

`-Dno-format` sounds like it disables all formatting checks. It only disables the *apply* phase. The *validate* phase still runs. Developers assume `-Dno-format` means "don't touch my files" but it actually means "don't reformat, but do still check". The error message names the formatter plugin but not the `-Dno-format` interaction.
