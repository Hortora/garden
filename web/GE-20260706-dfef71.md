### Symptom
Column headers do not align with body column contents. The last header column appears shifted right, and the column picker button occupies its own column-width space in the header row.

### Context
A data table renders header and body rows with identical grid-template-columns values. A column picker trigger button is placed inside the header grid as a direct child alongside the columnheader cells.

### What was tried (didn't work)
Using display:inline-block on the picker wrapper — CSS Grid ignores display on direct children and assigns them a grid track regardless.

### Root cause
Every direct child of a CSS Grid container becomes a grid item and receives its own grid track. The picker button becomes an extra column in the header grid that has no corresponding column in body rows, causing misalignment.

### Fix
Wrap the header grid and the picker in a flex container. The grid takes flex:1 (full data width), the picker is positioned absolutely relative to the flex container:

```css
.header-container { position: relative; }
.header { display: grid; /* takes full width */ }
.column-picker-wrapper { position: absolute; right: 8px; top: 50%; transform: translateY(-50%); }
```

This way the grid occupies the same width as body rows, and the picker floats over the right edge.

### Why this is non-obvious
The grid-template-columns values are identical between header and body — inspecting them shows no difference. The misalignment comes from the header having one more grid item (the picker) than the body, which shifts all tracks. The symptom looks like a template mismatch but is actually a child-count mismatch.
