### Symptom
Column renderers defined in a parent component (similarity-panel) return HTML
with CSS classes (`.bar-fill`, `.outcome-badge`). The rendered cells show raw
text values instead of styled progress bars and badges. Tests in jsdom pass.

### Context
Web component composition where a parent component passes render callbacks to
a child component (pages-table). The callback returns `html` template results
using CSS classes defined in the parent's `static styles`. The child component
renders the callback output inside its own shadow DOM.

### What was tried (didn't work)
- Checked property binding timing (`.columnRenderers` set before/after `dataSet`)
- Checked `ColumnId` type identity (branded string — Map lookup should match)
- Checked `_renderCell` logic in pages-table (renderer lookup is correct)
- Compared with work-item-inbox (identical pattern — same bug, unnoticed)
- Verified tests pass in jsdom (elements exist, styles don't render)

### Root cause
Shadow DOM CSS scoping. The parent's `static styles` stylesheet is scoped to
the parent's shadow root. The callback output HTML lands in the child's shadow
root, where those CSS class definitions don't exist. The HTML elements ARE
rendered (test queries find them), but they have no styling — visually
indistinguishable from raw text.

### Fix
Renderer output must use inline styles instead of CSS classes. Inline styles
travel with the element regardless of which shadow root it lives in.

```typescript
// BROKEN — class defined in parent's stylesheet, invisible in child's shadow DOM
return html`<span class="outcome-badge ${cls}">${value}</span>`;

// FIXED — inline styles are self-contained
return html`<span style="display: inline-block; padding: 4px 8px; border-radius: 4px; font-size: 12px; ${colors}">${value}</span>`;
```

Remove the now-dead CSS class definitions from the parent's `static styles`.

### Why this is non-obvious
The symptom — "cells show raw text instead of styled content" — points toward
renderer execution failure (property binding, timing, type mismatch). The
actual cause is purely visual (CSS scoping). jsdom tests compound the
misdirection because element existence checks pass. A developer would exhaust
all renderer-execution hypotheses before considering CSS scoping.

### Alternatives considered
- `::part()` — requires pages-table to add `part` attributes to cells; couples the child to parent styling expectations
- `adoptedStyleSheets` injection — parent injects its stylesheet into child's shadow root; fragile and breaks encapsulation
- Constructable stylesheets — Lit has no built-in cross-shadow style mechanism
- Inline styles (chosen) — renderer output is self-contained; no coupling between components
