### Symptom

Need incremental cross-package type checking in a monorepo but `tsc --build --noEmit` fails with TS6310 "Referenced project may not disable emit."

### Context

A TypeScript monorepo with 12 packages using `tsc --noEmit` per package for type checking. Per-package checking rebuilds everything from scratch (~6.6s). Project references with `tsc --build` would give incremental caching, but `--noEmit` is incompatible with references.

### Fix

Set `emitDeclarationOnly: true` with a gitignored `.typecheck/` outDir in each package's typecheck tsconfig. This emits only `.d.ts` files (needed for cross-package type resolution) without JS output:

```json
// tsconfig.json (typecheck)
{
  "extends": "@my/tsconfig-base",
  "compilerOptions": {
    "composite": true,
    "rootDir": "src",
    "outDir": ".typecheck",
    "emitDeclarationOnly": true
  },
  "include": ["src"],
  "references": [{ "path": "../other-package" }]
}

// tsconfig.build.json (production build)
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    "composite": false,
    "outDir": "dist",
    "emitDeclarationOnly": false
  },
  "exclude": ["**/*.test.ts"]
}
```

Root tsconfig.json:
```json
{
  "files": [],
  "references": [
    { "path": "packages/package-a" },
    { "path": "packages/package-b" }
  ]
}
```

Root typecheck: `tsc --build` (not `--noEmit`).

Result: second-run typecheck drops from ~6.6s to ~0.8s via `.tsbuildinfo` caching.

### Why this is non-obvious

The natural approach is `tsc --build --noEmit`, which fails because project references need declaration files for downstream type resolution. The workaround of emitting only declarations to a cache directory isn't documented in TypeScript's project references guide. Most guides show `tsc --build` for production builds, not for type checking.
