### Symptom
Rows of duplicate charts appear below the tab area in multi-page dashboards. The Kitchensink showed all Displayer charts twice — once inside the Displayers tab, once below the tabs at the page root.

### Context
DashBuilder dashboards define pages in a `pages:` array and organise them via `navTree:` groups. Navigation components (TABS, TREE, MENU, CAROUSEL) reference groups via `navGroupId`. The GWT version uses a Router that shows only one page at a time. The casehub runtime renders all top-level pages simultaneously.

### What was tried (didn't work)
1. `collectNavTreePageNames` — extracted page names from navTree groups and filtered them from the top-level content slot. This fixed pages explicitly IN the navTree but missed orphaned pages.
2. `containsNavGroupRef` — scanned pages for navigation components with `navGroupId` referencing a navTree group. This caught the Carousel but missed pages with no navGroupId at all (Forms, Map).

### Root cause
Pages can exist in the YAML without being in any navTree group — they're "orphaned" from navigation. The Kitchensink's Carousel page was commented out of the Navigation navTree group but still had `type: CAROUSEL, navGroupId: Displayers`. Forms and Map pages were never in any group. All three rendered at the top level, with Carousel duplicating every Displayer chart.

### Fix
When a navTree exists, only the first page (root/entry point) is top-level. All other pages are accessed via navigation — matching GWT Router behaviour.

```typescript
if (navTreePageNames.size > 0) {
  topLevelPages = patchedPages.length > 0 ? [patchedPages[0]!] : [];
}
```

### Why this is non-obvious
The initial fix (#32) correctly identified that navTree-embedded pages should be filtered from the top level. The symptom was identical (duplicate charts), but the root cause was different — orphaned pages outside the navTree, not pages inside it. The fix appeared complete because the specific pages it targeted (Bar Chart, Line Chart, etc.) were correctly filtered. The Carousel was a different category of duplication.

### Alternatives considered
- Filter only pages with `navGroupId` referencing navTree groups (misses pages without navGroupId like Forms/Map)
- Filter all non-first pages when navTree exists (correct — matches GWT Router model)
