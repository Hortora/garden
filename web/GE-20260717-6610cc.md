### Symptom
Emoji picker popover overflows the viewport on narrow screens. The component
has a `.flip` CSS class with correct `bottom: auto; top: 100%` rules that
should open the picker downward when insufficient space above.

### Context
A web component (`channel-reaction-bar`) defines CSS positioning variants as
classes (`.picker-popover.flip`) but no code path adds the class based on
viewport measurements. The component opens the picker with `left: 0` always
and `bottom: 100%` always — no `getBoundingClientRect()` check, no class toggle.

### What was tried (didn't work)
Investigation initially targets the CSS layer — looking for missing rules,
incorrect `position` values, or missing `overflow` handling. The CSS is correct.
The rules exist. They just never activate.

### Root cause
The CSS fallback class is defined but no JavaScript code computes viewport
position or toggles the class. The `.flip` block was written as part of the
initial CSS (anticipating the need) but the corresponding JS was never
implemented. Dead CSS that looks like a working fallback.

### Fix
Add viewport-aware positioning logic that runs before the popover opens:

```typescript
private _computePickerPosition() {
  const container = this.shadowRoot?.querySelector('.picker-container');
  if (!container) return;
  const rect = container.getBoundingClientRect();
  this._flipVertical = rect.top < PICKER_HEIGHT;
  this._flipHorizontal = rect.left + PICKER_WIDTH > window.innerWidth;
}
```

Apply classes in the template based on the computed state. Key insight: measure
the always-present container element, not the conditionally-rendered picker.

### Why this is non-obvious
The symptom — "popover overflows viewport" — suggests the CSS positioning is
wrong or missing. Reading the stylesheet shows a `.flip` class with the correct
rules, which appears to confirm the CSS is handled. The actual bug is in a
different layer entirely (JS never toggles the class). A developer can read the
CSS, conclude "flipping is implemented", and look elsewhere for the overflow cause.
