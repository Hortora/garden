## Problem

Using `Object.assign` to set properties on Three.js objects (lights, cameras, meshes) at construction time throws a TypeError silently, aborting the init function with no rendered output.

## Symptom

Blank canvas — no terrain, no units, no error in the console unless DevTools is open. Entire `init()` function aborts mid-execution.

## Root cause

Three.js vector/matrix properties (`.position`, `.rotation`, `.scale`, `.quaternion`) are getter-only on the class prototype in r128 — they return `Vector3`/`Quaternion` instances that are set up in the constructor. `Object.assign` tries to write a new `Vector3` object to the property name, which throws "Cannot assign to read only property 'position'".

```javascript
// Breaks silently in r128:
scene.add(Object.assign(new THREE.DirectionalLight(0xaabbff, 1.3), {
  position: new THREE.Vector3(20, 40, 20)  // ← TypeError
}));
```

## Fix

Always use `.set()` on vector properties:

```javascript
const sun = new THREE.DirectionalLight(0xaabbff, 1.3);
sun.position.set(20, 40, 20);
scene.add(sun);
```

## Why this is non-obvious

`Object.assign` is a standard JS pattern for constructor shorthand. Three.js objects look like plain objects; nothing about the API surface signals that position is a getter. The TypeError aborts the entire async `init()` with no visual feedback — just a blank canvas.
