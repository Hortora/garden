### Symptom

Global keyboard shortcuts (e.g. `?` for help overlay, `n`/`p` for navigation) fire while the user is typing in an `<input>` inside a Shadow DOM Web Component. The `isEditable()` guard in the keydown handler returns false because `e.target` is the shadow host element (e.g. `<drafthouse-diff>`), not the focused inner `<input>`.

### Context

Any application using Shadow DOM Web Components with global keyboard shortcut handlers. The DOM spec says composed events retarget to the shadow host when they cross the shadow boundary — `e.target` is the host, not the originating element inside the shadow tree.

### Root cause

Keyboard events are composed events. When a `keydown` fires on an `<input>` inside a shadow tree, the event crosses the shadow boundary and is retargeted — `e.target` becomes the shadow host element. A handler checking `e.target.tagName === 'INPUT'` sees the host's tag (e.g. `DRAFTHOUSE-DIFF`), not `INPUT`.

### Fix

```javascript
function isEditable(el) {
  const tag = el.tagName;
  if (tag === 'INPUT' || tag === 'TEXTAREA' || tag === 'SELECT' || el.isContentEditable) return true;
  let active = document.activeElement;
  while (active && active.shadowRoot && active.shadowRoot.activeElement) {
    active = active.shadowRoot.activeElement;
  }
  if (active && active !== el) {
    const t = active.tagName;
    if (t === 'INPUT' || t === 'TEXTAREA' || t === 'SELECT' || active.isContentEditable) return true;
  }
  return false;
}
```

### Why this is non-obvious

The standard keyboard guard pattern (`if (e.target.tagName === 'INPUT') return`) works universally in non-Shadow-DOM apps. Developers don't expect it to break when they wrap their UI in Web Components. The failure is silent — no error, no warning, just shortcuts firing at the wrong time.

### Alternatives considered

- Checking `e.composedPath()[0]` — returns the originating element but includes non-element nodes and is less readable than the activeElement walk
- Stopping propagation inside the shadow tree — breaks the global handler pattern and requires every Web Component to know about every shortcut
