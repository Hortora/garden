### Symptom
`tsc -p tsconfig.build.json` exits with code 0 but produces no `.js` files in `outDir`.
Only `.d.ts.map` files appear (or nothing at all). No error, no warning.

### Context
A composite TypeScript project (`composite: true`) with two tsconfig files:
- `tsconfig.json` — `emitDeclarationOnly: true` (for IDE type checking)
- `tsconfig.build.json` — extends `tsconfig.json`, overrides `emitDeclarationOnly: false`

A previous `tsc -p tsconfig.json` run (or IDE background build) created a
`.tsbuildinfo` file recording that all sources were up to date with declaration-only
emission. When `tsc -p tsconfig.build.json` runs next, it reads the stale
`.tsbuildinfo`, sees all sources unchanged, and skips emission entirely — even though
the emission mode changed from declaration-only to full.

### Root cause
`tsc` with `composite: true` uses `.tsbuildinfo` for incremental compilation. The
build info records what was emitted and when. If the `.tsbuildinfo` was created by a
declaration-only run, it records that declarations were emitted. A subsequent full-emit
run reads the same `.tsbuildinfo`, sees all inputs unchanged, and concludes nothing
needs to be done — it does not detect that the emission mode changed.

### Fix
Delete all `.tsbuildinfo` files before building with a different emission config:
```bash
rm -f tsconfig.tsbuildinfo tsconfig.build.tsbuildinfo
tsc -p tsconfig.build.json
```

Or use a dedicated `outDir` per tsconfig to avoid `.tsbuildinfo` collisions.

### Why this is non-obvious
- Exit code 0 — no signal that anything went wrong
- No warning about skipped emission
- The config resolves correctly (`--showConfig` confirms `emitDeclarationOnly: false`)
- The issue only manifests when switching between tsconfig files with different
  emission modes on the same source tree
- Most developers don't know `.tsbuildinfo` records emission mode, not just source timestamps

### Why this fix
Deleting `.tsbuildinfo` forces a full rebuild from scratch. A dedicated `outDir` per
tsconfig avoids the problem entirely but requires restructuring the build.

### Alternatives considered
- Checking `--showConfig` output — confirms the config is correct but doesn't explain
  why emission is skipped
- Adding `--force` flag — tsc has no `--force`; `--clean` is only available in
  `--build` mode
- Using `tsc --build --clean` — works but requires project references setup
