### Symptom

Runtime error: `Cannot read properties of undefined (reading 'detail')` when handling a `pages-event` via `onPagesEvent`. The error appears to mean the event wasn't dispatched, but the event fires correctly — the problem is in how the callback interprets its argument.

### Context

`pages-component` provides two complementary APIs for the `pages-event` system:

- `emitPagesEvent(target, topic, payload)` — dispatches a `CustomEvent` with `detail: { topic, payload }`
- `onPagesEvent(target, topic, handler)` — listens for `pages-event`, filters by topic, and calls `handler(detail.payload)` — passing the **payload directly**, not the event

The trap: if a component dispatches raw `CustomEvent` with `detail: { topic, data: {...} }` instead of using `emitPagesEvent`, the field is named `data` not `payload`. The `onPagesEvent` handler receives `undefined` because `detail.payload` doesn't exist.

### What was tried (didn't work)

1. Typed the callback as `(e: CustomEvent) => { e.detail.data... }` — fails because the callback argument is the payload, not the event
2. Used `e.detail.data ?? e.detail` as a fallback — fails because `e` has no `.detail` property (it's already the payload)

### Root cause

`onPagesEvent` unwraps the event internally:
```typescript
const detail = (e as CustomEvent<PagesEventDetail<T>>).detail;
if (matchesTopic(topicOrPattern, detail.topic)) {
  handler(detail.payload);  // ← passes payload, not event
}
```

Components using raw `CustomEvent` with `detail.data` instead of `detail.payload` produce events that `onPagesEvent` can filter by topic but delivers `undefined` as the payload.

### Fix

Two changes needed:

1. **Emitting component:** Use `emitPagesEvent` instead of raw `CustomEvent`:
```typescript
// Wrong — uses 'data' field, onPagesEvent reads 'payload'
this.dispatchEvent(new CustomEvent('pages-event', {
  bubbles: true, composed: true,
  detail: { topic: 'my:event', data: { key: 'value' } },
}));

// Right — emitPagesEvent uses 'payload' field
emitPagesEvent(this, 'my:event', { key: 'value' });
```

2. **Receiving component:** Type the `onPagesEvent` callback to receive payload directly:
```typescript
// Wrong — treats argument as CustomEvent
onPagesEvent(document, 'my:event', (e: CustomEvent) => {
  const data = e.detail.data;  // TypeError: e.detail is undefined
});

// Right — argument IS the payload
onPagesEvent<{ key: string }>(document, 'my:event', (payload) => {
  const data = payload.key;  // works
});
```

### Why this is non-obvious

The error message (`Cannot read properties of undefined`) points at the handler line, suggesting the event wasn't dispatched. The event IS dispatched and the topic filter matches — the problem is the field name mismatch (`data` vs `payload`) one level deeper. Components written before `emitPagesEvent` existed use the wrong field name and appear to work until consumed via `onPagesEvent`.
