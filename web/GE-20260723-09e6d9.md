### Symptom
Tests for a toolbar removal (casehub-pages #199) all passed — kebab zone inside header-container, filter bar input present — but the toolbar was still visually rendered as a dedicated row above the column headers.

### Context
Lit Web Component (`pages-table`) with shadow DOM. The refactor was supposed to eliminate a `.toolbar` div and reposition its contents. Tests used `querySelector` to verify new elements existed in the correct parent containers.

### What was tried (didn't work)
- `expect(headerContainer.querySelector('.kebab-zone')).not.toBeNull()` — passed, but kebab was still in a toolbar row
- `expect(shadowRoot.querySelector('.filter-bar-input')).not.toBeNull()` — passed, but filter was still always visible
- All structure tests green; commit message claimed "eliminates toolbar row"

### Root cause
Tests checked **presence** of new structure but never checked **absence** of old structure. The `.toolbar` div was still in the template and the CSS still rendered it as a row. DOM structure tests prove the template rendered — they don't prove the CSS positioned it correctly.

### Fix
Always include negative assertions for removed structure alongside positive assertions for new structure:

```typescript
// Positive: new structure exists
const headerContainer = el.shadowRoot!.querySelector('.header-container');
expect(headerContainer!.querySelector('.column-picker-trigger')).not.toBeNull();

// Negative: old structure is gone
const toolbar = el.shadowRoot!.querySelector('.toolbar');
expect(toolbar).toBeNull();
```

For layout changes, test both:
1. The new element exists in the right parent
2. The old element/container no longer exists at all

### Why this is non-obvious
DOM structure tests feel complete — "element X is inside container Y" reads as proof of correct layout. But in Web Components with shadow DOM, CSS can render structurally-correct DOM in any visual arrangement. The test proved the template, not the layout. A `.kebab-zone` can exist inside `.header-container` while CSS still positions it as a full-width row above the headers.
