## Problem

Fog overlay planes (`PlaneGeometry`, `renderOrder=5`) visually render through `THREE.Sprite` units when the camera is near-horizontal. No error is thrown — sprites simply disappear under the fog overlay.

## Symptom

At steep or near-horizontal camera angles, fog of war covers units that should be visible. The symptom looks like a renderOrder or z-fighting issue, not a depth buffer issue.

## Root cause

`THREE.SpriteMaterial` defaults to `depthWrite: false` (unlike `MeshLambertMaterial` and most other materials which default to `depthWrite: true`). When sprites do not write to the depth buffer, fog planes rendered after them (higher `renderOrder`) have no depth values to test against — they always pass and draw on top.

## Fix

```javascript
new THREE.SpriteMaterial({
  map: texture,
  transparent: true,
  depthWrite: true,   // must be explicit — default is false
  alphaTest: 0.1      // skip depth write for fully transparent border pixels
});
```

The `alphaTest` prevents transparent sprite borders from writing to depth, which would cause z-fighting with terrain tiles below.

## Why this is non-obvious

The default `depthWrite: false` on `SpriteMaterial` is intentional (sprites are typically UI overlays), but it's surprising when you have 3D sprites that need to occlude geometry rendered after them. The symptom (fog through sprites) looks like a renderOrder problem, not a depth buffer problem — leading to the wrong diagnosis.
