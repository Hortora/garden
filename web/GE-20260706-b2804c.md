### Symptom
Component renders but shows no data. Zero errors in console. No network requests visible.

### Context
A Lit Web Component has an `endpoint` string property used to construct fetch URLs. The examples app passes `endpoint=""` (empty string) because a mock-fetch interceptor handles requests on the same origin. The component guards its data loading with `if (!this.endpoint) return;`.

### What was tried (didn't work)
- Checked if the component was rendering at all (it was — the shell was visible)
- Checked if the mock-fetch was installed (it was — other components loaded data)
- Checked if the component was in the DOM (it was — visible in shadow root)

### Root cause
Empty string `""` is falsy in JavaScript. `!this.endpoint` evaluates to `true` when endpoint is `""`, causing the fetch guard to bail before making any request. The existing inbox component used `this.endpoint == null` (loose equality — `null` and `undefined` are falsy, but `""` is not), which correctly allows empty-string endpoints.

### Fix
Use `== null` (loose equality) instead of `!` (strict falsy) for optional string properties that may legitimately be empty:

```typescript
// WRONG — rejects empty string
if (!this.endpoint) return;

// RIGHT — only rejects null and undefined
if (this.endpoint == null) return;
```

### Why this is non-obvious
The symptom (no data, no errors) gives zero signal about the root cause. The component renders its template, the mock-fetch is installed, and the DOM structure is correct. The guard clause silently returns before any fetch, so there's no network request to debug. The fix is a one-character change (`!` → `== null`), but the debugging path requires tracing through connectedCallback → loadQueues → guard clause.

### Alternatives considered
- Using `=== undefined` — too narrow, doesn't catch null
- Using `typeof this.endpoint !== 'string'` — overly specific, doesn't match the existing codebase pattern
- Using `this.endpoint?.length > 0` — semantically wrong, empty string is a valid endpoint for mock-fetch
