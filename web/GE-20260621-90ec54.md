### Symptom
Clicking Next/Previous in a form view does nothing — the form stays on the same record with no error.

### Context
A form bound to a dataset via `dataScope` with `idColumn: Product`. The dataset has multiple rows with the same Product value (e.g., "Laptops" appears 3 times with different Regions).

### What was tried (didn't work)
The navigate handler correctly moved to the next row and updated the filter to the new row's idColumn value. But since the next row had the same Product value ("Laptops"), the filter `Product = "Laptops"` still matched the first Laptops row.

### Root cause
The navigate handler filters by `idColumn` value to select a record. When `idColumn` is not unique (e.g., Product has duplicates), the filter matches multiple rows and the data pipeline returns the first match — which is often the same row the user was already on.

### Fix
Use a truly unique column as `idColumn`. Add a synthetic ID column if the dataset doesn't have one:
```yaml
dataScope:
  dataset: sales
  idColumn: id    # must be unique per row
```

### Why this is non-obvious
No error is thrown. The handler runs, the filter updates, the data re-pushes — everything succeeds. The form just doesn't visually change because the filter still matches the same first row.
