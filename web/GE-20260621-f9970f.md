### Symptom

`tsc --noEmit` works fine on a composite project. `tsc --build --noEmit` on the same project fails with TS6310 "Referenced project may not disable emit."

### Context

A TypeScript monorepo with `composite: true` in all packages and `references` arrays linking dependencies. Per-package `tsc --noEmit` typechecks correctly. Root-level `tsc --build --noEmit` fails because `--build` mode requires declaration files from referenced projects for cross-package type resolution.

### Root cause

`tsc` and `tsc --build` have different emit requirements:
- `tsc --noEmit`: command-line `--noEmit` overrides the config. Type checking runs normally. `composite: true` is accepted — it means "this project *can* be referenced" but doesn't enforce emit.
- `tsc --build --noEmit`: `--build` mode walks the reference graph. Each referenced project must have emitted `.d.ts` files for downstream projects to resolve types. `--noEmit` prevents this, creating a contradiction.

### Fix

Use `emitDeclarationOnly: true` instead of `--noEmit` for `tsc --build` workflows. See GE-20260621-d5e7d4 for the full pattern.

### Why this is non-obvious

The TypeScript documentation mentions `--noEmit` and `--build` separately but does not clearly document their interaction with composite projects. A developer adding project references to an existing `tsc --noEmit` workflow will hit this error with no guidance on the correct alternative.
