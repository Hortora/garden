### Symptom
Navigating between pages in an SPA examples app causes API calls to return
HTML 404 pages instead of JSON. The error manifests as:
```
Failed to execute 'json' on 'Response': Unexpected token '<', "<!DOCTYPE "...
```

### Context
Each example page component sets `globalThis.fetch = this._mockFetch` in its
`connectedCallback` to provide page-specific mock data. When navigating to a
different page, the new page's mock replaces the previous one. The old page's
mock only handles its own endpoints — all other API calls fall through to the
real fetch (which hits the Vite dev server and gets HTML).

### Root cause
`globalThis.fetch` is a single global. Per-component overrides in
`connectedCallback` create a last-write-wins race. Components don't restore
the original fetch in `disconnectedCallback`, so navigation permanently
installs the last-visited page's mock.

### Fix
Centralise all mock routing in a single fetch interceptor installed once at
app startup. Each page's endpoints are routes in the central router, not
per-component overrides:

```typescript
// mock-fetch.ts — installed once in app bootstrap
const realFetch = window.fetch.bind(window);

export function installMockFetch(state: MockState): void {
  window.fetch = async (input, init) => {
    const mock = resolveMock(url, method, state);
    if (mock) return mock;
    return realFetch(input, init);
  };
}

function resolveMock(url, method, state) {
  // All pages' endpoints handled here
  if (path.match(/\/cases\/.*\/events$/)) return json(caseEvents);
  if (path.match(/\/ledger\/entries/)) return json(entries);
  if (path.match(/\/trust\//)) return json(trustData);
  // ...workitem routes...
  return null;
}
```

### Why this is non-obvious
Setting `globalThis.fetch` in `connectedCallback` appears to work in isolation —
each page loads its own data correctly. The bug only manifests when navigating
between pages, and the error message ("unexpected token '<'") points at JSON
parsing, not at the fetch mock itself.
