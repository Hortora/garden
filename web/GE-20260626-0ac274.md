### Symptom

An inline dataset with `accumulate: true`, `expression`, and `refresh` on the component renders the initial content correctly but never generates new rows. The chart shows static data — no live updates despite the expression being designed to produce new rows on each refresh cycle.

### Context

The Accumulate Flag example (`examples/dashboards/Basic Usage/Accumulate Flag.ts`) uses this exact pattern:

```typescript
inlineDataset("products", productsData, {
  accumulate: true,
  cacheMaxRows: 20,
  expression: '[["test", $now() ~> $toMillis(), ($random() * 100)]]'
});
```

The expression is a JSONata generator that should produce a new row with a timestamp and random value. Combined with `refresh: { interval: 1 }` on the component, this should simulate live streaming data.

### Root cause

Three independent mechanisms conspire to prevent expression evaluation:

1. **Schema validation rejects `content + accumulate`** — `schema.ts:53` has a Zod `.refine()`: `"accumulate is only valid when url is set"`. Similarly for `refreshTime` at line 56. However, the resolver (`resolver.ts:32-55`) does NOT apply Zod validation — it only checks sourceCount. So the combination doesn't crash; it just doesn't work.

2. **Extraction skips expression for `content + accumulate`** — `extraction.ts:214`:
   ```typescript
   if (def.expression !== undefined && !(def.content !== undefined && def.accumulate)) {
   ```
   The comment says "expression is a refresh generator, not a transform. The content provides the seed; expression generates on refresh." But no refresh path evaluates the expression.

3. **Component refresh doesn't re-resolve** — `data-pipeline.ts:187-193`: when the data manager already has the dataset (after first load), `handleDataRequest` just calls `pushData` with cached data. `scheduleRefresh` only fires if `refreshTime` is set (rejected by schema for inline data).

### Fix

None known. Filed as [#36](https://github.com/casehubio/casehub-pages/issues/36).

The intended fix would need to:
- Allow the schema to accept `content + accumulate + expression`
- Make the extraction or refresh path evaluate the expression (without the content) on subsequent resolution cycles
- Either use `refreshTime` on the dataset or have the component refresh trigger dataset re-resolution

### Why this is non-obvious

The code comment in `extraction.ts` explicitly describes the intended behavior ("expression generates on refresh"), making it appear implemented. The Accumulate Flag example exists in the gallery and renders without error — the initial data displays fine. The live-update failure is silent; no error, no warning, no console output.
