## Symptom

A file added to `.gitignore` like:

```
application.properties
```

Silently excludes `src/test/resources/application.properties` in every Maven module. `git status` doesn't show the file; `git add` says it's ignored. No error — the file simply isn't tracked.

## Context

Adding a root-level stray file (e.g. `application.properties` accidentally generated by a Maven plugin run from the project root) to `.gitignore` to prevent it being committed. The intention is to ignore only the root-level file.

## Root Cause

A bare pattern (no leading `/`) in `.gitignore` matches that filename **anywhere** in the repository tree — including `module-a/src/test/resources/application.properties`, `module-b/src/test/resources/application.properties`, etc.

Git pattern matching: a pattern without a `/` applies recursively to all directories.

## Fix

Add a leading `/` to match only the root-level file:

```gitignore
# Wrong — matches application.properties anywhere in the tree
application.properties

# Correct — matches only /application.properties at repository root
/application.properties
```

To verify before committing:
```bash
git check-ignore -v src/test/resources/application.properties
# If it prints a match → your pattern is too broad
```

## Why This Is Non-Obvious

The `.gitignore` entry looks correct at a glance. The failure is silent — `git status` doesn't show the file and `git add` silently does nothing. You only discover it when the test configuration is missing from the repository. The fix is a single `/` character.
