## Symptom

A workflow is manually triggered via `workflow_dispatch`. The build step passes and the workflow shows green. But the artifact is never published to the package registry — the deployment step was skipped with no warning.

## Context

Publish workflow has:
```yaml
on:
  push:
    branches: [main]
  workflow_dispatch:

steps:
  - name: Build and test
    run: mvn --batch-mode verify

  - name: Publish to GitHub Packages
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    run: mvn --batch-mode deploy -DskipTests
```

Triggering via workflow_dispatch → build passes → deploy step is skipped because `github.event_name` is `workflow_dispatch`, not `push`. Workflow exits green. Nothing published.

## Root cause

`if: github.event_name == 'push'` is false when triggered via `workflow_dispatch`. Skipped steps don't cause workflow failure — they silently succeed. The green tick gives no indication that the deploy was skipped.

## Fix

Use the negative guard instead — publish on everything except PRs:

```yaml
- name: Publish to GitHub Packages
  if: github.event_name != 'pull_request'
  run: mvn --batch-mode deploy -DskipTests
```

This covers `push`, `workflow_dispatch`, and `repository_dispatch` without needing to enumerate them.

## Why this is non-obvious

The workflow shows green. The build step ran. The developer assumes publishing happened. The only way to discover the skip is to check the step list and notice the deploy step has a grey (skipped) icon rather than green — easy to miss on a passing workflow.
