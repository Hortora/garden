## GitHub stacked PRs from a fork: base branch must exist on the upstream repo, not the fork

**ID:** GE-20260416-eb035d
**Stack:** GitHub, git
**Symptom:** `gh pr create --repo upstream/repo --base fork-owner:branch-name --head fork-owner:head-branch` fails with `GraphQL: Head sha can't be blank, Base sha can't be blank, No commits between upstream:branch and fork:branch, Base ref must be a branch`. Even if the branch exists on the fork and the commits are correct.
**Context:** Fork-based contribution workflow. You want to stack PR B on PR A, where both branches are on your fork. You try to use `fork-owner:branch-A` as the base for PR B targeting the upstream repo.

### What was tried (didn't work)
- `gh pr create --repo casehubio/engine --base mdproctor:feat/persistence/spi --head mdproctor:feat/persistence/hibernate-entities` — GitHub resolves the base against the upstream repo (`casehubio/engine`), not the fork; `mdproctor:feat/persistence/spi` doesn't exist there → fails

### Root cause
When creating a PR against an upstream repo, GitHub resolves the `--base` branch in the context of the *upstream* repo, not the fork. Fork-prefixed branch names (`owner:branch`) are only valid for the `--head` parameter (specifying the source of the diff), not for `--base`.

### Fix
Push the intermediate branches directly to the upstream repo, then use them as bases:

```bash
# Push intermediate branch to upstream (requires push access)
git push upstream feat/persistence/spi
git push upstream feat/persistence/hibernate-entities

# Now stacked PRs work
gh pr create --repo casehubio/engine \
  --base feat/persistence/spi \           # exists on upstream now
  --head mdproctor:feat/persistence/hibernate-entities \
  --title "..."
```

If you don't have push access to upstream, the alternative is opening all PRs against the same base branch and describing the stack relationship in each PR description.

### Why this is non-obvious
The fork workflow makes you think of branches as living on your fork. The `fork-owner:branch` syntax works for `--head` but not `--base` — the asymmetry is not documented clearly in `gh pr create --help`. The error message (`Base ref must be a branch`) gives no indication that the problem is which repo the branch needs to exist on.
