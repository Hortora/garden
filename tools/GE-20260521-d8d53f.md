### Context
A handover or planning doc contains a table of GitHub issues (e.g. "What's Next").
Before picking work from it, you want to know whether any items are already closed
without opening each one in the browser.

### Technique
Run a `gh issue view` loop with `--jq` to extract state and title for each issue number
in a single terminal pass:

```bash
for n in 19 22 23 26 28 29 32 33; do
  echo "=== #$n ==="
  gh issue view $n --repo casehubio/parent \
    --jq '.state + ": " + .title' \
    --json state,title 2>/dev/null || echo "not found"
done
```

Output: one `OPEN: <title>` or `CLOSED: <title>` line per issue — instantly shows
which items in the table are still actionable.

### Why this is non-obvious
The natural approach is to open each issue in GitHub's web UI or run `gh issue view N`
one at a time and scan for the state field. The `--jq` + `--json` combination lets you
extract exactly the fields you need and compose them into a single readable line,
and a bash loop applies it to a whole table at once. Most developers know `jq` from
API calls; reaching for it against `gh issue view` output is less obvious.
