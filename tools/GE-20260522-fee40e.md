### Context

Need to check whether N branches all contain a specific file (e.g. `design/EPIC-CLOSED.md`) without switching to each one. Switching branches 18 times modifies the working tree, requires stashing WIP, and leaves state to restore.

### Technique

`git show <branch>:path` reads any file from any branch ref without touching the working directory. Loop over all branches in one pass:

```bash
for b in $(git branch | tr -d '* '); do
  result=$(git show "$b:design/EPIC-CLOSED.md" 2>/dev/null && echo "closed" || echo "OPEN")
  echo "$b: $result"
done
```

Exits non-zero silently for missing files — no error output clutters results when the file is absent. Works on any ref: branch name, tag, or commit SHA.

### Why This Is Non-Obvious

Most developers reach for `git checkout <branch>` per branch, which modifies the working tree and requires restoring state afterward. `git show` is purely read-only and operates on any ref — it never changes HEAD, the index, or any working tree file. The pattern extends naturally to extracting specific fields from YAML frontmatter, checking config files, or comparing file content across branches in bulk.
