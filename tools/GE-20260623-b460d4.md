## ide_refactor_rename on an import statement renames the class globally, not the import

When positioned on a class name inside an import statement and calling
`ide_refactor_rename`, IntelliJ follows the symbol reference to the class
declaration and renames the class globally — including renaming the `.java`
file. It does NOT update just the import path.

### Symptom

`git status` shows `R OldClassName.java -> NewName.java`. The tool renamed
the class (and file) to match `newName`, not updated the package reference
in the import.

### Context

Attempting to fix a moved class import — e.g. changing
`import io.casehub.api.spi.PlannedAction;` to
`import io.casehub.worker.api.PlannedAction;` — by positioning on the class
name in the import statement and calling `ide_refactor_rename`.

### Root cause

IntelliJ's rename refactoring at an import position resolves the class name
as a symbol reference, follows it to the declaration, and performs a full
semantic rename — which includes renaming the `.java` file. Import statements
are not a rename target in IntelliJ's refactoring model; they are read as
references to the declared symbol.

### Fix

Use the `Edit` tool directly to update import statements. Only use
`ide_refactor_rename` when positioned within the class/method body, on the
class declaration line, or on a symbol you actually intend to rename across
the entire project.

### Recovery

```bash
git mv NewName.java OldClassName.java   # restore the file name
git checkout -- .                        # revert other staged changes
```

Note: `git checkout -- .` alone is insufficient because the rename is staged
as a tracked rename operation. The `git mv` is required first.

### Why this is non-obvious

An import statement looks like the natural place to "fix the reference" to a
moved class — you're pointing at the broken import and asking to rename it.
The tool appears to understand what you intend but takes completely different
action: it renames the class declaration itself rather than correcting the
package path in the import.
