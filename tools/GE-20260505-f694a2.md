## Context

Copying a file from one repository to another where a field in the heading (e.g. an ADR number) needs updating as part of the copy — for example renumbering an ADR from `0001` to `0008` when the target repo uses different numbering.

## Technique

Pipe through `sed` during the copy rather than copying then editing:

```bash
sed 's/^# 0001 — /# 0008 — /' source/adr/0001-foo.md > dest/adr/0008-foo.md
```

The `^` anchors the pattern to line-start, preventing accidental matches in the body. The destination filename is set in the redirect.

## Why This Is Non-Obvious

The natural instinct is `cp source dest && sed -i 's/.../.../' dest`. The single-pipeline form does it in one operation, avoids an intermediate file state where the destination has the wrong content, and makes the transformation intent explicit in the command itself.
