## Technique

Use `mvn --batch-mode validate` via the IntelliJ MCP terminal (`execute_terminal_command` with `executeInShell: true`) to verify that Maven module directory renames are structurally correct before running a full build.

```bash
mvn --batch-mode validate 2>&1 | tail -20
```

The `validate` phase resolves the reactor build order from all `pom.xml` files, checks that all `<module>` paths exist, and verifies parent-child relationships — all without compiling any code or triggering IntelliJ's stale module cache.

## Why non-obvious

The natural verification after module renames is either `mvn install -DskipTests` (slow, triggers compilation) or `build_project` via IntelliJ MCP (fast but stale until Maven reimport). `mvn validate` is the correct middle ground: it catches broken module paths and missing `<module>` entries in seconds, completely independent of IntelliJ's project model.

## When to use

- After any `git mv` of a Maven module directory
- After editing `<module>` entries in a parent `pom.xml`
- Before triggering a full IntelliJ Maven reimport to confirm the pom structure is valid first

## Output to look for

```
[INFO] Reactor Summary:
[INFO] CaseHub Work - Parent .................... SUCCESS
[INFO] CaseHub Work :: API ...................... SUCCESS
...
[INFO] BUILD SUCCESS
```

All modules appearing in the Reactor Summary confirms directory resolution succeeded.
