## What It Is

When running `git rebase --continue` in a script or automated context, git opens an editor for the commit message. To auto-accept the existing message without opening an editor:

```bash
GIT_EDITOR=true git rebase --continue
```

Setting `GIT_EDITOR=true` makes git invoke `/usr/bin/true` as the editor — which exits immediately with success, accepting the commit message unchanged.

## Context

Useful during scripted rebases where you've already resolved conflicts and want to continue without interactive prompts. Common in Claude Code sessions where interactive editor prompts are not supported.

## Why This Is Non-Obvious

The common approach is `git rebase --continue --no-edit`, but `--no-edit` is **not a valid flag** for `git rebase --continue` (it's valid for `git commit --amend` and `git merge`). Running it produces a usage error. The `GIT_EDITOR=true` pattern works by exploiting the editor contract rather than a dedicated flag.
