## garden_db_migrate.py leaves an empty GE-20260421-test999.md artifact after migration

**Stack:** soredium, garden_db_migrate.py
**Symptom:** After running `python3 scripts/garden_db_migrate.py ~/.hortora/garden` to migrate from CHECKED.md to garden.db, an empty file `quarkus/GE-20260421-test999.md` appears in the garden working tree as an untracked file.

**Context:** Running the migration script on a garden that uses CHECKED.md for pair tracking. The script creates garden.db successfully but also leaves this test artifact.

**Root cause:** The migration script contains a test file creation call that was not removed before release. The empty file is created as a side effect of the migration run.

**Fix:** After running `garden_db_migrate.py`, check for and remove the artifact:
```bash
rm -f ~/.hortora/garden/quarkus/GE-20260421-test999.md
```

Or add it to a cleanup commit:
```bash
git -C ~/.hortora/garden add .
git -C ~/.hortora/garden status  # verify what's untracked
```

**Why non-obvious:** The migration reports success with a clean summary (`Migrated N checked pairs, M discarded entries`) and gives no indication that a side-effect file was created. The artifact only shows up in `git status`.
