### Context

When generating or editing a multi-mode document (a README, ARC42STORIES.MD, or any document where different sections serve different rhetorical purposes), LLMs default to applying a single mode globally. This causes the wrong mode to strip or reformat sections where it doesn't apply.

### The technique

Define an explicit section-to-mode map in the document form specification. Each section is assigned the mode that fits its purpose:

```markdown
| Section | Mode | Rules |
|---|---|---|
| Lead | Reference / lookup | 1–3 sentences, dense, no padding |
| Per-capability explanations | Explanation / discursive | Bold lead-in + 2–4 sentences each |
| Module structure table | Reference / inventory | One line per entry |
| Status table | Reference / inventory | ✅ / 🔲 only |
| Documentation links | Reference / pointer | One clause per link |
```

The LLM switches mode section by section rather than applying one mode globally. The map is the mechanism — it makes the mode boundary explicit rather than leaving it to inference.

### Why this is non-obvious

The standard approach is to select one mode (Brief, Article/Explanation, etc.) for the whole document. The insight is that many common document types are not internally homogeneous — they concatenate sections that belong to different modes. A README lead is Reference; a per-capability explanation is Explanation/discursive; a module table is Reference/inventory. Without an explicit map, the selected mode bleeds into all sections and the wrong constraint is applied uniformly.

### Where it applies

Any document with heterogeneous sections:
- **README**: Reference tables + Explanation sections + Reference links
- **ARC42STORIES.MD**: Argumentation decisions + Reference tables + Explanation narratives
- **ADRs**: Brief context + Argumentation decision + Reference alternatives table

The section-to-mode map pattern generalises to any multi-mode document in any content generation context.

### Constraints

Requires the document form to be explicitly specified with the map. Works when the LLM has the map in context; defaults to global mode without it.
