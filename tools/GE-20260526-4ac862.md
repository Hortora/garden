## Context
You have a branch with scaffold or temporary files you want to discard. The files are tracked on the branch but not on main.

## Technique
Switch to main (or any branch that doesn't have those files). Git automatically removes from the working tree any files that are tracked on the current branch but absent on the target branch — no manual `rm` needed.

```bash
git checkout main
# files tracked only on the branch are removed from working tree automatically
```

## Why non-obvious
Developers typically think of `git checkout` as a navigation command, not a cleanup tool. The instinct is to explicitly delete unwanted files before switching. In practice, git's branch switch is surgical — it removes exactly the files that are branch-exclusive and nothing else.

## Constraints
Only removes *tracked* files absent on the target branch. Untracked files are left untouched.
