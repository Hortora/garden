## Symptom

A UI element never appears. No error is shown anywhere — not in the console, not in test output. Adding `async/await` to the same code path immediately surfaces a `TypeError` that had been silently swallowed for the entire lifetime of the feature.

## Context

`showUnitPanel()` fetched unit data and populated a DOM panel:

```javascript
function showUnitPanel(tag, isEnemy) {
  fetch(`/qa/unit/${tag}`)
    .then(r => r.ok ? r.json() : null)
    .then(data => {
      if (!data) return;
      // ... DOM updates ...
      window[fnName](pCtx, 32, 32, 0, tColor);  // ← wrong arg count
      document.getElementById('unit-panel').classList.add('visible');
    });
}
```

The call `window[fnName](pCtx, 32, 32, 0, tColor)` passed 5 args to a 4-arg function `drawX(ctx, size, dir, teamColor)`. `teamColor` received `0` (integer) instead of `'#4488ff'`. `hexToRgba(0, 0.3)` called `(0).slice(1,3)` → `TypeError: hex.slice is not a function`. The `.then()` chain caught this and suppressed it. The `classList.add('visible')` line never ran.

## What was tried

- Verified the fetch returned 200 ✓
- Verified the DOM element existed ✓  
- Verified `classList.add('visible')` worked when called directly ✓
- Increased `waitForFunction` timeout — still timed out
- The bug existed silently from the day the portrait drawing was added

## Root cause

An unhandled rejection inside a `.then()` callback does not propagate unless the chain ends with `.catch()` or the Promise is explicitly awaited. The TypeError terminated the callback early and the panel never showed — with zero observable error.

## Fix

```javascript
// Option 1: add .catch() to the chain
fetch(...)
  .then(...)
  .then(data => { ... })
  .catch(e => console.error('showUnitPanel failed:', e));

// Option 2: use async/await — errors propagate naturally
async function showUnitPanelAsync(tag, isEnemy) {
  const r    = await fetch(`/qa/unit/${encodeURIComponent(tag)}`);
  const data = r.ok ? await r.json() : null;
  if (!data) return;
  // ... DOM updates ...  ← TypeErrors here will now propagate to caller
  document.getElementById('unit-panel').classList.add('visible');
}
```

The actual bug was the wrong arg count: `window[fnName](pCtx, 32, 0, tColor)` not `window[fnName](pCtx, 32, 32, 0, tColor)`.

## Why this is non-obvious

`.then()` chains feel similar to try-catch blocks, but they aren't. An unhandled rejection inside `.then()` is silently discarded unless you add `.catch()`. The symptom — "UI element never appears, no error" — is indistinguishable from a network failure, a missing DOM element, or a logic bug that skips the `classList.add` call. Async/await converts this class of bug into a propagated exception immediately visible in stack traces.
