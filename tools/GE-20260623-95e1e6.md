## Context

You want to detect which `claude` processes are idle (no user interaction for N minutes) so they can be evicted to free memory. CPU usage is always near 0 when waiting for input, so it's not a useful signal.

## Technique

Claude Code writes conversation messages to `~/.claude/projects/<hash>/conversations/*.jsonl` as they happen. The mtime of those files is the last-activity timestamp for that session.

To map a PID to its project directory and last activity:

```bash
# Step 1: get the process's CWD
CWD=$(lsof -p "$PID" 2>/dev/null | awk '$4 == "cwd" {print $NF; exit}')

# Step 2: convert to Claude's project key (path with / → -)
PROJ_KEY=$(echo "$CWD" | sed 's|/|-|g')
PROJ_DIR="$HOME/.claude/projects/${PROJ_KEY}"

# Step 3: find the most recently modified conversation file
LAST_MTIME=$(find "$PROJ_DIR" -name "*.jsonl" 2>/dev/null \
  | xargs stat -f "%m %N" 2>/dev/null | sort -rn | head -1 | awk '{print $1}')

AGE=$(( $(date +%s) - LAST_MTIME ))
echo "PID=$PID idle=${AGE}s project=$(basename $CWD)"
```

## Why this is non-obvious

The natural approach is to check CPU usage or lsof activity — both fail (always near 0, or too noisy). The conversation files in `~/.claude/projects/` are not documented as a source of truth for session activity, but they're the most reliable signal available. The project key encoding (forward slashes become hyphens, starting from the root) has to be reverse-engineered from directory names in `~/.claude/projects/`.
