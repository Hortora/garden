## Context

You want to remove a method from a Java interface (sealed or otherwise) but need to confirm no code calls it polymorphically through the interface type before deleting it.

## Technique

Call `ide_find_references` on the interface method declaration:

```
ide_find_references(
  file: "src/main/java/.../MyInterface.java",
  line: <line of method declaration>,
  column: <column of method name>,
  project_path: "/path/to/project"
)
```

If the result is `totalCount: 0`, no code anywhere in the project calls this method through the interface type. The method is dead on the interface and safe to remove along with any `@Override` implementations.

## Why This Is Non-Obvious

`grep` finds ALL call sites — typed references (`myConcreteObj.method()`) and polymorphic ones (`interfaceRef.method()`) alike, requiring manual inspection of each hit to determine whether it's a true polymorphic caller. `ide_find_references` resolves reference types semantically: it returns only usages that actually reference the symbol you pointed at. Zero results on the interface method means zero polymorphic callers — definitive, not approximate.

## When It Applies

The result is only trustworthy when the IDE index is fully built (`ide_index_status` returns `isDumbMode: false`). Run the status check first if there is any doubt.
