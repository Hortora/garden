### Symptom
After renaming a GitHub repo (e.g. `neural-text` → `neocortex`) and updating
the workspace remote to the project fork URL, `git fetch` shows a forced update
on `origin/main`. Subsequent `git pull --rebase` fails with "deleted in HEAD"
conflicts on workspace-only files (e.g. `blog-routing.yaml`).

### Context
Multi-repo setups where methodology artifacts (blog, plans, specs, handover)
live in a dedicated workspace repo separate from the project repo. Both are
local git repos, but they have different GitHub remotes:
- Project fork: `mdproctor/<project-name>`
- Workspace: `mdproctor/wsp-casehub-<project-name>`

### What was tried (didn't work)
Updated the workspace remote to `mdproctor/neocortex.git` (the project fork URL),
assuming project and workspace share the same GitHub repo. This pulled the project's
commit history as `origin/main`, completely diverging from the workspace's history.

### Root cause
Workspace repos have their own separate GitHub repos using a `wsp-casehub-*` naming
convention. The workspace remote was never the same URL as the project fork — they
are independent repos with independent commit histories that happen to share a
remote name (`origin`).

### Fix
Before updating any remote during a rename, check peer workspaces to discover the
naming convention:

```bash
git -C ~/claude/public/casehub/platform remote -v
# → origin  https://github.com/mdproctor/wsp-casehub-platform.git
```

Then rename the correct workspace repo and set the remote accordingly:
```bash
gh repo rename wsp-casehub-neocortex --repo mdproctor/wsp-casehub-neural-text --yes
git -C ~/claude/public/casehub/neocortex remote set-url origin \
  https://github.com/mdproctor/wsp-casehub-neocortex.git
```

### Why this is non-obvious
The natural assumption is that workspace and project share a remote — both are
"the same project" conceptually. Nothing in the local directory structure reveals
that they point to different GitHub repos. The error only surfaces after the fetch,
when `origin/main` contains commits from a completely different repo's history.
