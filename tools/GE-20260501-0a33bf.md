## What It Is

When bulk-adding an XML block (e.g. a Maven plugin) to multiple `pom.xml` files where the `<build><plugins>` section may or may not exist, a short Python regex script is more reliable than `sed`:

```python
import sys, re

PLUGIN = """
            <plugin>
                <groupId>io.smallrye</groupId>
                <artifactId>jandex-maven-plugin</artifactId>
                <executions>
                    <execution>
                        <id>make-index</id>
                        <goals><goal>jandex</goal></goals>
                    </execution>
                </executions>
            </plugin>"""

BUILD_BLOCK = "\n    <build>\n        <plugins>{}\n        </plugins>\n    </build>\n".format(PLUGIN)

for pom in sys.argv[1:]:
    with open(pom) as f:
        content = f.read()
    if 'jandex-maven-plugin' in content:
        continue  # already present
    if '<build>' in content and '<plugins>' in content:
        content = re.sub(r'(\s*</plugins>)', PLUGIN + r'\n        </plugins>', content, count=1)
    elif '<build>' in content:
        content = re.sub(r'(<build>)', r'\1\n        <plugins>' + PLUGIN + r'\n        </plugins>', content, count=1)
    else:
        content = content.replace('</project>', BUILD_BLOCK + '</project>')
    with open(pom, 'w') as f:
        f.write(content)
```

Apply to any number of files: `python3 script.py module1/pom.xml module2/pom.xml ...`

## Context

Needed when a plugin must be added to 10+ library JAR modules across multiple repos, where some have no `<build>` section, some have `<build>` without `<plugins>`, and some have existing plugins. Handles all three cases cleanly.

## Why This Is Non-Obvious

`sed` with XML is fragile — whitespace, attribute ordering, and multiline patterns make it unreliable. A full XML parser (e.g. `lxml`) preserves structure but destroys formatting and comments. The regex approach is the middle ground: structure-aware enough to handle the three cases, simple enough to write inline.
