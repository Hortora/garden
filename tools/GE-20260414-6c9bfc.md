## gh pr create requires --body even when --title is provided in non-interactive mode

**ID:** GE-20260414-6c9bfc
**Stack:** GitHub CLI (gh), any version
**Symptom:** `gh pr create --title "..." --label "..." --head branch` exits with error even though a title is supplied:
```
must provide `--title` and `--body` (or `--fill` or `fill-first` or `--fillverbose`) when not running interactively
```
**Context:** Any automated or scripted PR creation where the terminal is non-interactive (CI, Claude Code Bash tool, subshell).

### What was tried (didn't work)
- `gh pr create --title "submit(GE-XXXX): slug" --label "garden-submission" --head submit/$GE_ID` — fails with the above error

### Root cause
`gh pr create` requires both `--title` AND `--body` (or one of the fill flags) when stdin is not a TTY. The error message says "must provide --title and --body" but when you've already provided --title, the natural reading is that body alone is missing — which is correct. The error message is technically accurate but the natural assumption ("title alone is enough for a simple PR") is wrong.

### Fix
Always provide `--body` explicitly in non-interactive contexts:
```bash
gh pr create \
  --title "submit($GE_ID): slug" \
  --body "Brief description of the submission." \
  --label "garden-submission" \
  --head submit/$GE_ID
```

Or use `--fill` to auto-populate body from the commit message:
```bash
gh pr create --title "..." --fill --label "..." --head branch
```

### Why this is non-obvious
The assumption is that title is the required field and body is optional (matching GitHub's web UI behaviour where body is optional). In non-interactive mode, `gh` requires both to avoid partially-formed PRs being created accidentally.
