### Symptom
After modifying multiple files via `ide_replace_text_in_file`, subsequent edit operations
(`ide_replace_member`, `ide_edit_member`, `ide_replace_text_in_file`) hang indefinitely
until the MCP client times out (300s default). No error is returned — the operation
simply never completes.

### Context
Occurs when bulk file modifications (11 files in this case) trigger IntelliJ's background
re-indexing. The IDE enters "dumb mode" (`isDumbMode=true`) and edit operations queue
behind the index rebuild. The re-index can take 10+ minutes for a multi-module project.

### What was tried (didn't work)
1. Retrying the same edit call — hangs again
2. Using `ide_sync_files` to force refresh — also hangs during dumb mode
3. Waiting 15 seconds and retrying — still in dumb mode

### Root cause
IntelliJ MCP edit tools require PSI (Program Structure Interface) access, which is
unavailable during dumb mode. The tools block waiting for smart mode instead of
returning an error. `ide_index_status` uses a different code path and remains responsive.

### Fix
Poll `ide_index_status` until `isDumbMode=false` before attempting edit operations
after bulk changes:

```
// Pseudo-workflow:
// 1. Make batch changes via ide_replace_text_in_file
// 2. Poll ide_index_status (responds even in dumb mode)
// 3. Wait until isDumbMode=false
// 4. Resume edit operations
```

### Caveats
- `ide_replace_text_in_file` may **silently succeed** despite the timeout — the change
  applies to the document but no response is sent back. Always re-read the file after
  a timeout to check if the change landed.
- `ide_index_status` is the only reliably responsive tool during dumb mode.

### Why this is non-obvious
The silent success case is the trap — a timed-out call that actually worked is
indistinguishable from one that failed. Without checking, you either retry (creating
a duplicate edit) or abandon (losing a successful change).

### Why this fix
Polling `ide_index_status` is the only reliable readiness signal. The MCP plugin
does not expose a "wait for smart mode" API, and no other tool responds during
dumb mode.
