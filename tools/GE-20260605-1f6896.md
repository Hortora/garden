### Symptom
`git cherry-pick` fails with `CONFLICT (content)` on `design/JOURNAL.md` or blog
entries when promoting a workspace epic branch to main during `work-end`.

### Context
Two-repo workspace setup. Session wrap runs while on the epic branch: stash, checkout
main, commit artifacts (blog entry, HANDOFF.md), push, checkout epic branch, pop
stash. Later, `work-end` cherry-picks the epic branch commits onto main.

### What was tried (didn't work)
Attempted to cherry-pick all branch commits in sequence — failed on the journal and
blog commits because they were already present on main from the session-wrap commit.

### Root cause
The session-wrap stash-commit-pop cycle commits artifacts to main using the branch's
stashed state. The epic branch then also has independently committed those same files
during the session (journal completion note, blog entry). When `work-end` cherry-picks
the branch onto main, those commits see the file as already present with different
content (the main version from the wrap commit), causing a conflict.

### Fix
Before cherry-picking, inspect what the session-wrap commit on main added vs what the
branch has. Cherry-pick only the additive commits (the primary deliverable —
e.g. ARC42STORIES.MD, HANDOFF.md updates), and skip commits that duplicate what's
already on main using `git cherry-pick --skip`.

```bash
# Identify divergence
git log --oneline main..epic-branch   # branch commits
git log --oneline epic-branch..main   # main-only commits (session-wrap)

# Cherry-pick selectively — skip the duplicate on conflict
git cherry-pick <primary-sha>         # e.g. ARC42STORIES.MD commit
git cherry-pick <secondary-sha>       # may succeed cleanly
# On conflict:
git cherry-pick --skip                # skip the duplicate blog/HANDOFF add
```

### Why this is non-obvious
The session-wrap stash/commit/pop is designed to land artifacts on main without
losing branch work. The side effect — that the branch may independently commit the
same files — only becomes visible at work-end when those commits are promoted. The
symptom (JOURNAL.md conflict) misleads: JOURNAL.md conflicts because it diverged
between branch and main, but the real issue is the blog or handover file that was
committed twice via two different paths.
