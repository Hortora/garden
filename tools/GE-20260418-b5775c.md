## What happened

Writing a Playwright E2E test that needs to wait until a DOM element has non-empty text content:

```java
// Intended: wait up to 5 seconds for condition to be true
page.waitForFunction(
    "() => document.getElementById('mesh-body').textContent.trim().length > 0",
    new com.microsoft.playwright.Page.WaitForFunctionOptions().setTimeout(5000));
```

This compiles fine but throws `RuntimeException: Unsupported type of argument` at runtime.

## Root cause

`Page.waitForFunction` in Playwright for Java has this signature:

```java
JSHandle waitForFunction(String expression, Object arg, WaitForFunctionOptions options)
```

`arg` is the second parameter (a value passed into the page function as its argument). When `WaitForFunctionOptions` is passed as the second argument, Playwright cannot serialise it to a JSON-serialisable JS argument and throws at runtime.

## Fix

Pass `null` explicitly as `arg` and move options to the third position:

```java
page.waitForFunction(
    "() => document.getElementById('mesh-body').textContent.trim().length > 0",
    null,   // arg — null means no argument is passed to the JS expression
    new com.microsoft.playwright.Page.WaitForFunctionOptions().setTimeout(5000));
```

## Why this is non-obvious

The Playwright for Java docs and IDE autocomplete suggest the two-argument form naturally. The error message ("Unsupported type of argument") points at serialisation, not at argument ordering. Nothing indicates that `WaitForFunctionOptions` landed in the wrong slot.
