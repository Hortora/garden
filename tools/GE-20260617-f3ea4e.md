## Symptom

LLM quality eval test fails on factual fidelity dimension despite renders being visually accurate. Claude Sonnet 4.6 scoring its own MARKDOWN/PROSE renders gives FACTUAL_FIDELITY 1.88–2.13 (scale 0–5). The same renders scored by an independent model (Qwen 8B via Ollama) give FACTUAL_FIDELITY 5.00.

## Context

Any eval harness where Claude is both the renderer (producing the rendered output) and the judge (scoring the output). Specifically observed when evaluating LLM-enriched disposition narratives: the renderer elaborates raw axis values (`riskAppetite=conservative`) into contextual prose ("You weigh regulatory exposure before recommending any change"), which is accurate — but Claude's judge penalises this as a claim not grounded in the descriptor JSON.

## What was tried (didn't work)

- Lowering the PROSE quality floor assuming the renders were genuinely lower quality
- Assuming the eval infrastructure had a bug

## Root cause

Claude's FACTUAL_FIDELITY criterion in its judge role applies a strict literalism: any claim in the rendered output that is not a verbatim or near-verbatim echo of the descriptor JSON is flagged as potentially hallucinated. Contextual elaborations — even accurate ones inferred from the descriptor semantics — score as "absent from descriptor". An independent judge with no connection to the renderer evaluates the elaborations as accurate and grounded, giving 5.00.

This is an **inverse** self-evaluation bias: Claude penalises its own outputs, not inflates them.

## Fix

Use an independent judge model for FACTUAL_FIDELITY evaluation when Claude is the renderer:

1. Run Claude as renderer to produce renders → save to cache (`renders-cache.json`)
2. Load cache and re-judge with a different model (Qwen 8B, Mistral, Llama) via `evaluateWithIndependentJudge()`
3. Calibrate score floors separately for each judge model — Sonnet self-judging PROSE floor is ~3.5; independent judge floors can be ~4.5

The independent judge approach catches real quality regressions without inheriting the renderer's self-penalisation bias.

## Why this is non-obvious

The naive assumption is that self-evaluation bias, if it exists, would inflate scores (model likes its own output). The observed direction is the opposite: Claude penalises its own elaborations for going beyond the descriptor, while an independent model accepts them as accurate. Nobody would design an eval harness expecting their judge to systematically under-score its own renderer.
