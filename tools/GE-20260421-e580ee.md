## What

Services with many CDI injections need Quarkus boot for `@QuarkusTest` — expensive.
The two-constructor pattern gives a second package-private constructor that accepts
dependencies directly, enabling pure unit tests with Mockito or stubs in milliseconds.

## How to apply

```java
@ApplicationScoped
public class WorkItemAssignmentService {

    private final WorkerSelectionStrategy fixedStrategy; // null in CDI path
    private final WorkerRegistry workerRegistry;
    private final WorkItemStore workItemStore;
    private WorkItemsConfig config;
    // ... other CDI-injected fields

    /** CDI constructor — full wiring. */
    @Inject
    public WorkItemAssignmentService(
            final WorkItemsConfig config,
            final WorkerRegistry workerRegistry,
            final WorkItemStore workItemStore,
            final ClaimFirstStrategy claimFirst,
            final LeastLoadedStrategy leastLoaded) {
        this.config = config;
        this.workerRegistry = workerRegistry;
        this.workItemStore = workItemStore;
        this.fixedStrategy = null; // CDI path: strategy resolved dynamically
        // ...
    }

    /** Package-private: for unit tests — bypasses CDI and config. */
    WorkItemAssignmentService(final WorkerSelectionStrategy strategy,
            final WorkerRegistry workerRegistry, final WorkItemStore workItemStore) {
        this.fixedStrategy = strategy;  // used directly, no config lookup
        this.workerRegistry = workerRegistry;
        this.workItemStore = workItemStore;
    }

    private WorkerSelectionStrategy activeStrategy() {
        if (fixedStrategy != null) {
            return fixedStrategy; // unit-test path — short-circuit
        }
        // CDI path: resolve from @Alternative or config
        ...
    }
}
```

Unit test:
```java
@ExtendWith(MockitoExtension.class)
class WorkItemAssignmentServiceTest {
    @Mock WorkItemStore workItemStore;
    @Mock WorkerRegistry workerRegistry;

    private WorkItemAssignmentService service;

    @BeforeEach
    void setUp() {
        service = new WorkItemAssignmentService(
            new LeastLoadedStrategy(), workerRegistry, workItemStore);
    }
    // runs in <100ms, no Quarkus boot
}
```

## Why this is non-obvious

Most developers either test everything with `@QuarkusTest` (slow, heavy) or use a CDI
test alternative (`@InjectMock`, `QuarkusUnitTest`) which still bootstraps the container.
The package-private constructor is an escape hatch that gives pure isolation without
any framework overhead. The `fixedStrategy != null` null-check in `activeStrategy()`
is the only coupling between the two paths and is obviously testable.

### Why this fix

Pure Mockito unit tests run in ~50ms vs 6+ seconds for `@QuarkusTest`. For services
with complex conditional logic (trigger gates, candidate resolution, capability filtering),
fast unit tests catch regressions in the test loop rather than in CI.
