## Context

Permuplate's Maven plugin supports `@Permute(inline=true)` on nested static classes. A template class generates sibling classes inside the same outer class. But when you need `@PermuteMethod` on a top-level class (to generate method overloads without an arity family), there's no enclosing outer class to hold the siblings.

## The technique

Name the **source file** after the **generated output class**, but name the **template class** differently (conventionally, with a `Template` suffix):

```
src/main/permuplate/
  com/example/RuleBuilder.java   ← file name matches generated class
```

```java
// Inside RuleBuilder.java:
@Permute(varName = "i", from = "1", to = "1",
         className = "RuleBuilder",      // ← output class name
         inline = true, keepTemplate = false)
public class RuleBuilderTemplate<DS> {  // ← template class name (different)
    
    @PermuteMethod(varName = "j", from = "2", to = "7", name = "extendsRule")
    @PermuteReturn(className = "JoinBuilder.Join${j-1}First", ...)
    public <@PermuteTypeParam(...)  A> Object extendsRule(
            @PermuteDeclr(type = "RuleExtendsPoint${j}<DS, ...>") Object ep) {
        // body using reflection for the return value
    }
}
```

The plugin:
1. Reads `RuleBuilder.java` from `src/main/permuplate/`
2. Finds `RuleBuilderTemplate` annotated with `@Permute(className="RuleBuilder")`
3. Processes the template (renames class, applies `@PermuteMethod`, etc.)
4. Outputs `target/generated-sources/permuplate/.../RuleBuilder.java` containing `public class RuleBuilder`

Because the file name (`RuleBuilder.java`) matches the generated public class name (`RuleBuilder`), Java's one-public-class-per-file rule is satisfied.

## Why this is non-obvious

The standard pattern requires `@Permute` on a **nested** class inside an outer container. Without this trick, `@PermuteMethod` on top-level classes seems impossible — there's no outer class for the plugin to augment. The file-naming technique sidesteps this by making the template file *itself* the "container" while using a different class name for the template so it can be renamed to the target.

## Constraint

`className` validation checks that the leading literal of the `className` value appears as a substring in the template class name. `"RuleBuilder"` is a substring of `"RuleBuilderTemplate"` — this passes. The template name must contain the target name as a substring.

## Related technique

Use `@Permute(from="1", to="1", className="OriginalName")` as a minimal wrapper when the only goal is to enable `@PermuteMethod` — the outer loop runs once (i=1), doing nothing itself, while the inner `@PermuteMethod` generates all the real overloads.
