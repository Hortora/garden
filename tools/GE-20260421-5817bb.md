## What exists

The superpowers brainstorm server (started via `scripts/start-server.sh`) serves any file in the `content/` session directory at the path `/files/<filename>`. This allows HTML pages in the brainstorm session to load local assets (JS libraries, images) without CDN dependencies.

```html
<!-- In a brainstorm HTML file -->
<script src="/files/three.min.js"></script>
```

```bash
# Download the asset into the content directory first
curl -s https://cdnjs.cloudflare.com/.../three.min.js \
  -o "$SCREEN_DIR/three.min.js"
```

## Why undocumented

The brainstorming skill documentation (`visual-companion.md`) describes only HTML content fragments and the frame template. Static asset serving via `/files/` is not mentioned anywhere in the skill docs or the server startup output.

## Use case

Loading Three.js, Chart.js, or other large libraries locally during interactive brainstorm sessions — avoids CDN latency/blocking, works fully offline, and eliminates CORS issues.
