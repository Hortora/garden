### What exists

A `hooks/hooks.json` file in a Claude Code plugin directory can reference scripts using `${CLAUDE_PLUGIN_ROOT}`. The variable resolves to the plugin's installed directory at runtime — making hook script paths portable across machines, users, and install methods (marketplace or cloned).

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "\"${CLAUDE_PLUGIN_ROOT}\"/scripts/check_project_setup.sh"
          }
        ]
      }
    ]
  }
}
```

The plugin system installs the full plugin directory (including subdirectories and arbitrary file types) to `~/.claude/skills/<plugin-name>/`. `${CLAUDE_PLUGIN_ROOT}` resolves to that path.

### Why undocumented

The main Claude Code plugin authoring documentation describes hooks as a supported component type and lists the `hooks/` directory in the plugin structure, but does not explain `${CLAUDE_PLUGIN_ROOT}`. It appears only in the plugins reference page under an example snippet, with no dedicated explanation that it is the mechanism for portable script paths. Without it, any `command` referencing a bundled script requires a hardcoded absolute path — which breaks across machines.

### Why this matters

Without `${CLAUDE_PLUGIN_ROOT}`, you cannot ship a plugin that includes a helper script and references it from a hook — the path would be hardcoded to one machine's `~/.claude/skills/` location and break for any other user.
