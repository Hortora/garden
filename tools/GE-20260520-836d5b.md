### What it does

When `python3 scripts/claude-skill sync-local --all -y` runs in cc-praxis, it does two things:

1. Copies all skill directories from `~/claude/cc-praxis/` to `~/.claude/skills/`
2. **Also copies `~/claude/cc-praxis/hooks/check_project_setup.sh` to `~/.claude/hooks/check_project_setup.sh`**

The second action — updating the deployed session-start hook — is not documented in the sync-local SKILL.md. The only indication it happened is the status line:

```
✅ Session-start hook updated from repo
```

### Why this matters

If you edit `~/.claude/hooks/check_project_setup.sh` directly (e.g. to test a fix), running `sync-local` will overwrite your edit. The deployed hook is always overwritten from the repo source. Any changes to hook logic must be made to `~/claude/cc-praxis/hooks/check_project_setup.sh` and then deployed via sync-local.

### The relevant code in scripts/claude-skill

```python
HOOK_SOURCE = Path(__file__).parent.parent / "hooks" / "check_project_setup.sh"
HOOK_DEST = Path.home() / ".claude" / "hooks" / "check_project_setup.sh"
```

### Why this is non-obvious

The sync-local SKILL.md describes syncing "skills" — not hooks. The hook update is a side effect of the same command with no documentation. The status message is the only hint, and it doesn't include a path to show which file was updated. Discovering this required tracing why a directly-patched hook kept reverting after sync-local ran.
