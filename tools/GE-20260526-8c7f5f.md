Claude Code exposes two distinct IntelliJ MCP tool namespaces that are NOT interchangeable:

- `mcp__intellij__*` — lower-level IDE operations (open file, build, get modules, find problems, run configurations, debugger). Uses the IDE's internal APIs directly.
- `mcp__intellij-index__*` — semantic index operations (find class, find references, go to definition, diagnostics, type hierarchy, search text). Uses the IDE's search and indexing engine.

When multiple projects are open, `mcp__intellij__*` tools require `projectPath` and will fail with "multiple projects open" if not specified. `mcp__intellij-index__*` tools require `project_path`. The parameter name differs between namespaces.

### Why this is non-obvious

Both namespaces are presented as "IntelliJ MCP tools" but they have different parameter names for the same concept (project path), different capabilities, and different failure modes. Using `mcp__intellij__get_file_problems` on a closed file gives different results than `mcp__intellij-index__ide_diagnostics` — the former returns nothing useful, the latter gives semantic errors. The distinction is not documented anywhere in the tool descriptions.
