## Symptom

`git mv source dest` fails with:

```
fatal: not under version control, source=<file>
```

even though the file is in a git-tracked directory.

## Context

Occurs when a file was created with `cp` (or any non-git means) and never `git add`'ed. `git mv` requires the source to be known to git — an untracked file in a tracked directory doesn't qualify.

## Root Cause

`git mv` is a compound operation: `mv` + `git rm` + `git add`. The `git rm` step fails because git has no record of the source file.

## Fix

```bash
git add <file>        # register the file with git
git mv <file> <dest>  # now git mv works
```

## Why This Is Non-Obvious

The error says "not under version control" but the file is inside a tracked repository directory. The distinction between "file exists in a git repo" and "file is tracked by git" is subtle — especially when the copy was done as part of a larger operation where other files in the same glob were already tracked and `git mv` on those succeeded.
