## What it is

Inside a `.SC2Map` MPQ archive, `t3SyncCliffLevel` is the authoritative terrain height map. It encodes discrete cliff tiers as multiples of 64, with non-multiples representing ramp transitions.

**Binary format:**
- Bytes 0–3: magic `CLIF` (ASCII)
- Bytes 4–7: version (uint32 LE)
- Bytes 8–11: width in tiles (uint32 LE)
- Bytes 12–15: height in tiles (uint32 LE)
- Bytes 16+: `width × height` little-endian uint16 values, row-major (y outer, x inner)

**Cliff tier encoding:**

| Value | Meaning |
|-------|---------|
| 0 | Void (out of bounds) → impassable |
| 64 | Lowest walkable tier (ground level) |
| 128 | Mid tier |
| 192 | Upper tier (player base level in standard maps) |
| 256 | Very high |
| 320+ | Impassable border cliffs |
| Non-multiples of 64 (e.g. 72–112, 136–176) | Ramp transition between tiers |

**Mapping to game terrain types:**

```java
static Height toHeight(int cliff) {
    if (cliff == 0 || cliff >= 320) return Height.WALL;
    if (cliff % 64 != 0)           return Height.RAMP;
    if (cliff == 64)               return Height.LOW;
    return Height.HIGH;
}
```

## Context

Used to extract terrain grids from AI Arena `.SC2Map` files (themselves MPQ archives) using the scelight-mpq library: `MpqParser.getFile("t3SyncCliffLevel")`.

## Discovered via

Binary analysis: extracted the raw bytes from TorchesAIE_v4.SC2Map (160×208 tile map), confirmed the CLIF magic, inferred width/height from header, correlated cliff values with visible terrain features and ASCII art visualisation.

## Why this is non-obvious

This format is not documented in any SC2 modding reference, SC2 Map Editor documentation, or sc2mapanalysis Python library docs. The file name appears in SC2 map editor internals but the binary format is only discoverable by reading raw bytes.
