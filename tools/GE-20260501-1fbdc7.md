## Symptom

`DELETE /orgs/{org}/packages/maven/{package_name}/versions/{version_id}` returns HTTP 400:

```
{"message":"You cannot delete the last version of a package. You must delete the package instead."}
```

## Context

Attempting to clear SNAPSHOT package versions from GitHub Packages via the API. Packages that have only one version (common after a rename or fresh publish) fail every deletion attempt.

## Root cause

GitHub Packages enforces a rule: a package must always have at least one version. If you try to delete the only remaining version via the version endpoint, it returns 400. You must instead delete the entire package using the package-level endpoint.

## Fix

Detect the 400 "last version" error and fall back to the package-level DELETE:

```bash
if resp=$(gh api -X DELETE \
  "/orgs/$ORG/packages/maven/$pkg_name/versions/$ver_id" 2>&1); then
  echo "ok"
elif echo "$resp" | grep -q "delete the last version"; then
  # Last version — delete the whole package
  if resp2=$(gh api -X DELETE \
    "/orgs/$ORG/packages/maven/$pkg_name" 2>&1); then
    echo "ok (deleted whole package)"
  else
    echo "FAILED (package delete): $resp2"
  fi
else
  echo "FAILED: $resp"
fi
```

## Why this is non-obvious

The version endpoint and the package endpoint are different resources. A developer trying to "delete all versions" naturally iterates over versions and deletes each one — the last iteration always fails. The error message is clear once visible, but is often suppressed by `--silent 2>/dev/null` in automation scripts.
