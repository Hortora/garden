### Symptom
Editing `_layouts/post.html` or `_includes/note-list.html` and saving — Jekyll reports a rebuild but the browser still shows the old layout. Changes appear to be silently ignored.

### Context
Running `jekyll serve --incremental` during active development of a Jekyll site. Editing template files (layouts, includes) and expecting them to trigger a full rebuild.

### What was tried
Modified `post.html`, saved, waited for Jekyll's incremental rebuild. Browser showed stale output. Repeated the edit — still stale. Assumed caching issue.

### Root cause
Jekyll's `--incremental` mode tracks **content file** changes only (markdown posts, collection items). It does not track template dependency changes — edits to `_layouts/`, `_includes/`, `_config.yml`, or CSS files do not trigger rebuilds of pages that depend on them.

### Fix
Run a full rebuild whenever templates or includes change:

```bash
# Kill the serve process, rebuild fully, restart
bundle exec jekyll build
bundle exec jekyll serve --livereload --incremental
```

Or simply don't use `--incremental` during active template development — only enable it once layout work is done.

### Why this is non-obvious
`--incremental` sounds like "rebuild anything that changed." In practice it's "rebuild content files whose front matter changed." The documentation mentions "experimental" status but doesn't clearly explain that template dependency tracking is absent.
