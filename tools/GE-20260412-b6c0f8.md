## Claude Code Read tool fails on files over ~256KB — use offset/limit or Bash grep

**ID:** GE-20260412-b6c0f8
**Stack:** Claude Code (all versions)
**Symptom:** Attempting to read a large file returns:
```
File content (350.4KB) exceeds maximum allowed size (256KB).
Use offset and limit parameters to read specific portions of the file,
or search for specific content instead of reading the whole file.
```
**Context:** Any file in the working repo that grows large through accumulated writes — generated indexes, pair logs, large YAML files. The error fires before any content is returned.

### Fix

**Option A — targeted read:** Provide `offset` and `limit` parameters to the Read tool to read a specific line range.

**Option B — content search:** Use the Grep tool or Bash `grep`/`git show` when you need to search for specific content rather than read a known line range.

```bash
# Search committed content in a large file
git -C $GARDEN grep "search-term" HEAD -- CHECKED.md

# Or read a section via git show + grep
git -C $GARDEN show HEAD:CHECKED.md | grep -A 10 "GE-0174"
```

### Why this is non-obvious
The 256KB limit is not documented anywhere. The natural assumption is that the Read tool can read any file in the repo regardless of size. The error message does hint at offset/limit, but if the goal is to find specific content at an unknown line, switching to Grep or Bash is the better path — and that redirect is not obvious from the error.
