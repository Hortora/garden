### Symptom
`gh repo list mdproctor --json name,isFork,parent` returns `isFork: true`
but `parent: null` for known forks. Querying `parent.nameWithOwner` gives
null even when the repo is definitely a fork of an upstream.

### Context
When listing multiple repos in bulk via `gh repo list`, the `parent` field
is not reliably populated. Occurs even when the upstream repo is public and
accessible.

### What was tried
Used `gh repo list --json name,isFork,parent` to map forks to their upstreams
in bulk. The parent field was null for all forks in the listing.

### Root cause
The `gh repo list` endpoint appears to omit parent details from the bulk
response. The single-repo endpoint (`gh repo view`) does return `parent`
correctly.

### Fix
Use per-repo queries for reliable parent information:
```bash
gh repo view mdproctor/<repo> --json name,isFork,parent
```
Do not rely on `gh repo list --json parent` for fork detection or upstream mapping.

### Why this is non-obvious
`isFork: true` and `parent: null` appearing together looks like a bug or
missing data, but it is a consistent limitation of the bulk listing endpoint.
The single-repo view is the only reliable source.
