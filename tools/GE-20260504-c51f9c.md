## Context

A GitHub organisation has accumulated repos where squash merges are enabled,
causing history loss. You want to enforce a consistent merge policy (no squash,
rebase allowed) across all repos at once.

## Technique

Use `gh api --method PATCH` in a loop over all repos:

```bash
for repo in engine ledger work qhorus claudony parent; do
  gh api --method PATCH repos/myorg/$repo \
    -f allow_squash_merge=false \
    -f allow_rebase_merge=true \
    -f allow_merge_commit=true \
    --jq '{repo:.full_name, squash:.allow_squash_merge, rebase:.allow_rebase_merge}'
done
```

The `--jq` flag confirms the new settings in the output without a separate
verification step. To also enable auto-delete of merged branches:
```bash
-f delete_branch_on_merge=true
```

## Why this is non-obvious

GitHub's UI requires navigating to each repo's Settings → General → Pull Requests
section individually. There is no bulk org-level policy enforcement in the UI for
merge strategy. The API approach takes seconds for any number of repos and produces
a verifiable output row per repo.
