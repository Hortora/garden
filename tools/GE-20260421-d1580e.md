## Problem

3D objects (meshes, groups) accumulate in a scene over time, causing memory growth and visual artefacts in 3D mode. No error or warning is thrown.

## Symptom

In toggle between 2D/3D rendering modes, stale 3D objects from previously-removed entities remain visible in 3D mode. `scene.children.length` grows indefinitely.

## Root cause

A `syncUnitLayer(spriteMap, meshMap, units, isEnemy)` function was called with `new Map()` as the `meshMap` argument for staging units:

```javascript
// Bug: transient map is discarded after each call
syncUnitLayer(stagingSprites, new Map(), state.enemyStagingArea || [], true);
```

Inside `syncUnitLayer`, when a new unit is seen, its 3D mesh is added to `group3d` and tracked in `meshMap`. When a unit disappears, `meshMap.get(tag)` returns `undefined` (the map is empty on every call), so `group3d.remove()` is never called. The mesh leaks.

## Fix

Promote the map to module scope:

```javascript
const stagingMeshes = new Map(); // module-level, persists across calls

// In syncUnits():
syncUnitLayer(stagingSprites, stagingMeshes, state.enemyStagingArea || [], true);
```

## Why this is non-obvious

The `instanceof Map` guard in the removal path (`if (meshMap instanceof Map)`) evaluates `true` for the transient `new Map()`, so it looks like cleanup will happen. But `meshMap.get(tag)` always returns `undefined` because the map is always freshly empty. The cleanup code runs but does nothing. No error, no warning — just silent accumulation.
