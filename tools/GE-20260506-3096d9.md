### Behaviour
`rustkyll build --incremental` works correctly — rebuilds only changed pages.  
`rustkyll serve --incremental` throws an error:

```
error: unexpected argument '--incremental' found
Usage: rustkyll serve --source <SOURCE>
```

The `serve` subcommand silently omits the `--incremental` flag from its CLI interface. There is no documentation for this asymmetry.

### Workaround
Use `rustkyll serve` without `--incremental`. The `serve` command does its own internal change detection and rebuilds automatically on file save — the incremental flag is not needed for the serve workflow.

### Where this matters
Anyone switching from `jekyll serve --incremental` to `rustkyll serve` and copying the flag will get an immediate error. The error message doesn't explain that the flag exists only on `build`.
