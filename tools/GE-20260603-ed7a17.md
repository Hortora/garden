## Context

When syncing local branches across many repos (e.g. aligning fork main with upstream main), the standard approach is: checkout the branch, pull or reset, checkout back. This is disruptive if the repo has an active working branch — it switches context unnecessarily.

## Technique

`git update-ref` is a plumbing command that moves a branch ref to any commitish directly, without checking out the branch:

```bash
# Fast-forward local main to match upstream/main — no checkout needed
git update-ref refs/heads/main upstream/main

# Equivalent to: git checkout main && git reset --hard upstream/main && git checkout -
# but without any branch switch
```

Works for any ref update — local branch to remote ref, to a specific SHA, or to another local branch. Safe to run when the repo is on a different branch entirely.

## Why this is non-obvious

Most developers know only the porcelain path: `git checkout <branch> && git pull`. `update-ref` is a plumbing command not mentioned in everyday git tutorials. Its use for branch fast-forwarding without checkout is undocumented outside man pages.

## Caveats

- Only works for fast-forwards (or forced moves) — `update-ref` does not merge or rebase
- Does not update the working tree or index — the filesystem is unchanged
- If used to non-fast-forward a branch, the branch tip moves but history is rewritten for that ref; use with intent
