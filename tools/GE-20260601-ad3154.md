## What it does

Reads a compiled `.class` file from a JAR and parses its constant pool (JVMS §4.4) to extract
all UTF-8 string constants — including `public static final String` values — without needing
the source code or a decompiler. Useful when source isn't available and `javap -verbose` output
is too noisy to scan manually.

## Why this is non-obvious

Most developers reach for a decompiler (CFR, Fernflower, `javap -c`), but these produce large
output that requires scanning for the value you want. The constant pool is a direct binary
structure at the start of every class file — Python's `int.from_bytes` and a small state machine
can walk it in ~20 lines. Constant values for `static final String` fields are stored as
UTF-8 constants directly in the pool (JVMS §4.7.2 ConstantValue attribute), not in `<clinit>`.

## Technique

```python
import zipfile

z = zipfile.ZipFile('/path/to/library.jar')
data = z.read('io/example/MyConstants.class')

# Walk the constant pool (JVMS §4.4)
# Skip: magic (4) + major/minor version (4)
i = 8
count = int.from_bytes(data[i:i+2], 'big')
i += 2

constants = {}
idx = 1
while idx < count:
    tag = data[i]; i += 1
    if tag == 1:          # CONSTANT_Utf8
        length = int.from_bytes(data[i:i+2], 'big'); i += 2
        constants[idx] = data[i:i+length].decode('utf-8', errors='replace')
        i += length
    elif tag in (7, 8, 16, 19, 20):   # 2-byte index
        i += 2
    elif tag in (9, 10, 11, 12, 17, 18):  # 4-byte
        i += 4
    elif tag in (3, 4):   # Integer / Float
        i += 4
    elif tag in (5, 6):   # Long / Double — takes two pool slots
        i += 8; idx += 1
    elif tag == 15:        # MethodHandle
        i += 3
    idx += 1

# All string constants are now in: constants (dict index→str)
# Filter for values of interest:
interesting = {k: v for k, v in constants.items()
               if len(v) > 5 and not v.startswith('(') and not v.startswith('L')}
print(interesting)
```

Key insight: `static final String` constants are stored as `CONSTANT_Utf8` entries (tag=1)
in the pool and referenced by a `ConstantValue` attribute on the field. You do not need to
parse attributes — just dumping all UTF-8 constants and filtering by length/pattern is usually
enough to find named values.

## Constraints

Works for constants stored at compile time (primitive literals and `String` literals folded by
`javac`). Does NOT work for values computed at runtime (e.g. `UUID.randomUUID()` in a static
initializer). Encoding is modified UTF-8 (JVMS §4.4.7) — use `errors='replace'` to handle
supplementary characters safely.

## Alternatives considered

- `javap -verbose MyClass.class`: shows constant pool but in a verbose format that's hard to
  grep. Good for understanding structure, noisy for finding one value.
- Java decompilers (CFR, Fernflower): heavy tool for a simple lookup; also not always
  available on a developer machine.
- Running the class in a JVM and instrumenting it: overkill; requires a compatible JDK and
  classpath setup.
