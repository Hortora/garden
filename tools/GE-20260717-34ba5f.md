### Symptom

Mid-session, `ide_edit_member`, `ide_replace_text_in_file`, or any `mcp__intellij-index__*`
tool call returns:

```
error: project_not_found
message: No open project matches the specified path
```

The workspace was open minutes ago and no user action closed it.

### Root cause

The IntelliJ Index MCP plugin's lifecycle manager automatically transitions idle workspaces
through `active → background → dormant → closed`. The `closed` transition happens after
~8 minutes of MCP inactivity (no tool calls to that workspace). Once closed, all project
references are invalidated.

Any gap in tool calls — a background task running, the user thinking, a long compilation —
can exceed the idle threshold and silently close the workspace.

### Fix

After opening a workspace with `ide_open_workspace`, immediately set it to `active` mode:

```java
ide_set_project_mode(mode = "active", project_path = "<any-module-in-workspace>")
```

Active mode disables the lifecycle manager's automatic transitions for that workspace.
The workspace stays open until explicitly closed or the IDE exits.

### Why this is non-obvious

The error message ("No open project matches the specified path") suggests the path is wrong
or the project was never opened. Nothing indicates the project was *automatically closed by
the lifecycle manager*. The fix (`ide_set_project_mode`) is in a different API surface than
the tools that fail — you have to know the lifecycle management system exists to find it.
