### Symptom
After deleting a skill directory from the cc-praxis source repo, running
`python3 scripts/claude-skill sync-local --all -y` reports success and shows
the correct skill count — but the deleted skill remains installed in
`~/.claude/skills/` and continues appearing in the skill listing.

### Context
Occurs any time a skill is deleted from the source repository. The sync
script uses copy/update semantics, not mirror semantics — it only adds or
updates files that exist in the source, never removes files at the
destination. No warning is printed.

### Root cause
`sync-local` iterates over skills present in the source directory and copies
them to `~/.claude/skills/`. It has no awareness of skills that were
previously synced but no longer exist in the source.

### Fix
After deleting a skill from the source repo, manually remove it from the
installed location:
```bash
rm -rf ~/.claude/skills/<deleted-skill-name>
```

This must be done separately from the sync — there is no flag to enable
mirror/delete behaviour.

### Why this is non-obvious
The command reports "N/N skills synced ✅" with no mention of deleted skills.
A developer would reasonably expect `--all` to produce a complete mirror of
the source, matching both additions and deletions.
