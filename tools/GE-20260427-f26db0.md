## Context

Playwright tests for a Three.js WebGL visualizer need to trigger an inspect action (fire raycaster → fetch unit data → populate DOM panel → show panel) and then assert on the panel content. The naive approach fires the action and then polls with `waitForFunction`:

```java
// Fragile — fetch may not complete before waitForFunction first polls
page.evaluate("() => window.__test.clickUnit(tag, false)");
page.waitForFunction("() => window.__test.panelVisible()");
```

This is a race: `page.evaluate()` returns as soon as `clickUnit()` returns (synchronously), but the fetch inside `clickUnit` is still in flight. `waitForFunction` polls at 100ms intervals. In practice it timed out consistently even though the fetch completed in <10ms.

## Technique

Make the `window.__test` helper `async` and internally `await` the full side-effect chain. Then call it with `page.evaluate("async () => ...")` — Playwright awaits the returned Promise before returning to Java.

```javascript
// In visualizer.js window.__test:
clickUnit: async (tag, isEnemy = false) => {
  const sp = isEnemy ? enemySprites.get(tag) : unitSprites.get(tag);
  if (!sp || !camera) return false;
  const ndc = sp.position.clone().project(camera);
  ndcMouse.x = ndc.x; ndcMouse.y = ndc.y;
  raycaster.setFromCamera(ndcMouse, camera);
  const hits = raycaster.intersectObjects([...unitSprites.values(), ...enemySprites.values()]);
  if (hits.length === 0) return false;
  await showUnitPanelAsync(hits[0].object.userData.unitTag, hits[0].object.userData.isEnemy);
  return true;  // resolves AFTER panel is visible
},
```

```java
// In test — page.evaluate awaits the async function's Promise:
boolean hit = (boolean) page.evaluate(
    "async () => window.__test.clickUnit('" + tag + "', false)");
assertTrue(hit);
// Panel is guaranteed visible here — no waitForFunction needed
assertTrue((Boolean) page.evaluate("() => window.__test.panelVisible()"));
```

## Why this is non-obvious

Most Playwright tutorials use `waitForFunction` for async side effects — it's the documented pattern. The alternative (making the test helper itself async and awaiting inside `page.evaluate`) requires knowing that Playwright's Java `evaluate()` automatically awaits Promises returned by async functions. The result is deterministic: the evaluation doesn't return until the full pipeline completes, eliminating any need for polling or sleeps.

The companion `showUnitPanelAsync` function uses `await fetch(...)` rather than `.then()` chains — this is what makes errors visible rather than silently swallowed (see related gotcha on `.then()` error suppression).
