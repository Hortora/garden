## Technique

Before writing any Java (or other typed-language) parsing code against an unknown ZIP/JSON format, use a Python one-liner to inspect actual field names, nesting, compression method, and sample values directly from the file.

This catches: field name mismatches (e.g. `evtTypeName` vs assumed `_event`), encoding differences (e.g. raw integers vs fixed-point ×4096), and unexpected compression methods (e.g. BZip2 outer ZIP that `java.util.zip` can't handle) — before a line of production code is written.

```bash
# Inspect outer ZIP structure and compression
python3 -c "
import zipfile
with zipfile.ZipFile('path/to/outer.zip') as z:
    for info in z.infolist():
        print(info.filename, 'compress_type:', info.compress_type)  # 8=DEFLATE, 12=BZip2
"

# Inspect nested ZIP + sample JSON field names
python3 -c "
import zipfile, io, json
with zipfile.ZipFile('outer.zip') as outer:
    inner_bytes = outer.read('inner_data.zip')
    with zipfile.ZipFile(io.BytesIO(inner_bytes)) as inner:
        sample = json.loads(inner.read(inner.namelist()[0]))
        print('Top-level keys:', list(sample.keys()))
        events = sample.get('trackerEvents', [])
        print('Event type names:', set(e.get('evtTypeName') for e in events[:50]))
        print('First 3 events:', json.dumps(events[:3], indent=2)[:600])
"

# Verify specific field encoding (e.g. is food fixed-point or raw?)
python3 -c "
import zipfile, io, json
with zipfile.ZipFile('outer.zip') as outer:
    inner_bytes = outer.read('inner.zip')
    with zipfile.ZipFile(io.BytesIO(inner_bytes)) as inner:
        sample = json.loads(inner.read(inner.namelist()[0]))
        for e in sample['trackerEvents']:
            if e.get('evtTypeName') == 'PlayerStats':
                print('foodUsed:', e['stats']['scoreValueFoodUsed'])
                # 12 = raw integer; 49152 = fixed-point ×4096
                break
"
```

## Why this is non-obvious

Most developers writing typed-language parsers draft code against assumed field names from documentation or related formats, then debug failures at runtime. The Python stdlib one-liner approach takes 2 minutes and eliminates an entire class of trial-and-error. Python's `zipfile` and `json` are zero-setup. The `compress_type` field on `ZipInfo` reveals BZip2 before any Java code is written. Value inspection catches encoding assumptions (fixed-point vs raw) that documentation may not state.
