### Context
Facing an architecture decision (e.g. LLM agent identity model for an audit ledger) with
multiple viable options but no clear winner from first principles. Standard web search
returns surface-level results; ChatGPT-style prompting produces plausible-sounding but
unverified recommendations.

### Technique
Spawn a dedicated research subagent with a precise multi-query brief covering:
- Specific academic sources (arxiv, Google Scholar)
- Standards bodies (W3C, NIST, ISO, EU regulatory text)
- Industry framework implementations (LangChain, AutoGen, CrewAI docs)
- Named papers and RFCs where known

Structure the brief as explicit numbered search targets, not open-ended questions:
```
1. Web search: "LLM agent identity accountability audit"
2. Web search: "W3C PROV agent identity"
3. arxiv search: "agent identity accountability provenance" academic papers
4. Search: "NIST AI agent identity", "ISO AI traceability agent"
5. Search: "LangChain agent identity", "AutoGen agent accountability"
```

Return format: findings per source, consensus view, gaps/open questions.

### Why this is non-obvious
The instinct is to ask the model directly ("what's the best approach to X?") or do
ad-hoc web searches. Spawning an agent with a structured multi-source brief produces
a qualitatively different result: actual citations, contradictions between sources
surfaced, and awareness of what is vs isn't standardised. The synthesised output
directly informs design decisions with academic backing rather than model priors.

In this pattern, the design work stays in the main context; the research subagent
handles the retrieval. The human synthesises from the agent's report — delegation
of search, not delegation of understanding.
