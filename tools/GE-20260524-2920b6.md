### Symptom
Arrowhead marker on a vertical SVG line appears rotated 90° — pointing sideways instead of along the line.

### Context
Using `<marker orient="auto">` with a `marker-end` on a vertical line (e.g. an upward-pointing architecture diagram arrow). The marker path was drawn visually pointing "up" in marker coordinates.

### Root cause
`orient="auto"` rotates the marker so its **positive x-axis** aligns with the line direction. For an upward line (−y direction), the marker is rotated −90°. A path drawn pointing "up" in marker space (toward y=0) ends up pointing sideways after rotation.

### Fix
Draw the marker path pointing to the **right (+x direction)** in marker coordinates. Set `refX` to the tip's x-coordinate (rightmost point) and `refY` to the tip's y-coordinate (vertical center):

```svg
<marker id="arr" markerWidth="9" markerHeight="8" refX="8" refY="4" orient="auto">
  <path d="M0,0 L8,4 L0,8" fill="none" stroke="#1a2e38" stroke-width="1.2"/>
</marker>
```

For an upward line, `orient="auto"` rotates this rightward chevron so it points upward. The tip at `(8,4)` lands exactly on the line endpoint.

### Why this is non-obvious
The intuition is to draw the arrowhead in the direction you want it to appear. But `orient="auto"` defines "forward" as the marker's positive x-axis — the same as a horizontal line pointing right. The visual direction only emerges after rotation. Drawing "up" in marker space produces a leftward arrow on screen for an upward line.
