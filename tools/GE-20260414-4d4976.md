## Symptom

A `CROSS_CHECK` function always returns `[]` in tests. No error, no warning — tests
expecting it to flag a problem pass silently for the wrong reason. Invoking the check
function directly returns the expected issue; calling `validate()` does not.

## Context

A `validate(md, slug, html_path=None)` function runs `MD_CHECKS` unconditionally and
`CROSS_CHECKS` only when `html_path is not None and html_path.exists()`. Tests written
without providing `html_path` — because "the check is about the Markdown, right?" —
never trigger cross-checks at all.

## What Was Tried

- Checking the function in isolation — works correctly
- Verifying the function is registered in `CROSS_CHECKS` — it is
- Printing intermediate results — the cross-check loop body never executes

## Root Cause

`validate()` gates the entire `CROSS_CHECKS` loop on `html_path and html_path.exists()`.
No individual check runs without HTML — even checks that only incidentally reference
the HTML (e.g. for false-positive suppression) are skipped entirely. The gate is at
the loop level, not per-function.

## Fix

In tests for cross-checks, always provide an HTML file even if the check's primary
logic doesn't depend on it. Write a minimal HTML fixture and pass it as `html_path`:

```python
with tempfile.NamedTemporaryFile(mode='w', suffix='.html', delete=False) as f:
    f.write('<html><body><article><p>content</p></article></body></html>')
    html_path = Path(f.name)
try:
    issues = validate(md, 'slug', html_path)
finally:
    html_path.unlink()
```

## Why This Is Non-Obvious

The check function itself doesn't need HTML to produce output — it works in isolation.
The gating happens in `validate()`, not in the check. The function signature
(`html_path=None`) implies HTML is optional for *all* checks, but it's actually
required for the entire cross-check category to run. Tests get empty results with
zero indication that anything was skipped.
