## Symptom

`mvn deploy` on a SNAPSHOT version fails with `403 Forbidden` when deploying a child module. The error URL points to the *parent* POM's registry, not the child's. CI succeeds for release versions but fails for SNAPSHOTs.

```
Could not transfer artifact io.example:child-parent:pom:0.2-20260427.144117-1
  from/to github (https://maven.pkg.github.com/org/parent-repo): 403 Forbidden
```

## Context

Child POM has a `distributionManagement` block with `<repository>` (releases) but no `<snapshotRepository>`. Parent POM has both pointing to the parent's registry. The CI token only has write access to the child's registry.

## Root cause

Maven uses `<repository>` for release versions and `<snapshotRepository>` for SNAPSHOT versions. When `<snapshotRepository>` is absent from the child POM, Maven inherits it from the parent — which points to a different registry (the parent's). The token cannot write there → 403.

## Fix

Explicitly declare both elements in the child's `distributionManagement`:

```xml
<distributionManagement>
  <repository>
    <id>github</id>
    <url>https://maven.pkg.github.com/org/child-repo</url>
  </repository>
  <snapshotRepository>
    <id>github</id>
    <url>https://maven.pkg.github.com/org/child-repo</url>
  </snapshotRepository>
</distributionManagement>
```

## Why this is non-obvious

Maven's `<repository>` and `<snapshotRepository>` are independent settings that inherit separately. A child POM that defines `<repository>` still inherits `<snapshotRepository>` from the parent. The 403 error message shows the wrong URL (the parent's), which points to a permissions problem rather than a configuration problem — leading developers to investigate token scopes first.
