### Symptom
Calling `mcp__intellij__search_file` with a `globPattern` parameter fails:
`MCP tool call has been failed: No argument is passed for required parameter 'q'`

### Context
Two IntelliJ MCP tools both search for files by glob pattern but use different
parameter names: `mcp__intellij__search_file` uses `q`; `mcp__intellij__find_files_by_glob`
uses `globPattern`. The error message doesn't name the correct parameter.

### Root cause
Two separate MCP tools from different IntelliJ MCP servers with overlapping
functionality but inconsistent parameter naming conventions.

### Fix
Use `q` for `mcp__intellij__search_file`:
```
// Wrong
mcp__intellij__search_file(globPattern: "LedgerEntry.java")

// Right
mcp__intellij__search_file(q: "LedgerEntry.java")

// Or switch to the other tool which uses globPattern
mcp__intellij__find_files_by_glob(globPattern: "**/*.java")

// Third alternative — mcp__intellij-index__ide_find_file uses `query`
mcp__intellij-index__ide_find_file(query: "LedgerEntry.java")
```

### Why non-obvious
All three tools are described as glob/name-based file searches. The parameter name
mismatch only surfaces at runtime. Three tools, three parameter names: `q`,
`globPattern`, `query` — with no cross-reference in any tool description.
