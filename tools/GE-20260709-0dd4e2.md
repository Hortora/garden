### Symptom
IntelliJ Project view shows only "External Libraries" — no source directories, no packages, no files. The project appears to have no content roots despite the directory containing TypeScript sources, package.json, and yarn.lock.

### Context
Discovered when opening casehub-blocks-ui (TypeScript/Yarn workspace monorepo) and casehub-pages via `ide_open_project` through the IntelliJ MCP Index Plugin. Both projects opened successfully (`ide_project_status` showed them as active) but the Project tab had no source tree — only the External Libraries node.

### What was tried (didn't work)
1. `ide_open_project` — project opens but no source tree visible
2. `ide_sync_files` — no effect, still only External Libraries
3. Assumed indexing issue — `ide_index_status` showed `isDumbMode: false`

### Root cause
IntelliJ requires a `.iml` module file and `.idea/modules.xml` to define content roots. For Maven/Gradle projects, IntelliJ auto-generates these on import. For TypeScript/Yarn projects opened programmatically via `ide_open_project`, no auto-detection runs — the project opens with no module definition, so IntelliJ has no content roots and shows nothing in the Project view. This also causes all search/navigation tools to return empty (see GE-20260701-f1c67f).

### Fix
Create two files in `.idea/`:

**`.idea/<project-name>.iml`:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<module type="WEB_MODULE" version="4">
  <component name="NewModuleRootManager" inherit-compiler-output="true">
    <exclude-output />
    <content url="file://$MODULE_DIR$">
      <excludeFolder url="file://$MODULE_DIR$/node_modules" />
    </content>
    <orderEntry type="inheritedJdk" />
    <orderEntry type="sourceOrder" />
  </component>
</module>
```

**`.idea/modules.xml`:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<project version="4">
  <component name="ProjectModuleManager">
    <modules>
      <module fileurl="file://$PROJECT_DIR$/.idea/<project-name>.iml"
              filepath="$PROJECT_DIR$/.idea/<project-name>.iml" />
    </modules>
  </component>
</project>
```

Then close and reopen the project (`ide_close_project` + `ide_open_project`).

**For polyglot projects** (TypeScript root + Maven subdirectory), create the web module for the root, then after reopening use `ide_import_modules` to add Maven subprojects:
```
ide_import_modules paths=["/path/to/project/app"] project_path="/path/to/project"
```

### Why this is non-obvious
- The project opens successfully via MCP — `ide_project_status` reports it as active with no errors
- The symptom (empty Project view) looks like a UI or indexing bug, not a configuration issue
- Maven/Gradle projects never have this problem because IntelliJ auto-generates module config on import
- `ide_index_status` reports no indexing in progress, suggesting everything is fine
- The fix requires knowledge of IntelliJ's internal module system (.iml files) which most TypeScript developers never encounter
