### What it does

The Semantic Scholar public API exposes a `/citations` endpoint that returns all papers citing a given paper — no authentication required, no scraping needed.

### How to use it

```python
import requests, json

paper_id = "e3fb46acded22a5a2ebe632425317cede730d7e4"  # from Semantic Scholar URL
url = f"https://api.semanticscholar.org/graph/v1/paper/{paper_id}/citations"
params = {
    "fields": "title,year,authors,venue,abstract",
    "limit": 100
}
response = requests.get(url, params=params)
data = response.json()

for paper in data["data"]:
    citing = paper["citingPaper"]
    print(f"{citing.get('year','?')} | {citing.get('title','?')} | {citing.get('venue','?')}")
```

The paper ID is the hex string in the Semantic Scholar URL:
`https://www.semanticscholar.org/paper/<title>/<ID>`

### Why this is non-obvious

The natural approach to "find all papers citing X" is either:
1. Manually browse Semantic Scholar's web UI (not scriptable)
2. Try to scrape the web page (brittle, rate-limited)
3. Use Google Scholar scraping tools (fragile, against ToS)

The public REST API exists, is free, requires no API key for basic usage, and returns structured JSON. It handles pagination via `offset` and `limit` params. Not prominently documented — discoverable only from the API reference pages, not from the main Semantic Scholar site.

### Limits

- 100 results per request (paginate with `offset`)
- Rate limits apply for high-volume use (register for an API key)
- `abstract` field may be empty for older papers
