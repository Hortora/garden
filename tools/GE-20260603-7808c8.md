### What it is
`mcp__intellij-index__ide_find_class` with `scope: "project_and_libraries"` finds classes inside Maven dependency JARs that IntelliJ has indexed. Following up with `ide_find_definition` on the found class name opens the source (from `-sources.jar` if attached, or IntelliJ's decompiler if not). This works for beta/snapshot JARs in the local `.m2` repository.

### Why a skilled developer wouldn't reach for this
When needing to understand an upstream library's internals, the instinct is to clone the upstream repo, find the file, and read it directly. The IntelliJ MCP approach skips that entirely — if the JAR is in the Maven local repo and the project has been opened in IntelliJ, the IDE has already indexed it and can navigate into it immediately.

### How to apply
```json
// Step 1: Find the class in libraries
{
  "tool": "mcp__intellij-index__ide_find_class",
  "query": "DeclarativeUtil",
  "scope": "project_and_libraries"
}
// Returns: file path including jar:// URL or library source path

// Step 2: Navigate to definition (triggers decompilation if no sources)
{
  "tool": "mcp__intellij-index__ide_find_definition",
  "language": "Java",
  "symbol": "dev.langchain4j.agentic.declarative.DeclarativeUtil"
}
// Returns: decompiled source with method signatures, field declarations, etc.
```

### What this enables
- Confirming upstream API contracts for libraries with no published documentation
- Discovering undocumented SPIs and extension points by reading the actual class structure
- Verifying that a beta library method signature matches what your code calls
- Understanding private implementation details without cloning the upstream repo

### Constraints
- Requires the project to be open in IntelliJ and Maven sync to have run (so the JAR is in the IDE's library index)
- Works best when the local `.m2` repository contains the JAR — remote-only dependencies may not be fully indexed
- `ide_read_file` does NOT accept the `jar://` paths that `ide_find_class` returns (see gotcha GE-20260603-a4e92f); use `ide_find_definition` instead
