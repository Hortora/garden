## Read any branch's files without checkout using git show and git ls-tree

### Technique

Inspect branch content, journal entries, artifact presence, and closure markers across multiple branches without switching — no stash, no lost work, no context switch.

```bash
# Read a specific file from a branch
git show epic-foo:design/JOURNAL.md

# List directory contents on a branch
git ls-tree -r --name-only epic-foo -- specs/

# Check if a file exists on a branch
git show epic-foo:EPIC-CLOSED.md 2>/dev/null || echo "(absent)"

# Check unmerged commits
git log main..epic-foo --oneline

# Run all hygiene checks for a branch without touching it:
branch="epic-foo"
echo "=== Unmerged ===" && git log main..$branch --oneline
echo "=== Specs ===" && git ls-tree -r --name-only $branch -- specs/
echo "=== Journal ===" && git show $branch:design/JOURNAL.md 2>/dev/null | grep "^### "
echo "=== Closure ===" && git show $branch:EPIC-CLOSED.md 2>/dev/null | head -4
```

### Why this is non-obvious

Most developers reach for `git checkout` or `git stash && git checkout` when they need to inspect another branch. `git show <branch>:<path>` is rarely the first instinct but avoids all the overhead of a context switch — no index update, no working tree modification, no lost in-progress state.
