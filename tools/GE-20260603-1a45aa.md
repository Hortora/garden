### Symptom
Calling `mcp__intellij-index__ide_find_references` with `file` and `line` but without `column` returns a parameter error. No results returned.

### Context
Position-based tools in `mcp__intellij-index__` require all three of `file`, `line`, and `column` when using position-based lookup. Tools that offer an alternative `language + symbol` lookup can work without position, but the position-based path requires all three values.

### What was tried
Calling with `file` + `line` only, assuming `column` might default to 1.

### Root cause
`column` has no default. The tool schema marks it as required when `file` is provided. The error message on failure does not clearly state that `column` is the missing field.

### Fix
Always supply all three position parameters:
```json
{
  "tool": "mcp__intellij-index__ide_find_references",
  "file": "agentic/runtime/src/main/java/io/quarkiverse/langchain4j/agentic/runtime/AgenticRecorder.java",
  "line": 99,
  "column": 1
}
```

Column is 1-based. If you want to find references to a symbol starting on a line, use column 1 and the IDE resolves the symbol at that position.

### Why this is non-obvious
Many position-based IDE tools make column optional (defaulting to start of line). `ide_find_references` does not. The tool description says "Required when using file or line" but this is easy to miss when calling programmatically.
