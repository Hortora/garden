macOS /var/folders has two directory levels — single-glob rm silently matches nothing

### Symptom

`rm -rf /var/folders/*/T/casehub-ingestion-cursors/` exits 0 with no output. The directory still exists. No error, no "No such file or directory" — the glob simply expanded to nothing.

### Context

macOS per-user temp directories use two levels of hashing:
```
/var/folders/<2-char>/<26-char>/T/
# e.g. /var/folders/8m/0v940n7x1m5gy95zvnl3ms700000gn/T/
```

A single `*` glob matches one directory level only. `/var/folders/*/T/` tries to match `/var/folders/8m/T/` which doesn't exist.

### Fix

Use `$TMPDIR` (always set by macOS to the correct two-level path) or double-glob:

```bash
# Correct — use the environment variable
rm -rf "$TMPDIR/casehub-ingestion-cursors/"

# Also correct — double glob
rm -rf /var/folders/*/*/T/casehub-ingestion-cursors/
```

### Why this is non-obvious

The glob silently expands to nothing — no error, no warning. `rm -rf` with zero arguments exits 0. The command appears to succeed. You only discover the failure when the supposedly-deleted directory is still there, and even then the diagnosis is "but I deleted it" — the glob mismatch is not the first suspect.
