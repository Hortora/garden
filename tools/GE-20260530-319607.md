## What it achieves

When a public Java method gains new trailing parameters (e.g. a `@Tool` method adds 4 optional `@ToolArg` params), every test file calling the old N-arg version needs N trailing `null` args added. For 50+ files with single-line and multi-line call sites, regex fails because call arguments can span lines.

## The technique

Walk the source file character-by-character tracking balanced parentheses and string literals. When a `createChannel(` (or any target method) opening paren is found, extract the full call text up to the matching `)`, count top-level commas (=arg count), and if it matches the old arity, inject trailing nulls.

```python
def count_top_level_args(inner: str) -> int:
    depth, in_str, count = 0, False, 1
    for ch in inner:
        if ch == '"' and not in_str: in_str = True
        elif ch == '"' and in_str: in_str = False
        elif not in_str:
            if ch == '(': depth += 1
            elif ch == ')': depth -= 1
            elif ch == ',' and depth == 0: count += 1
    return count

def fix_file(path: str, method: str = 'createChannel', old_arity: int = 9, add: int = 4):
    with open(path) as f:
        content = f.read()
    result, i, changed = [], 0, False
    while i < len(content):
        idx = content.find(f'{method}(', i)
        if idx == -1:
            result.append(content[i:]); break
        result.append(content[i:idx + len(method)])
        j = idx + len(method)
        depth, k = 0, j
        while k < len(content):
            if content[k] == '(': depth += 1
            elif content[k] == ')':
                depth -= 1
                if depth == 0:
                    call = content[j:k+1]
                    if count_top_level_args(call[1:-1].strip()) == old_arity:
                        result.append(call[:-1] + ', ' + ', '.join(['null']*add) + ')')
                        changed = True
                    else:
                        result.append(call)
                    i = k + 1; break
            k += 1
        else:
            result.append(content[j:]); break
    if changed:
        with open(path, 'w') as f:
            f.write(''.join(result))
```

Run with `os.walk()` over the test source tree:

```python
import os
for dirpath, _, files in os.walk('src/test'):
    for fname in files:
        if fname.endswith('.java'):
            fix_file(os.path.join(dirpath, fname))
```

## Why this is non-obvious

Most developers reach for `sed` with a regex like `s/null, null, null, null, null, null);/null, null, null, null, null, null, null, null, null, null);/g`. This works for single-line calls with a predictable suffix but breaks when:
- The last few args are non-null (e.g. `"EVENT"` for `allowed_types`)  
- The call spans multiple lines with the closing `)` on a different line
- The old regex suffix doesn't account for every variant

The balanced-paren approach counts arity precisely regardless of line breaks or non-null trailing values. It correctly updates `createChannel("name", "APPEND", null*7)` AND `createChannel("name", "desc", "APPEND", null*5, "EVENT")` in the same pass.

### Alternatives considered

- **IntelliJ structural search** — works for single-project use within the IDE; not scriptable for CI or batch-from-terminal; doesn't handle multi-module source trees without per-module runs
- **`sed` regex** — fast but brittle for multiline calls and non-null-only trailing args
- **JavaParser / javaparser library** — correct AST-based approach but 10× more setup for a one-off migration
