## Technique

Bulk-add issues from multiple repos to a GitHub Projects V2 board using GraphQL `addProjectV2ItemById` mutations, driven by node IDs fetched with `gh issue list --json id`.

To remove items from a project, use `deleteProjectV2Item` with the *project item ID* (not the issue node ID) â€” fetched via querying `projectV2.items.nodes[].id`.

## When to use

- Adding 10+ issues across multiple repos to a single org-level project
- Need explicit confirmation that each item was added (the `gh project item-add --url` command gives no signal)
- Removing specific items from a project by issue identity

## How

**Collect node IDs:**
```bash
gh issue list --repo org/repo --label my-label --state all --limit 200 \
  --json id --jq '.[].id' >> /tmp/node_ids.txt
```

**Add to project (write to temp file first â€” avoid subshell counter issue):**
```bash
PROJECT_ID="PVT_kwDO..."   # from: gh project list --owner org --format json
SUCCESS=0; FAILED=0

while IFS= read -r node_id; do
  [[ -z "$node_id" ]] && continue
  result=$(gh api graphql -f query='
mutation($project: ID!, $item: ID!) {
  addProjectV2ItemById(input: {projectId: $project, contentId: $item}) {
    item { id }
  }
}' -f project="$PROJECT_ID" -f item="$node_id" 2>&1)
  echo "$result" | grep -q '"id"' && ((SUCCESS++)) || { echo "FAILED: $node_id"; ((FAILED++)); }
done < /tmp/node_ids.txt

echo "Added: $SUCCESS, Failed: $FAILED"
```

**Get project item IDs (needed for deletion):**
```bash
gh api graphql -f query='
{
  organization(login: "myorg") {
    projectV2(number: 2) {
      items(first: 100) {
        nodes {
          id
          content {
            ... on Issue { number repository { nameWithOwner } }
          }
        }
      }
    }
  }
}' --jq '.data.organization.projectV2.items.nodes[] | "\(.id) \(.content.repository.nameWithOwner)#\(.content.number)"'
```

**Remove an item:**
```bash
gh api graphql -f query='
mutation($project: ID!, $item: ID!) {
  deleteProjectV2Item(input: {projectId: $project, itemId: $item}) {
    deletedItemId
  }
}' -f project="$PROJECT_ID" -f item="$ITEM_ID"
```

## Why this is non-obvious

`gh project item-add --url` is the documented CLI path but gives no confirmation signal (exit 0, no output, on both success and silent failure). The GraphQL path â€” getting node IDs with `--json id` and using `addProjectV2ItemById` â€” is not in the main `gh project` docs. The distinction between issue node ID and project item ID (different namespaces, needed for different operations) is also undocumented.
