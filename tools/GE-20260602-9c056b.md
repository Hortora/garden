## What happens

Anti-slop guidance accumulates banned words and structural patterns ("no delve", "no 'it's important to note'"). Despite the lists, AI-sounding prose keeps appearing. Adding more banned words doesn't solve it. The lists treat symptoms.

## Root cause

LLMs default to the prose end of the structure–prose spectrum due to statistical pressure from training corpora dominated by prose. When asked to write a Gotcha section, the LLM generates explanation-mode prose because that's what the training distribution pulls toward — not because it's banned-word-free.

## The technique

Identify the required mode before generating. Generate to that mode's structural constraints. The mode determines the structure; the structure prevents the banned words from appearing.

- How-to/diagnostic mode → Symptom → Cause → Fix (forced structure prevents flowing prose)
- Reference/inventory mode → `path — one sentence` (forces telegraphic format)
- Tutorial mode → numbered imperative steps (forces sequential, not narrative)

If the mode is wrong, fixing individual word choices is fighting downstream effects.

## Why non-obvious

The instinct is to add more banned words to anti-slop lists. The non-obvious fix is to go upstream: identify the mode required by the content, then constrain the generation to that mode's structural form. The banned words become self-enforcing side effects of correct mode selection.

**Validated empirically:** arc42stories sections with explicit structural prescriptions (Symptom → Cause → Fix) stayed in mode across every implementation. Sections with only narrative instructions ("teaching narrative, explain what this layer adds") drifted to prose. Same authors, same tool, same document — the structural prescription was the only differentiator.

### Why this fix

The prose default is a distributional artifact, not a vocabulary problem. Constraining structure at the generation prompt level overrides the distributional pull before it has any effect. Word-level filtering operates on the output of a wrong-mode generation and can never fully compensate.
