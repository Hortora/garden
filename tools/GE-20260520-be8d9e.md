## Symptom
After running `git filter-repo --force --refs refs/heads/<branch>`, the range
`origin/<branch>..HEAD` returns an enormous number of commits (17,000+), as if
the entire repo history is ahead of origin. The merge-base computation similarly
returns the repository root rather than the actual branch divergence point.

## Context
Running filter-repo with `--force` on a working branch in an existing (non-fresh)
clone to strip a specific file (e.g. session handover artifacts). The intent is
to clean only the 4–5 unpushed commits on the working branch.

## What was tried (didn't work)
```bash
# After filter-repo, this returns ~17,000 commits instead of 4:
git log --oneline origin/droolsvol2..HEAD

# And this returns the repo root SHA instead of the branch base:
git merge-base droolsvol2 HEAD
```

## Root cause
`git filter-repo --force` rewrites **all commits in the entire repository** when
used on a non-fresh clone, even with `--refs refs/heads/<branch>`. The `--refs`
flag limits which refs are *updated* to point at the new SHAs, but the rewrite
itself touches the full commit graph. After the rewrite, the working branch's
SHAs share no common ancestry with origin's SHAs — so `origin/branch..HEAD`
spans the entire rewritten history.

## Fix
After filter-repo runs on an existing repo, **do not use remote-relative ranges**.
Count from HEAD backwards instead:

```bash
# If you know you had N unpushed commits before filter-repo, use:
git log --oneline HEAD~N..HEAD

# Or verify by counting up from the expected base content:
git log --oneline HEAD~5..HEAD   # check which N is correct
```

Cross-reference with the pre-filter-repo commit subjects (known from context)
to confirm you have the right window.

## Why this is non-obvious
The `--refs` flag strongly implies the operation is scoped to the named branch.
"I'm only rewriting refs/heads/my-branch, so only those commits change" is the
natural reading — but filter-repo's rewrite is not scope-aware at the graph
traversal level. The scope only controls which ref pointers are updated at the
end. All parents in the graph are rewritten, disconnecting the working branch
from any shared history with origin.
