### What it is
Dispatching 6 independent parallel agents, each assigned a specific audit dimension and explicitly required to use `mcp__intellij-index__` tools, produces findings that bash/grep cannot replicate. The agents decompile upstream library JARs, follow type hierarchies, trace all callers of internal methods, and verify findings against actual source code — not text patterns.

### Why a skilled developer wouldn't reach for this
The instinct for a codebase audit is to grep for patterns, read key files, and reason from there. Grep operates on text and misses semantic relationships (e.g. which classes implement a particular interface, what methods override a base class method, which exact code path calls a given internal API). The IntelliJ MCP tools resolve these semantically, including into library JARs the project depends on but doesn't own.

### How to apply
In each agent prompt, make MCP tool usage **mandatory and explicit**:
```
You MUST use mcp__intellij-index__* tools for ALL code navigation.
Do NOT use Bash grep/find as a substitute — use the IDE tools which are
faster and semantically accurate.
```

Assign each agent a focused dimension (e.g. "concurrency and context propagation", "CDI injection patterns", "observability — OTel, metrics, health"). Six dimensions run concurrently and complete in the time one sequential pass would take.

### Key capabilities this unlocks
- **Library JAR decompilation**: `ide_find_class scope: "project_and_libraries"` + `ide_find_definition` navigates into beta JARs and decompiles on demand — confirms upstream API limitations by reading bytecode, not documentation
- **Reference tracing**: `ide_find_references` across the whole project finds every call site, not just text occurrences of a name
- **Type hierarchy**: `ide_type_hierarchy` and `ide_find_implementations` confirm interface/implementation relationships that grep would miss through aliases and imports
- **Cross-cutting search**: `ide_search_text` with `filePattern` scopes searches to specific file types without path gymnastics

### Constraints
Requires the project to be open in IntelliJ and fully indexed before agents run. Agents must be instructed to use `project_path` when the project is not the default open project. The `mcp__intellij-index__` server (not the built-in `mcp__intellij__`) supports auto-opening via `project_path`.
