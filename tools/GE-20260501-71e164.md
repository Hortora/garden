## What It Is

`gh api` exits with code **4** specifically to indicate an authentication failure (invalid token, expired token, or missing scope). This is distinct from other exit codes:

- `0` — success
- `1` — general error
- `2` — invalid usage / bad arguments
- `4` — authentication error

## Context

Useful when writing diagnostic scripts that need to distinguish between "the API call failed because the token is bad" vs "the call failed for another reason". Without knowing this, you might loop on retry logic or report misleading errors to the user.

## Why It's Undocumented

The `gh` CLI manual (`gh help api`) describes the `--fail` flag and general error handling but does not enumerate exit codes. The exit code 4 / authentication distinction is not in the official GitHub CLI documentation as of 2026.

## Usage

```bash
if gh api /user > /dev/null 2>&1; then
  echo "authenticated"
elif [ $? -eq 4 ]; then
  echo "token is invalid or expired — check GH_TOKEN"
else
  echo "API error"
fi
```
