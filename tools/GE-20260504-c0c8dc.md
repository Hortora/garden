### Symptom

Mid-session, both `mcp__intellij__get_project_modules` and `mcp__intellij-index__ide_index_status` return errors:
- `mcp__intellij`: HTTP 404 "Session not found"
- `mcp__intellij-index`: "Unable to connect. Is the computer able to access the url?"

All subagents that check MCPs as mandatory immediately report BLOCKED. Execution halts.

### Context

Happens when IntelliJ is restarted while Claude Code is running. The MCPs are tied to the
IntelliJ process session, not to the Claude session. Any restart — voluntary (update, settings
change) or involuntary (crash) — drops both MCP connections.

### What was tried

Retrying the same MCP calls from a subagent — fails identically.

### Root cause

The IntelliJ MCPs are served by an in-process plugin that registers a local HTTP server on
IntelliJ startup. When IntelliJ is restarted, the old server process terminates and a new
one starts — with a new session ID. Claude's existing MCP connection references the dead session.

### Fix

Wait for IntelliJ to fully restart and re-index, then verify from the main session:
```python
# Verify both MCPs are live before dispatching any subagent
mcp__intellij__get_project_modules(projectPath=...)    # must succeed
mcp__intellij-index__ide_index_status(project_path=...)  # isDumbMode must be false
```

If either fails, stop and surface to the user before dispatching subagents. Do not continue
with bash-only fallbacks when the user's standing instruction is "check MCPs first".

### Why this is non-obvious

The Claude session and the IntelliJ MCP session look independent — Claude is still running,
the terminal is still open. There's no visible indication that the MCP connection was severed
until a subagent tries to use it and receives HTTP 404. The error message "Session not found"
is ambiguous — it sounds like a coding error, not an IDE restart.

### Constraints

Applies to any workflow that uses `mcp__intellij` or `mcp__intellij-index` in subagents
with mandatory pre-flight MCP checks.

### Invalidation triggers

Revisit if IntelliJ MCPs are redesigned to reconnect automatically after IDE restart.
