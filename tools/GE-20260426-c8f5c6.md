### Symptom
`defaults:` entries in `_config.yml` (layout, title, permalink,
`render_with_liquid`) silently have no effect on specific markdown files.
The files are copied verbatim to `_site/` rather than rendered as pages.

### Context
Occurs when adding existing repo-root markdown docs (e.g. `OVERVIEW.md`,
`ARCHITECTURE.md`) as Jekyll pages. The natural approach — add a `defaults:`
entry pointing at the file — produces no output and no error.

### What was tried
Added `defaults:` entries with `layout: doc`, `title:`, `permalink:`, and
`render_with_liquid: false` targeting specific `.md` files. Jekyll built
successfully but treated the files as static assets, copying them unchanged.

### Root cause
Jekyll only processes files as *pages* (applying layouts, defaults,
Liquid) if they contain a YAML frontmatter block. Files without any
frontmatter block are static assets — `defaults:` is evaluated only
after Jekyll classifies the file as a page.

### Fix
Add an empty frontmatter block to each file:

```yaml
---
---
```

GitHub renders this as invisible (no visible YAML content). Pair with
`render_with_liquid: false` in `defaults:` to prevent Jekyll from
processing `${ }` expressions in the content as Liquid templates.

### Why this is non-obvious
The Jekyll docs describe `defaults:` as applying to "pages, posts, and
documents" without making clear that the frontmatter block is a
prerequisite for a file being classified as a page at all. The silent
failure (no error, file just copies verbatim) makes this especially
hard to diagnose.
