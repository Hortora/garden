### Symptom

Editing the hook detection logic in `install-skills/SKILL.md` has no effect on the deployed hook at `~/.claude/hooks/check_project_setup.sh`. After running `sync-local`, the deployed hook still contains the old detection code despite the SKILL.md template being updated.

### Context

When fixing the workspace detection logic in the cc-praxis session-start hook. The hook template in `install-skills/SKILL.md` (the bash code block that shows the hook content) looks authoritative — it's where you'd naturally edit the hook logic.

### What was tried (didn't work)

1. Edited the bash code block in `install-skills/SKILL.md` to use `wksp` symlink detection
2. Committed to cc-praxis
3. Ran `sync-local --all`
4. Hook reverted to old detection code immediately — sync-local regenerated it from the canonical source

Repeated this cycle three times before finding the actual source.

### Root cause

`sync-local` (the `scripts/claude-skill` Python script) maintains a separate constant:

```python
HOOK_SOURCE = Path(__file__).parent.parent / "hooks" / "check_project_setup.sh"
HOOK_DEST = Path.home() / ".claude" / "hooks" / "check_project_setup.sh"
```

It copies `hooks/check_project_setup.sh` directly to the deployment location. The bash code block in `install-skills/SKILL.md` is documentation/reference only — it is NOT used to generate the hook.

### Fix

Edit `~/claude/cc-praxis/hooks/check_project_setup.sh` directly. Then run `sync-local` to deploy it. The SKILL.md template can be updated for documentation consistency but does not affect deployment.

```bash
# Edit the canonical source
vim ~/claude/cc-praxis/hooks/check_project_setup.sh

# Deploy
python3 ~/claude/cc-praxis/scripts/claude-skill sync-local --all -y

# Verify the deployed hook updated
grep -n "wksp\|Session Start" ~/.claude/hooks/check_project_setup.sh
```

### Why this is non-obvious

The `install-skills/SKILL.md` contains a full bash code block showing the hook content, with comments about what the hook does. It reads exactly like the template from which the hook is generated. There is no comment saying "this is documentation only — see hooks/check_project_setup.sh". The only hint that sync-local updates the hook is the message "✅ Session-start hook updated from repo" — with no path shown.
