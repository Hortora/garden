### Symptom
Need to test components that subscribe to Kubernetes Watch events (Watcher interface)
using fabric8's mock server, but it's unclear whether CRUD operations trigger Watch
event delivery.

### Context
When building a K8s Watch-based drift detection component (e.g. a WatchManager that
subscribes to Deployment/Service/ConfigMap changes), integration tests need to verify
that Watch subscribers receive events when resources are created, modified, or deleted.

### What was tried (didn't work)
1. Manually constructing Watch responses on the mock server — complex and brittle
2. Using raw `@EnableKubernetesMockClient` without `crud = true` — Watch requests
   return empty responses

### Root cause
The `KubernetesCrudDispatcher` (enabled by `crud = true`) internally maintains a
resource store. When resources are created/updated/deleted through the CRUD API,
the dispatcher automatically notifies any active Watch subscriptions with the
appropriate `ADDED`/`MODIFIED`/`DELETED` events.

### Fix
Use `@EnableKubernetesMockClient(crud = true)` and exercise the CRUD API directly.
Watch subscribers receive events automatically:

```java
@EnableKubernetesMockClient(crud = true)
class K8sWatchManagerTest {
    KubernetesClient client;

    @Test
    void emitsDriftedOnModification() throws Exception {
        // Start watching
        watchManager.startWatching("cluster-1", "casehub");
        Thread.sleep(200); // Allow Watch subscription to establish

        // Create via CRUD — Watch subscriber receives ADDED
        client.apps().deployments().inNamespace("casehub")
            .resource(deployment).create();
        Thread.sleep(200);

        // Update via CRUD — Watch subscriber receives MODIFIED
        deployment.getSpec().setReplicas(3);
        client.apps().deployments().inNamespace("casehub")
            .resource(deployment).update();
        Thread.sleep(500);

        // Assert Watch-triggered event was emitted
        assertThat(events).anySatisfy(e ->
            assertThat(e.newStatus()).isEqualTo(NodeStatus.DRIFTED));
    }
}
```

Note: `Thread.sleep()` delays are needed because Watch event delivery is asynchronous.
Use 200ms for subscription establishment and 500ms for event propagation.

### Why this is non-obvious
fabric8's mock server documentation focuses on HTTP request/response mocking, not on
the CRUD dispatcher's Watch event delivery. A developer would typically try to manually
construct Watch responses or use a real K8s cluster for Watch testing. The CRUD mode's
automatic Watch dispatch is not documented but provides a much simpler testing model.
