## Technique

Create a GitHub org team, add members, and grant write access to multiple repos entirely via `gh api` — no browser or GitHub web UI required:

```bash
# 1. Create the team
gh api -X POST orgs/MY-ORG/teams \
  -f name="developers" \
  -f description="Core developers" \
  -f privacy="closed" \
  -f permission="push" \
  --jq '.slug'

# 2. Add a member
gh api -X PUT orgs/MY-ORG/teams/developers/memberships/USERNAME \
  -f role="member" \
  --jq '.state'

# 3. Grant write (push) access to each repo
for repo in repo1 repo2 repo3; do
  gh api -X PUT orgs/MY-ORG/teams/developers/repos/MY-ORG/$repo \
    -f permission="push"
done
```

This covers all repos in a single scripted session, equivalent to: Org Settings → Teams → Create team → Add repos.

## Why This Is Non-Obvious

The GitHub Teams API endpoints are not prominently documented in the `gh` CLI help (`gh api --help` shows the generic interface). Most developers navigate to the org settings UI. The `-X PUT` + `-f` pattern for teams membership and repo access is only discoverable from the REST API reference, not the CLI docs.

## Context

Useful when setting up CI access for a fork-based workflow — granting write access to an org means the fork's branches can push directly to the org repo, eliminating the need for per-PR approval of fork workflows.
