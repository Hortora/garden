## Symptom

A Playwright test asserting `sceneObjectCount >= N` passes cleanly. In the live browser, fewer than N objects are visible — some are completely hidden by fog or outside the camera view.

## Context

Occurs in any visualizer using Three.js (or similar scene graph) where:
1. Objects are added to the scene unconditionally
2. Fog, culling, or out-of-bounds positioning renders some objects invisible in the browser
3. The test counts scene graph objects rather than visually-rendered pixels

Specific trigger: units placed outside a sight-range radius in a fog-of-war system are added to the Three.js scene but rendered with fully opaque fog planes in front of them.

## What was tried

Placed all N units in a grid across the full map. Test counted N units in `enemyCount()`. Browser showed only a fraction of them.

## Root cause

The test API (`window.__test.enemyCount()`) counts Three.js sprite objects in the scene graph. Fog is applied as separate opaque plane meshes in front of units — the unit sprites are present in the graph but visually occluded. The count API has no awareness of visual occlusion.

## Fix

Two options:

**Option 1 — Constrain unit positions to within the sight radius** so fog never hides them. Calculate the radius from all observation points and verify corners:

```
distance = sqrt((x - obs_x)² + (z - obs_z)²) < sight_radius
```

**Option 2 — Add observer units** scattered across the map to provide sight coverage for the larger area:

```java
// Place friendly observer units at strategic positions
simulatedGame.spawnFriendlyUnitForTesting(UnitType.PROBE, new Point2d(4, 5));
simulatedGame.spawnFriendlyUnitForTesting(UnitType.PROBE, new Point2d(11, 5));
```

## Why this is non-obvious

Scene graph count is a standard way to test renderer output — it's natural to assume "in the scene" means "visible". Fog planes are transparent to count-based tests because they're separate objects. You only discover the mismatch when a human looks at the browser.
