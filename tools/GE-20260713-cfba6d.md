### Symptom
Attempting to edit a Python file using IntelliJ MCP structural editing tools (`ide_edit_member`, `ide_replace_member`, `ide_insert_member`, `ide_file_structure`) returns "Language not supported" — even when the Python project has been successfully opened via `ide_open_project`.

When Claude Code hooks enforce IntelliJ-first editing by blocking the `Edit` tool on source files (`.py`, `.java`, `.ts`, etc.), this creates a deadlock: IntelliJ MCP cannot edit the file (language not supported), and the Edit tool is blocked by the hook.

### Context
Occurs when:
- A standalone Python project (not Maven/Gradle) is opened in IntelliJ via `ide_open_project`
- Claude Code hooks block `Edit`/`Write` on existing source files, enforcing `ide_edit_member` etc.
- The file to edit is `.py`

Does NOT occur for Java, Kotlin, TypeScript, or JavaScript files — these are fully supported by IntelliJ MCP structural editing.

### Root cause
The JetBrains Index MCP plugin's structural editing tools (`ide_edit_member`, `ide_replace_member`, `ide_insert_member`) and `ide_file_structure` use IntelliJ's PSI (Program Structure Interface) for language-aware editing. The plugin only implements PSI support for Java, Kotlin, TypeScript, JavaScript, PHP, Rust, and Markdown. Python PSI support is not implemented, so all structural operations fail with "Language not supported."

Opening the project via `ide_open_project` succeeds (the project is indexed for file search, text search, etc.) but does not enable PSI-based editing.

### Fix
Adjust the Claude Code hook that blocks `Edit` on source files to exempt `.py` files in non-IntelliJ-indexed projects. The hook should check whether the target file's language is supported by IntelliJ MCP before blocking.

Alternatively, the hook can check the project type: if no Maven/Gradle build file is present in the project root, allow `Edit` for Python files.

### Why this is non-obvious
The `ide_open_project` call succeeds, and text-based tools (`ide_search_text`, `ide_find_file`) work on the Python project. The failure only surfaces when attempting structural editing — a developer would reasonably assume that opening a project enables all MCP operations on its files.
