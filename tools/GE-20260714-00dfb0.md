### Symptom
`yarn npm publish --access restricted` sends a PUT to `registry.yarnpkg.com` instead of
`npm.pkg.github.com`, returning 404 Not Found. The `.yarnrc.yml` has the correct
`npmScopes.casehubio.npmRegistryServer` pointing to GitHub Packages.

### Context
Yarn 4 monorepo publishing `@casehubio/*` scoped packages. `.yarnrc.yml` configures the
scope for reading (install works fine). `setup-node` action configures `registry-url` and
`scope`. But `yarn npm publish` does not use the scope registry for writes.

### Root cause
Yarn 4's `npm publish` resolves the publish registry differently from install. The
`npmScopes` configuration in `.yarnrc.yml` governs package resolution (reads) but not
publishing (writes). For publishing, Yarn uses `publishConfig.registry` in the individual
`package.json`, falling back to the default npm registry.

### Fix
Add `publishConfig.registry` to every package.json that needs to publish to GitHub Packages:

```json
{
  "publishConfig": {
    "registry": "https://npm.pkg.github.com"
  }
}
```

### Why this is non-obvious
The `.yarnrc.yml` scope configuration works perfectly for `yarn install`. The assumption
that it also governs `yarn npm publish` is natural — the docs don't explicitly state the
publish path ignores scope configuration. The 404 from `registry.yarnpkg.com` is the only
clue that the wrong registry is being used.
