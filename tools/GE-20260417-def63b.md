## Python one-liner to batch-resolve merge conflicts taking one side uniformly

**ID:** GE-20260417-def63b
**Stack:** Git, Python 3
**Context:** A file has 5+ merge conflicts and all should resolve the same way (all take HEAD, or all take incoming). Manually editing each `<<<<<<`/`=======`/`>>>>>>>` block is tedious.

### The technique

Take the **incoming** side (the commit being applied/rebased) for all conflicts:

```python
python3 -c "
import re

with open('path/to/conflicted/file.java', 'r') as f:
    content = f.read()

def resolve(m):
    return m.group(2)  # group(1) = HEAD, group(2) = incoming

pattern = r'<<<<<<< HEAD\n(.*?)=======\n(.*?)>>>>>>> [^\n]+\n'
content = re.sub(pattern, resolve, content, flags=re.DOTALL)

with open('path/to/conflicted/file.java', 'w') as f:
    f.write(content)
print('Done')
"
```

Swap `m.group(2)` → `m.group(1)` to take HEAD (ours) instead.

After running, `git add path/to/file` and `git cherry-pick --continue` / `git rebase --continue`.

### Why this is non-obvious

`git checkout --ours file` or `git checkout --theirs file` only works for whole-file resolution — it picks one side's entire file, discarding all content from the other side. When there are N conflicts each needing the same *side* but different *content*, those commands don't apply. The regex approach targets only the conflict markers while preserving everything else.

### Caveat

Only use this when you've verified *all* conflicts in the file genuinely resolve the same way. If some conflicts need HEAD and others need incoming, this will get some wrong. Review the diff after running.
