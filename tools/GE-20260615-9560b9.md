### What

Two distinct failure modes corrupt PRs after rebase or squash. Run both checks before every push.

**Step 1 — Scope scan (accidental inclusions)**

```bash
git diff origin/main..HEAD --name-only
```

Review every file. Any file outside the expected directories is an unintended inclusion — catch it here, not in code review. Automate with a global pre-push hook (see Enforcement section below).

**Step 2 — Deletion review (accidental removals and modifications)**

```bash
git diff origin/main..HEAD | grep "^-" | grep -v "^---"
```

Scan every removed line. Look for:
- Methods, classes, or constants removed without reason
- Visibility changes (`public` → `private`) without justification
- Modifications to shared files outside the PR's core module — each needs explicit justification

This is judgement, not automation. It must be a conscious act.

### Why this is non-obvious

Developers check `git diff` for content correctness but not for *what was silently lost*. Two failure modes exist and are independent:

- **Inclusions**: rebase replays commits in sequence; a commit that touched a CI workflow file lands silently in the squashed result
- **Deletions**: an edit to a shared file drops a method, constant, or registration that compiled fine but silently broke behaviour at runtime — no warning, no error, reviewers catch it expensively

Both happened in the same PR: a CI workflow regression slipped in, and `toString()`, `JsonAnyOfSchemaObjectSubstitution`, and `SkipToolBoxProcessingBuildItem` were silently removed.

### Root cause

Rebase and squash both replay and compress history. The developer's mental model is "my changes" but the actual diff is "everything between origin/main and HEAD" — which includes more than intended when branches share history.

### Enforcement — global pre-push hook

Discipline fails under pressure. Automate it instead with a global `pre-push` hook that blocks the push when always-suspicious files (`.github/`, `.gitignore`, `LICENSE`, `CODEOWNERS`, `.editorconfig`, `.mvn/`) appear in the diff:

```sh
# ~/.config/git/hooks/pre-push  (global via git config --global core.hooksPath)
#!/bin/sh
SUSPICIOUS=$(git diff origin/main..HEAD --name-only 2>/dev/null | grep -E \
  "^\.github/|^\.gitignore$|^\.gitattributes$|^LICENSE|^CODEOWNERS$|^\.editorconfig$|^\.mvn/")

if [ -n "$SUSPICIOUS" ]; then
  echo "⚠ Out-of-scope files detected:"
  echo "$SUSPICIOUS"
  exit 1
fi
```

Register globally:
```bash
mkdir -p ~/.config/git/hooks
chmod +x ~/.config/git/hooks/pre-push
git config --global core.hooksPath ~/.config/git/hooks
```

Use `--no-verify` only when the out-of-scope file is genuinely intentional (e.g. a PR that actually updates CI).
