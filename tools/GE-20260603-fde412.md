# claude-code-sdk AsyncSpec.claudePath(String) sets the claude binary path — undocumented on the builder

## The feature

`ClaudeClient.async()` returns an `AsyncSpec` builder. The builder has a `.claudePath(String)` method that sets the path to the `claude` CLI binary, overriding the default PATH-based resolution:

```java
ClaudeAsyncClient client = ClaudeClient.async()
    .workingDirectory(Path.of("."))
    .systemPrompt("Be concise")
    .claudePath("/custom/path/to/claude")   // undocumented
    .build();
```

## Why undocumented

The README and SDK Javadoc examples show only `.workingDirectory()`, `.model()`, `.systemPrompt()`, and `.timeout()`. The `.claudePath()` method is not listed in any documentation section — only discoverable by reading the `AsyncSpec` source.

## When needed

Required when:
- The `claude` binary is not on the system PATH (containers, CI with custom install paths)
- Multiple Claude versions are installed and the non-default one is needed
- Integration testing against a specific binary

Without this, the only alternative is to manipulate PATH at the process level before the JVM starts.
