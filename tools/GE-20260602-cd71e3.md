## What happens

A documentation standard defines a section with a narrative instruction: "write a teaching narrative explaining what this layer introduces and the gap it closes relative to the previous layer." Across multiple implementations and authors, the section drifts to flowing prose — even when the intent was a tight comparative structure.

## Root cause

Narrative instructions describe intent; they don't constrain structure. Without a structural form to follow, the LLM's (or author's) default is prose. The mode is determined by the structural prescription, not the content description.

## The technique

For any section of a documentation standard that must stay in a specific mode: prescribe the structure explicitly, not just the content.

**Narrative instruction (causes drift):**
> "Teaching narrative: what this layer introduces, what gap it closes."

**Structural prescription (locks mode):**
```
**Before:** [previous state] — what existed, in one clause.
**After:** [component] — what displaced/extended it, in one clause.

What this adds:
- **[Named capability]** — [mechanism]; [what it prevents or enables]

Not closed here: [Layer N] ([what still lacks]), [Layer M] ([what still lacks]).
```

## Why non-obvious

The instinct is to write better instructions. The non-obvious insight: **the prescription IS the mode enforcement mechanism.** A well-written narrative instruction is still a narrative instruction — it tells authors what to achieve, not what structure to produce. Structural prescriptions constrain generation before content is produced.

**Empirically validated:** arc42stories sections with explicit format prescriptions (`Format: **Symptom** → Cause → Fix`, `numbered steps`, `path — one-line description`) held their mode across every implementation. The "What it adds" section, given only a narrative instruction, drifted to prose in every implementation — same spec, same tool, different structural discipline.

### Alternatives considered

- Better narrative instructions: doesn't work — even excellent narrative instructions leave structure open
- Example-based guidance: helps but examples can be interpreted loosely; explicit prescriptions cannot
