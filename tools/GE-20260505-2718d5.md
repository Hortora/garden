## Symptom

A git command (e.g. `git push`) reports success but operates on the wrong repository — e.g. "Everything up-to-date" when changes were expected to be pushed.

## Context

Occurs when running multiple bash commands in a Claude Code session where an earlier command `cd`'d to a directory, and a later parallel or subsequent command omits an explicit `cd`.

## Root Cause

Bash tool calls in Claude Code share working directory state from the previous call. If one call `cd`'s to `/repo-a` and a subsequent call doesn't `cd` to `/repo-b`, the command runs in `/repo-a` silently.

## Fix

Always use absolute paths or an explicit `cd` at the start of each bash command, especially when operating across multiple repositories:

```bash
cd /absolute/path/to/repo && git push
```

Never rely on the working directory being in the expected state from a previous call.

## Why This Is Non-Obvious

"Everything up-to-date" looks like a success message. There is no error, no warning — the operation completed correctly, just in the wrong repository. Easy to miss when running several repo operations in quick succession across multiple directories.
