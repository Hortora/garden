## What

After shipping a new abstraction to a consumer (e.g. `ActorTypeResolver.resolve()` replacing local derivation across four repos), dispatch parallel subagents to audit each consumer: read recently-changed source files and run `mvn test` / equivalent. Surfaces pre-existing failures and code smells invisible from the primary repo.

## Technique

Dispatch two subagents in parallel, each covering two repos:

```
Agent A: read changed files in repo1 and repo2, run tests, report:
  - files recently touched (git log --oneline -10)
  - code issues spotted in the changed files
  - test results (pass/fail counts + failure details)
  - any pre-existing failures unrelated to this change

Agent B: same for repo3 and repo4
```

Track findings in a GitHub issue in the primary repo so they survive session end.

## Why This Is Non-Obvious

Step 6 of the Platform Coherence Protocol says "grep all repos for code now superseded" — but grep only finds text. Running tests and reading files catches pre-existing failures (invisible from the primary repo), silent bugs introduced by the abstraction change, and code smells that exist independently. In this session it surfaced 8 pre-existing test failures in quarkus-work and two production bugs in claudony — none visible from quarkus-ledger.
