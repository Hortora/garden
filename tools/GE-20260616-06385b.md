## Symptom

You know a file exists somewhere in the repo but it's not on the current branch and you don't know which branch it came from — or you need to find when and where it was first committed.

## Context

Multi-branch repos where files (specs, configs, migration scripts) are created on feature branches and may never have been merged. `git log -- path` only searches the current branch's history.

## What was tried (didn't work)

- `git log -- path/to/file.md` — only shows history reachable from HEAD; misses branches that diverged
- `git grep "filename"` — searches content across commits, not file creation events
- Manually checking each branch — impractical with dozens of branches

## The technique

```bash
git log --all --oneline --diff-filter=A -- "path/to/file.md"
# or with a glob
git log --all --oneline --diff-filter=A -- "specs/*inbound*"
```

`--all` searches all refs (every branch and tag). `--diff-filter=A` limits results to commits where the path was **A**dded. The result shows the commit hash — then read the file without checking out:

```bash
git show <hash>:path/to/file.md
```

Or find which branch contains the commit:

```bash
git branch --all --contains <hash>
```

## Why this is non-obvious

`--diff-filter` is an obscure git flag with multiple letter codes (`A`=Added, `M`=Modified, `D`=Deleted, `R`=Renamed). Most developers reach for `git grep` or manual branch iteration. Combining `--all` with `--diff-filter=A` finds any file's birth commit in one command regardless of how many branches exist.
