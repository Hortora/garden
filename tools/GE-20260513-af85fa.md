### Symptom

`publish-blog` skill fails to find blog entries, or errors immediately because `scripts/blog_router.py` does not exist.

### Context

Invoking `publish-blog` from a cc-praxis workspace session where blog entries are stored in the standard `workspace/blog/` directory. The skill expects entries in `docs/_posts/` (Jekyll convention) and depends on `scripts/blog_router.py`, which is not scaffolded by `workspace-init`.

### What was tried

Invoking the `publish-blog` skill directly from a workspace session after confirming blog entries exist in `blog/`.

### Root cause

Two structural mismatches:
1. `publish-blog` hardcodes `docs/_posts/` as the source directory; cc-praxis workspaces use `blog/`.
2. The skill requires `scripts/blog_router.py` which is not created by `workspace-init` and has no documented source.

### Fix

Manually copy the blog entry to the correct external destination based on `entry_type` in frontmatter, then commit and push:

- `entry_type: note` → `~/claude/mdproctor.github.io/_notes/`
- `entry_type: article` → `~/claude/mdproctor.github.io/_articles/`

The underlying structural fix (`workspace-init` scaffolding `blog_router.py`, correcting the source path in `publish-blog`) is tracked in skills Claude.

### Why this is non-obvious

The `publish-blog` skill description makes no mention of `docs/_posts/` — it reads as if it works from any blog directory. The missing `blog_router.py` dependency gives no hint about where it should come from or how to create it.
