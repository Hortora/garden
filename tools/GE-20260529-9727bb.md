## What happened

Research specs and conceptual OpenClaw docs show pseudocode like:

```python
@agent.on("before_prompt_build")
def inject_context(ctx):
    return { "appendSystemContext": "..." }
```

This is not real Python code. The Python SDK (`openclaw_sdk`, `from openclaw import OpenClawClient`)
is an external REST API wrapper — it manages agents via HTTP, schedules cron jobs, and handles
pipelines. It has no `agent.on()` method, no hook registration, and no in-process lifecycle access.

The `before_prompt_build` hook only exists in OpenClaw's **TypeScript Plugin SDK** (`openclaw/plugin-sdk`),
which is for in-process Gateway plugins written in TypeScript/JavaScript. Every real-world OpenClaw
plugin using this hook (`openclaw-sticky-context`, `openclaw-memory-auto-recall`, `openclaw-observability-plugin`,
MemOS Cloud plugin) is 100% TypeScript.

## Why it's non-obvious

OpenClaw's docs show Python SDK examples and TypeScript plugin examples side-by-side without clearly
labelling the architectural boundary. The pseudocode in integration docs and LLM outputs uses Python
syntax for hook registration because the surrounding discussion is Python-centric. The Python SDK
package name (`openclaw`) is the same prefix as the plugin system, making them look like parts of
the same API.

A developer starting from OpenClaw Python SDK examples would spend significant time before discovering
that `agent.on()` simply doesn't exist in that SDK.

## The fix

Implement `before_prompt_build` hooks as TypeScript/JavaScript plugins:

```typescript
// plugin/src/index.ts
export function register(api: OpenClawPluginApi): void {
  // register() is synchronous — hooks snapshotted at registration time
  api.on("before_prompt_build", async (ctx) => {
    return { appendSystemContext: "..." };
  });
}
```

The plugin must have an `openclaw.plugin.json` manifest and be loaded via the user's `openclaw.json`
`plugins.load.paths` config. The Python SDK can still be used separately for external orchestration
(triggering agents, managing cron jobs) — it just cannot register in-process hooks.

## Signal to recognise this

If you're trying to implement `before_prompt_build` in Python and can't find `agent.on()` in the
OpenClaw Python SDK — you're looking in the wrong SDK. The hook system is TypeScript-only.
