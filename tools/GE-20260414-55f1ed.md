## Context

Need to add issue-tracking footers (`Refs #N`, `Closes #N`) to a sequence of already-committed messages. Interactive rebase (`git rebase -i`) is not available in Claude Code (requires interactive terminal input). `git filter-branch` is deprecated and requires complex message-matching logic for per-commit footers.

## Technique

Create a temp branch at the base commit, cherry-pick each commit in order with an immediate amend, then reset the target branch:

```bash
# 1. Identify the base (commit before the range to rewrite)
BASE=$(git rev-parse FIRST_COMMIT^)

# 2. Create temp branch
git checkout -b rewrite-temp "$BASE"

# 3. Cherry-pick + amend each commit (oldest first)
git cherry-pick abc1234
git commit --amend -m "feat: original subject line

Closes #18"

git cherry-pick def5678
git commit --amend -m "style: original subject line

Closes #19"

# ... repeat for each commit

# 4. Reset target branch and clean up
git checkout main
git reset --hard rewrite-temp
git branch -d rewrite-temp
```

Key constraint: write the full amended message as a literal string (do NOT use `$(git log -1 --format='%B')` — bash strips trailing newlines and the footer concatenates to the last line).

## Why Non-Obvious

Most developers reach for `git rebase -i` (unavailable in automated contexts), `git filter-branch` (complex per-commit logic), or `git filter-repo` (may not be installed). The cherry-pick loop is simple, transparent, and requires no external tools. Each commit is independently visible and correctable if an amend is wrong.
