### Symptom
`ide_create_file` returns success. `ide_build_project` passes — the new class is fully
resolved, references compile, diagnostics are clean. But `mvn compile` fails with
`cannot find symbol` for the class that was just created.

### Context
When creating a new `.java` file via `mcp__intellij-index__ide_create_file`, the file
is written to IntelliJ's Virtual File System (VFS) and immediately indexed. All
IntelliJ-based operations (build, diagnostics, find references) see the file. But
the VFS write does not always flush to the physical filesystem — Maven, which reads
directly from disk, cannot see it.

### Root cause
`ide_create_file` operates through IntelliJ's `VirtualFileManager` which maintains an
in-memory file tree. The VFS-to-disk flush is asynchronous and may not complete before
Maven is invoked. `ide_sync_files` does not force a VFS→disk flush for new files — it
syncs disk→VFS (the opposite direction).

### Fix
Use the `Write` tool (Claude Code's native file writer) to create new source files
instead of `ide_create_file`. `Write` persists directly to disk. After writing, call
`ide_sync_files` to make IntelliJ's VFS aware of the new file.

```
# For NEW files: Write tool → disk, then sync to IntelliJ
Write(file_path="/path/to/NewClass.java", content="...")
ide_sync_files(paths=["path/to/NewClass.java"])

# For EXISTING files: IntelliJ MCP tools work correctly
ide_edit_member(...)   # reads and writes through VFS, which IS flushed for edits
ide_insert_member(...) # same — edits to existing files persist
```

### Why this is non-obvious
`ide_create_file` reports success and the file passes all IntelliJ verification.
The IntelliJ build (`ide_build_project`) compiles cleanly. The failure only surfaces
when an external build tool (Maven, Gradle CLI) is invoked — which may be many steps
later. The symptom (`cannot find symbol`) points at a missing import or typo, not a
missing file.

### Alternatives considered
- `ide_sync_files` after `ide_create_file` — does not help; sync is disk→VFS, not VFS→disk
- Waiting/retrying — unreliable; the flush timing is non-deterministic
