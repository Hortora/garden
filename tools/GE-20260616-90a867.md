## Technique

When you need to verify whether a class in a compiled JAR carries a specific
annotation (e.g. `@Alternative`, `@Default`, `@ApplicationScoped`) without
access to source code or a Java decompiler:

```python
import zipfile

jar = '/path/to/library.jar'
with zipfile.ZipFile(jar) as z:
    data = z.read('com/example/MyClass.class')
    if b'Alternative' in data:
        print('Has @Alternative')
    if b'Default' in data:
        print('Has @Default')
```

The annotation type names are stored as UTF-8 constants in the class file constant
pool. A simple `bytes in bytes` search is sufficient for presence detection —
no bytecode parser needed.

## Extended use: list all internal string constants

To see everything a class references (useful for understanding delegation chains):

```python
import zipfile

jar = '/path/to/library.jar'
with zipfile.ZipFile(jar) as z:
    data = z.read('com/example/MyClass.class')
    strings = []
    i = 0
    while i < len(data):
        if data[i:i+1] == b'\x01':  # CONSTANT_Utf8 tag
            length = (data[i+1] << 8) | data[i+2]
            s = data[i+3:i+3+length]
            try:
                decoded = s.decode('utf-8')
                if len(decoded) > 3 and '/' in decoded:
                    strings.append(decoded)
            except:
                pass
            i += 3 + length
        else:
            i += 1
    for s in sorted(set(strings)):
        print(s)
```

This reveals method signatures, type references, and field descriptors — enough
to map a class's full dependency graph from bytecode alone.

## Why this is non-obvious

The standard approach for inspecting class metadata is `javap -verbose`, which
requires extracting the class file from the JAR first and running a separate JDK
tool. The Python `zipfile` approach works directly on the JAR without any Java
toolchain present, and is scriptable in the same session that needs the answer.
It's particularly useful when working with SNAPSHOT JARs that have no published
source or Javadoc.
