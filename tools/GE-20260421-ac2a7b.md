## Technique

`gh run view --repo <owner/repo> <run-id> --log-failed` prints only the log lines from failing steps of a GitHub Actions run, directly in the terminal.

## When to Use

When a CI run fails and you want to see the error without opening a browser and clicking through the GitHub UI.

## Steps

```bash
# 1. Find the failing run ID
gh pr checks <PR-number> --repo <owner/repo>
# output includes the Actions run URL, e.g.:
# Build and test  fail  2m32s  https://github.com/.../runs/24740863935/job/...

# 2. View only the failing logs
gh run view --repo <owner/repo> 24740863935 --log-failed 2>&1 | head -100
```

## Why This Is Non-Obvious

Most developers navigate to the GitHub Actions UI to see CI failures. `--log-failed` is a flag on `gh run view` that is not prominently documented and is easy to miss. It reduces the diagnosis cycle from "open browser, find run, expand failing step, read logs" to a single terminal command.

## Notes

- Use `| head -100` to avoid overwhelming terminal output for verbose builds
- The run ID appears in the Actions URL and in `gh pr checks` output
- Works for any GitHub Actions run, not just PRs
