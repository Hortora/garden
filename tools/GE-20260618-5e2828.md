### Symptom
`git rebase -i` with `GIT_SEQUENCE_EDITOR="cp <todo-file>"` silently applies the wrong
todo when the todo file path contains spaces. Git exits 0, rebase completes, but the
result is wrong.

### Context
When automating non-interactive rebase from Python via subprocess, the todo file is
typically written to a temp path and passed via GIT_SEQUENCE_EDITOR:
```python
env_cmd = f"cp {todo_file}"
subprocess.run(["git", "rebase", "-i", base],
    env={**os.environ, "GIT_SEQUENCE_EDITOR": env_cmd})
```

### Root cause
Shell word splitting on the unquoted path. `cp /tmp/my plan.txt /path/to/git-rebase-todo`
becomes three arguments. `cp` copies `/tmp/my` to `plan.txt` (or errors silently depending
on the shell), and git uses whatever ended up at the todo path.

### Fix
Quote the path in the shell command:
```python
env_cmd = f"cp '{todo_file}'"
```
Or avoid the shell entirely with a Python wrapper script.

### Why this is non-obvious
The failure is completely silent — git rebase exits 0, the working tree changes, but the
squash plan applied is wrong. Debugging requires comparing the expected todo against what
git actually received, which is not visible in any output.
