### Symptom
Calling `ide_edit_member`, `ide_replace_member`, or `ide_insert_member` on a `.ts` file returns:
`Member editing not supported for TypeScript. Supported: Java, Kotlin.`

### Context
When editing TypeScript source files through the IntelliJ MCP plugin. The tool descriptions list no language restrictions, so the natural assumption is they work on any language the IDE indexes.

### What was tried (didn't work)
Called `ide_edit_member` to replace a TypeScript interface declaration — received the unsupported language error.

### Root cause
The structural edit tools rely on IntelliJ's PSI (Program Structure Interface) member resolution, which the MCP plugin only implements for Java and Kotlin PSI trees. TypeScript PSI exists in the IDE but the plugin doesn't wire it up for member-level edits.

### Fix
Use `ide_replace_text_in_file` instead — it operates at the text level (IntelliJ's Document API), not the PSI/structural level, so it works on all file types. For targeted member edits, use a unique text anchor from the member's signature as the search pattern.

### Why this is non-obvious
The tool descriptions for `ide_edit_member` etc. mention "Replace an entire member or class declaration" with examples in Java and Kotlin but don't state a language restriction. A developer seeing these tools available would naturally try them on TypeScript before discovering the limitation.

### Alternatives considered
- `Edit` tool — blocked by project hook enforcing IntelliJ-first policy on source files
- Manual `ide_replace_text_in_file` with precise anchors — this is the working approach
