### Symptom

`mcp__intellij-index__ide_search_text` (and all other `mcp__intellij-index__*` tools) returns `project_not_found` with hint "Path exists but has no .idea directory" when `project_path` points to the monorepo root.

### Context

In the Melviz monorepo, the root directory (`/Users/dev/melviz`) has no `.idea` directory. The IntelliJ project is set up inside `core/` (`/Users/dev/melviz/core/.idea`). The `mcp__intellij-index__*` plugin can only auto-open projects that have a `.idea` directory at the specified path.

### What was tried (didn't work)

1. Passing `project_path: "/Users/dev/melviz"` — fails because no `.idea` at root.
2. Falling back to bash `grep` — works but loses semantic code intelligence.

### Root cause

The IntelliJ MCP plugin resolves projects by looking for `.idea` at the exact `project_path` provided. It does not search subdirectories. In monorepos where the IDE project is a subdirectory, the `project_path` must point to that subdirectory explicitly.

### Fix

Use the subdirectory containing `.idea` as `project_path`:
```
project_path: "/Users/dev/melviz/core"  // not "/Users/dev/melviz"
```

### Why this is non-obvious

The error message is clear once you see it, but the initial instinct is to use the repo root as `project_path`. In monorepos with multiple IDE projects in subdirectories, you need to know which subdirectory has `.idea` — and there may be multiple (each with different Java modules visible).
