### Symptom
`git status` shows "You are currently editing a commit while rebasing branch
'<branch>' on '<sha>'" even though you just checked out a completely different
branch. `git rebase origin/main` then fails with:

```
fatal: It seems that there is already a rebase-merge directory, and
I wonder if you are in the middle of another rebase.
```

### Context
A previous session started a `git rebase -i` (or similar), completed all
cherry-picks (no commands remaining, working tree clean), but the session
ended before `git rebase --continue` was run to finalise it. The
`.git/rebase-merge/` directory survives indefinitely — across checkouts,
reboots, and new sessions.

### What was tried (didn't work)
- Assumed it was a multi-user conflict (wrong — single-user repo)
- Checking out the target branch: orphaned state persists across checkouts

### Root cause
Git's rebase state lives in `.git/rebase-merge/`. It is only cleaned up by
`git rebase --continue`, `git rebase --abort`, or `git rebase --skip`. If a
session ends without running one of those, the directory stays forever.
`git checkout` does not clear it.

### Fix
**If the rebase work is already done (commits applied and pushed):**
```bash
rm -fr .git/rebase-merge
```

**If the rebase was interrupted mid-way and you want to finish it:**
```bash
git rebase --continue   # if working tree is clean
git rebase --skip       # to skip the current commit
```

**If you want to abandon the rebase:**
```bash
git rebase --abort      # reverts HEAD to pre-rebase state
```

To prevent it: before ending any session that started a rebase, always run
`git rebase --abort` if the rebase isn't complete.

### Why this is non-obvious
The stale state shows up on any branch checkout — not just the branch that was
being rebased. This makes it look like a repository-wide or multi-user problem
rather than an orphaned local state. The error message does hint at the solution
(`rm -fr .git/rebase-merge`) but only if you read it carefully and trust that
the "other rebase" is stale, not in-progress.
