### Symptom

`git rebase -i --root` with `GIT_SEQUENCE_EDITOR="cp /tmp/todo"` exits immediately:

```
error: could not parse '<sha>'
error: invalid line N: pick <sha> <message>
You can fix this with 'git rebase --edit-todo' and then run 'git rebase --continue'.
Or you can abort the rebase with 'git rebase --abort'.
```

No indication of which SHA is wrong or what the correct value is.

### Context

Occurs when using the non-interactive rebase pattern for automated squash plans —
writing a todo file manually and injecting it via `GIT_SEQUENCE_EDITOR`. A single
character typo in any SHA causes the entire rebase to abort before any commit is
replayed.

### Root cause

git validates all SHAs in the todo file upfront before applying any changes.
The error names the line number and content but does not show the correct SHA,
making the typo hard to spot — especially in a 60+ line todo.

### Fix

Generate the todo programmatically from `git log` output rather than transcribing
SHAs by hand:

```python
import subprocess

result = subprocess.run(
    ['git', 'log', '--format=%H %s', '--reverse'],
    capture_output=True, text=True
)

fixups = {
    'cbf5dd759b245f365a85356c2c0cb52b1efa7c17',  # mark these as fixup
    # ... add full SHAs from git log output
}

lines = result.stdout.strip().split('\n')
todo_lines = []
for line in lines:
    sha, msg = line.split(' ', 1)
    action = 'fixup' if sha in fixups else 'pick'
    todo_lines.append(f'{action} {sha[:7]} {msg}')

with open('/tmp/squash_todo.txt', 'w') as f:
    f.write('\n'.join(todo_lines) + '\n')
```

Then run:

```bash
GIT_SEQUENCE_EDITOR="cp /tmp/squash_todo.txt" git rebase -i --root
```

### Why this is non-obvious

The obvious workflow is to write the todo file by hand, copy-pasting SHA prefixes
from `git log --oneline`. A single transposition (e.g. `4027dcf` → `4027dcc`) is
invisible to the eye and produces an error that names the line but not the typo.
Generating programmatically from `--format=%H` eliminates the transcription step
entirely and is not a natural first approach.
