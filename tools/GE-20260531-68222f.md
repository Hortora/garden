### Context

Squashing a group of commits with `fixup` in a `git rebase -i` todo and then
wanting to set a custom final commit message — without triggering an interactive
editor for each `squash` line, and without a second rebase.

### Technique

Add an `exec` line immediately after the last `fixup` in a group. The `exec`
runs in the working tree after all fixups complete, with `HEAD` pointing to the
just-created squashed commit:

```
# Group 1: IDLE feature
pick bc75896 feat(email-inbound): replace IMAP polling with IDLE
fixup be036ae refactor(email-inbound): rename pollIntervalSeconds
fixup 9575106 docs: fix QuarkusTest comment; sync DESIGN.md
exec git commit --amend -F /tmp/idle-msg.txt

# Group 2: Attachment feature
pick 4dad533 feat(email-inbound): wire attachment extraction
fixup 4959606 refactor: single-pass ContentExtractor
fixup 0c77a3d feat(core): add Attachment record
exec git commit --amend -F /tmp/attachments-msg.txt
```

Pre-write the message files before invoking rebase:
```bash
echo "feat(email-inbound): IMAP IDLE replaces polling — Closes #9" > /tmp/idle-msg.txt
echo "feat(core,email-inbound): binary attachment support — Closes #10" > /tmp/attachments-msg.txt

GIT_SEQUENCE_EDITOR="cp /tmp/squash-todo.txt" git rebase -i <base>
```

The entire sequence runs non-interactively. Two commits emerge with exact messages,
no editor prompts.

### Why This Is Non-Obvious

The standard approach for setting a custom message after multiple fixups is to
use `squash` (not `fixup`) which opens an editor for message editing — interactive
and not scriptable. The `exec` mechanism is documented but not commonly paired with
`commit --amend` for message control. Most squash guides either accept the KEEP
commit's message (via `fixup`) or accept the editor interaction (via `squash`).
`exec + amend` combines the silence of `fixup` with full message control.
