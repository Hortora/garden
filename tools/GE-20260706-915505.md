### Symptom

@QuarkusTest appears to hang during bootstrap — surefire prints the "T E S T S" header then produces no output. No timeout, no error, no test execution. Looks like a CDI wiring or Quarkus startup issue.

### Context

Project pom.xml has a GitHub Packages `<repository>` configured for snapshot dependencies. No `GITHUB_TOKEN` environment variable is set (common in local dev without GitHub auth configured).

### What was tried (didn't work)

- Adding CDI exclusions to fix suspected wiring issues — made things worse
- Running with `-X` debug logging — showed surefire fork starting but provided no useful diagnostics about the hang
- Checking surefire reports — no test report files generated

### Root cause

The surefire forked JVM attempts to resolve snapshot metadata from GitHub Packages at test time. GitHub Packages returns 401 Unauthorized for anonymous requests. The Maven resolver blocks in `RunnableErrorForwarder.awaitTerminationOfAllRunnables` waiting for the metadata response — confirmed via `jstack` thread dump of the hanging process. The test framework never starts because Maven dependency resolution hasn't completed.

### Fix

Run tests with offline mode:
```bash
mvn --batch-mode -o test -pl app
```

The `-o` flag skips all remote repository checks, using only the local `.m2/repository` cache. Since the dependencies are already resolved during `mvn compile`, offline test execution is safe.

### Why this is non-obvious

The symptom (test bootstrap hang) looks like a CDI or Quarkus issue, not a Maven resolution issue. The hang occurs inside the forked JVM where normal Maven logging isn't visible. Only a `jstack` thread dump reveals the real cause — the forked process is blocked in Maven's artifact resolver, not in Quarkus startup.
