## What

Use a Python script to verify that every box-border line in a Markdown code block is exactly the same character length, then fix specific lines by line number. This catches alignment bugs that `awk`, `grep`, and `wc -c` all miss because they count bytes, not display columns.

## The technique

```python
with open('file.md') as f:
    lines = f.readlines()

in_block = False
target_width = 82  # expected total width including borders

for i, line in enumerate(lines, 1):
    s = line.rstrip('\n')
    if s.strip() == '```':
        in_block = not in_block
        continue
    if in_block and s and s[0] in '|+':
        actual = len(s)
        if actual != target_width:
            print(f"Line {i}: {actual} chars (expected {target_width})")
```

Fix specific lines with direct index assignment:

```python
def row(content="", width=80):
    return "|" + content.ljust(width) + "|\n"

lines[22] = row("   Tamper-evident. Queryable. Proof of what happened.")
```

## Why non-obvious

`awk '{print length}'` counts bytes in UTF-8. A single Unicode character like `—` (em dash, U+2014) is 3 bytes but 1 display column. `len()` in Python 3 counts Unicode code points — each character is 1, regardless of byte representation. This is the correct unit for display alignment.

Most developers reach for `awk` or `wc` to measure line lengths; Python `len()` on a decoded string is the right tool for display-width problems in ASCII-safe content.
