### Symptom

During `work-end` for an epic branch, inspecting `blog/` from the branch shows entries only up to the branch creation date. Entries written to workspace main AFTER the branch diverged appear missing — looks like a coverage gap requiring a new blog entry.

### Context

The cc-praxis workspace pattern keeps blog entries on workspace `main`. Blog entries are often written during handover (end of session) directly to main, not to the current epic branch. An epic branch `git checkout` shows the blog directory as it was at branch creation time — any entries added to main afterward are not visible.

### What was tried (didn't work)

Inspected `blog/` from the `epic-phase-6` workspace branch. Saw entries only up to 2026-05-13. Concluded that phase-6 work had no blog coverage and wrote a new entry. In reality, workspace main had entries for 2026-05-16, 2026-05-20, and 2026-05-22 covering the same phase.

### Root cause

Git branches are isolated snapshots. Blog entries committed to workspace `main` after branch divergence exist only on `main`. The epic branch filesystem shows the branch's own blog history, which is frozen at creation. There is no way to "see" main's later commits from the branch without explicitly checking.

### Fix

When checking for missing blog coverage during `work-end`, always inspect workspace `main`'s blog directory, not the epic branch:

```bash
git -C /path/to/workspace show main:blog/ | grep "YYYY-MM"
# or simply switch to main first, then ls
git -C /path/to/workspace checkout main
ls blog/ | sort
```

Never use the epic branch's working tree blog directory as the authoritative source of blog history.

### Why this is non-obvious

Git branch isolation is well-understood. The non-obvious part is the specific workflow: blog entries are written to workspace `main` (not to the epic branch), so they look "newer than the branch" from the branch's perspective. The natural action during `work-end` — "check if we have blog coverage" — uses the current working directory, which is the epic branch. The false absence is indistinguishable from a real gap without explicitly checking main.
