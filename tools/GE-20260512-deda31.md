## Use Python str.replace() over sed -i '' for safe multi-file text replacement on macOS

### Why reach for this

`sed -i ''` on macOS silently empties files that are already empty, exits 0, and gives no warning. Python's `read_text()` / `write_text()` reads first and only writes if content changed — providing an implicit guard against operating on empty input.

### The pattern

```python
import pathlib

files = [
    pathlib.Path("docs/PLATFORM.md"),
    pathlib.Path("CLAUDE.md"),
]

for f in files:
    if not f.exists():
        print(f"MISSING: {f}")
        continue
    content = f.read_text()
    updated = content.replace("old/path/", "new/path/")
    if updated != content:
        count = content.count("old/path/")
        f.write_text(updated)
        print(f"Fixed {f.name}: {count} occurrence(s)")
    else:
        print(f"No change: {f.name}")
```

### Why this is non-obvious

Most developers reach for `sed -i ''` for quick multi-file replacements. Python feels heavier. But `sed` offers no guard against empty input, no per-file change count, and no easy way to skip missing files gracefully. The Python pattern is only marginally more verbose and significantly safer for scripted multi-file edits where working tree state is uncertain.

### When to use

- Replacing paths or strings across multiple files in a scripted context
- Any situation where working tree files might be empty or missing
- When you need confirmation of how many occurrences were replaced per file
