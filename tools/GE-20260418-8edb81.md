## What

`TuiTestRunner` (the Tamboui Pilot test harness for headless TUI testing) lives in
`tamboui-tui:test-fixtures`. But the class it depends on — `dev.tamboui.terminal.TestBackend`
— is published in `tamboui-core:test-fixtures`, not `tamboui-tui:test-fixtures`.

The Pilot documentation only mentions `tamboui-tui:test-fixtures`. Following the docs produces
a `NoClassDefFoundError: dev/tamboui/terminal/TestBackend` at runtime even though compilation
succeeds.

## Symptom

```
java.lang.NoClassDefFoundError: dev/tamboui/terminal/TestBackend
    at dev.tamboui.tui.pilot.TuiTestRunner.runTest(TuiTestRunner.java:...)
```

Tests compiled fine (because `TuiTestRunner.class` is present in `tamboui-tui:test-fixtures`)
but failed at runtime (because `TestBackend` itself is in a different module's test-fixtures).

## Root cause

Tamboui's Gradle build places `TestBackend` in `tamboui-core`'s `testFixtures` source set —
not `tamboui-tui`'s. Architecturally this makes sense (`TestBackend implements Backend`, which
is a `tamboui-core` type), but it creates a split-module dependency invisible to Maven
consumers because Gradle `testFixtures` API dependencies are not exported in the published
Maven POM.

## Fix

Declare **both** test-fixtures artifacts as test dependencies:

```xml
<!-- TuiTestRunner + TuiPilot -->
<dependency>
  <groupId>dev.tamboui</groupId>
  <artifactId>tamboui-tui</artifactId>
  <version>0.2.0-SNAPSHOT</version>
  <classifier>test-fixtures</classifier>
  <scope>test</scope>
</dependency>

<!-- TestBackend — used internally by TuiTestRunner -->
<dependency>
  <groupId>dev.tamboui</groupId>
  <artifactId>tamboui-core</artifactId>
  <version>0.2.0-SNAPSHOT</version>
  <classifier>test-fixtures</classifier>
  <scope>test</scope>
</dependency>
```

Also note: if your main code uses `TuiRunner`, `KeyEvent`, `Renderer` etc. directly
(not via the Toolkit DSL), you must also add `tamboui-tui` as a **compile** dependency —
`tamboui-toolkit` does not transitively export it.

## Why this is non-obvious

The Pilot documentation only lists `tamboui-tui:test-fixtures`. There is no documentation
mentioning `tamboui-core:test-fixtures`. The runtime error points to `TestBackend` but nothing
tells you which artifact it lives in. Confirmed by Max Anderston (Tamboui author) only after
exhaustive search through all Tamboui JARs.
