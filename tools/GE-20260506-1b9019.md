### Symptom
A long paragraph in a Jekyll article renders as a giant `<h2>` heading instead of normal body text. The paragraph itself is correct prose — no heading markers — but it displays in heading size and weight.

### Context
Jekyll site using Kramdown as the markdown renderer. Article content has a `---` separator between sections.

### What was tried
Checked the markdown for accidental `##` heading markers — none found. The paragraph looked completely normal. Suspected a CSS issue first.

### Root cause
Kramdown implements **Setext-style headings**: a line of text followed *immediately* (no blank line) by `---` is parsed as an `<h2>` heading. The entire preceding paragraph becomes the heading text.

```markdown
This is a very long paragraph of prose text that continues
for several lines and looks completely normal in the editor.
---               ← Kramdown reads this as Setext h2 underline
```

The above produces `<h2>This is a very long paragraph...</h2>`.

### Fix
Always put a **blank line** before `---` horizontal rules:

```markdown
This is the paragraph of prose text.

---

## Next Section
```

### Why this is non-obvious
Setext headings are a CommonMark legacy from the 1990s. Most developers only know ATX-style headings (`##`). A `---` that visually looks like a divider — and is used as one throughout the document — silently becomes a heading underline if the preceding blank line is accidentally omitted. The paragraph being long makes it even less likely to be suspected as a heading.
