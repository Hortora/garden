## Context

A multi-repo ecosystem (e.g. `quarkus-ledger` → `quarkus-work` → `quarkus-qhorus` → `casehub-engine`) where rebuilding everything on every push is expensive but necessary for correctness. A full rebuild takes 20–30 minutes; most changes only affect one repo.

## Technique

Compare each repo's current HEAD SHA against the SHA recorded in the previous successful build log. Classify each repo into one of three states:

| State | Condition | Action |
|---|---|---|
| **BUILD** | Own SHA changed | Full compile + install (`mvn install`) |
| **TEST** | Own SHA unchanged, but a transitive casehub dep is in BUILD state | Test-only (`mvn test`) — no recompile needed, but verify integration against new dep |
| **SKIP** | Own SHA and all dep SHAs unchanged | Nothing — artifact in local .m2 is still valid |

The dependency graph is declared explicitly in the build script:

```bash
declare -A DEPS
DEPS[quarkus-ledger]=""
DEPS[quarkus-work]="quarkus-ledger"
DEPS[quarkus-qhorus]="quarkus-ledger quarkus-work"
DEPS[casehub-engine]="quarkus-langchain4j quarkus-ledger quarkus-work"
```

Each run writes a timestamped SHA log (`build-logs/YYYYMMDDTHHMMSS.shas`). The most recent log is the cache manifest. A companion `replay.sh` script can reproduce any prior build exactly by checking out each repo's recorded SHA.

The **TEST state** is the key insight: if `quarkus-ledger` changes but `quarkus-work`'s own code doesn't, `quarkus-work`'s bytecode is unchanged — no point recompiling. But its tests should run against the new `quarkus-ledger` to catch integration regressions. This halves build time in the common case of a single-repo change.

## Why this is non-obvious

Standard CI tools (Maven, Gradle, Bazel) perform incremental compilation within a single repo but have no built-in mechanism for cross-repo dependency awareness. Most multi-repo setups rebuild everything or use a full dependency matrix. The SHA-log approach is simple Bash — no CI framework, no cache server — and gives you both incremental builds and a permanent audit trail of exactly what built together.
