## gh issue create does not support --json flag — capture issue number by parsing stdout URL

**ID:** GE-20260420-89f0c6
**Stack:** GitHub CLI (gh)

**Symptom:** `gh issue create ... --json number` fails with `unknown flag: --json`. Scripts that try to capture the new issue number inline with `--json` break silently or with a confusing error.

### Root cause
`gh issue create` outputs a plain URL to stdout (`https://github.com/owner/repo/issues/123`) and does not support `--json`. The `--json` flag is available on `gh issue view`, `gh issue list`, and other read commands — but not create/edit operations.

### Fix
Parse the issue number from the stdout URL:

```bash
# Option A — tail the URL
NUMBER=$(gh issue create --title "..." --body "..." | grep -oE '[0-9]+$')

# Option B — awk on the URL
NUMBER=$(gh issue create --title "..." --body "..." | awk -F'/' '{print $NF}')
```

Or create first, then query:
```bash
URL=$(gh issue create --title "..." --body "...")
NUMBER=$(gh issue view "$URL" --json number -q '.number')
```

### Why non-obvious
`gh issue list` and `gh issue view` both support `--json`, so the flag feels like it should work on `create` too. The pattern `gh issue create ... --json number -q '.number'` looks syntactically correct and produces no helpful error — just "unknown flag".
