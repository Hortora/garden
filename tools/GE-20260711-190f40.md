### Symptom
`ide_edit_member(file, member="extractDataSet")` returns `"member_not_found"` for a
top-level exported function in a TypeScript file, even when `ide_file_structure` correctly
shows the function at the expected line number. Adding the `line` parameter does not help.

### Context
Occurs when trying to replace or modify a top-level exported function declaration
(not a class member) via `ide_edit_member`. The function is visible to `ide_file_structure`
and `ide_find_symbol`, but `ide_edit_member` cannot resolve it. Class members, interface
members, and type aliases work correctly.

Tested with `extractDataSet` at line 450 in `packages/pages-data/src/dataset/external/extraction.ts`.

### What was tried (didn't work)
1. `ide_edit_member(file, member="extractDataSet")` — "member_not_found"
2. `ide_edit_member(file, member="extractDataSet", line=450)` — "member_not_found"
3. `ide_sync_files` then retry — same result

### Root cause
The plugin's member resolution likely only searches class/interface member scopes, not
the file's top-level scope for standalone function declarations. `ide_file_structure`
uses a different code path (PSI tree traversal) that does find top-level functions.

Related: GE-20260710-0b8c2a (ide_edit_member fails on Java records — same plugin,
different failure mode).

### Fix
Use the `Edit` tool for top-level function signature changes. For body-only changes,
`ide_replace_member` may work if it resolves differently. For new functions, `ide_insert_member`
works at the file level.

### Why this is non-obvious
`ide_file_structure` showing the function creates a reasonable expectation that
`ide_edit_member` can find it. The "member_not_found" error doesn't distinguish between
"member doesn't exist" and "member exists but is not in a resolvable scope."
