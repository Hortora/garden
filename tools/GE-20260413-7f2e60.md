## Chain Liquid `replace` filters in Jekyll layouts to fix relative paths without touching source files

**ID:** GE-20260413-7f2e60
**Stack:** Jekyll (all versions)

When Jekyll's `permalink: pretty` style breaks relative image and cross-page link paths in Markdown content (because pretty URLs create `folder/index.html` files, shifting the path depth), the fix is a one-liner in the layout template — not a mass edit of every source file.

### The technique

In `_layouts/doc.html` (or any layout), replace the bare `{{ content }}` with a chained filter expression:

```liquid
{{ content
  | replace: 'src="images/', 'src="../images/'
  | replace: 'href="01', 'href="../01'
  | replace: 'href="02', 'href="../02'
  | replace: 'href="03', 'href="../03'
  | replace: 'href="04', 'href="../04'
  | replace: 'href="05', 'href="../05'
  | replace: 'href="06', 'href="../06'
  | replace: 'href="07', 'href="../07'
  | replace: 'href="08', 'href="../08'
  | replace: 'href="09', 'href="../09'
  | replace: 'href="10', 'href="../10'
  | replace: 'href="11', 'href="../11'
  | replace: 'href="features', 'href="../features'
  | replace: '.md"', '/"'
}}
```

Jekyll processes Liquid before Markdown renders to HTML, so this operates on the final rendered HTML string. The chain:
1. Prepends `../` to all image `src` attributes (so `images/file.png` → `../images/file.png`)
2. Prepends `../` to known cross-page `href` prefixes
3. Converts `.md"` link targets to directory-style `/"` links

### Why this is non-obvious
The natural fix is to edit every source Markdown file — change `images/file.png` to `../images/file.png` across dozens of files. The Liquid filter approach requires knowing that: (a) Jekyll applies Liquid to layout files; (b) `{{ content }}` is a string you can pipe; (c) Liquid's `replace` filter is chainable and works on rendered HTML. Most Jekyll documentation shows `{{ content }}` as an opaque block, not as something to post-process.

### When to use this
- You have many source files with consistent relative path patterns
- You can't (or don't want to) change the `permalink` style
- Switching to absolute paths via `relative_url` would require Liquid syntax in raw Markdown, which is ugly and brittle

### Limitations
- The replace targets must be specific enough not to match external URLs or other unintended content
- Order matters: prepend `../` before stripping `.md` extensions
