## Symptom

A PR is opened early based on a branch. More commits are pushed to the branch later.
When the PR is squash-merged, the resulting commit on main contains only the content
from when the PR was opened — the later commits are silently absent from main.
No error, no warning. The additional commits still exist on the branch but are not
on main.

## Context

GitHub squash merges take a snapshot of the PR's diff at merge time. If the branch
has diverged from its original state (new commits pushed after PR creation), those
additional commits are folded into the squash — but only if GitHub detects them as
part of the PR's tip. In some cases, particularly when a PR is rebased or the base
is updated, GitHub's squash merge takes the original base diff rather than the tip.

## Root cause

Squash merge collapses all PR commits into one. If the merge was initiated when the
branch was at an earlier commit (e.g. the PR was already marked "ready to merge" and
someone merged without pulling the latest), the squash commit reflects the earlier
branch state, not the tip.

## Fix

Always verify what was actually merged immediately after a squash merge:

```bash
git show --stat <merge-sha>   # check which files and how many lines changed
git log --oneline <branch> ^main  # check if any commits remain above main
```

If commits are missing: they still exist on the source branch. Cherry-pick or
replay them in a new PR.

Long-term fix: disable squash merges. Use rebase merges (all commits preserved,
linear history) or merge commits (full branch history preserved). Configure via
GitHub repo settings or API:
```bash
gh api --method PATCH repos/<org>/<repo> \
  -f allow_squash_merge=false \
  -f allow_rebase_merge=true
```

## Why this is non-obvious

`git status` and the PR interface both show "merged" with no indication that commits
were omitted. The branch still exists with the missing commits, which makes it look
like everything is intact. Discovery requires explicitly checking the merge commit's
stat against the branch's full commit log.
