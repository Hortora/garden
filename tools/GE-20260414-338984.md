## Symptom
After running sed to add a trailing `# comment` to a line like:
```python
f = make_file(tmp_path, "file.md", "content\n")
```
the result is:
```python
f = make_file(tmp_path, "file.md", "content\n"  # comment)
```
This is a SyntaxError — the `)` closing the function call is now inside the comment.

## Context
Using sed to append `  # nocheck:project-types` to lines that contain Python string literals ending with `\n")`. The sed pattern matched the string content and appended the comment, but the closing `)` that ends the function call remained on the same line and was consumed by the comment.

## Root cause
sed's substitution replaces the matched pattern and appends the comment at that position. The `)` that follows the string on the same line is NOT part of the sed match — it stays where it is — but since `#` starts a comment in Python, everything after `# comment` including `)` is now a comment. The function call is never closed.

## Fix
Extract the string to a named variable first, then put the comment on that line:
```python
# Instead of:
f = make_file(tmp_path, "file.md", "content\n")  # comment

# Do:
content = "content\n"  # comment
f = make_file(tmp_path, "file.md", content)
```

Or use a nocheck comment on a line that doesn't contain syntax-critical trailing characters.

## Why non-obvious
The sed substitution looks correct when applied: the pattern matches and the comment appears at the end. The error only becomes visible when Python parses the file — the syntax error message points to the argument line, not to the comment insertion, making the root cause non-obvious.
