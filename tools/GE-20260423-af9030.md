## Context

When writing implementation plans for batches of 10–20+ canvas sprite draw functions, including the actual Canvas 2D code in the plan steps causes the plan to exceed LLM token limits (observed at ~32k tokens for 20 units × 50–100 lines of draw code each).

## Technique

In plan steps, replace the full Canvas 2D implementation with a structured visual description. The subagent implementing the task writes the actual code from the description.

**Instead of:**
```javascript
function drawSCV(ctx, S, dir, teamColor) {
  if (dir === 3) { ctx.save(); ctx.translate(S, 0); ctx.scale(-1, 1); ... }
  ctx.fillStyle = '#4a4a4a';
  ctx.fillRect(cx - S*0.12, cy - S*0.18, S*0.24, S*0.28);
  // ... 40 more lines
}
```

**Write:**
```markdown
**Visual design — implement using Canvas 2D API:**
- Stocky industrial worker suit, grey-gunmetal palette (~`#4a4a4a` body).
- Body: rounded rectangular torso centred at (S/2, S/2). Must cover pixel (64,64).
- Helmet: rounded dome on top, visor slit in team colour, shadowBlur ~6.
- Tool arm: dir 1 — welding arm extends right with bright spark tip in team colour.
- Two short rectangular legs below torso.
```

The subagent writes idiomatic Canvas 2D from the description in its own context window, keeping each task self-contained.

## Why this is non-obvious

The natural instinct when writing a technical plan is to include the full implementation code to be precise. The token limit is only hit when you try to run the plan — not when writing it. The fix (visual descriptions) feels less rigorous but works better in practice because:
1. Plans stay small enough to write and run in one session
2. Subagents write higher-quality draw code than copy-pasted plan code anyway (they optimize for the current canvas state)
3. The visual description is sufficient specification for a skilled agent

## Constraints

Applies specifically to tasks where the implementation is code that the subagent can write correctly from a description (graphics, CSS, templates). Does not apply to algorithmic logic where exact signatures or data structures must match existing interfaces.
