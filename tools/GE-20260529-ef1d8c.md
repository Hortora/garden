## What it is

When tracking per-session cursor state (sequence numbers, page tokens, watermarks) for an agent
or user process where sessions reset periodically, the intuitive key is `agentId:sessionKey`.
But this creates an unbounded Map — old session entries are never cleaned up, and the Map grows
indefinitely as sessions rotate.

**Better:** key by `agentId` only, store `sessionKey` alongside the cursor value. When the stored
`sessionKey` differs from the current one, the session has reset — treat `since` as `0` (fresh
start). The Map is bounded by distinct agents, not by total session count.

## How to use it

```typescript
private readonly cursors = new Map<string, { cursor: number; sessionKey: string }>();

private getCursor(agentId: string, sessionKey: string): number {
  const entry = this.cursors.get(agentId);
  // Session changed → reset cursor to 0 (new session gets full window)
  return (entry?.sessionKey === sessionKey) ? (entry?.cursor ?? 0) : 0;
}

private setCursor(agentId: string, sessionKey: string, cursor: number): void {
  this.cursors.set(agentId, { cursor, sessionKey });
}
```

## Why it's non-obvious

The natural decomposition is "one cursor per agent-session combination" → key by
`agentId:sessionKey`. This is correct on day 1. It silently leaks on day 30 when the same 20
agents have each had 30 sessions and the Map holds 600 entries, none of which are the current
session except the 20 most recent.

The session-change detection also adds semantic value: when `sessionKey` changes, `since=0`
is the right response (a new session should get the full buffered context, not continue from
where the old session left off). The Map-bounding and the semantic correctness solve together.

## Context

Originally developed for an OpenClaw `before_prompt_build` plugin cursor tracking sequences
from a Qhorus ChannelContextWindow REST API. OpenClaw sessions reset daily and on idle timeout;
each reset produces a new `sessionKey`. The plugin process runs continuously, making unbounded
growth a real concern.

Applies to any long-running process with recurring sessions: WebSocket connections, OAuth
token cursors, polling watermarks.
