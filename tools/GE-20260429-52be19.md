## What

Amend the commit message of any commit in the unpushed history — not just HEAD — without using `git rebase -i`, which requires interactive shell input unavailable in agentic sessions.

## Technique

```bash
# Target: amend message of COMMIT_SHA (not HEAD)
# Prerequisite: COMMIT_SHA and HEAD are the only two commits ahead of the shared base

C1=COMMIT_SHA
C2=$(git rev-parse HEAD)     # the commit on top
PARENT=$(git rev-parse $C1^) # shared base

NEW_MSG="$(git log -1 --format=%B $C1)

Refs #69"  # append to existing message

# Create new commit object for C1 with amended message
NEW_C1=$(git commit-tree $(git rev-parse ${C1}^{tree}) -p $PARENT -m "${NEW_MSG}")

# Restack C2 on top of new C1 (same tree, new parent)
NEW_C2=$(git commit-tree $(git rev-parse ${C2}^{tree}) -p $NEW_C1 -m "$(git log -1 --format=%B $C2)")

git reset --hard $NEW_C2
```

## Why This Is Non-Obvious

`git rebase -i` is the standard answer but opens an interactive editor — unusable in non-interactive shells or agentic sessions. `git commit --amend` only works on HEAD. `git commit-tree` is a low-level plumbing command most developers never touch; it creates commit objects directly without any editor or interactive prompt.
