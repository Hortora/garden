## Context

Non-interactive rebase where a `squash` step produces a combined message (both commit subjects concatenated) but you want a specific unified subject — different from either individual message.

## Technique

Add an `exec git commit --amend -m "..."` line immediately after the `squash` line in the rebase todo:

```
pick abc1234 chore: register repo — BOM, docs, badge
squash def5678 chore: add repo to CI workflows, build scripts, dashboard
exec git commit --amend -m "chore: register repo — BOM, docs, badge, CI workflows, dashboard"
pick ghi9012 feat: next unrelated commit
```

Git executes the `exec` line as a shell command at that point in the rebase. The amend fires before any subsequent `pick` steps, leaving no follow-up work.

Works with `GIT_SEQUENCE_EDITOR="cp /tmp/plan.txt"` for fully non-interactive rebases.

## Why This Is Non-Obvious

Most developers either accept the combined squash message, or do a separate `git rebase -i` / amend pass after the main rebase completes — requiring navigation back to the right commit in history. The `exec` approach is self-contained: the custom message is set inline as part of the same rebase operation.
