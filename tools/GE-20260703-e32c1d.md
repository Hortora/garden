### Symptom
Configuring `.mcp.json` with `"type": "url"` and a Streamable HTTP endpoint causes Claude Code to silently skip the server. Running `/mcp` shows: `Skipped — unknown MCP server type "url" for server "drafthouse"`.

### Context
The MCP specification (2025-06-18) defines Streamable HTTP as the `url` transport type. `quarkus-mcp-server-http` exposes this at `/mcp`. When configuring Claude Code to connect to a Quarkus MCP server, the natural choice is `"type": "url"` — but Claude Code doesn't recognise it.

### What was tried (didn't work)
```json
{
  "mcpServers": {
    "drafthouse": {
      "type": "url",
      "url": "http://localhost:9001/mcp"
    }
  }
}
```
Server was healthy — `curl` to the endpoint worked. The error was purely client-side.

### Root cause
Claude Code (as of July 2025) only recognises two MCP transport types: `"stdio"` and `"sse"`. The `"url"` type from the MCP spec is not yet implemented.

### Fix
Use `"type": "sse"` with the SSE fallback endpoint that `quarkus-mcp-server-http` also exposes:

```json
{
  "mcpServers": {
    "drafthouse": {
      "type": "sse",
      "url": "http://localhost:9001/mcp/sse"
    }
  }
}
```

### Why this is non-obvious
The MCP spec says `url` is the transport type for HTTP servers. The Quarkus extension is named `mcp-server-http`. A developer would naturally use `"type": "url"` — the error message ("unknown MCP server type") doesn't suggest trying `"sse"` as a fallback.
