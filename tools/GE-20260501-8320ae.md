## Symptom

GitHub Actions workflow reads `${{ secrets.MY_SECRET }}` as empty string. The `gh secret set --org` command reported success. Re-setting the secret still fails. The workflow is in a **public repository**.

## Context

When setting an org-level secret via `gh secret set --org <org>`, the default visibility is **private repositories only**. Public repos in the org cannot see the secret — the runner simply gets an empty string with no error or warning.

## What Was Tried

- Re-running `gh secret set --org casehubio` — reported `✓ Set Actions secret GH_PAT for casehubio` each time
- Checking PAT token validity — token was valid
- Adding scope check workflow — secret read as empty

## Root Cause

`gh secret set --org` defaults to `--visibility private`. Public repos are excluded from this visibility level. GitHub Actions runners in public repos silently receive an empty value — there is no error at workflow runtime.

## Fix

```bash
gh secret set GH_PAT --org casehubio --visibility all
```

The `--visibility all` flag makes the secret available to all repositories in the org, including public ones.

## Why This Is Non-Obvious

The `gh secret set` command reports success regardless of visibility setting. There is no indication at set-time or run-time that the secret is inaccessible. The PAT check workflow fails with "token is invalid or expired" — a symptom that points to token validity, not visibility. Hours can be spent rotating and recreating tokens before the visibility flag is discovered.
