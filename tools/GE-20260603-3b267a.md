### Symptom
Calling `mcp__intellij-index__ide_search_text` with parameter `q` returns a parameter validation error and no results — silently fails with no fallback or hint that the parameter name is wrong.

### Context
All other `mcp__intellij-index__` search and find tools (`ide_find_file`, `ide_find_class`, `ide_find_symbol`, `ide_find_references`) use `q` as the search parameter. `ide_search_text` is the exception — it uses `query`.

### What was tried
Passing `q: "guardrail"` as with other tools — error returned.

### Root cause
The `ide_search_text` tool was authored with a different parameter convention from the rest of the MCP server. The error message does not name the correct parameter.

### Fix
Use `query` instead of `q`:
```json
{
  "tool": "mcp__intellij-index__ide_search_text",
  "query": "guardrail",
  "filePattern": "*.java"
}
```

### Why this is non-obvious
Every other `mcp__intellij-index__` search tool uses `q`. There is no documentation listing the parameter names; they must be inferred from tool descriptions or error messages. The error message on failure does not suggest the correct parameter name.
