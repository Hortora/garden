### Symptom
CI fails with `Could not find artifact io.casehub:casehub-worker-api:jar:0.2-SNAPSHOT in github (https://maven.pkg.github.com/casehubio/*)`. The artifact exists in GitHub Packages and resolves locally.

### Context
GitHub org with multiple repos publishing Maven SNAPSHOTs to GitHub Packages. All CI workflows use `GITHUB_TOKEN` with `packages: write` permission and a wildcard repository URL (`https://maven.pkg.github.com/<org>/*`). Works perfectly until a dependency is added from a **private** repo.

### What was tried (didn't work)
- Verifying the artifact exists in GitHub Packages (it does)
- Checking Maven repository configuration (wildcard URL is correct)
- Re-triggering CI (consistent failure across multiple runs)
- Adding `write:packages` scope to tokens (irrelevant — the issue is repo visibility, not token scope)

### Root cause
GitHub Packages Maven artifacts inherit visibility from the **repository**, not the package. Unlike container packages, Maven packages have no independent visibility setting. `GITHUB_TOKEN` with `packages:read` can only resolve artifacts from **public** repos cross-repo. Private repo → private packages → invisible to other repos' `GITHUB_TOKEN`.

The wildcard URL gives no indication this is happening — Maven just reports "artifact not found" as if it doesn't exist.

### Fix
Make the source repository public. This is the only fix that preserves the `GITHUB_TOKEN` + wildcard pattern. Alternative: use a PAT with org-wide `read:packages` scope in CI, but this adds token management overhead across all consuming repos.

Verify with: `gh api 'orgs/<org>/packages/maven/<package-name>' --jq '.visibility'` — if `private`, the package is invisible to other repos' `GITHUB_TOKEN`.

### Why this is non-obvious
The wildcard URL works for all public repos in the org. When one private repo is added to the dependency chain, only that dep fails — everything else continues to resolve. The error message (`Could not find artifact`) gives zero indication that visibility is the issue. You naturally debug Maven config, token scopes, and SNAPSHOT metadata before considering repo visibility.
