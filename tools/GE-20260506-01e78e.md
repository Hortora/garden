### Symptom
Flex container has `gap: 0` and all bounding boxes appear to touch (0px gap), but visually there are obvious gaps between elements. Measuring distances between bounding boxes shows 0px even though the browser renders clear spacing.

### Context
Debugging CSS layout with Playwright — reading element positions via `element.bounding_box()` to diagnose nav item spacing issues.

### What was tried
Set `gap: 0` on the flex container. Measured `bounding_box()` for each nav item — all appeared to be adjacent (0px between right edge of one and left edge of next). Visually the browser still showed significant spacing.

### Root cause
`bounding_box()` returns the element's **border box** only — it does not include CSS `margin`. Margins exist outside the border box and are invisible to `bounding_box()` measurements. A `margin-right: 20px` on every element produces 20px gaps that are completely undetectable by measuring bounding boxes.

In this case, minima's compiled CSS set `.site-nav .page-link:not(:last-child) { margin-right: 20px }` — but that rule's higher specificity (due to `:not()` pseudo-class) meant a `margin: 0` override was silently losing.

### Fix
Use `window.getComputedStyle()` to read the actual computed margin values:

```python
margin = await page.evaluate(
    "el => window.getComputedStyle(el).marginRight", el
)
# Returns "20px" — the real culprit
```

Once the margin is identified, override it with a selector of equal or higher specificity.

### Why this is non-obvious
`bounding_box()` feels like it should capture "how much space this element occupies" — but margins are not part of the box model's border box. The W3C box model includes margin as the outermost layer, but Playwright (and DevTools' layout API) only reports up to the border box. You can stare at position data all day and never see the gap.
