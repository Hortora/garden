## OpenClaw POST /api/sessions/main/messages does not exist — widely cited but returns 404

**ID:** GE-20260614-8c0371
**Stack:** OpenClaw v2026.6.x

### Symptom

```
POST http://localhost:18789/api/sessions/main/messages
→ HTTP 404 Not Found
```

Also 404: `GET /api/sessions`, `GET /api/status`, `POST /hooks/agent`, `POST /hooks/wake`.

### Context

Multiple third-party articles, API guides, and community posts cite
`POST /api/sessions/main/messages` as the primary REST endpoint for triggering an
OpenClaw agent turn. This endpoint does not exist in the running gateway.

### Root cause / Undocumented behaviour

The OpenClaw gateway REST API is more limited than third-party docs suggest. By default,
only two endpoints respond with non-404 status:

- `GET /healthz` → `{"ok":true,"status":"live"}`
- `GET /readyz` → readiness check

All `/api/*` paths, `/hooks/*` paths, and session endpoints return 404 with default config.

**Working alternatives:**

1. **OpenAI-compatible endpoint** (recommended) — enable in `openclaw.json`:
   ```json
   {"gateway": {"http": {"endpoints": {"chatCompletions": {"enabled": true}}}}}
   ```
   Then call:
   ```
   POST /v1/chat/completions  (model: "openclaw")
   ```

2. **Webhooks** — `POST /hooks/agent` and `POST /hooks/wake` may require channel/hook
   plugin configuration to activate (not tested exhaustively at v2026.6.5).

### Why this is non-obvious

The endpoint appears in multiple authoritative-looking third-party integration guides.
The OpenClaw official docs describe hook endpoints as existing without stating they
require additional config to activate. Nothing in the 404 response or gateway logs
explains which config is missing or what the correct endpoint is.
