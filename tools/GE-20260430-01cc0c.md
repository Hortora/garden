## Context

You need to re-trigger CI on a branch without making any code changes — e.g. to pick up newly published upstream artifacts, to work around a CI flake, or to re-run after an infrastructure issue.

## Technique

```bash
git commit --allow-empty -m "ci: retrigger — <reason>"
git push
```

This creates a commit with no file changes, which GitHub Actions treats as a new push and triggers the workflow. Cleaner than touching a dummy file.

## Why This Is Non-Obvious

The `--allow-empty` flag is easy to miss and most developers' first instinct is to touch a file (e.g. add a blank line to README). The empty commit approach leaves no noise in the working tree and is self-documenting via the commit message.
