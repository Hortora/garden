### The technique
Use `ide_read_file` with `qualifiedName` parameter to read library/dependency
source code directly from Maven repository JARs — no downloading, extracting,
or navigating JAR files manually.

### How to use it
```
ide_read_file(qualifiedName="io.smallrye.mutiny.groups.UniZip",
              project_path="/path/to/project")
```

Returns the full source from the sources JAR in the Maven local repository
(e.g. `~/.m2/repository/io/smallrye/reactive/mutiny/3.1.1/mutiny-3.1.1-sources.jar`).
Works for any dependency on the project's classpath.

### When to use it
When you need to verify a library's actual behavior — not what the docs say,
but what the code does. Common scenarios:
- Verifying exception conditions (does this API really throw on empty input?)
- Understanding internal validation logic
- Checking default values and fallback behavior
- Tracing call chains through library internals

### Why this is non-obvious
Most developers — and AI agents — reach for `mvn dependency:copy` or
manually extract JARs to read library source. The IntelliJ MCP's
`qualifiedName` parameter resolves the class through the project's
dependency graph and reads it directly, with no filesystem gymnastics.
The parameter exists on `ide_read_file` but is not prominently documented.
