### Symptom
After rebasing a feature branch onto a rewritten main, git emits warnings like:

```
warning: skipped previously applied commit 1bcdb2c
warning: skipped previously applied commit ed2b9b9
hint: use --reapply-cherry-picks to include skipped commits
```

Only 1 of 5 commits survives. Looks like the other 4 feature commits were dropped and their code is gone.

### Context
Feature branch was squashed (11→5 commits), then rebased onto main. Main's history had been rewritten (prior squash) since the feature branch was created. Git computes patch IDs for each commit and skips any that match patches already present in the target branch.

### Root cause
`git rebase` detects "previously applied" commits by comparing patch IDs — a hash of the diff content, independent of commit SHA or message. When main was rewritten, some commits' patch content was absorbed into the rewritten history. Git correctly skips re-applying content that's already there, but the warning message gives no indication of whether this is safe (content already in target) or a problem (content genuinely dropped).

### Fix
After the rebase, verify by comparing trees — not commit counts:

```bash
# Zero = all content preserved, the skipped commits were already in target
git diff original-branch rebased-branch | wc -l

# See which files differ (if any)
git diff --name-only original-branch rebased-branch
```

If the two-dot diff is 0, the rebase is correct — the "skipped" commits' content was already in the target branch. The apparent commit count drop is real but not a content loss.

If the diff is non-zero, use `--reapply-cherry-picks` to force all commits through, then resolve any conflicts.

### Why this is non-obvious
The warning message "skipped previously applied commit" says nothing about safety. A developer expecting 5 commits to land and seeing 1 appear will assume the other 4 were dropped — this looks exactly like content loss. The only reliable check is the tree diff, not the commit count. The hint to use `--reapply-cherry-picks` implies something went wrong, but in the common case (rewriting main absorbed the content) it's the correct outcome.
