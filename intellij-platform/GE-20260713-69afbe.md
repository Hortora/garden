### Symptom
Calling `ide_edit_member`, `ide_replace_member`, or `ide_insert_member` on a
TypeScript file returns: "Member editing not supported for TypeScript.
Supported: Java, Kotlin."

### Context
The IntelliJ MCP plugin's structural editing tools (`ide_edit_member`,
`ide_replace_member`, `ide_insert_member`) work by resolving PSI member
elements. The implementation is Java/Kotlin-specific — TypeScript PSI
resolution is not wired up.

### Root cause
The plugin's member-editing endpoints use Java/Kotlin PSI tree traversal to
locate class members by name. TypeScript's PSI tree structure differs (ESTree
vs JVM class model), and the plugin does not implement the TypeScript-specific
member resolution.

### Fix
Use `ide_replace_text_in_file` for targeted text replacements in TypeScript
files. This tool works across all languages since it operates on document text,
not PSI structure.

For new TypeScript files, use the `Write` tool (or `ide_create_file`).

For refactoring operations (rename, move), `ide_refactor_rename` and
`ide_move_file` DO work for TypeScript — only the member-level editing
tools are unsupported.

### Why this is non-obvious
The IntelliJ MCP tool documentation does not mention language limitations.
The tools appear generic — "replace a member" sounds language-agnostic. You
only discover the limitation when you call the tool on a .ts file mid-
implementation, forcing an unplanned tool switch.
