## Symptom

An `LocalInspectionTool` that checks `PsiAnnotation.getQualifiedName()` against a fully-qualified name fires correctly in production but never fires in `BasePlatformTestCase` tests, even when the annotation is present. No error is thrown — the inspection silently does nothing.

## Context

`BasePlatformTestCase` tests use `myFixture.configureByText()` with Java source that imports annotation types. The annotation JARs are typically not on the test module's classpath (only `provided` scope in production), so PSI cannot resolve the import. In this state, `getQualifiedName()` returns the bare simple name — e.g. `"PermuteAnnotation"` instead of `"io.quarkiverse.permuplate.PermuteAnnotation"`.

## Root cause

`PsiAnnotation.getQualifiedName()` resolves through the import. When the import is unresolved, the FQN cannot be determined and PSI returns the simple name only. The common defensive check `fqn.endsWith(".SimpleAnnotationName")` also fails because `"PermuteAnnotation"` contains no dot, so `endsWith(".PermuteAnnotation")` is false.

```java
// BROKEN — both checks fail when import is unresolved:
if (!fqn.equals("com.example.MyAnnotation")
        && !fqn.endsWith(".MyAnnotation")) return;

// "MyAnnotation".equals("com.example.MyAnnotation") → false
// "MyAnnotation".endsWith(".MyAnnotation") → false  ← no dot prefix
```

## Fix

Add a third guard for the no-dot bare simple name case:

```java
String fqn = annotation.getQualifiedName();
if (fqn == null) return;
if (!fqn.equals("com.example.MyAnnotation")
        && !fqn.endsWith(".MyAnnotation")
        && !fqn.equals("MyAnnotation")) return;   // ← bare simple name fallback
```

The bare-name branch is safe in production — when imports resolve, `getQualifiedName()` always returns the FQN and hits the first two branches. The third branch only fires in test/unresolved contexts.

## Why this is non-obvious

The symptom — inspection silently never fires — gives no signal that FQN resolution is the cause. The `endsWith(".MyAnnotation")` pattern is specifically designed as a fallback, so developers assume it covers unresolved imports. It doesn't, because `"MyAnnotation".endsWith(".MyAnnotation")` is false (no leading dot). Only discovering that `getQualifiedName()` returns the bare simple name (not null, not a partial name) reveals the fix.
