### Symptom

`ide_search_text` with query `"drafthouse-diff"` returns zero matches in Java E2E test files, even though the string appears verbatim in Playwright locator calls like `page.locator("drafthouse-diff .entry")`.

### Context

During a Web Component rename migration (6 panels renamed), `ide_search_text` was used to find all old tag-name references in Java E2E test files. It consistently reported zero matches, leading to the belief that no test files needed updating. Multiple E2E test suites then failed at runtime because Playwright selectors still referenced the old tag names.

### Root cause

IntelliJ's word index tokenises Java source at the lexical level. When a string like `"drafthouse-diff"` appears inside a Java string literal, the index sees it as a single token. But when the same string appears inside a concatenated expression — e.g. `"const el = document.querySelector('drafthouse-diff');"` built with `+` — the index may not reconstruct the full string across segments. The search query `"drafthouse-diff"` does not match partial tokens within longer string literals.

Additionally, `page.evaluate("() => { ... document.querySelector('drafthouse-timeline') ... }")` embeds the search target inside a JavaScript string within a Java string — the index does not decompose nested string content.

### Fix

Do not rely solely on `ide_search_text` for rename migrations in test files with inline JavaScript/Playwright selectors. After the IDE search, verify with direct file reads on the test files most likely to contain the old names. For comprehensive coverage, dispatch a subagent to read every file in the test directory and search for the literal string.

### Why this is non-obvious

`ide_search_text` works correctly for most Java string searches — it finds class names, method names, and simple string constants reliably. The failure mode is specific to strings embedded within longer concatenated expressions or nested language strings (JavaScript inside Java), which is exactly the pattern Playwright E2E tests use extensively.
