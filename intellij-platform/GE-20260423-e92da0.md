### Symptom
`myFixture.testParameterInfo()` causes `NoSuchMethodError` or is unavailable in IntelliJ 2023.2 when testing a custom `ParameterInfoHandler`.

### Context
Testing a `ParameterInfoHandler<PsiElement, MyBuiltin>` for a custom language in a `BasePlatformTestCase` with `intellijIdeaCommunity("2023.2")` in build.gradle.kts.

### Fix
Use `MockCreateParameterInfoContext` from `com.intellij.testFramework.utils.parameterInfo`:

```java
import com.intellij.testFramework.utils.parameterInfo.MockCreateParameterInfoContext;

myFixture.configureByText("Foo.java",
    "... typeArgList(1, <caret>i, 'T') ...");
MyParameterInfoHandler handler = new MyParameterInfoHandler();
MockCreateParameterInfoContext ctx =
    new MockCreateParameterInfoContext(myFixture.getEditor(), myFixture.getFile());
PsiElement element = handler.findElementForParameterInfo(ctx);
if (element != null) {
    handler.showParameterInfo(element, ctx);
    assertNotNull(ctx.getItemsToShow());
}
```

### Why This Is Non-Obvious
`testParameterInfo()` is the documented approach in IntelliJ plugin SDK examples. `MockCreateParameterInfoContext` is in the test framework jar but absent from official docs. Its availability and API differ by platform version — this pattern works in 2023.2 but the equivalent `testParameterInfo()` method appears only in later platform versions.
