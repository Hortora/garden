### Symptom
`ide_create_file` with 80+ lines of Java content hangs for 300 seconds then times out with "MCP server sent no response or progress for 300s; aborting." The MCP server remains responsive to read operations (`ide_index_status`, `ide_diagnostics`, `ide_read_file`) — only the write stalls.

### Context
Occurs when creating new source files with substantial initial content (full class with imports, fields, methods). Small files (5–10 lines) succeed immediately. The failure is consistent — retrying the same large content times out again.

### Root cause
The intellij-index MCP plugin's file creation path appears to block on VFS refresh or indexing when the initial content is large. The 300s timeout is the Claude Code MCP idle timeout, not an IntelliJ timeout — the plugin never responds.

### Fix
Create a minimal skeleton file first, then build up incrementally:

```
// Step 1: Create minimal file
ide_create_file(file: "path/To/MyClass.java", content: "package com.example;\n\nclass MyClass {\n}\n")

// Step 2: Replace class declaration with full content
ide_edit_member(file: "path/To/MyClass.java", member: "MyClass", content: "class MyClass implements Foo {\n    // full body here\n}")

// Step 3: Add imports if needed (ide_edit_member doesn't manage imports)
ide_replace_text_in_file(file: "...", searchText: "package com.example;\n\n", replaceText: "package com.example;\n\nimport ...;\n\n")
```

### Why this is non-obvious
The timeout is silent — no error message appears until the 300s MCP idle timeout fires. Read operations continue working normally, which misleads you into thinking the MCP server is healthy and the specific file creation is broken for a different reason (wrong path, permissions, etc.). The size-dependent nature is not obvious because smaller files work instantly.
