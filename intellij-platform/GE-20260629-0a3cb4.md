### Symptom

All IntelliJ Index MCP search and navigation tools return empty results for a TypeScript project: `ide_find_file`, `ide_search_text`, `ide_find_class`, `ide_find_references`, `ide_find_definition` all return zero matches. `ide_file_structure` explicitly errors: "Language not supported for file structure. Supported languages: JAVA, kotlin, Markdown."

The project is open and in active mode (`ide_project_status` confirms `mode: active`, `isDumbMode: false`). Syncing files (`ide_sync_files`) does not help.

### Context

A TypeScript monorepo (Yarn workspaces, Vitest) opened in IntelliJ IDEA. The project has no `pom.xml` or `build.gradle` — it uses `package.json` + `tsconfig.json`. `ide_reload_project` fails with "No Maven or Gradle build system found."

`ide_open_file` works (opens files in the editor), and `ide_read_file` works for reading content. Only the index-based tools are empty.

### What was tried (didn't work)

1. Setting project to active mode (`ide_set_project_mode active`)
2. Syncing files (`ide_sync_files` with full project sync)
3. Opening a file to trigger indexing (`ide_open_file`)
4. Checking index status (reports `isDumbMode: false, isIndexing: false`)
5. Reloading project (`ide_reload_project` — fails, no Maven/Gradle)

### Root cause

The JetBrains Index MCP plugin's search and navigation tools rely on IntelliJ's Java/Kotlin indexing infrastructure, not the TypeScript language service. While IntelliJ IDEA itself has full TypeScript support (rename refactoring, find usages, etc. via the TypeScript language service plugin), the MCP plugin does not bridge to this — it only exposes the Java/Kotlin PSI-based index.

`ide_file_structure` confirms this directly: its supported languages list is "JAVA, kotlin, Markdown" — despite the MCP plugin's documentation claiming support for "Java, Kotlin, Python, JavaScript, TypeScript, PHP, Markdown."

### Fix

None — this is a plugin limitation. For TypeScript projects:

1. **Cannot use** `ide_find_references`, `ide_find_definition`, `ide_find_class`, `ide_search_text`, `ide_file_structure`, `ide_refactor_rename` via MCP
2. **Can use** `ide_open_file`, `ide_read_file`, `ide_index_status`, `ide_sync_files`, `ide_project_status`
3. **For semantic operations**: use the TypeScript compiler (`yarn typecheck`) for type checking, and text-based search with verification for navigation
4. **For refactoring**: IntelliJ's native TypeScript refactoring works in the IDE directly (Shift+F6 for rename) — just not through MCP

### Why this is non-obvious

The MCP plugin's documentation and tool descriptions claim TypeScript support. The project appears to be properly indexed (`isDumbMode: false`). All tools accept the `project_path` parameter and return successfully — they just return empty results. There is no error message indicating the language is unsupported (except `ide_file_structure` which explicitly lists supported languages). A developer can spend significant time troubleshooting indexing before discovering it's a fundamental plugin limitation.
