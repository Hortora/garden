## Symptom

Adding `@PermuteSource` to a template and running the Maven plugin generates classes that fail to compile with `cannot find symbol: class PermuteSource`. The generated `.java` files contain the raw annotation.

## Context

Permuplate has `@Retention(SOURCE)` on its annotations, meaning they're stripped by `javac` at compile time. However, in inline/Maven plugin mode, Permuplate generates source files using JavaParser — it doesn't go through `javac` itself. The `removeIf` pass that strips Permuplate annotations from generated output must be explicitly updated every time a new annotation is added.

## Root cause

When `@PermuteSource("Callable${i}")` is added to a template class, InlineGenerator clones the class and transforms it. Without explicitly stripping `@PermuteSource` and `@PermuteSources` from the generated AST, they pass through into the output source file. When the user's `javac` compiles that output, it cannot resolve the annotation types (which are stripped from the output classpath in practice).

## Fix

Add `PermuteSource` and `PermuteSources` (both simple and FQ names) to the annotation-stripping `removeIf` call in `InlineGenerator.generate()`:

```java
// In the per-generated-class cleanup block:
generated.getAnnotations().removeIf(ann -> {
    String n = ann.getNameAsString();
    return "Permute".equals(n) || n.endsWith(".Permute")
        || "PermuteFilter".equals(n) || n.endsWith(".PermuteFilter")
        || "PermuteSource".equals(n) || n.endsWith(".PermuteSource")    // NEW
        || "PermuteSources".equals(n) || n.endsWith(".PermuteSources"); // NEW
});
```

## Why this is non-obvious

The annotation has `@Retention(SOURCE)` which implies it disappears at compile time — but "compile time" here means the user's javac, not Permuplate's JavaParser-based generation. Permuplate operates entirely in the AST before javac sees anything, so SOURCE retention provides no protection during Permuplate's own processing. Every new annotation added to Permuplate must be manually added to the strip list.
