## Symptom

`@PermuteAnnotation(type="FunctionalInterface")` (or any `@PermuteAnnotation`) on a
non-inline template class has no effect — the target annotation is absent from all
generated classes. No error or warning is emitted.

## Context

Occurs when using `@PermuteAnnotation` on templates with `inline=false` (the default
APT-style templates processed by `PermuteMojo.generateTopLevel()`). Inline templates
(`inline=true`) are unaffected — they go through `InlineGenerator.generate()` which
correctly calls `PermuteAnnotationTransformer`.

## What was tried

- Verified the annotation syntax is correct and it works on inline templates
- Verified the template compiles and other annotations (`@PermuteTypeParam`, etc.) work
- Checked the generated output — `@FunctionalInterface` completely absent

## Root cause

`PermuteMojo.generateTopLevel()` had a minimal annotation-stripping block that only
removed `@Permute` itself, never calling `PermuteAnnotationTransformer.transform()`.
The transformer that adds the target annotation before stripping `@PermuteAnnotation`
was simply never invoked for non-inline templates.

## Fix

In `PermuteMojo.generateTopLevel()`, call `PermuteAnnotationTransformer.transform()`
**before** `InlineGenerator.stripPermuteAnnotations()`:

```java
// In generateTopLevel(), after renaming and before stripping:
PermuteAnnotationTransformer.transform(typeDecl, ctx, null, null);
InlineGenerator.stripPermuteAnnotations(typeDecl);
// Previously: only removed @Permute itself, skipped all transformers
```

Also replace the hand-rolled minimal strip with `InlineGenerator.stripPermuteAnnotations()`
to ensure `@PermuteAnnotation` itself (and `@PermuteVar`, `@PermuteThrows` etc.) are
stripped from the generated output.

## Why this is non-obvious

The inline and non-inline pipelines look symmetric from the outside. Since `@PermuteAnnotation`
works on inline templates, a developer would expect it to work everywhere. There is no
compile error — the annotation is simply silently ignored. The pipeline gap was a latent
bug present since `@PermuteAnnotation` was first added.
