## The technique

When using `typeArgList()` inside a `@PermuteDeclr` type string to expand generic type argument lists (e.g., `Callable3<A, B, C>`), use **single quotes** for the style argument rather than escaped double quotes:

```java
// Clean — single quotes inside the JEXL expression:
private @PermuteDeclr(
    type = "Callable${i}<${typeArgList(1,i,'alpha')}>",
    name = "c${i}") Callable2<A, B> c2;

// Also works but noisier — escaped double quotes:
private @PermuteDeclr(
    type = "Callable${i}<${typeArgList(1,i,\"alpha\")}>",
    name = "c${i}") Callable2<A, B> c2;
```

Both produce `Callable3<A, B, C>` for i=3. The single-quote form is cleaner and avoids backslash clutter in the Java source.

## Why this is non-obvious

JEXL3 supports both single and double quoted strings. Inside a Java annotation attribute value (already surrounded by double quotes), using single quotes for inner JEXL string literals avoids escaping entirely. The alternative — `\"alpha\"` — requires remembering that the backslash is part of the Java string literal, not the JEXL expression. Many developers reach for `\"` by default and end up with noisy annotation strings.
