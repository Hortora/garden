### Symptom
`wrong number of type arguments; required N` errors at call sites that reference the template class with N-1 type args — call sites look correct but the compiler disagrees.

### Context
A Permuplate `@Permute` template class declares a sentinel type param with `@PermuteTypeParam`:

```java
@Permute(varName = "n", from = 2, to = 10, className = "ScopeGate${n}",
         inline = true, keepTemplate = true)
public static class ScopeGate1<CTX, B,
        @PermuteTypeParam(varName = "j", from = "3", to = "${n+1}") C> { ... }
```

For `n=1` (the template class itself, kept via `keepTemplate=true`), the `@PermuteTypeParam` range is `from=3, to=2` — empty. The expectation is that `C` is dropped, giving `ScopeGate1<CTX, B>` (2 type params). Code referencing `ScopeGate1<CTX, B>` then fails with "wrong number of type arguments; required 3".

### Root cause
`keepTemplate=true` retains the template class verbatim from the source file. The template source has `ScopeGate1<CTX, B, C>` (3 type params). The empty `@PermuteTypeParam` range drops `C` only in the *generated* classes (ScopeGate2..ScopeGate10), not in the kept template. So the actual `ScopeGate1` class still has 3 type params.

### Fix
Use `keepTemplate=false` when the template class must not appear in the compiled output — only the generated classes should exist. Renumber the template if needed:

```java
// Template named ScopeGate1, generates ScopeGate1..ScopeGate10 (no kept template)
@Permute(varName = "n", from = 1, to = 10, className = "ScopeGate${n}",
         inline = true, keepTemplate = false)
public static class ScopeGate1<CTX, B,
        @PermuteTypeParam(varName = "j", from = "3", to = "${n+1}") C> { ... }
```

With `keepTemplate=false`, `ScopeGate1` is generated with `n=1` → empty range → `C` dropped → `ScopeGate1<CTX, B>` (2 type params). ✓

### Why this is non-obvious
`keepTemplate=true` is described as "keep the template class in the output." The natural reading is "keep it with the same transformations applied." The actual behaviour is "keep it verbatim from source" — which means the empty-range type param stays. The error appears at call sites with the wrong arity count, pointing away from the `keepTemplate` setting.
