### Summary
Use `@PermuteReturn(when="i == N")` (with `Object` declared return type) to generate a method only on the class produced at arity N, suppressing it for all other arities.

### Context
A Permuplate template generates classes at arities i=2..10. A method should exist only at arity 2 (e.g. `path5()` as a stub that makes sense for `Join2Gate` specifically, inherited by `Join2First`), but should be absent from `Join3Gate..Join10Gate`.

The naive approach `@PermuteReturn(className="void", when="false")` fails when the method has a non-`Object` declared return type (see related gotcha: `when` is ignored for non-Object return types). Even with `className="void"`, `when="false"` suppresses the method from all generated classes including i=2.

### Technique
Declare the method with `Object` return type so `applyPermuteReturn` processes it, then use `when="i == 2"`:

```java
@PermuteReturn(when = "i == 2")
public Object path5(Function2<PathContext<Tuple5<C, PB, PC, PD, PE>>, C, ?> fn2,
                    Predicate2<PathContext<Tuple5<C, PB, PC, PD, PE>>, PB> flt2) {
    // body — kept verbatim on the i=2 generated class
    Path5<...> path5 = new Path5<>(null, null, null);
    return path5.path(fn2, flt2);
}
```

- For i=2: `when = "2 == 2"` = true → method kept with original body ✓
- For i=3..10: `when = "3 == 2"` = false → method suppressed ✓

With `keepTemplate=false`, the i=2 generated class retains the method body and the return type is the declared `Object` (callers get `Object` back). Works well for stubs whose value is compilation-time typing (the body is not called at runtime in production use).

### Why this is non-obvious
Most uses of `@PermuteReturn` provide `className` to set the return type. Using it without `className` but with `when` purely for suppression is an unusual pattern. The combination of "Object sentinel + when condition" as an arity gate is not documented anywhere.
