### What's surprising

Two independent silent breaking changes in the casehub-ledger SPI migration:

1. `ActorIdentityProvider` moved from `io.casehub.ledger.runtime.privacy` (in `casehub-ledger` jar) to `io.casehub.ledger.api.spi` (in `casehub-ledger-api` jar).
2. `tokeniseForQuery(String)` return type changed from `String` to `Optional<String>`.

### Symptom

```
[ERROR] cannot find symbol
  symbol:   class ActorIdentityProvider
  location: package io.casehub.ledger.runtime.privacy

[ERROR] tokeniseForQuery(java.lang.String) in TestActorIdentityProvider
  does not override or implement a method from a supertype
```

The `@Override` on `tokeniseForQuery` becomes invalid because the return type no longer matches — but this error only appears AFTER fixing the import, making it look like a second unrelated problem.

### Root cause

The `ActorIdentityProvider` interface was promoted from `casehub-ledger` runtime to `casehub-ledger-api` as part of the ledger SPI externalization. Simultaneously the `tokeniseForQuery` signature changed to return `Optional<String>` for null-safety.

### Fix

```java
// Before
import io.casehub.ledger.runtime.privacy.ActorIdentityProvider;

@Override
public String tokeniseForQuery(String rawActorId) {
    return rawActorId;
}

// After
import io.casehub.ledger.api.spi.ActorIdentityProvider;

@Override
public Optional<String> tokeniseForQuery(String rawActorId) {
    return Optional.ofNullable(rawActorId);
}
```

### Why this is non-obvious

Two independent breaking changes in the same migration — the package change is visible immediately but the return type change only surfaces as a second error after fixing the first. Nothing in the error message about `tokeniseForQuery` suggests the return type changed; the `does not override or implement` message is the same for signature mismatch as for a completely missing interface. A developer sees two separate errors and may not realize they share the same root cause (SPI migration).
