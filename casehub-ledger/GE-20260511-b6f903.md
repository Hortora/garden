**Resolved in: casehub-ledger 0.2-SNAPSHOT (post 2026-05-11)** ‚Äî `@PrePersist` was added
to `LedgerEntry` (runtime model, line 261) and now auto-assigns `id` (UUID.randomUUID()
when null) and `occurredAt` (Instant.now() when null). Writers must NOT set these fields ‚Äî
the entity listener handles them. Setting them explicitly before save() is harmless but
redundant. Unit tests that mock `LedgerEntryRepository` bypass `@PrePersist` ‚Äî do not
assert on `entry.id` or `entry.occurredAt` in mocked tests.

The required caller-set fields are now **5** (down from 7):

```java
AdverseEventLedgerEntry entry = new AdverseEventLedgerEntry();
// id and occurredAt: auto-set by @PrePersist ‚Äî do NOT set manually
entry.subjectId = domainEntity.id;        // the aggregate being audited; scopes the sequence chain
entry.sequenceNumber = nextSeq;           // 1-based per subject; caller computes this
entry.entryType = LedgerEntryType.EVENT;  // EVENT (fact), COMMAND (intent), or ATTESTATION
entry.actorId = "system";                 // actor identity string
entry.actorType = ActorType.SYSTEM;       // HUMAN, AGENT, or SYSTEM
entry.actorRole = "MyRole";               // descriptive role, e.g. "AdverseEventReporter"
// domain-specific fields...
ledgerRepo.save(entry);
```

### What was undocumented (original ‚Äî pre 2026-05-11)
`LedgerEntryRepository.save(entry)` required 7 base fields to be manually populated.
No builder, factory, or documentation stated which fields were required. Missing any
resulted in a DB constraint violation at runtime.

### Computing sequenceNumber
The `sequenceNumber` is the position of this entry in the per-subject chain (1-based). casehub-ledger does **not** auto-increment it. Caller pattern:

```java
int nextSeq = ledgerRepo.findLatestBySubjectId(subjectId)
    .map(e -> e.sequenceNumber + 1)
    .orElse(1);
```

### Key surprises (still current)
- `entryType`, `actorType` are enums from `io.casehub.ledger.api.model` (the API module), not `io.casehub.ledger.runtime.model`.
- `sequenceNumber` is still caller-managed ‚Äî `@PrePersist` does not set it.

### Still-current surprises that were previously undocumented
- `id` was not auto-assigned at submission time ‚Äî now resolved.
- `occurredAt` was not auto-set ‚Äî now resolved.
