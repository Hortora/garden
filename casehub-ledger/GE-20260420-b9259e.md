## Symptom

`AgentMessageLedgerEntryRepository.java` fails to compile with:

```
cannot find symbol: method list(String, UUID)
  location: class io.quarkiverse.ledger.runtime.model.LedgerAttestation
cannot find symbol: method persist()
  location: variable attestation of type LedgerAttestation
```

The error points at your own repository file, not at the quarkus-ledger library. The file's Javadoc comment may say "LedgerAttestation is still a PanacheEntityBase; its Panache static methods are used as-is" — this comment is wrong (or was accurate on an earlier version).

## Root Cause

`LedgerAttestation` in quarkus-ledger 1.0.0-SNAPSHOT is a plain `@Entity` with `@NamedQuery` annotations — it does **not** extend `PanacheEntityBase`. Panache static methods (`list()`, `persist()`, `findById()`, etc.) are not available. The misleading Javadoc is a copy-paste from an earlier design.

## Fix

Replace Panache static calls with `EntityManager` + named queries:

```java
// Instead of: LedgerAttestation.list("ledgerEntryId = ?1 ...", ledgerEntryId)
em.createNamedQuery("LedgerAttestation.findByEntryId", LedgerAttestation.class)
    .setParameter("entryId", ledgerEntryId)
    .getResultList();

// Instead of: attestation.persist()
em.persist(attestation);

// Instead of: LedgerAttestation.list("ledgerEntryId IN ?1", entryIds)
em.createNamedQuery("LedgerAttestation.findByEntryIds", LedgerAttestation.class)
    .setParameter("entryIds", entryIds)
    .getResultList();
```

Named queries are declared as `@NamedQuery` annotations on the `LedgerAttestation` class — check the source jar to find the exact names.

## Why This Is Non-Obvious

The compile error points at your own code (the repository implementation), not at the library. A developer reading the file would see the Javadoc comment asserting Panache is available and assume the issue is elsewhere. The library API changed from Panache to plain JPA without the consuming code being updated.
