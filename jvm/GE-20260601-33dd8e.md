## Symptom

`@Alternative @Priority(1) @ApplicationScoped` bean in a test helper JAR (e.g. `InMemoryWorkItemStore` from `casehub-work-testing`) is not selected despite having `@Priority`. The default non-alternative `@ApplicationScoped` bean (`JpaWorkItemStore`) wins instead, causing `@QuarkusTest` to fail at startup or produce incorrect behaviour when the JPA store can't connect.

## Context

CDI 4.0 spec allows `@Alternative @Priority(N)` to activate globally without explicit registration via `beans.xml` or equivalent. This works within the same application module but does **not** work for `@Alternative` beans declared in external library JARs in Quarkus ARC 3.x.

## Root cause

Quarkus ARC 3.x requires `quarkus.arc.selected-alternatives` for external-JAR `@Alternative` beans to win, even when `@Priority` is set. The CDI spec auto-activation only applies to beans within the same augmentation unit (the application module).

## Fix

Add the bean's fully-qualified class name to `quarkus.arc.selected-alternatives` in `src/test/resources/application.properties`:

```properties
quarkus.arc.selected-alternatives=\
  io.casehub.work.testing.InMemoryWorkItemStore,\
  io.casehub.persistence.memory.MemorySubCaseGroupRepository
```

Also exclude the non-alternative default to prevent ambiguity:

```properties
quarkus.arc.exclude-types=io.casehub.work.runtime.repository.jpa.JpaWorkItemStore
```

## Why this is non-obvious

`@Alternative @Priority(1)` is documented as "globally activating the alternative" without beans.xml. Developers familiar with CDI spec expect this to work for library JARs too. The failure manifests as the JPA bean winning (often causing a different error like datasource not found) rather than an explicit "alternative not activated" message. Checking `selected-alternatives` is not an obvious diagnostic step when `@Priority` is already set.

### Alternatives considered

- Using `@QuarkusTestProfile.getEnabledAlternatives()`: activates the alternative but **replaces** (not appends to) `quarkus.arc.selected-alternatives` — requires re-declaring all other needed alternatives in the same profile method.
