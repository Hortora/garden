## Situation

Adding JSON Schema validation to a Quarkus project. The networknt
json-schema-validator library has three active version lines.

## Gotcha

Version 3.x requires Jackson 3 and JDK 17+. Quarkus (as of 3.32.2) uses
Jackson 2.x. Using networknt 3.x produces runtime classpath conflicts.

## Resolution

Use the 2.x line (Jackson 2 compatible) or 1.x line (also Jackson 2,
well-established API with `JsonSchemaFactory`/`JsonSchema`/`ValidationMessage`).

- 1.0.83: stable, `JsonSchemaFactory.getInstance(SpecVersion.VersionFlag.V202012)`
- 2.0.3: redesigned API (`SchemaRegistry`/`Schema`/`Error`)
- 3.0.5: Jackson 3 only — do not use with Quarkus

## Evidence

Confirmed during casehub-worker#7 implementation (2026-07-06). The networknt
GitHub releases page documents the Jackson 3 upgrade in 3.0.0.
