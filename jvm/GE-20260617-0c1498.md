## The technique

Disable `SseEventSource` auto-reconnect in integration tests by setting an astronomically long reconnect delay:

```java
SseEventSource source = SseEventSource.target(target)
    .reconnectingEvery(Long.MAX_VALUE, TimeUnit.MILLISECONDS)
    .build();
```

**Why `MILLISECONDS` and not `DAYS`:** `TimeUnit.DAYS.toMillis(Long.MAX_VALUE)` overflows to a negative long, which may cause immediate reconnection. `MILLISECONDS` requires no conversion — `Long.MAX_VALUE` ms ≈ 292 million years.

## Why this is non-obvious

`SseEventSource` has a default reconnect delay of 3 seconds. After the server closes a terminal SSE stream, the client reconnects 3 seconds later. In tests where a task is dispatched and the connection is expected to close cleanly after a terminal event:

1. Server sends terminal event, closes sink
2. Client receives terminal event — test latch fires, assertion passes  
3. 3 seconds later, client reconnects to the same URL
4. Server responds with `event:error` (task not found — no new COMMAND dispatched)
5. Error event fires the event handler again, appending to `events` list
6. If the test is slow or the latch was `CountDownLatch(1)`, the second event may corrupt the events list or fire the latch a second time

The failure is intermittent and timing-dependent — sometimes the assertion completes before the reconnect, sometimes not.

## Pattern

```java
CountDownLatch latch = new CountDownLatch(1);
CopyOnWriteArrayList<String> events = new CopyOnWriteArrayList<>();
Client client = ClientBuilder.newClient();
try {
    WebTarget target = client.target(baseUri).path("/stream/" + taskId);
    try (SseEventSource source = SseEventSource.target(target)
            .reconnectingEvery(Long.MAX_VALUE, TimeUnit.MILLISECONDS)  // key line
            .build()) {
        source.register(event -> {
            events.add(event.getName() + "|" + event.readData(String.class));
            latch.countDown();
        });
        source.open();
        // ... dispatch terminal message, await latch ...
        assertTrue(latch.await(10, TimeUnit.SECONDS));
        assertThat(events).hasSize(1); // safe — no reconnect contamination
    }
} finally {
    client.close();
}
```
