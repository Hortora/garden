### Symptom
Consumer `@QuarkusTest` fails with `UnsatisfiedResolutionException` for a dependency
(`ChatModel`) that only the production bean needs — even though an `@Alternative @Priority(1)`
test stub exists for the same SPI and would be selected.

### Context
`LlmQueryExpander` is `@ApplicationScoped` with `@IfBuildProperty(enableIfMissing=true)` — registered by default.
Its constructor requires `ChatModel`. `InMemoryQueryExpander` is `@Alternative @Priority(1)` implementing
the same `QueryExpander` SPI. In a `@QuarkusTest` without a `ChatModel` bean, the expectation is that
`InMemoryQueryExpander` replaces `LlmQueryExpander` and no `ChatModel` is needed.

### What was tried (didn't work)
Adding `InMemoryQueryExpander` as `@Alternative @Priority(1)` to the test classpath. The alternative
is correctly selected for `QueryExpander` injection points, but `LlmQueryExpander` is still a
registered bean and its `ChatModel` constructor parameter is still validated.

### Root cause
Quarkus Arc validates injection points of ALL registered beans at build time — not just the
selected bean for a given type. `@Alternative` controls which bean is injected at consumer
injection points, but does NOT suppress validation of other registered beans' own injection
points. A registered bean with an unsatisfied dependency causes a build failure regardless
of whether it would ever be instantiated.

### Fix
Two complementary approaches:

1. **Remove `enableIfMissing = true`** from `@IfBuildProperty` — require explicit mode
   selection so the heavy bean is only registered when the consumer explicitly opts in.

2. **Add a `@DefaultBean` no-op** that implements the SPI with zero dependencies.
   `@DefaultBean` is displaced by any `@ApplicationScoped` bean (when mode is set)
   or `@Alternative` (in tests). The no-op ensures a bean always exists without
   dragging in heavy transitive dependencies.

### Why this is non-obvious
The mental model of `@Alternative` suggests it "replaces" the original bean. In standard
CDI (Weld), this is closer to true — alternatives affect typesafe resolution. But Quarkus
Arc performs aggressive build-time validation of ALL registered beans' injection points,
including beans that will never be instantiated because a higher-priority alternative exists.
The distinction between "not selected" and "not registered" is invisible until it breaks.
