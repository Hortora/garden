## Symptom

`Awaitility.untilAsserted()` times out with `ConditionTimeoutException` — a downstream assertion (lineage query, event count, etc.) is empty. The `Suppressed:` exception inside the timeout reads:

```
java.lang.NullPointerException: Cannot invoke "CaseInstance.getCaseContext()" because "caseInstance" is null
    at WorkflowExecutionCompletedHandler.onWorkflowExecutionCompletedHandler(...)
```

## Context

Integration test using `InMemoryCaseInstanceRepository` (or `TestCaseInstanceRepository` which extends it). Engine SNAPSHOT added a `tenancyId` parameter to `findByUuid`, making the signature `findByUuid(UUID, String)`. The test profile has no tenancy configured — the engine stores the case instance with `tenancyId=null`. The test then calls `findByUuid(caseId, "default")` to retrieve the instance before publishing `WorkflowExecutionCompleted`.

## Root Cause

`findByUuid(UUID id, String tenancyId)` filters by tenancy using:

```java
if (!tenancyId.equals(instance.tenancyId)) {
    return Uni.createFrom().nullItem();
}
```

`"default".equals(null)` returns `false`, so the method returns `Uni.nullItem()` even though the UUID exists in the store. The null `CaseInstance` is passed into `new WorkflowExecutionCompleted(instance, ...)` — the handler NPEs when it calls `instance.getCaseContext()` — the Vert.x event bus swallows the exception — the lifecycle event (`WorkerExecutionCompleted`) is never fired — the Awaitility condition never satisfies.

## Fix

Inject `CrossTenantCaseInstanceRepository` instead of `CaseInstanceRepository` in tests that don't have a tenancy context:

```java
// Before
@Inject CaseInstanceRepository caseInstanceRepository;
// ...
CaseInstance instance = caseInstanceRepository.findByUuid(caseId, "default")
        .await().atMost(Duration.ofSeconds(5));

// After
@Inject CrossTenantCaseInstanceRepository caseInstanceRepository;
// ...
CaseInstance instance = caseInstanceRepository.findByUuid(caseId)
        .await().atMost(Duration.ofSeconds(5));
```

`CrossTenantCaseInstanceRepository.findByUuid(UUID)` performs a pure UUID lookup with no tenancy filtering. `InMemoryCaseInstanceRepository` implements both interfaces; `TestCaseInstanceRepository extends InMemoryCaseInstanceRepository` inherits both correctly.

## Why This Is Non-Obvious

Three layers of indirection hide the actual cause:

1. **Silent null** — `findByUuid(UUID, String)` returns `Uni.nullItem()` with no exception and no log warning. Nothing signals that the UUID was found but the tenancyId didn't match.

2. **NPE buried as Suppressed** — the Vert.x event bus handler catches the NPE and attaches it as a `Suppressed:` exception inside Awaitility's `ConditionTimeoutException`. It doesn't appear in the main exception message — only visible on close inspection of the stack trace output.

3. **Misleading symptom** — the empty result points to the ledger write or the lineage query, not the instance lookup. Debugging naturally starts in the wrong place. In one session this was misdiagnosed as a `PlanExecutionContext` constructor change because the engine SNAPSHOT had introduced other unrelated changes at the same time.
