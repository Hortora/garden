### Symptom / Context

Building SC2 tactic detection (enemy build classification) with no awareness of external structured data sources. Our `EnemyArchetype` enum has 10 early-game archetypes hand-crafted from replay observation — but 50-90 compositions exist across all game phases (early/mid/late) and 6 matchups.

### What exists

**Academic datasets with formal classification:**
- **TacticCraft (2025)** — arxiv.org/html/2507.15618 — used GPT-4/DeepSeek to classify ~50k ZvZ replays into 8-category tactical archetypes with softmax confidence scores. Methodology transfers to all matchups.
- **STARDATA Classification** — annals-csis.org/Volume_25/drp/pdf/48.pdf — identified 90 strategies (30/race) in StarCraft: Brood War via build order + army composition signatures.
- **SC2EGSet** — nature.com/articles/s41597-023-02510-7 — 17,930 esports tournament replays with extracted game-state data (Nature Scientific Data, 2023).
- **SC-Phi2 (2024)** — arxiv.org/html/2409.18989v1 — fine-tuned LLM on SC2 concepts; authors explicitly plan commentary system.

**Community databases with structured build orders:**
- **Spawning Tool** — lotv.spawningtool.com/build/ — community build order database. Python library `spawningtool` (github.com/StoicLoofah/spawningtool) parses replays into structured build orders.
- **Liquipedia** — liquipedia.net/starcraft2/Strategy — formal taxonomy per matchup: Opening → Transition → Counter → Endgame, with strong/weak counter classifications.
- **SC2 AI Replay Coach** — starcraft2.ai/en/build-orders — 50+ pro build orders with timing benchmarks.

**What does NOT exist:**
- No SC2-specific commentary-replay paired dataset (caster transcript aligned to replay timestamps). LoL19-21 methodology (YouTube ASR + timestamp alignment) would need to be applied to SC2.

### Why this is non-obvious

A developer building tactic detection would likely start from scratch — observing replays and hand-coding detection rules (as we did for 10 archetypes). The academic literature has already solved the classification problem at scale using LLM-based approaches, and community databases provide structured data in parseable formats. The gap is awareness, not capability.

### Alternatives considered

- **Hand-coded Drools rules only** — what we have now (10 archetypes). Works for early-game but doesn't scale to mid/late-game without exponential rule growth.
- **Pure LLM prompting** — pass raw game state to an LLM and trust its SC2 training data. Works but is uncontrollable and not verifiable.
- **Hybrid** — Drools for detection (what IS happening), structured data for taxonomy (what it's CALLED + counters), LLM for deeper context. Most promising approach.
