### Symptom
Project pins `mockito-junit-jupiter` at an older version (e.g. 5.11.0). The pin works silently — no warning — but the BOM would provide a newer version (5.21.0) if left unmanaged. The explicit pin shadows the BOM without any compiler or Maven output indicating a conflict.

### Context
Any Quarkus project using the `io.quarkus.platform:quarkus-bom` BOM and explicitly declaring `<version>` for `org.mockito:mockito-junit-jupiter` in `pom.xml`.

### Root cause
The Quarkus BOM (`quarkus-bom-3.34.2.pom`) includes `org.mockito:mockito-junit-jupiter` in its `<dependencyManagement>` section at version `5.21.0`. When a project explicitly declares `<version>5.11.0</version>`, Maven uses the project-level declaration and silently ignores the BOM version. No warning is emitted.

### Fix
Remove the explicit `<version>` tag from the `mockito-junit-jupiter` dependency in `pom.xml` and let the BOM manage it:

```xml
<!-- Before -->
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-junit-jupiter</artifactId>
    <version>5.11.0</version>
    <scope>test</scope>
</dependency>

<!-- After -->
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-junit-jupiter</artifactId>
    <scope>test</scope>
</dependency>
```

Verify with: `grep -A 3 "mockito-junit-jupiter" ~/.m2/repository/io/quarkus/platform/quarkus-bom/<version>/quarkus-bom-<version>.pom`

### Why this is non-obvious
The Quarkus documentation lists the BOM as the dependency management strategy but does not enumerate which third-party test libraries are included. The only way to discover that `mockito-junit-jupiter` is BOM-managed is to grep the BOM POM file directly. Maven gives no warning when an explicit pin shadows a BOM version.
