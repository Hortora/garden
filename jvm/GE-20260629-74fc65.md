### Symptom
`MissingMethodInvocationException: when() requires an argument which has to be 'a method call on a mock'` when using `mockStatic(ExternalActor.class)` and stubbing `ExternalActor.findByIdOptional(id)`.

### Context
Unit-testing a service that calls Panache Active Record static methods (`Entity.findByIdOptional()`, `Entity.list()`, `Entity.count()`). The test uses `@ExtendWith(MockitoExtension.class)` with `MockedStatic<ExternalActor>`.

### What was tried (didn't work)
```java
MockedStatic<ExternalActor> mock = mockStatic(ExternalActor.class);
mock.when(() -> ExternalActor.findByIdOptional(id))
    .thenReturn(Optional.of(actor));
// → MissingMethodInvocationException
```

### Root cause
`findByIdOptional()`, `list()`, `count()`, and other query methods are defined on `PanacheEntityBase`, not on the entity class itself. Java compiles `ExternalActor.findByIdOptional(id)` as a call to `PanacheEntityBase.findByIdOptional(id)`. Mockito's `mockStatic` only intercepts methods whose declaring class matches the mocked class — inherited static methods are dispatched to the parent class at bytecode level.

### Fix
Use `@InjectMock` from `quarkus-junit5-mockito` with `@QuarkusTest` instead of pure Mockito `mockStatic`. Add the dependency:
```xml
<dependency>
  <groupId>io.quarkus</groupId>
  <artifactId>quarkus-junit5-mockito</artifactId>
  <scope>test</scope>
</dependency>
```
Then inject the bean that wraps the Panache calls and mock it at the CDI level, or mock the collaborator beans that the service under test depends on.

### Why this is non-obvious
Mockito's documentation says `mockStatic(Class)` mocks "all static methods of the given class." A developer reading that naturally expects inherited static methods to be included — they are part of the class's API surface. The failure only appears at runtime with a misleading error message that suggests the call isn't on a mock, not that the method's declaring class doesn't match.
