## Symptom

`cannot find symbol` on a `Channel` variable even after adding an import:

```java
import io.casehub.qhorus.runtime.model.Channel;  // wrong — class does not exist at this path
```

The error gives no hint that the package path is wrong — it just says the class can't be found.

## Root Cause

The `Channel` JPA entity is at `io.casehub.qhorus.runtime.channel.Channel`, NOT `io.casehub.qhorus.runtime.model.Channel`. The sub-package name matches the class name (`channel/Channel.java`), which makes the wrong guess (`model.Channel`) natural for developers who expect JPA entities in a `model/` package.

## Fix

```java
import io.casehub.qhorus.runtime.channel.Channel;  // correct
```

## Why This Is Non-Obvious

The class name and its parent package share the same word (`channel/Channel.java`), making `model.Channel` the first guess for any developer who expects JPA entities in a conventional `model/` directory. The compile error gives no information about where the class actually lives.
