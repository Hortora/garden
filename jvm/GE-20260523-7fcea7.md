## Symptom
Code guarding `typeArgs.isEmpty()` after a successful `instanceof JavaParameterizedType`
check is dead — it can never execute.

## Context
When writing an ArchUnit rule that inspects generic return types (e.g. verifying `Uni<T>`
wraps the correct type argument), the natural pattern is:

```java
JavaType returnType = method.getReturnType();
if (returnType instanceof JavaParameterizedType paramType) {
    List<JavaType> typeArgs = paramType.getActualTypeArguments();
    if (typeArgs.isEmpty()) {
        // ← this branch is unreachable
        events.add(SimpleConditionEvent.violated(...));
        return;
    }
    JavaClass arg = typeArgs.get(0).toErasure();
    ...
}
```

## Root Cause
`DomainBuilders$JavaParameterizedTypeBuilder.build()` contains:

```java
checkArgument(typeArguments.size() > 0,
    "Parameterized type must have at least one type argument");
```

An object that satisfies `instanceof JavaParameterizedType` is guaranteed by construction
to have at least one element in `getActualTypeArguments()`. Raw types (e.g. `Uni` without
`<T>`) are represented as `JavaClass`, not `JavaParameterizedType`, so they are caught by
the `instanceof` check failing — they never reach the `typeArgs.isEmpty()` path.

## Fix
Remove the `isEmpty()` guard entirely. After `instanceof JavaParameterizedType` succeeds,
call `typeArgs.get(0)` directly:

```java
if (returnType instanceof JavaParameterizedType paramType) {
    // ArchUnit guarantees JavaParameterizedType always has ≥1 type arg
    JavaClass arg = paramType.getActualTypeArguments().get(0).toErasure();
    ...
}
// Raw Uni is JavaClass, not JavaParameterizedType — caught by instanceof failing above
```

## Why This Is Non-Obvious
The constructor invariant is enforced internally and not documented in ArchUnit's public
API or Javadoc. The parallel with Java's standard library (where `ParameterizedType.getActualTypeArguments()`
can return an empty array for some implementations) makes the defensive check feel reasonable,
but it reflects the JDK contract, not ArchUnit's stricter builder-enforced invariant.
