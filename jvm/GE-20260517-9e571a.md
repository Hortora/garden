### Symptom

Quarkus fails to start with:

```
AmbiguousResolutionException: Ambiguous dependencies for type com.framework.SomeInterface
```

Occurs when a custom CDI bean implements an interface that a framework component already
provides a bean for.

### Context

Implementing a framework SPI interface (e.g. `AgentChannelBackend` in casehub-qhorus)
to create a custom backend or handler. The framework ships its own `@ApplicationScoped`
implementation of the same interface.

### Root cause

CDI sees two beans satisfying the same interface type — the framework's default bean and
the custom implementation. Without disambiguation, CDI cannot choose. Unlike `@DefaultBean`
(which requires the custom bean to displace it), `@Typed` restricts which types the custom
bean is visible as, removing it from contention for the framework's injection point.

### Fix

Add `@Typed` to the custom bean, listing only the interfaces it should be discoverable as:

```java
// ❌ CDI sees this as AgentChannelBackend — conflicts with framework's QhorusChannelBackend
@Dependent
public class PushAgentDispatch implements AgentDispatchMechanism, AgentChannelBackend {
    // ...
}

// ✅ CDI only sees this as AgentDispatchMechanism and PushAgentDispatch
@Dependent
@Typed({AgentDispatchMechanism.class, PushAgentDispatch.class})
public class PushAgentDispatch implements AgentDispatchMechanism, AgentChannelBackend {
    // ...
}
```

Always include the concrete class itself in `@Typed` to preserve direct-type injection.

### Why this is non-obvious

The error message names the interface type but gives no hint that `@Typed` exists or
would fix it. Developers typically try `@Alternative`, `@DefaultBean`, or `@Priority`
— all wrong in this case. `@Typed` is rarely needed in application code (it's mainly
a framework-authoring tool) and doesn't appear in most Quarkus tutorials. The Java
`implements` declaration is the cause: removing one interface from `implements` would
fix it, but `@Typed` does this at the CDI level without changing the implementation
hierarchy.
