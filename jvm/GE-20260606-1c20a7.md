## What it does

In a SmallRye Config `@ConfigMapping` interface, a method with return type `java.nio.file.Path` and a `@WithDefault` annotation containing a property expression will have the expression resolved **before** the string-to-Path conversion is applied:

```java
@ConfigMapping(prefix = "myapp")
interface MyConfig {
    Storage storage();

    interface Storage {
        @WithDefault("${user.home}/.myapp/data")
        Path root();
    }
}
```

At Quarkus startup, `${user.home}` is expanded to the actual home directory (e.g. `/Users/alice`), and the resulting string `/Users/alice/.myapp/data` is then converted to a `Path` by SmallRye's built-in `PathConverter`. The caller gets a fully resolved `Path` object — no `@PostConstruct`, no `Optional<String>`, no `Path.of()` call needed.

## Why this isn't in the docs

The SmallRye Config documentation covers `@WithDefault` for simple types and property expressions separately. The combination — expression in `@WithDefault` with a non-String converted return type — is not documented or demonstrated in any official example. The behaviour works because `@WithDefault` registers the value as a proper config entry that passes through the expression interceptor chain before the type converter is invoked.

This is unlike `@ConfigProperty(defaultValue = "...")` from MicroProfile, which the MicroProfile specification **explicitly excludes** from expression expansion. That distinction makes developers assume `@WithDefault` has the same limitation — it does not.

## Discovery

Found during DraftHouse #28 when evaluating whether `@PostConstruct` + `Optional<String>` was required to resolve `user.home` at runtime. Testing confirmed that `@WithDefault("${user.home}/...")` on a `Path` return resolves correctly without any additional runtime code.

## Contrast

| Approach | Works? | Code required |
|----------|--------|---------------|
| `@WithDefault("${user.home}/...")` on `Path` | ✅ | None — SmallRye handles it |
| `@WithDefault("${user.home}/...")` on `String` | ✅ | `Path.of(config.storage().root())` at use site |
| `@ConfigProperty(defaultValue="${user.home}/...")` | ❌ | MicroProfile spec excludes expressions from defaultValue |
| `Optional<String>` + `@PostConstruct Path.of(...)` | Works but unnecessary | Mutable field, lifecycle coupling |
