## JavaMethod.isSynthetic() does not exist in ArchUnit 1.4.1 — filter by PUBLIC modifier only

### Symptom

Compile error when calling `m.isSynthetic()` on a `com.tngtech.archunit.core.domain.JavaMethod` object:

```
cannot find symbol
  symbol:   method isSynthetic()
  location: variable m of type com.tngtech.archunit.core.domain.JavaMethod
```

### Context

When writing an `ArchCondition` to enforce structural contracts (e.g. blocking/reactive method parity), the natural instinct is to filter synthetic bridge methods the same way Java reflection does: `m.isSynthetic()`. This method exists on `java.lang.reflect.Method` but not on ArchUnit's `JavaMethod`.

### Root cause

ArchUnit's `JavaMethod` (and `JavaCodeUnit`) does not expose `isSynthetic()` as a method. The only modifier-based filter available is `m.getModifiers().contains(JavaModifier.PUBLIC)` — there is no `SYNTHETIC` value in `JavaModifier`.

### Fix

Filter by visibility only. Quarkus augmentation adds synthetic CDI proxy methods at runtime after bytecode compilation — ArchUnit reads pre-augmentation `.class` files, so synthetic methods are not present at the point of analysis anyway.

```java
// Wrong — compile error
private static Stream<JavaMethod> publicMethods(final JavaClass cls) {
    return cls.getMethods().stream()
            .filter(m -> m.getModifiers().contains(JavaModifier.PUBLIC))
            .filter(m -> !m.isSynthetic());  // ← does not exist
}

// Correct
private static Stream<JavaMethod> publicMethods(final JavaClass cls) {
    return cls.getMethods().stream()
            .filter(m -> m.getModifiers().contains(JavaModifier.PUBLIC));
}
```

### Why this is non-obvious

Java reflection's `Method.isSynthetic()` is a standard API that works on compiled classes. Developers familiar with reflection-based structural tests expect the same filter to be available in ArchUnit's domain model — the compile error is the only signal that it doesn't exist.
