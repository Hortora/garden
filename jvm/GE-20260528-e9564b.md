## Symptom

Passing `ResponseFormat.JSON` (type only, no schema) to `ChatRequest.builder().responseFormat(...)` for an Anthropic-backed `ChatModel` throws `UnsupportedFeatureException` rather than silently degrading or prompting.

## Context

`InternalAnthropicHelper.validate(ChatRequestParameters)` is called inside `AnthropicChatModel.doChat()` before any API call is made. The validation checks:

```java
if (parameters.responseFormat() != null
        && parameters.responseFormat().type() == ResponseFormatType.JSON
        && parameters.responseFormat().jsonSchema() == null) {
    throw new UnsupportedFeatureException(...);
}
```

This means JSON mode without a schema is explicitly rejected at the LangChain4j layer — it never reaches the Anthropic API.

## Root cause

Anthropic's native API requires a JSON schema to enforce structured output. Unlike OpenAI (which supports a bare JSON mode that produces valid but unstructured JSON), Anthropic has no schema-less JSON mode. LangChain4j's Anthropic integration reflects this correctly by refusing the schemaless request rather than silently ignoring the constraint.

## Fix

Always pair `ResponseFormat.JSON` with a `JsonSchema` when using Anthropic:

```java
ResponseFormat responseFormat = ResponseFormat.builder()
    .type(ResponseFormatType.JSON)
    .jsonSchema(JsonSchema.builder()
        .name("MySchema")
        .rootElement(JsonObjectSchema.builder()
            .addStringProperty("field", "Description")
            .required("field")
            .build())
        .build())
    .build();
```

For a broad catch on provider-specific failures (including older Claude models that don't support structured output at all), wrap the `llm.chat(request)` call in `catch (Exception e)` and degrade gracefully:

```java
try {
    ChatResponse response = llm.chat(request);
    return Optional.of(parse(response.aiMessage().text()));
} catch (Exception e) {
    log.warn("Structured enrichment failed ({}), falling back to structural rendering", e.getMessage());
    return Optional.empty();
}
```

## Why this is non-obvious

The LangChain4j docs describe `ResponseFormat.JSON` as a valid type value. There is no public documentation on which providers support it with or without a schema. The `UnsupportedFeatureException` is thrown before any network call, so there is no HTTP error to debug — it looks like an in-process configuration error with no obvious connection to the Anthropic API's structured output requirements.

The `supportedCapabilities()` mechanism exists on `ChatModel` for high-level AI Services to check before using JSON schema, but `ClaudeMarkdownRenderer`-style code calling the low-level `ChatModel.chat(ChatRequest)` API bypasses that guard entirely.

### Why this fix

`ResponseFormat.JSON` with `JsonSchema` is the only valid combination for Anthropic. Adding a schema is not optional — it is the required API surface.
