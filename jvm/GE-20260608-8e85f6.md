## Context

The Graphiti OSS REST server (`getzep/graphiti`). When storing episodes via `POST /messages`,
you might expect to attach structured metadata for later filtering (e.g. domain, caseId).

## What exists in graphiti-core

`EpisodicNode` in `graphiti_core/nodes.py` has:

```python
episode_metadata: dict[str, Any] | None  # "customer-defined metadata key-value pairs for filtering"
```

This field exists, is stored, and appears in the episode response from `GET /episodes/{group_id}`.

## The REST gap

The REST `Message` DTO (`server/graph_service/dto/common.py`) does NOT include `episode_metadata`:

```python
class Message(BaseModel):
    content: str
    uuid: str | None = None
    name: str = ''
    role_type: Literal['user', 'assistant', 'system']
    role: str | None
    timestamp: datetime
    source_description: str = ''  # ← only this is available for metadata carrier
```

There is no way to set `episode_metadata` on a stored episode via the REST API.

## Workaround

Use `source_description` as the metadata carrier — it is passed through from the REST `Message`
DTO to the stored `EpisodicNode` and appears in `GET /episodes/{group_id}` responses:

```java
"domain=" + input.domain().name() + (input.caseId() != null ? ";caseId=" + input.caseId() : "")
```

This is a key=value string stored in `source_description`, not a structured dict.
Note: `source_description` does NOT appear in `FactResult` (search results) — only in
`EpisodicNode` (episode queries).
