### Symptom
Drift checker reports a channel as PRESENT when it should be ABSENT — a channel in tenant A satisfies a spec intended for tenant B. No error, no warning. The `tenancyId` parameter is passed to `check()` but never reaches the lookup.

### Context
`ChannelDriftChecker` used a nested `@FunctionalInterface`:
```java
@FunctionalInterface
public interface ChannelLookup {
    Optional<Channel> findByName(String name);
}
```
The `check(NodeSpec spec, String tenancyId)` method received `tenancyId` but the `ChannelLookup` interface had no slot for it. The lookup was `channelLookup.findByName(channelSpec.name())` — tenancyId silently dropped.

### Root cause
The functional interface was designed before multi-tenancy was added. When `tenancyId` was threaded through the `check()` method signature, nobody updated the `ChannelLookup` interface to accept it. The abstraction looked correct — clean single-method interface, easy to test with lambdas — which made the gap invisible during review.

### Fix
Replace the functional interface with direct injection of the real store interface:
```java
// Before: abstraction hides tenancyId
private final ChannelLookup channelLookup;
channelLookup.findByName(channelSpec.name()); // tenancyId lost

// After: real interface makes all parameters visible
private final CrossTenantChannelStore channelStore;
channelStore.findByNameAndTenancy(channelSpec.name(), tenancyId); // tenancyId required
```

### Why this is non-obvious
The functional interface pattern is widely recommended for testability — lambda stubs in tests, clean constructor injection. The abstraction looked correct at every level: the interface compiled, the implementation compiled, tests passed (because test stubs didn't enforce tenancy scoping either). The bug was invisible at the type level — only visible by tracing the data flow from `check()` through the lookup to the store.

### Alternatives considered
- **Evolve the functional interface** to accept tenancyId: `Optional<Channel> findByName(String name, String tenancyId)`. Rejected — adds a CDI producer glue layer and the abstraction provides no value over the real interface.
- **Keep ChannelLookup, add a second BindingLookup**: doubles the custom interfaces without benefit. The sibling `AgentDriftChecker` injects `AgentRegistry` directly — following that established pattern is simpler.
