### Symptom
After migrating `Worker` and `Capability` imports from `io.casehub.api.model` to `io.casehub.worker.api`, the build fails on `.function(workflow)` calls with "cannot find symbol". Import migration appears complete — all `Worker`/`Capability` references resolve — but the method signature changed.

### Context
The Worker Foundation Extraction (engine#543) moved `Worker` and `Capability` from `casehub-engine-api` to `casehub-worker-api`. The old `Worker.Builder.function(Workflow)` overload accepted a Serverless Workflow directly. The new `Worker.Builder.function(WorkerFunction)` requires a `WorkerFunction` — a sealed interface in worker-api.

### What was tried (didn't work)
1. Changed imports to `io.casehub.worker.api.Worker` and `io.casehub.worker.api.Capability` — build still fails on `.function()` call
2. Searched worker-api for a `function(Workflow)` overload — doesn't exist
3. Checked engine-api for `FlowWorkerFunction` — not there (moved to engine-flow in engine#567)

### Root cause
Three-way dependency split: `Worker`/`Capability` → worker-api, `FlowWorkerFunction` (bridges `Workflow` to `WorkerFunction`) → engine-flow, `CaseDefinition`/`Binding` → engine-api. The bridge type lives in a third module that isn't an obvious dependency for the migration.

### Fix
Wrap `Workflow` objects in `new FlowWorkerFunction(workflow)` before passing to `Worker.builder().function()`. Import from `io.casehub.engine.flow.FlowWorkerFunction` (casehub-engine-flow module).

```java
// Before (engine-api Worker)
Worker pruneWorker = Worker.builder()
    .function(pruneWorkflow)  // Workflow accepted directly
    .build();

// After (worker-api Worker + engine-flow bridge)
Worker pruneWorker = Worker.builder()
    .function(new FlowWorkerFunction(pruneWorkflow))  // Must wrap
    .build();
```

### Why this is non-obvious
The import migration looks complete after changing `Worker` and `Capability` packages. The `.function()` method still exists with the same name — the compiler error is "cannot find symbol" for `FlowWorkerFunction`, not a type mismatch on `.function()`. The bridge type is in `casehub-engine-flow`, not alongside `Worker` in worker-api or alongside `CaseDefinition` in engine-api.
