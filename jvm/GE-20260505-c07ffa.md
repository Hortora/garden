## Context

You have a Quarkus REST endpoint that is fail-closed: if a config property (e.g. `webhook-secret`) is not set, the endpoint returns 401 rather than processing the request. Your `@QuarkusTest` `application.properties` always sets this property (needed for other tests), so you can't test the "no secret" path without a separate profile.

## Technique

**Step 1** — Use `.filter(s -> !s.isBlank()).orElse(null)` instead of `.orElse(null)` in the production code:

```java
// In the resource
final String secret = config.webhookSecret()
    .filter(s -> !s.isBlank())   // treats "" the same as absent
    .orElse(null);
if (secret == null) {
    return Response.status(401).build();
}
```

**Step 2** — Create a `@QuarkusTestProfile` that overrides the property to `""`:

```java
@QuarkusTest
@TestProfile(MyResourceNoSecretTest.NoSecretProfile.class)
class MyResourceNoSecretTest {

    public static class NoSecretProfile implements QuarkusTestProfile {
        @Override
        public Map<String, String> getConfigOverrides() {
            return Map.of("casehub.work.issue-tracker.github.webhook-secret", "");
        }
    }

    @Test
    void noSecretConfigured_returns401() {
        given()
            .header("Content-Type", "application/json")
            .body("{}")
        .when()
            .post("/workitems/github-webhook")
        .then()
            .statusCode(401);
    }
}
```

`@QuarkusTestProfile.getConfigOverrides()` lets each test class override specific properties. Quarkus restarts between test classes with different profiles — `quarkus.http.test-port=0` in `application.properties` prevents TIME_WAIT conflicts on restart.

## Why This Is Non-Obvious

Quarkus test profiles are typically used to switch datasources or enable features — not to test absence of a property. The blank-string trick works because `Optional.filter(s -> !s.isBlank())` treats `""` as absent, mimicking the "not configured" branch without needing a separate `application.properties` file. Most developers try to solve this with `@QuarkusTestProfile.getEnabledAlternatives()` or by removing the property from the main test properties, which either fails or affects all tests in the suite.
