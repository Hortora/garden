### Context
A field like `capabilityTag` on an entity scopes a record to a specific domain. Some records
apply to all domains — the "global" case. The obvious choice is NULL (no tag = applies to
all). The better choice is `"*"`.

### What developers usually do
Use NULL for the "no specific scope" case, then query with `WHERE tag IS NULL` for global
records and `WHERE tag = :tag` for scoped ones.

### The technique
Define a constant for the global sentinel:

```java
public static final String GLOBAL = "*";  // in a CapabilityTag constants class
```

Set it as the column default and Java field default:

```sql
capability_tag VARCHAR(255) NOT NULL DEFAULT '*'
```

```java
@Column(name = "capability_tag", nullable = false)
public String capabilityTag = CapabilityTag.GLOBAL;
```

Queries are now uniform `= :tag` — no IS NULL special case:

```java
// Global records
em.createQuery("... WHERE a.capabilityTag = '*' ...")

// Scoped records
em.createQuery("... WHERE a.capabilityTag = :tag ...")
        .setParameter("tag", "security-review")
```

### Why this is non-obvious
NULL semantics in SQL are subtle: `NULL != NULL`, IS NULL requires special handling in
indexes, NULLS NOT DISTINCT is needed for unique constraints, and JPQL has its own IS NULL
syntax. The `"*"` sentinel eliminates all of this — normal equality, normal indexes, normal
unique constraints, no Java null checks.

The `"*"` value is already familiar from IAM/RBAC systems: Kubernetes RBAC, AWS IAM, and
Casbin policies all use `"*"` for "all resources/verbs".

### Constraints
Works best when the sentinel value cannot be a real domain value (i.e. no actual capability
will ever be named `"*"`). Requires all insertion paths to set the field or rely on the
column DEFAULT.

### Alternatives considered
- **NULL**: classic but requires IS NULL queries, NULLS NOT DISTINCT for unique constraints, and null checks throughout Java code
- **Separate boolean `isGlobal` column**: creates an enforcing invariant problem (what if isGlobal=true but tag is also set?)
- **Mandatory tag, no global concept**: shifts burden to callers (must write N attestations for N capabilities)

### Invalidation triggers
Not version-sensitive — this is a design pattern, not a library feature.
