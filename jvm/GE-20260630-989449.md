### Symptom

Compilation error: `cannot find symbol — method capabilities(java.util.List<io.casehub.worker.api.Capability>)` on `Worker.Builder`.

### Context

Any code creating `Worker` instances via the builder that uses `.capabilities(List.of(cap(...)))` breaks on the new SNAPSHOT. The `Capability` record itself appears to have been removed or relocated.

### Root cause

`Worker` was refactored from using `List<Capability>` (where `Capability` had `name`, `inputSchema`, `outputSchema`) to `Set<String>` capability names. The builder API changed accordingly:

- **Removed:** `capabilities(List<Capability>)`
- **Added:** `capabilityName(String name)` — single capability (returns `Set.of(name)`)
- **Added:** `capabilityNames(String... names)` — multiple capabilities
- **Added:** `capabilityNames(Collection<String> names)` — collection variant
- **Changed:** `Worker` record field from `capabilities` to `capabilityNames` (returns `Set<String>`)

### Fix

```java
// Before (broken)
private static Capability cap(String name) {
    return Capability.builder().name(name).inputSchema(".").outputSchema(".").build();
}

Worker.builder()
    .name("my-agent")
    .capabilities(List.of(cap("entity-resolution")))
    .function(...)
    .build();

// After (works)
Worker.builder()
    .name("my-agent")
    .capabilityName("entity-resolution")
    .function(...)
    .build();
```

In tests, replace `w.capabilities().get(0).name()` with `w.capabilityNames().iterator().next()`.

### Why this is non-obvious

No changelog or migration guide. The `Capability` class removal means the compilation error doesn't suggest the replacement — it just says the method doesn't exist. The new API simplification (strings instead of objects) is only discoverable by decompiling the SNAPSHOT jar.
