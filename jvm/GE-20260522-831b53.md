## Context

A `@Observes StartupEvent` handler needs testing — specifically the behaviour that it calls `serviceA.listAll()` and invokes `initX()` for each result. The obvious approach is a `@QuarkusTest` that prepares data and calls the handler.

## Problem with @QuarkusTest approach

In `@QuarkusTest`, data created inside `@TestTransaction` is not visible to non-`@Transactional` JPA reads called from within the handler:

```java
@Test
@TestTransaction
void onStart_registersBackend_forPersistedChannels() {
    channelStore.put(ch);  // committed within @TestTransaction

    gateway.onStart(new StartupEvent());  // calls channelService.listAll()
    // → listAll() opens its own Hibernate session, sees no data
    // → channel not in registry → assertion fails
}
```

`channelService.listAll()` is not `@Transactional`, so Hibernate may open a fresh auto-session that doesn't see the test's uncommitted data. Using `QuarkusTransaction.requiringNew()` to commit first works but requires manual cleanup (committed data isn't rolled back).

## The technique

Make the handler method **package-private** and write a plain unit test in the same package:

```java
// Production class (package-private method — CDI @Observes works with any visibility)
@ApplicationScoped
public class ChannelGateway {
    void onStart(@Observes StartupEvent ev) {
        for (Channel ch : channelService.listAll()) {
            try {
                initChannel(ch.id, new ChannelRef(ch.id, ch.name));
            } catch (Exception ex) { ... }
        }
    }
}
```

```java
// Test in the same package (io.casehub.qhorus.runtime.gateway)
class ChannelGatewayStartupTest {

    @SuppressWarnings("unchecked")
    @Test
    void onStart_registersBackend_forEachPersistedChannel() {
        Channel ch1 = channel("case-1/work");
        Channel ch2 = channel("case-2/observe");
        ChannelService channelService = mock(ChannelService.class);
        when(channelService.listAll()).thenReturn(List.of(ch1, ch2));

        ChannelGateway gateway = new ChannelGateway(
                agentBackend, normaliser, messageService,
                channelService, mock(Event.class));

        gateway.onStart(new StartupEvent());

        assertTrue(gateway.listBackends(ch1.id).stream()
                .anyMatch(b -> "qhorus-internal".equals(b.backendId())));
    }
}
```

Key points:
- No `@QuarkusTest`, no Quarkus container, no transaction — test runs in milliseconds
- `ChannelService` is mocked: you control exactly what `listAll()` returns
- `Event<T>` is mocked: observer calls are no-ops
- The handler is called directly as a regular method
- Package-private visibility is sufficient for CDI `@Observes` — the spec doesn't require public

## Why this is non-obvious

Developers default to `@QuarkusTest` for anything touching CDI beans. The transaction visibility problem only manifests when the handler calls non-`@Transactional` reads — the first test attempt passes if test data happens to be committed already (e.g. from `@BeforeAll`). The unit test approach inverts the dependency direction entirely and eliminates the transaction question at the cost of not testing CDI wiring.

The CDI `@Observes` placement on a package-private method is underdocumented — most examples show `public`. It works because CDI uses proxy-based dispatch and the proxy is in the same package.
