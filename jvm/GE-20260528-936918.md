## Symptom

Code compiles with a `Response.Status.UNPROCESSABLE_ENTITY` reference in a `WebApplicationException` or `Response.status()` call, but the IDE shows an error and the build fails:

```
error: cannot find symbol
  symbol:   variable UNPROCESSABLE_ENTITY
  location: class Response.Status
```

## Root cause

`Response.Status.UNPROCESSABLE_ENTITY` (HTTP 422) was added in JAX-RS 3.1 / Jakarta EE 10. Projects targeting Jakarta EE 9 / JAX-RS 3.0 (which Quarkus 3.x may use depending on the BOM) don't have this constant.

## Fix

Use the raw integer `422` directly:

```java
// Wrong (JAX-RS 3.0 / Jakarta EE 9):
throw new WebApplicationException("Not valid", Response.Status.UNPROCESSABLE_ENTITY);

// Correct:
throw new WebApplicationException("Not valid", 422);
```

Or create a constant:
```java
private static final int HTTP_UNPROCESSABLE_ENTITY = 422;
```

## Why this is non-obvious

The HTTP 422 status is well-established (WebDAV, RFC 4918) and commonly used for validation errors. Developers reasonably expect it to be in the `Response.Status` enum. The version split between EE 9 and EE 10 isn't surfaced by IDE autocomplete which may show UNPROCESSABLE_ENTITY from a different JAR on the classpath.
