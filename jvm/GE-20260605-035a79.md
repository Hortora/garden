### Symptom
CI build hangs or is cancelled. Log output is truncated or misleading (pipe buffer, `redirectTestOutputToFile`). Need to determine which module actually hung.

### Context
GitHub Actions workflow uploads Surefire reports as an artifact (`actions/upload-artifact` with `if: always()`). The artifact captures `**/target/surefire-reports/*` across all modules.

### Technique
Download the Surefire reports artifact and check which modules have report directories:

```bash
gh run download <run-id> --repo org/repo --name surefire-reports --dir /tmp/surefire-reports
find /tmp/surefire-reports -type d | sort
```

Modules with `<module>/target/surefire-reports/` directories containing `.txt` files completed their test execution. The first module in the build order WITHOUT a reports directory is the one that hung.

Each `.txt` file shows the exact test count, failures, and timing:
```
Tests run: 26, Failures: 0, Errors: 0, Skipped: 0, Time elapsed: 1.068 s
```

### Why this is non-obvious
CI logs are the natural diagnostic tool — but they can be truncated (pipe buffer), silenced (`redirectTestOutputToFile`), or simply not available while a run is in progress. The Surefire reports artifact is available even for cancelled runs (because of `if: always()`) and provides ground truth that the CI log cannot.
