## @Priority is NOT @Inherited — CDI proxy classes silently drop it from reflection

### Symptom

Bean iteration order from `Instance<T>` is silently non-deterministic despite
`@Priority` annotations on all implementations. Sorting by
`resolver.getClass().getAnnotation(Priority.class)` returns `null` for every
bean, so all beans get `Integer.MAX_VALUE` and ordering is undefined.

### Context

When iterating CDI beans from `Instance<T>` and sorting by `@Priority` using
Java reflection. Common in composite/pipeline patterns where multiple
implementations of an SPI must execute in a defined order.

### Root cause

`@jakarta.annotation.Priority` is not meta-annotated with `@java.lang.annotation.Inherited`.
CDI proxy classes (e.g. `WebDIDResolver_ClientProxy`) are JDK subclasses of the
real bean class. `proxy.getClass().getAnnotation(Priority.class)` returns `null`
because the JDK only propagates annotations marked `@Inherited` to subclasses.

The proxy class itself carries no annotations — it is a synthetic subclass
generated by the CDI container solely for interception and scoping purposes.

### What was tried (didn't work)

```java
// Natural reflection approach — silently fails for all proxied beans
private static int priorityOf(DIDResolver resolver) {
    var priority = resolver.getClass().getAnnotation(Priority.class);
    return priority != null ? priority.value() : Integer.MAX_VALUE;
    // Always returns MAX_VALUE for proxied beans
}
```

### Fix

Use Quarkus Arc's `Instance.Handle` API to read bean metadata through the
CDI container rather than through reflection on the proxy class:

```java
@Inject
public CompositeDIDResolver(@DIDMethod Instance<DIDResolver> methodResolvers) {
    this.resolvers = methodResolvers.handles().stream()
            .sorted(Comparator.comparingInt(h -> priorityOf(h)))
            .map(Instance.Handle::get)
            .collect(Collectors.toUnmodifiableList());
}

private static int priorityOf(Instance.Handle<?> handle) {
    var bean = handle.getBean();
    if (bean instanceof InjectableBean<?> injectable) {
        Integer p = injectable.getPriority();
        if (p != null) return p;
    }
    return Integer.MAX_VALUE;
}
```

Arc stores priority metadata at container build time (augmentation), independent
of proxy reflection. `InjectableBean.getPriority()` reads this stored value.

### Why this is non-obvious

`getClass().getAnnotation()` is the natural Java reflection approach for reading
annotations. It works for many annotations and in many contexts. The `@Inherited`
requirement is a JDK detail that only matters when the object is a subclass of
the annotated class — a situation that is universal in CDI (every injected bean
is potentially proxied) but rarely encountered in direct reflection usage.

The failure is silent: no exception, no warning. The annotation simply isn't
found, and the fallback value (typically `Integer.MAX_VALUE` or a default
priority) is used for every bean, making ordering non-deterministic.

### Alternatives considered

- **Walk superclass chain:** `resolver.getClass().getSuperclass().getAnnotation(Priority.class)`
  — works for single-level proxies but fragile if proxy depth changes or if
  the container uses a different proxy strategy.
- **`@All List<InstanceHandle<T>>`:** Quarkus-specific injection that preserves
  metadata — equivalent to `Instance.handles()` but declared at the injection site.
