## Symptom

After projecting a channel through a filtering `ChannelProjection` decorator (e.g. one that skips messages with `round > maxRound`), calling `result.isEmpty()` returns `false` even though the resulting state is identical to `projection.identity()` — no messages contributed to the state.

Code that checks `result.isEmpty()` to branch to a "no content" sentinel fails silently, calling `renderer.render(identityState)` instead and producing output like `# Review Summary\n**Updated:** ...\n\n---\n\n` with no actual content.

## Context

`ProjectionResult.isEmpty()` is defined as:

```java
public boolean isEmpty() {
    return lastMessageId == null;
}
```

`lastMessageId` is the ID of the last message **scanned** by `ProjectionService.fold()`, not the last message that **contributed** to the state:

```java
for (final var msg : messages) {
    state = projection.apply(state, mapper.toMessageView(msg));
    lastId = msg.id;  // updated for EVERY message, even if apply() returned state unchanged
}
return new ProjectionResult<>(state, lastId);
```

When a decorator projection returns `state` unchanged for every message (because all messages have `round > maxRound`), the cursor still advances to the last scanned message ID. `isEmpty()` therefore returns `false` — the channel was not empty, it was fully scanned.

## Root Cause

`isEmpty()` answers "was the channel empty at scan time?" not "did any message contribute to the state?". A filtering decorator that skips all messages leaves the cursor at the last scanned message ID, making the result look non-empty even though the materialized state is `identity()`.

## Fix

Check state content explicitly after a filtering projection:

```java
// Wrong — isEmpty() checks the cursor, not whether messages matched
String summary = result.isEmpty()
    ? "No activity."
    : renderer.render(result.state());

// Right — check actual state content for domain-specific emptiness
ReviewState state = result.state();
String summary = (state.points().isEmpty()
        && state.memos().isEmpty()
        && state.subTaskFindings().isEmpty())
    ? "No activity up to round " + round + "."
    : renderer.render(state);
```

The emptiness check must be domain-specific — the caller knows which fields constitute "empty" for its state type.

## Why Non-Obvious

`isEmpty()` is the standard Java idiom for "nothing here." Developers reading `ProjectionResult.isEmpty()` reasonably expect it to mean "the projection produced no content." The actual semantics — "no messages were scanned" — are cursor-based and only detectable by reading the `ProjectionResult` record definition. The Javadoc documents the contract correctly (`lastMessageId == null` when the channel was empty at scan time) but the mismatch with developer expectations is easy to miss when writing decorator projections that filter rather than accumulate.

## Alternatives Considered

- Adding a `hasContent()` method to `ProjectionResult` that delegates to the state — rejected because it would require `ProjectionResult` to know about the state's structure (loses genericity)
- Using the incremental overload (`project(channelId, previous, projection)`) — not applicable here since we want a fresh full scan, just filtered
