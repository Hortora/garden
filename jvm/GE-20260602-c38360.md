## Symptom

Compilation error: `cannot find symbol — method stream()` on a
`io.quarkus.hibernate.reactive.panache.PanacheQuery<T>` instance.

## Context

Trying to implement cursor-backed reactive streaming from a Panache repository,
expecting the same `.stream()` API as the blocking `PanacheQuery<T>` which returns
`Stream<T>`.

## What was tried (didn't work)

```java
return repo.find(jpql, params).stream();  // ← does not compile
```

Inspecting the Panache reactive JAR via `javap` confirms only these methods exist on
`PanacheQuery<T>`: `list()`, `firstResult()`, `singleResult()`, `count()`, `page()`,
`lastPage()`, `hasNextPage()`, `pageCount()`.

## Root cause

The reactive Panache `PanacheQuery<T>` in Quarkus 3.32.x has no `.stream()` method.
Hibernate Reactive does not expose a cursor-backed streaming API in the Panache surface
at this version — only full list materialisation is available.

## Fix

Wrap the materialised list as a `Multi`:

```java
// Still materialises the full list, but gives Multi<T> shape for downstream fold:
return scan(query).toMulti().flatMap(list -> Multi.createFrom().iterable(list));
```

For true cursor-backed streaming, use the native Hibernate Reactive session API
(not Panache) — significantly more complex. Revisit when/if Quarkus exposes
`PanacheQuery.stream()` on the reactive side.

## Why this is non-obvious

The blocking `PanacheQuery<T>` has `.stream()` → `Stream<T>`. Developers familiar with
blocking Panache naturally reach for the same API on the reactive side. The missing method
only surfaces as a compile error with no helpful message pointing at the version gap.
