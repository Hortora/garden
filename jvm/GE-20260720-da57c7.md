### Symptom
`NullPointerException: Cannot invoke "InMemoryCommitmentStore.findByChannel(UUID)" because "this.delegate" is null` when calling any method on `ReactiveCommitmentStore` in a `@QuarkusTest`.

### Context
`InMemoryReactiveCommitmentStore` delegates all operations to an `InMemoryCommitmentStore` via a package-private `delegate` field. The sibling reactive stores (`InMemoryReactiveChannelStore`, `InMemoryReactiveMessageStore`) have `@Inject` on their delegate fields — CDI wires them automatically in Quarkus test contexts. `InMemoryReactiveCommitmentStore` was missing `@Inject`, so CDI left the field null.

### What was tried (didn't work)
Injecting `ReactiveCommitmentStore` in a `@QuarkusTest` and calling `findByChannel()` — NPE with no useful context about the missing wiring.

### Root cause
The `delegate` field on `InMemoryReactiveCommitmentStore` lacked the `@Inject` annotation. In plain unit tests, the delegate is set manually (`store.delegate = blocking`), so the bug never surfaces there. It only appears when CDI manages the bean lifecycle.

### Fix
Add `@Inject` to the delegate field:

```java
@Inject
InMemoryCommitmentStore delegate;
```

### Why this is non-obvious
The field is package-private with no annotation — it looks like it's meant to be set manually. The sibling stores have `@Inject` but you wouldn't check them unless you already suspected a CDI wiring issue. Plain unit tests pass, so CI doesn't catch it. The NPE message says "delegate is null" but doesn't hint at CDI — you'd first suspect a bean resolution issue or missing `@Alternative` priority.
