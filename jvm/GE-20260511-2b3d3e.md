### What it is
When a system has trust-based routing that must degrade gracefully during cold start (no trust history), express the phase logic — "is this agent in bootstrap mode?" and "is this score borderline?" — as methods on the routing policy value object itself, not in the routing service.

### How it works

```java
public record RoutingPolicy(
    OptionalDouble threshold,
    OptionalInt minimumObservations,
    OptionalDouble borderlineMargin,
    Optional<String> fallbackType,
    String rationale
) {
    public boolean isBootstrap(int agentObservations) {
        return minimumObservations.isPresent()
            && agentObservations < minimumObservations.getAsInt();
    }

    public boolean isBorderline(double trustScore) {
        return borderlineMargin.isPresent() && threshold.isPresent()
            && trustScore >= threshold.getAsDouble()
            && trustScore < threshold.getAsDouble() + borderlineMargin.getAsDouble();
    }
}
```

Routing logic stays policy-agnostic:

```java
// Routing service — no phase knowledge, no magic numbers
if (policy.isBootstrap(agent.observationCount())) {
    return availabilityRouting(candidates);
}
if (policy.isBorderline(agent.trustScore())) {
    flagForHumanOversight(assignment);
}
```

### Why this is non-obvious
The obvious placement for routing phase logic is the router or routing strategy, not the policy record. Putting it in the router scatters threshold knowledge across service code and requires the router to know about `minimumObservations` and `borderlineMargin` internals. Putting it on the policy record keeps the knowledge co-located with the data it operates on, the policy is self-documenting, and the routing code reads as pure intent.

### Context where it applies
Any system where routing quality depends on accumulated evidence (trust scores, reputation, observation counts) and must work correctly before that evidence exists. The `isBootstrap` gate says "this agent hasn't built enough history to be scored; treat as available." The system degrades to availability routing in Phase 0 and improves automatically as evidence accumulates.
