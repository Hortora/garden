### Symptom

`@AfterEach` calls `channelStore.clear()` on an injected CDI bean but state leaks between tests — subsequent tests see data from previous test runs despite the cleanup.

### Context

A Quarkus test class has:
```java
@Inject InMemoryChannelStore channelStore;   // CDI @Alternative @Priority(1)

@AfterEach void cleanUp() { channelStore.clear(); }
```
A reactive wrapper bean also exists:
```java
@Alternative @Priority(1) @ApplicationScoped
public class InMemoryReactiveChannelStore implements ReactiveChannelStore {
    private final InMemoryChannelStore delegate = new InMemoryChannelStore();
    ...
}
```

### Root cause

`InMemoryReactiveChannelStore` creates its delegate via `new InMemoryChannelStore()` — plain Java instantiation, not CDI injection (`@Inject`). The CDI container manages:
- One `InMemoryChannelStore` bean (injected into the test)
- One `InMemoryReactiveChannelStore` bean (used by reactive services)

The reactive wrapper's `delegate` field is a **third, separate object** — not the CDI bean. Services injecting `ReactiveChannelStore` write to the wrapper's private delegate. `clear()` on the CDI `InMemoryChannelStore` bean clears the wrong instance.

### Fix

Inject the type that the actual service uses:
```java
// If service uses blockingChannelService → ChannelStore:
@Inject InMemoryChannelStore channelStore;        // correct for blocking path

// If service uses ReactiveChannelStore:
@Inject InMemoryReactiveChannelStore channelStore; // has its own clear()
```

Verify which path the service under test takes by reading its source — field type determines which CDI bean it receives.

### Why this is non-obvious

Both `InMemoryChannelStore` and `InMemoryReactiveChannelStore` are `@ApplicationScoped @Alternative @Priority(1)` CDI beans — they appear symmetric. The internal `new InMemoryChannelStore()` delegation is an implementation detail only visible in source code. `clear()` returns void and throws no error, so state leakage is the only signal — and that's often masked by test ordering or unique identifiers.
