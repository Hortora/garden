### What exists
The `quarkus-mcp-server-http` extension simultaneously serves two MCP transport endpoints:
- **Streamable HTTP** at `/mcp` (the spec-compliant transport)
- **SSE** at `/mcp/sse` (backwards-compatible fallback)

The server logs this on startup:
```
MCP HTTP transport endpoints [streamable: http://0.0.0.0:9001/mcp, SSE: http://0.0.0.0:9001/mcp/sse]
```

### How it was discovered
The extension is named `mcp-server-http`, suggesting only Streamable HTTP. The SSE endpoint was found by curling `/mcp/sse` after `"type": "url"` was rejected by Claude Code — it returned a valid SSE `event: endpoint` response.

### Why this matters
MCP clients that don't support the newer Streamable HTTP transport (like Claude Code as of July 2025) can still connect via SSE without switching the server dependency to `quarkus-mcp-server-sse`. This dual-transport behaviour is not documented in the extension README or Quarkus guides.

### Why this is non-obvious
The extension name `mcp-server-http` implies HTTP-only transport. There is no configuration to enable SSE — it's always on. A developer who needs SSE would likely switch to the `quarkus-mcp-server-sse` artifact instead of discovering that `-http` already includes it.
