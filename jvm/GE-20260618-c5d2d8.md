### Symptom

Need a lightweight mock HTTP server for Quarkus `@QuarkusTest` integration tests that works with the Vert.x HTTP client. OkHttp MockWebServer doesn't work (see GE-20260618-2bc371). WireMock works but adds a large dependency tree.

### Context

Quarkus `@QuarkusTest` REST client tests need an HTTP server to verify request routing, header injection, and response deserialization. The mock server must:
1. Start on a dynamic port
2. Integrate with `QuarkusTestResourceLifecycleManager` to inject the URL into config
3. Support response queuing (enqueue before the test makes its call)
4. Record requests for post-call assertions

JDK's `com.sun.net.httpserver.HttpServer` (exported from `jdk.httpserver` module) satisfies all four requirements in ~90 lines of code with zero external dependencies.

### Fix

Implement a `QuarkusTestResourceLifecycleManager` backed by JDK HttpServer:

```java
public class TestHttpServerResource implements QuarkusTestResourceLifecycleManager {

    static volatile TestHttpServer INSTANCE;

    @Override
    public Map<String, String> start() {
        var server = new TestHttpServer();
        server.start();
        INSTANCE = server;
        return Map.of("quarkus.rest-client.\"configKey\".url",
                       "http://localhost:" + server.port());
    }

    @Override
    public void stop() {
        if (INSTANCE != null) { INSTANCE.shutdown(); INSTANCE = null; }
    }

    public static class TestHttpServer {
        private HttpServer server;
        private final ConcurrentLinkedQueue<StubbedResponse> responseQueue = new ConcurrentLinkedQueue<>();
        private final ConcurrentLinkedQueue<RecordedRequest> requestQueue = new ConcurrentLinkedQueue<>();

        void start() {
            server = HttpServer.create(new InetSocketAddress(0), 0);
            server.createContext("/", this::handleRequest);
            server.start();
        }

        public int port() { return server.getAddress().getPort(); }
        public void enqueue(StubbedResponse r) { responseQueue.add(r); }
        public RecordedRequest takeRequest() { return requestQueue.poll(); }
        public void drainRequests() { requestQueue.clear(); }
        void shutdown() { if (server != null) server.stop(0); }

        private void handleRequest(HttpExchange exchange) throws IOException {
            byte[] body = exchange.getRequestBody().readAllBytes();
            requestQueue.add(new RecordedRequest(
                exchange.getRequestMethod(), exchange.getRequestURI().getPath(),
                new String(body, UTF_8), exchange.getRequestHeaders()));
            StubbedResponse resp = responseQueue.poll();
            if (resp == null) {
                byte[] msg = "No response enqueued".getBytes(UTF_8);
                exchange.sendResponseHeaders(500, msg.length);
                exchange.getResponseBody().write(msg);
            } else {
                exchange.getResponseHeaders().set("Content-Type", resp.contentType());
                byte[] b = resp.body().getBytes(UTF_8);
                exchange.sendResponseHeaders(resp.statusCode(), b.length);
                exchange.getResponseBody().write(b);
            }
            exchange.close();
        }
    }

    public record StubbedResponse(int statusCode, String contentType, String body) {
        public static StubbedResponse json(int status, String body) {
            return new StubbedResponse(status, "application/json", body);
        }
    }

    public record RecordedRequest(String method, String path, String body,
                                   com.sun.net.httpserver.Headers headers) {}
}
```

Usage in tests:

```java
@QuarkusTest
@QuarkusTestResource(TestHttpServerResource.class)
class MyRestClientTest {
    @Inject @RestClient MyRestClient client;

    @Test
    void returnsData() {
        TestHttpServerResource.INSTANCE.enqueue(
            StubbedResponse.json(200, "[{\"id\":1}]"));
        var result = client.getData().await().indefinitely();
        assertThat(result).hasSize(1);
    }
}
```

### Why this is non-obvious

Most developers reach for OkHttp MockWebServer or WireMock for HTTP testing. `com.sun.net.httpserver.HttpServer` is overlooked because it's in the `com.sun` namespace (perceived as internal API) and lacks the polished API of dedicated mock libraries. But it's a stable, exported JDK module that handles the core use case — dynamic port, request recording, response queuing — with no dependencies and guaranteed compatibility with any HTTP client including Vert.x/Netty.
