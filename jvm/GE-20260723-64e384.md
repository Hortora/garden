## Symptom

Quarkus startup fails with `ConfigValidationException: SRCFG00050: casehub.cbr.tracking.enabled does not map to any root` when `@IfBuildProperty(name = "casehub.cbr.tracking.enabled")` is used on a `@Decorator` and `application.properties` contains `casehub.cbr.tracking.enabled=true`.

## Context

A `@ConfigMapping(prefix = "casehub.cbr.tracking")` interface declares runtime
properties (`retentionDays`, `sqlite.path`). A `@Decorator` uses
`@IfBuildProperty(name = "casehub.cbr.tracking.enabled")` for conditional activation.
The `enabled` key falls under the mapping's prefix but is not declared as a method.

SmallRye Config's strict validation (Quarkus 3.32+) sees the property in
`application.properties`, matches it to the `casehub.cbr.tracking` prefix, and
rejects it because no `enabled()` method exists in the mapping.

## What was tried (didn't work)

Setting the property only in `application.properties` without adding it to the config
mapping тАФ validation fires before the build-time property is consumed.

## Root cause

`@IfBuildProperty` properties are consumed during Quarkus augmentation (build phase).
`@ConfigMapping` validation runs during SmallRye Config construction (runtime phase).
The two mechanisms don't communicate тАФ SmallRye doesn't know that `@IfBuildProperty`
already consumed the property.

## Fix

Add the property to the `@ConfigMapping` interface with a default:

```java
@ConfigMapping(prefix = "casehub.cbr.tracking")
public interface CbrTrackingConfig {
    @WithDefault("false")
    boolean enabled();

    // ... other properties
}
```

The property is now mapped (satisfying SmallRye validation) and also consumed by
`@IfBuildProperty` (unchanged). The mapping method is never called at runtime тАФ
it exists purely to pass validation.

## Why this is non-obvious

`@IfBuildProperty` and `@ConfigMapping` operate at different lifecycle phases.
Developers naturally expect build-time properties to be invisible to runtime
validation. The error only appears when both mechanisms share a prefix AND the
property is set in `application.properties`.
