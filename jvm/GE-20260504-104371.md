## Symptom

`@ConfigProperty` fields on a CDI bean return `null` (or the primitive default) when the bean is instantiated with `new` rather than injected via CDI. The failure is silent — no exception is thrown, the field is simply never populated.

In the specific case that surfaced this: `OllamaAvailabilityCondition` (a JUnit 5 `ExecutionCondition`) called `new OllamaHealthChecker()`, expecting `@ConfigProperty String ollamaBaseUrl` to be `"http://localhost:11434"` (the `defaultValue`). Instead, `ollamaBaseUrl` was `null`, causing `URI.create(null + "/api/tags")` to throw, which was caught and returned `false` — silently reporting Ollama as unavailable when it was running fine.

## Context

JUnit 5 `ExecutionCondition` classes run outside the Quarkus CDI container. `@ConfigProperty` injection is a CDI/MicroProfile mechanism — it only fires when the container manages the bean lifecycle. Using `new` bypasses the entire injection chain.

The `defaultValue` attribute on `@ConfigProperty` is **not** a Java field default — it is a hint to the CDI container. Outside CDI, `defaultValue` has no effect and the field remains `null` for reference types.

## Root cause

`@ConfigProperty` is resolved by the CDI container's `@PostConstruct` / injection point processing. `new ClassName()` skips the container entirely. Java field defaults (`= "value"`) are the only initialisation that survives `new`.

## Fix

Two options:

**Option 1 — Hardcode the default in the class field (correct for non-CDI use)**
```java
// Before: field is null when instantiated with new
@ConfigProperty(name = "quarkus.langchain4j.ollama.base-url", defaultValue = "http://localhost:11434")
String ollamaBaseUrl;

// After: hardcoded default in the field; CDI still overrides via injection
String ollamaBaseUrl = "http://localhost:11434";
```
When CDI manages the bean, the injected value overrides the field default. When `new` is used, the field default applies.

**Option 2 — Inline the URL in the non-CDI code path**
```java
// In the JUnit extension that uses new:
private static final String OLLAMA_URL = "http://localhost:11434";
// Use OLLAMA_URL directly instead of calling new OllamaHealthChecker()
```

## Why this is non-obvious

The `defaultValue` attribute in `@ConfigProperty` reads like a Java field default — it says "if the config key isn't set, use this value." Developers reasonably expect it to apply universally. The CDI-only scoping is documented but easy to miss, especially when the pattern of injecting `@ApplicationScoped` beans via CDI works correctly everywhere else in the codebase.
