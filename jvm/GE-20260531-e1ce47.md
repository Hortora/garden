## CDI @Observes and @ObservesAsync are separate delivery channels — @Observes never receives fireAsync() events

### Symptom

A test CDI bean that captures events via `@Observes` receives events fired with `event.fire()`
but silently receives nothing from `event.fireAsync()`. Capture list stays empty; assertions
that check `events().isEmpty()` pass trivially and hide missing async delivery:

```java
// Fires on the sync channel — @Observes receives it
keyRotatedEvent.fire(new AgentKeyRotatedEvent(actorId, prev, next));

// Fires on the ASYNC channel — @Observes does NOT receive it
keyRotatedEvent.fireAsync(new AgentKeyRotatedEvent(actorId, prev, next));
```

### Root cause

CDI 4.0 §10.5.6 defines two distinct event delivery channels:
- **Synchronous channel**: observers declared with `@Observes` — receives only `event.fire()` events
- **Asynchronous channel**: observers declared with `@ObservesAsync` — receives only `event.fireAsync()` events

There is no cross-channel delivery. An `@Observes` method will never be called for `fireAsync()`,
regardless of return type or execution context.

### Fix

Test capture beans that need to observe async events must declare `@ObservesAsync`:

```java
@ApplicationScoped
public class AgentKeyRotatedEventCapture {

    private final List<AgentKeyRotatedEvent> syncEvents = new CopyOnWriteArrayList<>();
    private final List<AgentKeyRotatedEvent> asyncEvents = new CopyOnWriteArrayList<>();

    // Receives event.fire() calls
    void onRotatedSync(@Observes final AgentKeyRotatedEvent event) {
        syncEvents.add(event);
    }

    // Receives event.fireAsync() calls
    CompletionStage<Void> onRotatedAsync(@ObservesAsync final AgentKeyRotatedEvent event) {
        asyncEvents.add(event);
        return CompletableFuture.completedFuture(null);
    }

    public List<AgentKeyRotatedEvent> events() {
        // Combined list if both channels are needed
        List<AgentKeyRotatedEvent> all = new ArrayList<>(syncEvents);
        all.addAll(asyncEvents);
        return all;
    }
}
```

For async events: add `Thread.sleep(200)` or `Awaitility.await()` after the trigger
before asserting, as `fireAsync()` delivers on a separate thread after the caller returns.

### Why this is non-obvious

Developers familiar with sync CDI events assume all observers receive all events of the
matching type. The `@ObservesAsync` annotation looks like an optimization hint (receive
on a different thread) rather than a separate channel gate. The failure mode — empty
capture list — is silent and appears as a passing test that never actually verified anything.
