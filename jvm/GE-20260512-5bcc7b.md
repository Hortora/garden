### Context

`CaseContextImpl.deepCopy()` is `private`. A subclass overriding `snapshot()` to preserve its own type cannot call `deepCopy()` directly. The naive alternatives are: duplicate the logic, change visibility in the parent, or serialize/deserialize through JSON.

### Technique

Call `super.snapshot()` to obtain a correctly deep-copied `CaseContext`, then construct the subclass using `base.getData()`:

```java
@Override
public CaseContext snapshot() {
    final CaseContext base = super.snapshot(); // deep-copy done by CaseContextImpl
    return new MySubclass(base.getData());     // getData() returns a defensive copy
}
```

`CaseContextImpl.snapshot()` internally runs `deepCopy(data)` before constructing the result. `getData()` on that result returns a second defensive shallow copy of the already-deep-copied map — correct for all non-nested structures. No parent modification, no logic duplication.

### Why this is non-obvious

The three obvious approaches all have problems: (1) make `deepCopy` protected — requires modifying the parent class; (2) copy the deep-copy logic — duplication that can diverge; (3) serialize through JSON — expensive and lossy for non-serializable values. The two-step `super.snapshot()` + `getData()` avoids all three by exploiting that `getData()` is already a defensive copy of the deep-copied state.
