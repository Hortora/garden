### Symptom
A custom `InboundNormaliser` implementation intended for domain-specific channels (e.g. PI oversight channels) also applies to all other channels. Messages on unrelated channels get classified as DONE or DECLINE instead of QUERY, causing unexpected Commitment state transitions or ignored messages.

### Context
Occurs when implementing `InboundNormaliser` for a specific domain governance channel and deploying alongside other qhorus channels.

### Root cause
`ChannelGateway` injects `InboundNormaliser` as a single CDI bean. The `@ApplicationScoped` implementation that displaces `@DefaultBean DefaultInboundNormaliser` is global — there is no per-channel normaliser registration. All channels share the same normaliser.

```java
// In ChannelGateway — one normaliser instance for all channels
final InboundNormaliser normaliser;

public void receiveHumanMessage(ChannelRef channel, InboundHumanMessage raw) {
    NormalisedMessage normalised = normaliser.normalise(channel, raw);
    // ...
}
```

### Fix
Scope the domain-specific detection inside the normaliser by checking the channel name:

```java
@ApplicationScoped
public class ClinicalInboundNormaliser implements InboundNormaliser {

    @Override
    public NormalisedMessage normalise(ChannelRef channel, InboundHumanMessage raw) {
        // Only apply clinical decision parsing to PI oversight channels
        MessageType type = isOversightChannel(channel.name())
            ? detectDecision(raw.content())
            : MessageType.QUERY;  // default for all other channels
        return new NormalisedMessage(type, raw.content(), "human:" + raw.externalSenderId());
    }

    private boolean isOversightChannel(String name) {
        return name != null && name.contains("/pi-oversight");
    }
}
```

### Why this is non-obvious
The `InboundNormaliser` SPI receives the `ChannelRef` as a parameter — which implies per-channel behaviour is intended. But the CDI wiring is global, not per-channel. A developer implementing a domain-specific normaliser will assume it only applies to channels where it's relevant, when in fact it applies everywhere.
