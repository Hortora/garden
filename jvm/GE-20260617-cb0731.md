## Symptom

SSE integration test asserts that keepalive events don't reach the event handler (`events.isEmpty()`). The assertion fails — the event handler IS called, with empty name and empty data `""`, even though the server sent SSE comment lines (`: keepalive`).

## Context

Server sends keepalive SSE comments via:
```java
sink.send(sse.newEventBuilder().comment("keepalive").build());
```

Test registers an event handler expecting no calls:
```java
source.register(event -> events.add(event.readData(String.class)));
Thread.sleep(3_000);
assertThat(events).isEmpty(); // FAILS
```

## Root cause

RESTEasy's `SseEventSource` client is non-compliant with the SSE specification. The SSE spec (W3C EventSource spec) states that lines beginning with `:` are comments and MUST be silently ignored by clients. RESTEasy's implementation fires the registered event handler for comment-only frames, delivering them as events with empty name (`""`) and empty data (`""`).

Real SSE clients (browsers, `curl --no-buffer`, Python `sseclient`) correctly ignore comment lines.

## Fix

Use a **named event** instead of an SSE comment for keepalives. A named event sends bytes over the wire (satisfying proxy idle-timeout requirements) and is delivered to the event handler by name, making it filterable:

**Server:**
```java
// Instead of:
sink.send(sse.newEventBuilder().comment("keepalive").build());

// Use:
sink.send(sse.newEventBuilder().name("keepalive").data("").build());
```

**Test client — filter by event name:**
```java
source.register(event -> {
    if (!"keepalive".equals(event.getName())) {
        events.add(event.getName() + "|" + event.readData(String.class));
    }
});
```

## Why this is non-obvious

The SSE spec explicitly states comment lines must be ignored by clients. A developer implementing server-side keepalives with SSE comments (the recommended approach per the spec and Mozilla docs) would expect comment-based keepalives to be invisible to event handlers — that's the entire point of comments in the SSE protocol.

### Why this fix

Named events (`name("keepalive")`) send actual bytes over the wire, satisfying proxy/load balancer idle-timeout requirements (the original motivation for keepalives), while being filterable by name in the handler. SSE comments would have been cleaner architecturally but RESTEasy's non-compliance makes them unusable in test infrastructure.
