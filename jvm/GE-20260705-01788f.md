### Symptom
LLM prompts, CBR similarity queries, or log messages contain the literal text
`"null"` where they should contain nothing. No error, no warning — the string
passes all standard blank/empty checks.

### Context
Any code that accepts a `JsonNode`, converts it to a `String` via `.toString()`,
and uses the result in text processing. Common in routing strategies, prompt
builders, and context serialisers where the input may be `NullNode.instance`
(Jackson's representation of JSON `null`).

### What was tried (didn't work)
- `caseContext != null` — passes, because `NullNode.instance` is a non-null Java object
- `!caseContext.toString().isBlank()` — passes, because `"null"` is a 4-character non-blank string
- `!caseContext.toString().isEmpty()` — passes for the same reason

### Root cause
`NullNode.toString()` returns the **string** `"null"` (the JSON text representation of
a null value), not Java `null` and not an empty string. This is by design — Jackson's
`toString()` produces valid JSON text. But any code that treats the result as a
human-readable value or feeds it into text processing will silently propagate the
literal word "null".

### Fix
Check `JsonNode.isNull()` before calling `toString()`:

```java
String text = node != null && !node.isNull()
    ? node.toString()
    : null;
```

`JsonNode.isNull()` returns `true` for `NullNode` and `false` for all other node types.
This is the Jackson-idiomatic null check.

### Why this is non-obvious
The triple negative (`!= null`, `!isBlank()`, content check) gives false confidence —
every guard passes. The string `"null"` is indistinguishable from a legitimate value
in downstream text. The bug is only visible when someone reads the LLM prompt or query
and notices the word "null" where context should be absent.

### Why this fix
`isNull()` is the Jackson-native check. Alternatives like comparing `toString().equals("null")`
are fragile — they conflate the JSON null representation with a legitimate string value
that happens to be "null".

### Alternatives considered
- `node instanceof NullNode` — works but couples to the concrete class; `isNull()` is the
  abstract API method
- `node.getNodeType() == JsonNodeType.NULL` — equivalent to `isNull()` but more verbose
