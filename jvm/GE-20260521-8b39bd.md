## Symptom

`cannot find symbol` compilation error when importing
`io.quarkus.deployment.builditem.ArtifactResultBuildItem` in a third-party
Quarkus extension's deployment module.

## Context

Writing a void `@BuildStep` in a third-party Quarkus extension that produces
no output but must always run (e.g. a validation or warning step). The developer
wants to use `@Produce(ArtifactResultBuildItem.class)` to force the step into the
build graph so Quarkus does not elide it.

## Root cause

`ArtifactResultBuildItem` is in the `pkg` subpackage:

```
io.quarkus.deployment.pkg.builditem.ArtifactResultBuildItem   ← correct
io.quarkus.deployment.builditem.ArtifactResultBuildItem       ← does not exist
```

The `deployment.builditem` package contains most build items; the packaging
subpackage is easy to miss. IDEs may not autocomplete to the correct path
because the class name is absent from `deployment.builditem` entirely —
autocomplete returns nothing rather than the wrong suggestion.

## Fix

```java
import io.quarkus.deployment.pkg.builditem.ArtifactResultBuildItem;

@BuildStep
@Produce(ArtifactResultBuildItem.class)
void myValidationStep() {
    // always runs — anchored to the build artifact graph
}
```

## Why this is non-obvious

The Quarkus `deployment` package contains a top-level `builditem` subpackage
where the vast majority of build items live. Nothing in the error message or
IDE hints that the missing class is in `deployment.pkg.builditem` rather than
`deployment.builditem`. Searching the Quarkus extension guide does not surface
this — it only appears when reading extension source code directly.
