## Symptom

A `@ApplicationScoped` CDI bean defined in the test classpath observes `ChannelInitialisedEvent` with `@Observes`. Production code calls `Event<ChannelInitialisedEvent>.fire()` synchronously inside a `@TestTransaction` test. The observer's list of captured events remains empty — `onMessage()` is never called.

Two variants tried:

1. **Static inner class** inside the `@QuarkusTest` class — annotated `@ApplicationScoped`, injected into the test. Observer method never fires.
2. **Separate top-level class** in the test package — same result.

No error or warning. The observer bean is injected successfully (non-null), but events fired by production code do not reach it.

## Root Cause

Quarkus CDI's augmentation-time bean discovery may not activate `@ApplicationScoped` observer beans defined in the test classpath without explicit configuration, even when they appear to be discovered (injection succeeds). The `Event.fire()` call dispatches to observers known to the CDI container at augmentation time; test-classpath beans without `quarkus.arc.selected-alternatives` configuration are in a grey zone.

The CDI injection point (`@Inject BindingUpdateObserver observer`) resolves via a proxy that may be a valid CDI bean but whose observer method registration is incomplete.

## Fix

Use a **Mockito unit test** (no CDI, no `@QuarkusTest`) to verify that `Event<T>.fire()` is called with the correct payload:

```java
@SuppressWarnings("unchecked")
Event<ChannelInitialisedEvent> events = mock(Event.class);
// wire the service under test manually
service.channelInitialisedEvents = events;

service.updateConnectorBinding(channelId, "new-connector", "new-dest");

verify(events).fire(new ChannelInitialisedEvent(channelId, "my-channel", false));
```

Use `@QuarkusTest` integration tests to verify the state outcome (binding fields updated in store), not the event firing mechanics. Keep event-firing verification in pure unit tests where the `Event<T>` is a mock you fully control.

## Why this is non-obvious

- The observer bean IS injectable — `@Inject BindingUpdateObserver` does not throw `UnsatisfiedResolutionException`. The proxy resolves. But events don't reach it.
- Synchronous `Event.fire()` "should" always reach all active observers in the same CDI container. The gap between compile-time bean activation and runtime observer registration is not visible from application code.
- No warning is emitted when a `@Observes` method is registered on an observer that won't receive events in a `@QuarkusTest` context.
