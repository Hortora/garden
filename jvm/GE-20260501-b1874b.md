## Context

You want to unit-test a CDI bean that has `@Inject` fields without spinning up a full Quarkus
container (`@QuarkusTest`). The field is `private` so you can't set it from the test class.

## Technique

Declare the `@Inject` field as package-private (no access modifier). In the same package, the
test class can set it directly with a Mockito mock — no reflection, no container:

**Production class** (`engine/src/main/java/.../worker/`):
```java
@ApplicationScoped
public class EmptyWorkerContextProvider implements WorkerContextProvider {

    @Inject CaseChannelProvider caseChannelProvider; // package-private for test injection
    ...
}
```

**Test class** (same package under `src/test/`):
```java
@Test
void populatesChannelsFromProvider() {
    UUID caseId = UUID.randomUUID();
    CaseChannelProvider mock = mock(CaseChannelProvider.class);
    when(mock.listChannels(caseId)).thenReturn(List.of(someChannel));

    var provider = new EmptyWorkerContextProvider();
    provider.caseChannelProvider = mock;   // direct field access

    WorkerContext ctx = provider.buildContext("worker-1", caseId, WorkRequest.of("task", Map.of()));
    assertThat(ctx.channels()).containsExactly(someChannel);
}
```

## Why this is non-obvious

Most developers reach for `@QuarkusTest` + `@InjectMock`, Mockito's `@InjectMocks` annotation
(which uses reflection), or constructor injection refactoring. The package-private trick
requires no framework support, no reflection, and runs in milliseconds.
