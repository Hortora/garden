## Summary

Pass the caller's raw JWT as a key in the workflow input map. Workflow steps reference it via JQ expression in the HTTP step header. This gives downstream services the caller's identity without any OIDC client configuration.

## The Pattern

**REST endpoint — extract and pass the token:**

```java
@Inject JsonWebToken jwt;

@POST @Authenticated
public Uni<Response> handle(Map<String, Object> payload) {
    Map<String, Object> input = Map.of(
        "token",   jwt.getRawToken(),  // caller's raw bearer token
        "payload", payload
    );
    return workflow.startInstance(input)
        .onItem().transform(ignored -> Response.status(201).build());
}
```

**Workflow step — reference token via JQ in the Authorization header:**

```java
return workflow("myWorkflow")
    .tasks(
        http()
            .uri(downstreamUrl + "/resource")
            .header("Authorization", "${ \"Bearer \" + (.token) }")
            .GET()
    )
    .build();
```

The JQ expression `${ "Bearer " + (.token) }` is evaluated against the workflow's input data at execution time. The token is available to all steps in the workflow.

## When to Use

Use when a quarkus-flow workflow needs to call external services (Jira, GitHub, internal APIs) on behalf of the authenticated caller, and you want the external call to carry the user's identity.

Not suitable for long-running cases where the JWT may expire before the workflow completes — consider token refresh or service-account credentials instead.

## Why This Is Non-Obvious

The natural assumption is that downstream auth requires configuring an OIDC client or using the `secret()` DSL (which is for static credentials). The token-as-data approach — passing the JWT through the workflow's input map and referencing it with a JQ expression — is not covered in the main quarkus-flow documentation. The JQ expression syntax in HTTP step headers is mentioned briefly but the auth pattern is not shown as an example.

Verified in: `quarkus-flow` integration tests — `JwtWithinWorkflowTest` + `SubmissionWorkflow`.
