### Symptom
Attempting to create an `AgentProvider` test double with a lambda:
`AgentProvider provider = config -> Multi.createFrom().items(...)` fails
with a compile error — "AgentProvider is not a functional interface."

### Context
Writing tests for any class that takes `AgentProvider` (e.g. `LlmSelectedRouting`,
`SupervisorBuilder`). The interface appears single-purpose (invoke an LLM) so
lambdas seem natural.

### Root cause
`AgentProvider` has two abstract methods: `invoke(AgentSessionConfig)` returning
`Multi<AgentEvent>`, and `openSession(AgentSessionConfig)` returning a session
handle. Two abstract methods disqualifies it as a `@FunctionalInterface`.

### Fix
Use Mockito mocks instead of lambdas:
```java
var provider = Mockito.mock(AgentProvider.class);
when(provider.invoke(any())).thenReturn(
    Multi.createFrom().items(new AgentEvent.TextDelta("response text")));
```

### Why this is non-obvious
The interface name ("Provider") and its primary use (invoke an LLM) strongly
suggest a single-method functional interface. The second method (`openSession`)
is a session-management concern that most test scenarios don't need, but its
presence on the interface blocks lambda construction for all callers.
