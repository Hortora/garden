### What happened

A Quarkus extension has a framework-free `api/` module containing SPI contracts and DTO records. A new record (`DispatchResult`) needed `@JsonInclude(NON_NULL)` to omit null fields in MCP tool responses, and `@Nullable` to document nullable components. Both annotations failed: Jackson is not a dependency of the `api/` module, and `jakarta.annotation.Nullable` is similarly absent. The record is returned by a `@Tool`-annotated method in the `runtime/` module that is serialized to JSON.

### Symptom

Compilation fails with `cannot find symbol: class JsonInclude` and `cannot find symbol: class Nullable` when the annotations are placed on the `api/` module record.

### Context

Quarkus extension module structure:
- `api/` — pure-Java SPI contracts, zero framework dependencies. Consumed by any Quarkus app.
- `runtime/` — extension runtime, has Quarkus/Jackson on classpath.
- `deployment/` — build-time processor.

The `api/` module deliberately has no Jackson or `jakarta.annotation` dependencies to stay lightweight and framework-agnostic. Any record defined there cannot carry Jackson annotations directly.

### Root cause

Jackson (`com.fasterxml.jackson.annotation`) and `jakarta.annotation` are on the `runtime/` module classpath but not on `api/`. The `api/` module is the right place for the DTO record (consumed cross-module), but the wrong place for framework-specific annotations.

### Fix

**Option A — Global Jackson config** (simplest, affects whole app):

Add to `application.properties`:
```properties
quarkus.jackson.serialization-inclusion=non-null
```

This omits all null fields globally. Works correctly with Java records serialized via `@Tool` return types. Not per-type — affects all Jackson serialization in the consumer app.

**Option B — Jackson mixin in runtime module** (scoped, no global config):

```java
// In runtime/ module
@JsonInclude(JsonInclude.Include.NON_NULL)
abstract class DispatchResultMixin {}

// Register in Jackson ObjectMapper customizer
@Singleton
public class JacksonConfig implements ObjectMapperCustomizer {
    public void customize(ObjectMapper mapper) {
        mapper.addMixIn(DispatchResult.class, DispatchResultMixin.class);
    }
}
```

**Option C — Move the record to runtime/** (breaks cross-module usage):
Only valid if no other module needs to consume the record as a Java type. Not applicable when consumers (AML, engine, clinical) inject services that return the record.

### Why this is non-obvious

The `api/` module is the correct location for DTOs in Quarkus extension design (other modules depend on it). Jackson annotations feel like metadata, not framework coupling. But `@JsonInclude` requires the Jackson annotation jar as a compile dependency, which would break the framework-free constraint. The fix is counter-intuitive: configure serialization globally rather than annotating the type.
