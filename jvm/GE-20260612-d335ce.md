## @InjectMock ManagedExecutor works in @QuarkusTest — MicroProfile interface is fully replaceable via Quarkus mock machinery

**ID:** GE-20260612-d335ce
**Stack:** Quarkus 3.x, quarkus-smallrye-context-propagation, quarkus-junit5
**Symptom:** N/A — this is a positive capability, not a failure mode. Developers avoid `@InjectMock ManagedExecutor` assuming it won't work because `ManagedExecutor` is a MicroProfile interface (not a plain `@ApplicationScoped` CDI bean).
**Context:** Testing an `@ApplicationScoped` bean that injects `ManagedExecutor` for fire-and-forget async dispatch. The test needs to eliminate the async race condition by running the executor task synchronously on the test thread.

### What was tried (didn't work)
- Awaitility (not in test dependencies)
- `Thread.sleep` (flaky)
- Assuming `@InjectMock ManagedExecutor` would fail with CDI wiring errors

### Root cause
`quarkus-smallrye-context-propagation` registers `ManagedExecutor` as a CDI `@ApplicationScoped` bean backed by a SmallRye implementation. Quarkus's `@InjectMock` machinery replaces CDI beans — including interface-typed ones — with Mockito mocks. Since `ManagedExecutor` has a CDI bean registration, it is fully replaceable.

### Fix / Usage
```java
@QuarkusTest
class MyBridgeIntegrationTest {

    @InjectMock ManagedExecutor executor;

    @BeforeEach
    void setUp() {
        // Run the Runnable synchronously on the test thread — eliminates async race
        doAnswer(inv -> {
            inv.getArgument(0, Runnable.class).run();
            return null;
        }).when(executor).execute(any());
    }

    @Test
    void dispatchIsCalledWithCorrectArguments() {
        // ... no Awaitility or sleep needed
        verify(messageService).dispatch(argThat(d -> ...));
    }
}
```

### Why this is non-obvious
The `ManagedExecutor` interface is defined in `microprofile-context-propagation-api`. It looks like a pure API type, not a CDI bean — the documentation for `@InjectMock` focuses on `@ApplicationScoped` beans, not MicroProfile interfaces. Developers assume it won't work and reach for Awaitility or latches instead.
