### Symptom
`"ChatModel is not a functional interface"` compile error when trying to stub with a lambda:
```java
ChatModel stub = request -> ChatResponse.builder()...build();  // ← compile error
```

### Context
Writing a unit test for code that takes a `ChatModel`. Attempting to use a lambda stub instead of Mockito, following the pattern used for other single-method interfaces.

### Root Cause
`ChatModel` has no abstract methods. Every method — including `chat(ChatRequest)`, the primary API — is `default`. The real extension point is `doChat(ChatRequest)`, also `default`, which throws `RuntimeException("Not implemented")` unless overridden. The `chat(ChatRequest)` default delegates to `doChat(chatRequest, ...)`.

### Fix
Implement `ChatModel` as an anonymous class, overriding `doChat`:
```java
private static ChatModel stubModel(final String responseJson) {
    return new ChatModel() {
        @Override
        public ChatResponse doChat(final ChatRequest request) {
            return ChatResponse.builder()
                .aiMessage(AiMessage.from(responseJson))
                .finishReason(FinishReason.STOP)
                .build();
        }
    };
}
```

### Why This Is Non-Obvious
`chat(ChatRequest)` is documented as "the main API to interact with the chat model" and is the method callers invoke. Developers expect the main API method to be the abstract method. Instead, it is a `default` that orchestrates logging, listeners, and parameter merging before delegating to `doChat`. The `doChat` name suggests an implementation detail, not a public extension point.
