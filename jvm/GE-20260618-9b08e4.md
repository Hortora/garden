## Symptom

`mvn install` fails with artifact not found or BOM lookup failure when declaring `quarkus-smallrye-reactive-messaging-kafka` in pom.xml.

## Context

Adding Kafka reactive messaging to a Quarkus 3.x module. The Quarkus 3.x BOM no longer manages `io.quarkus:quarkus-smallrye-reactive-messaging-kafka` — the artifact was renamed as part of the messaging extension consolidation.

## What Was Tried

`<artifactId>quarkus-smallrye-reactive-messaging-kafka</artifactId>` — compile error or artifact not found at BOM version.

## Root Cause

Quarkus 3.x renamed the reactive messaging extensions. The `smallrye` prefix was dropped from the public artifact name. Same pattern applies to AMQP.

## Fix

Use the renamed artifact:

```xml
<!-- Quarkus 3.x — correct names -->
<dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-messaging-kafka</artifactId>
</dependency>
<dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-messaging-amqp</artifactId>
</dependency>

<!-- These no longer exist in Quarkus 3.x BOM -->
<!-- quarkus-smallrye-reactive-messaging-kafka -->
<!-- quarkus-smallrye-reactive-messaging-amqp -->
```

The old names may still compile if an old version is cached, but the Quarkus 3.x BOM does not manage them and they are not guaranteed to exist.

## Why This Is Non-Obvious

The old names followed a consistent `quarkus-smallrye-*` pattern used across many Quarkus extensions. The rename happened without a widely-promoted migration note. Most documentation and tutorials still show the old artifact names. The error message ("artifact not found") doesn't hint that the artifact was renamed.
