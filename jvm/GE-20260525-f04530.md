## Context

Writing a focused wiring test for a CDI `@Observes` handler — verifying that the handler
is correctly wired (right injection, right event routing, right downstream effect) without
going through the production event source.

## Technique

Inject `jakarta.enterprise.event.Event<T>` directly in the `@QuarkusTest` class. Construct
the event payload in the test, then call `fire()`. Since `@Observes` is synchronous CDI
(not `@ObservesAsync`), the handler completes before `fire()` returns. Assert the
downstream effect immediately (or with a short `await` if the handler triggers async engine
writes).

```java
@QuarkusTest
class SlaBreachHandlerWiringTest {

    @Inject Event<SlaBreachEvent> breachEvents;
    @Inject CaseInstanceRepository caseInstanceRepository;

    @Test
    void slaBreachHandler_onFail_signalsCaseContext() throws Exception {
        UUID caseId = caseHub.startCase(ctx).toCompletableFuture().get(5, SECONDS);
        assertThat(caseId).isNotNull();

        // Construct and fire the event directly — bypasses ExpiryLifecycleService entirely
        var task = new BreachedTask(UUID.randomUUID(), CallerRef.encode(caseId, "human-approval"),
                                    "PR approval", Set.of("pr-leads"));
        var breach = new SlaBreachContext(BreachType.CLAIM_EXPIRED, task,
                                          Path.root(), new MapPreferences(Map.of()));
        breachEvents.fire(new SlaBreachEvent(breach, new BreachDecision.Fail("sla-breach")));

        // @Observes is synchronous — handler ran before fire() returned
        // but handler calls caseHub.signal() which may trigger async engine writes
        await().atMost(5, SECONDS).pollInterval(100, MILLISECONDS).untilAsserted(() -> {
            var instance = caseInstanceRepository.findByUuid(caseId)
                    .await().atMost(Duration.ofSeconds(2));
            assertThat(instance.getCaseContext().getPath("humanApproval.status"))
                    .isEqualTo("sla-breach");
        });
    }
}
```

## Why This Is Non-Obvious

The natural instinct is to drive the full production stack (event source → SPI → CDI event
→ handler), which works but ties the test to multiple components and introduces async
races (e.g. the production event source also triggers binding activity on the case). The
inverse pattern — injecting `Event<T>` as the *driver* rather than the *subject* — is the
opposite of how event testing is usually framed (which is: capture what you receive). It
isn't documented as a testing approach in Quarkus or CDI docs.

Related: GE-20260519-114395 covers calling `@ObservesAsync` handlers directly via the
injected CDI proxy. This entry is the synchronous `@Observes` equivalent — using `fire()`
rather than direct invocation, so the CDI dispatch mechanism is also exercised.

## Constraints

Only applies to `@Observes` (synchronous). For `@ObservesAsync`, `fire()` dispatches
off-thread and the handler may not have completed when assertions run — use
`fireAsync().toCompletableFuture().join()` or see GE-20260429-3d4e35 instead.
