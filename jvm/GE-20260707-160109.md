### Symptom

Quarkus augmentation fails with:

```
It's not possible to automatically add a synthetic no-args constructor to an unproxyable bean class.
You need to manually add a non-private no-args constructor to
io.casehub.desiredstate.runtime.CdiNodeProvisionerRouter
```

This blocks `@QuarkusTest` execution and production builds of the app module.

### Context

`CdiNodeProvisionerRouter` is `@ApplicationScoped` (normal scope) and extends `DefaultNodeProvisionerRouter`. It only has a parameterized `@Inject` constructor — no no-args constructor. Quarkus CDI requires a no-args constructor for normal-scoped beans because they're proxied at build time.

The class is in a dependency jar (casehub-desiredstate-runtime) — you can't modify it.

### What was tried (didn't work)

- Adding the jar to `quarkus.index-dependency` — doesn't fix the constructor issue, just ensures the class IS discovered (which makes the problem worse)
- Using `quarkus.arc.exclude-types` — works but ONLY if added to BOTH production AND test `application.properties` (see related entry about test property override)

### Root cause

`CdiNodeProvisionerRouter` extends `DefaultNodeProvisionerRouter` which has constructors taking `Collection<NodeProvisioner>` and `PreferenceProvider`. The CDI subclass passes these via `@Inject` but never provides a no-args constructor. Quarkus ArC cannot add a synthetic no-args constructor because the superclass also lacks one.

### Fix

1. Exclude the unproxyable bean:
```properties
# application.properties AND test application.properties
quarkus.arc.exclude-types=io.casehub.desiredstate.runtime.CdiNodeProvisionerRouter
```

2. Provide a local replacement with a protected no-args constructor:
```java
@ApplicationScoped
public class AppNodeProvisionerRouter extends DefaultNodeProvisionerRouter {
    protected AppNodeProvisionerRouter() {
        super(List.of());
    }

    @Inject
    public AppNodeProvisionerRouter(Instance<NodeProvisioner> provisioners) {
        super(StreamSupport.stream(provisioners.spliterator(), false).toList());
    }
}
```

The protected no-args constructor satisfies the CDI proxy requirement. CDI calls the `@Inject` constructor at runtime — the no-args constructor is only used for proxy generation.

### Why this is non-obvious

The error message says "add a non-private no-args constructor" — but the class is in a dependency jar you can't modify. The real fix is exclude + replace, which requires understanding that CDI discovers beans from Jandex-indexed jars AND that the replacement needs to preserve the original's CDI discovery semantics (collect all `NodeProvisioner` beans via `Instance<>` iteration).
