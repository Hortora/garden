### Symptom
After editing TypeScript source files and running `npm run build` manually,
refreshing the browser still shows the old version. No errors — the build
succeeds, dist/ is updated, but the served content doesn't change.

### Context
Quinoa manages the frontend build lifecycle during `quarkus:dev`. It copies
the build output from the webui `dist/` directory to `target/quinoa/build/`
during the Quarkus build phase. The Quarkus dev server serves static files
from `target/quinoa/build/`, not from `src/main/webui/dist/`.

### Root cause
Manual `npm run build` updates `src/main/webui/dist/` but Quinoa's copy to
`target/quinoa/build/` only happens during Quarkus's build phase. In dev
mode, Quinoa watches for source file changes and triggers its own rebuild
cycle, but this watch may not detect changes to already-built output files.

### Fix
Restart `quarkus:dev` after making frontend changes to force Quinoa to
rebuild and copy fresh output to `target/quinoa/build/`. Alternatively,
touch a source file (`touch src/index.ts`) to trigger Quinoa's file watcher,
though this is not always reliable.

### Why this is non-obvious
A developer expects that rebuilding the frontend and refreshing the browser
is sufficient — the mental model is "dist/ is what gets served." The
indirection through `target/quinoa/build/` is invisible unless you know
Quinoa's internal copy mechanism.
