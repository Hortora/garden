### Symptom
Second `@QuarkusTest` method queries `CbrCaseMemoryStore.retrieveSimilar()` and finds cases stored by the first test method. Test fails with "Expecting empty but was: [ScoredCbrCase...]".

### Context
`InMemoryCbrCaseMemoryStore` is `@ApplicationScoped` â€” one instance per Quarkus test application lifecycle. It persists across all test methods in all `@QuarkusTest` classes within the same test run. The store has no `clearAll()` or `reset()` method.

### What was tried (didn't work)
- `purge(CbrRetentionPolicy)` â€” requires `maxAgeDays >= 1` (won't clear cases stored seconds ago) and `maxCasesPerType >= 1` (keeps at least one case per type). Both constraints are validated in the constructor with `IllegalArgumentException`.
- `eraseEntity(entityId, tenantId)` â€” requires knowing the exact entity IDs from prior tests, which are randomly generated.

### Root cause
The `CbrRetentionPolicy` record validates that `maxAgeDays` and `maxCasesPerType` must both be positive when non-null. There is no API surface to clear all cases unconditionally.

### Fix
Capture `Instant.now()` before the store operation under test, then query with `.withNotBefore(before)` to filter results to only cases stored during the current test:

```java
Instant before = Instant.now();
observer.onOutcome(event);  // stores a CBR case

CbrQuery query = CbrQuery.of(tenantId, domain, caseType, Map.of(), 10)
    .withNotBefore(before);
List<ScoredCbrCase<PlanCbrCase>> results = service.retrieveSimilar(query, PlanCbrCase.class);
assertThat(results).isEmpty();  // only sees cases stored after 'before'
```

### Why this is non-obvious
The natural expectation is that an in-memory store would either auto-reset between test methods or provide a `clearAll()` method. The `purge()` method looks like it should work but its validation constraints silently prevent clearing fresh data. The `withNotBefore` approach is a query-level workaround that doesn't require store-level changes.
