## Context

Two or more CDI `@Alternative @Priority(N)` strategy implementations share a complex algorithmic sub-step. The sub-step produces intermediate data structures that the downstream scoring step needs. Strategy interface composition (delegating to another `AgentRoutingStrategy`) would lose access to those intermediate results.

## Technique

Inject a shared `@ApplicationScoped` CDI bean (the "classifier" or "kernel") into each strategy implementation. The kernel encapsulates the shared algorithm; each strategy calls the kernel and then applies its own unique scoring logic to the results.

```java
// Shared kernel ‚Äî no CDI strategy annotations, just @ApplicationScoped
@ApplicationScoped
public class TrustCandidateClassifier {
    public List<ClassifiedCandidate> classify(
        List<AgentCandidate> candidates, String capability,
        TrustRoutingPolicy policy, TrustScoreCache cache) { ... }

    public static AgentAssignment decide(
        List<ClassifiedCandidate> classified,
        List<ScoredCandidate> scored, String capability) { ... }
}

// Strategy A ‚Äî injects the kernel
@Alternative @Priority(1) @ApplicationScoped
public class TrustWeightedAgentStrategy implements AgentRoutingStrategy {
    @Inject TrustCandidateClassifier classifier;
    // calls classifier.classify(), applies trust blend scoring, calls decide()
}

// Strategy B ‚Äî also injects the kernel
@Alternative @Priority(2) @ApplicationScoped
public class SemanticAgentRoutingStrategy implements AgentRoutingStrategy {
    @Inject TrustCandidateClassifier classifier;
    // calls classifier.classify(), applies semantic + trust blend scoring, calls decide()
}
```

CDI discovers `TrustCandidateClassifier` at boot as a regular bean; both strategies inject it without any `selected-alternatives` configuration. The strategy SPI interface remains clean ‚Äî no composition or chaining in the interface itself.

## Why This Is Non-Obvious

The natural impulse when two strategies share logic is:
1. **Duplicate the code** (easy, diverges over time)
2. **Compose strategies** ‚Äî `SemanticStrategy` wraps `TrustStrategy` via the SPI interface (loses intermediate `ClassifiedCandidate` data; `TrustStrategy.select()` only returns the final `AgentAssignment`)
3. **Extract a static utility** (works, but bypasses CDI ‚Äî harder to mock, no lifecycle)

Injecting a shared `@ApplicationScoped` bean into multiple `@Alternative` implementations is correct: it is discoverable, mockable in `@QuarkusTest`, singleton-safe when stateless, and preserves the full intermediate representation for downstream use.

## Constraints

The shared kernel must be stateless (or use thread-local state) ‚Äî it is a CDI singleton shared by concurrent routing decisions. If the kernel has mutable state, use `@Dependent` scope instead.
