## IntelliJ ide_refactor_rename with relatedRenamingStrategy:none renames one overload precisely — declaration and internal call sites updated, sibling overloads untouched

### Context

A Java class has multiple public methods with the same name — one annotated with `@Tool` and a
testability overload without. Goal: rename only the testability overload (4-param) without touching
the `@Tool` version (3-param), while still having IntelliJ update all call sites for the renamed
overload.

### Technique

Use `ide_refactor_rename` with `overrideStrategy: rename_only_current` and
`relatedRenamingStrategy: none`, targeting the specific overload by line/column:

```json
{
  "file": "runtime/src/main/java/.../QhorusMcpTools.java",
  "line": 930,
  "column": 24,
  "newName": "requestApprovalWithCorrelationId",
  "overrideStrategy": "rename_only_current",
  "relatedRenamingStrategy": "none"
}
```

Result: IntelliJ renamed the 4-param method declaration AND all call sites of that specific overload
(including internal call sites within the same file), while leaving the 3-param `@Tool` method
completely untouched. `changesCount: 1` referred to files changed, not individual rename occurrences.

### Why this is non-obvious

The natural approach when renaming one overload of many is text-based find-and-replace scoped to
the exact signature — fragile, misses internal call sites, and doesn't update references in other
files. IntelliJ's semantic rename naturally wants to rename ALL overloads together or none. The
`overrideStrategy` + `relatedRenamingStrategy` combination threads the needle: semantic accuracy
for the target overload, surgical non-involvement for the others. The parameters are documented in
the MCP tool schema but the combination for targeted overload rename is not explicitly described.

### When to apply

When a class has multiple overloads of the same method and you need to rename exactly one to
eliminate a naming conflict (e.g. a convenience overload that shadows a `@Tool`-annotated method
from a framework scanner) while all other overloads keep their names.
