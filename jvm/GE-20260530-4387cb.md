## Symptom

In a `@QuarkusTest` that relies on `InMemoryMessageStore` (from `casehub-qhorus-testing`), calling `messageService.findAllByCorrelationId(correlationId)` returns an empty list even though messages were dispatched to that correlationId moments before in the same test. The messages exist in the InMemory store but are invisible to this method.

## Context

`OversightGateService.fulfill()` calls `messageService.findAllByCorrelationId(gateId.toString())` to look up the original COMMAND message's Long ID (needed as `inReplyTo` for the RESPONSE dispatch). In `@QuarkusTest` with InMemory stores, this always returns empty — causing the service to log "no COMMAND message found" and fail open.

## Root Cause

`MessageService.findAllByCorrelationId()` is implemented with a Panache static call:

```java
public List<Message> findAllByCorrelationId(final String correlationId) {
    return Message.<Message> find("correlationId = ?1 ORDER BY id ASC", correlationId).list();
}
```

This bypasses the `MessageStore` interface entirely and queries Hibernate/H2 directly. The `InMemoryMessageStore` (the `@Alternative @Priority(1)` test bean) stores messages in a `ConcurrentHashMap` — it is never consulted by Panache static calls.

Most other `MessageService` methods inject and delegate to `MessageStore` — those work correctly with InMemory stores. Only the methods that use Panache static calls are affected.

## Fix

Wrap the problematic method with `@InjectSpy` in the test class and override only the bypassing method to scan the InMemory store:

```java
@InjectSpy
MessageService messageService;

@Inject
InMemoryMessageStore messageStore; // or inject the store interface directly

@BeforeEach
void mockFindAllByCorrelationId() {
    doAnswer(inv -> {
        String correlationId = inv.getArgument(0);
        return messageStore.findAll().stream()
            .filter(m -> correlationId.equals(m.correlationId))
            .sorted(Comparator.comparingLong(m -> m.id))
            .collect(Collectors.toList());
    }).when(messageService).findAllByCorrelationId(anyString());
}
```

All other `messageService` calls continue to use the real implementation; only `findAllByCorrelationId` is bridged to the InMemory store.

## Why This Is Non-Obvious

The `InMemoryMessageStore` gives the impression that all Qhorus data access is intercepted in tests. It isn't — only access that flows through the `MessageStore` interface is intercepted. Methods that call Panache directly bypass it silently: no error, just empty results. The real `InMemoryMessageStore` is injected correctly; it simply never receives the query.
