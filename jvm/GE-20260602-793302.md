## What Happened

Test class used `import static org.junit.jupiter.api.Assertions.*;` and `import static org.mockito.Mockito.*;` — standard test shortcuts. The build failed with checkstyle violations. The `AvoidStarImport` rule applies equally to static imports; there is no default exemption for test classes or static method imports.

## Why This Is Non-obvious

Static star imports for assertion methods (`assertThat`, `assertEquals`, etc.) are extremely common in Java test code and are often taught as idiomatic. The rule name "AvoidStarImport" suggests it targets `import java.util.*` patterns, not static method imports. The checkstyle error message says "Using a.* form of import" without distinguishing static from non-static.

## Fix

Replace star imports with explicit imports:
```java
// Wrong:
import static org.junit.jupiter.api.Assertions.*;
import static org.mockito.Mockito.*;

// Correct:
import static org.junit.jupiter.api.Assertions.assertEquals;
import static org.junit.jupiter.api.Assertions.assertNull;
import static org.mockito.Mockito.mock;
import static org.mockito.Mockito.when;
```

## Alternatives Considered

- Add `checkstyle.skip=true` to the build — breaks all checkstyle enforcement
- Suppress with `@SuppressWarnings` — not supported by checkstyle (annotation-based suppression requires a SuppressionXpathFilter)
