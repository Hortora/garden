### What It Is
In Vert.x 4.x, `writeBinaryMessage(Buffer)` with no completion handler is available on
`io.vertx.reactivex.core.http.WebSocket` (the RxJava2 binding) but NOT on
`io.vertx.core.http.WebSocket` (the core API):

```java
// Core (io.vertx.core.http.WebSocket) — Vert.x 4.x:
webSocket.writeBinaryMessage(buffer, handler);   // requires Handler<AsyncResult<Void>>
// webSocket.writeBinaryMessage(buffer);          // REMOVED — NoSuchMethodError

// RxJava2 binding (io.vertx.reactivex.core.http.WebSocket) — Vert.x 4.x:
webSocket.writeBinaryMessage(buffer);            // still exists — returns WebSocket
webSocket.rxWriteBinaryMessage(buffer);          // also available — returns Completable
```

### Context
Discovered when patching ocraft 0.4.21 for Vert.x 4.x compatibility. `Ping.java` and
`OnRequest.java` in ocraft called `webSocket.getDelegate().writeBinaryMessage(buffer)`,
accessing the core WebSocket via the rx wrapper's delegate. This failed with
`NoSuchMethodError` even though `writeBinaryMessage` looked available on both classes.

### Fix
Don't call `.getDelegate().writeBinaryMessage()`. Call the rx binding's method directly:
```java
// Instead of:
webSocket.getDelegate().writeBinaryMessage(io.vertx.core.buffer.Buffer.buffer(data));

// Use:
webSocket.writeBinaryMessage(io.vertx.reactivex.core.buffer.Buffer.buffer(data));
```
The rx `Buffer` wrapper (`io.vertx.reactivex.core.buffer.Buffer`) is required — not the
core `io.vertx.core.buffer.Buffer`.

### Why This Is Undocumented
The Vert.x 4.x migration guide says `writeBinaryMessage` changed but doesn't distinguish
between the core and rx-binding availability. javadoc on the rx binding class lists the
method without noting it's absent on the core class. The logical move — use `.getDelegate()`
to access the underlying core object — hits the exact removed method.
