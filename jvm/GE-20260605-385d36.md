## @InjectMock cannot mock @Singleton-scoped CDI beans — fails with 'not a normal scope' at test instantiation

### Symptom

`@QuarkusTest` fails at test class instantiation with:

```
java.lang.IllegalStateException: Invalid use of io.quarkus.test.InjectMock — the injected bean
does not declare a CDI normal scope but: jakarta.inject.Singleton.
Offending field is <fieldName> of test class class ...SomeTest
```

### Context

Occurs when using `@InjectMock` on a field whose CDI bean is declared `@Singleton` (or `@Dependent`) rather than `@ApplicationScoped`.

```java
// The Vert.x Mutiny EventBus is @Singleton-scoped — cannot be mocked
@InjectMock EventBus eventBus;  // ← IllegalStateException at test startup
```

### Root cause

`@InjectMock` works by replacing the CDI proxy with a Mockito mock. CDI proxying requires a **normal scope** (proxy-able). `@ApplicationScoped` is normal-scoped. `@Singleton` is **not** normal-scoped — it is a pseudo-scope without proxy support. CDI cannot intercept the bean to replace it with a mock. `@Dependent` is also pseudo-scoped and has the same limitation.

The Vert.x Mutiny `EventBus` is `@Singleton`-scoped by Quarkus's own extension — it cannot be mocked with `@InjectMock`.

### Fix

For `EventBus` specifically: instead of mocking, register a test consumer and capture messages:

```java
@Inject EventBus eventBus;  // real bus — inject directly, not @InjectMock

@Test
void myTest() {
    AtomicReference<MyEvent> captured = new AtomicReference<>();
    eventBus.consumer(EventBusAddresses.MY_ADDRESS,
            msg -> captured.set((MyEvent) msg.body()));

    // ... trigger code that sends to the event bus ...

    Awaitility.await().atMost(Duration.ofSeconds(5))
            .untilAsserted(() -> assertThat(captured.get()).isNotNull());
}
```

For other `@Singleton`-scoped beans you need to mock: there is no `@InjectMock` path. Alternatives:
- Change the bean's scope to `@ApplicationScoped` if you control it
- Use a CDI `@Alternative @Priority(1)` test double in the test classpath instead of Mockito
- Refactor so the dependency is injected at call-site rather than field-injected (enables manual injection in tests)

### Why this is non-obvious

The error message correctly names the scope (`jakarta.inject.Singleton`) but the connection to "cannot mock" is not documented. Developers assume `@InjectMock` mocks any injectable CDI bean. `@ApplicationScoped` (the most common scope) is mockable, so the limitation is never encountered until a less-common scoped bean is targeted. The Vert.x `EventBus` looks like any other injectable service — nothing in its injection point or the Quarkus docs warns that its scope makes it unmockable.
