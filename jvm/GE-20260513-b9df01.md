### Context

Moving an abstract method on a Java interface to a `default` method. The goal is to prove the contract lives on the interface itself — not just that the concrete class provides it. Testing on the concrete class alone does not prove this, because the concrete class override would still satisfy the test whether or not the method is abstract.

### Technique

Write a test with an anonymous implementation that provides only the genuinely abstract methods and deliberately omits the one being promoted to `default`. The compiler error ("does not override abstract method X in Y") is the RED state — it proves the method is still abstract. After adding `default`, the anonymous class compiles and the test passes GREEN.

```java
class CapabilityRegistrySpiTest {

    // Minimal implementation — only provides required abstract methods.
    // isKnown() is intentionally NOT overridden — proves it must be a default.
    private final CapabilityRegistry minimal = new CapabilityRegistry() {
        @Override
        public Set<String> capabilities() {
            return Set.of("code-analysis", "security-review");
        }

        @Override
        public Optional<RoutingPolicy> policy(String capability) {
            return Optional.empty();
        }
        // isKnown() deliberately omitted — compile error if still abstract (RED)
    };

    @Test
    void isKnownDefaultReturnsTrueForKnownCapability() {
        assertThat(minimal.isKnown("code-analysis")).isTrue();
    }

    @Test
    void isKnownDefaultReturnsFalseForUnknownCapability() {
        assertThat(minimal.isKnown("unknown")).isFalse();
    }

    @Test
    void isKnownDefaultReturnsFalseWhenCapabilitiesEmpty() {
        CapabilityRegistry empty = new CapabilityRegistry() {
            @Override public Set<String> capabilities() { return Set.of(); }
            @Override public Optional<RoutingPolicy> policy(String c) { return Optional.empty(); }
        };
        assertThat(empty.isKnown("anything")).isFalse();
    }

    @Test
    void isKnownDefaultThrowsNullPointerExceptionForNull() {
        assertThatThrownBy(() -> minimal.isKnown(null))
            .isInstanceOf(NullPointerException.class);
    }
}
```

### Why this is non-obvious

Most developers test a default method on the concrete class. That passes even if the method is abstract — the concrete class override satisfies it either way. The anonymous implementation trick isolates the interface contract. Crucially, the RED state is a **compile error**, not a test failure — which is different from the normal TDD cycle and easy to miss as intentional.
