## Context

You need the **last element** in a list that matches a predicate, where the list may have
non-matching elements after the last match. The naive approaches have problems:

- `list.stream().filter(pred).findFirst()` → gives the FIRST match, not the last
- `list.reversed().stream().filter(pred).findFirst()` → reverses the whole list; verbose
- `.toList().getLast()` after filter → creates an intermediate list; fails if result is empty

## Technique

Use `reduce((a, b) -> b)` — it replaces the accumulator with each new element, so the reduction
result is always the last element encountered:

```java
// Find the last ThreadEntry in a debate thread whose type is DISPUTE, QUALIFY, or COUNTER
// (thread may end with FLAG_HUMAN or AGREE after these — we want the last substantive response)

String lastResponse = thread.stream()
        .filter(e -> e.type() == EntryType.DISPUTE
                  || e.type() == EntryType.QUALIFY
                  || e.type() == EntryType.COUNTER)
        .reduce((a, b) -> b)          // accumulator = last element seen
        .map(ThreadEntry::content)
        .orElse("(no response yet)"); // safe default — stream may be empty after filter
```

The `reduce` runs through all filtered elements, always keeping the second argument (the later one),
ending with the last. Because `Stream.reduce` with a binary operator returns `Optional<T>`, the
empty-stream case is handled safely by `.orElse()`.

## Why this is non-obvious

Most developers reaching for "last element matching predicate" would reverse the list first, or
call `.toList()` and then index. `reduce((a, b) -> b)` works without creating an intermediate
collection and handles the empty case via `Optional` automatically. The pattern reads as
"replace accumulator with new value" — which happens to mean "last element".

## Constraints

The stream must be sequential (not parallel). For parallel streams, `reduce` order is not
guaranteed. This pattern is only correct when the stream is ordered and sequential.

## Alternatives considered

- `.toList().getLast()` after filter — clean but throws `NoSuchElementException` on empty list
- `.toList().isEmpty() ? default : list.getLast()` — verbose two-step
- `.reversed()` then `findFirst()` — reverses the whole backing list before filtering
