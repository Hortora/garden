## Symptom

`ExceptionInInitializerError` → `NoClassDefFoundError` for a class that is NOT directly
referenced by the subclass (e.g. `PostgreSQLAgroalConnectionConfigurer`). The error occurs
during Quarkus augmentation (`quarkus:build` phase), not at runtime.

## Context

Creating a CDI `@Alternative @Priority(1)` subclass of a framework-internal
`@ApplicationScoped` bean to override a single method. The superclass has `@Inject`
fields whose dependency chain includes optional extensions not on the classpath.

Example: subclassing `SchedulerService` (casehub-engine internal) to override
`registerScheduledTriggers()`. The superclass injects `JobScheduler`, whose Quartz
implementation transitively references `PostgreSQLAgroalConnectionConfigurer`. Even
though the subclass overrides the method that uses those dependencies, Quarkus
augmentation resolves the entire dependency chain of the `@Alternative` bean at
build time.

## Root cause

Quarkus Arc CDI resolves all injection points of a bean at augmentation time (build
phase), not lazily at first use. When an `@Alternative` subclass is registered, Arc
resolves ALL inherited `@Inject` fields — including those from the superclass that
the override will never call. If any transitive dependency in that chain is not on
the classpath, augmentation fails with `NoClassDefFoundError`.

## Fix

Do not subclass framework-internal beans with `@Alternative`. Instead:
- Provide a no-op implementation of the SPI the internal bean consumes (if the SPI
  exists and is public)
- Or file an issue on the framework to make the internal behavior configurable
- Or use `@InjectMock` (quarkus-junit5-mockito) for test-only overrides

## Why this is non-obvious

The `@Alternative` subclass compiles cleanly. The superclass is `public`. The method
being overridden is `public`. The failure occurs at augmentation time with a class
name from a completely different module (PostgreSQL JDBC) that appears nowhere in
the subclass code. The connection between "I subclassed SchedulerService" and
"PostgreSQLAgroalConnectionConfigurer not found" is invisible without understanding
Quarkus augmentation's eager dependency resolution.
