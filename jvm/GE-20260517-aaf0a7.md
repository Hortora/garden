## Context

Accessing the last element of a `List` in Java. Common pattern when scanning message histories, event logs, or any ordered sequence where only the most recent item determines state.

## Technique

Java 21 introduced `SequencedCollection`, which `List` implements. Use `List.getLast()` instead of a loop or index arithmetic:

```java
// Before Java 21 — loop accumulator
MessageType lastType = null;
for (Message m : messages) {
    lastType = m.messageType;
}

// Before Java 21 — index access (verbose, throws on empty)
MessageType lastType = messages.isEmpty() ? null
    : messages.get(messages.size() - 1).messageType;

// Java 21 — clean and O(1)
if (messages.isEmpty()) return "submitted";
MessageType lastType = messages.getLast().messageType;
```

Also available: `getFirst()`, `reversed()`, `addFirst()`, `addLast()`, `removeFirst()`, `removeLast()`.

## Why this is non-obvious

Java developers reaching for the last element habitually write a for-loop accumulator (`lastVal = item` on every iteration) or use `get(size()-1)`. Both are O(1) or O(n) depending on implementation, but neither expresses *intent*. `SequencedCollection` landed in Java 21 (JEP 431) and is not widely known outside developers who followed the JEP closely.

## Constraints

Requires Java 21+. The loop accumulator approach remains correct and is still the right choice in code that targets Java 17 or earlier.
