# DeserializationProblemHandler.handleUnknownTypeId() has no access to sibling JSON fields

### Symptom

Designing a polymorphic type hierarchy with graceful degradation — unknown subtypes should fall back to a common parent type. The fallback decision depends on a data field in the JSON (e.g., `deviceClass`). Attempted to use `DeserializationProblemHandler.handleUnknownTypeId()` to read the sibling field and resolve the fallback type.

### Context

`@JsonTypeInfo(use = Id.CUSTOM)` with a custom `TypeIdResolver`. When the resolver encounters an unknown type ID (e.g., a vendor supplement type not on the classpath), it throws. The `DeserializationProblemHandler` is the intended extension point for handling unknown type IDs.

### Root cause

The `handleUnknownTypeId` signature (verified in Jackson 2.21.1 source at `DeserializationProblemHandler.java:357-359`):

```java
public JavaType handleUnknownTypeId(DeserializationContext ctxt,
        JavaType baseType, String subTypeId, TypeIdResolver idResolver,
        String failureMsg) throws IOException
```

No `JsonParser` parameter. The handler receives only the type ID string (`subTypeId`), the base type, the resolver, and a failure message. It **cannot** read `deviceClass` or any other sibling field from the JSON.

### Fix

Embed the fallback information in the type ID itself. Use a compound format that carries the fallback key as a prefix:

```
@deviceType: "THERMOSTAT:HomeAssistantThermostat"
@deviceType: "COVER:OpenHabRollershutter"
```

The custom `TypeIdResolver.typeFromId()` splits on `:` — tries the full ID first (exact match), falls back to the prefix (maps to the common parent type). No `DeserializationProblemHandler` needed.

### Why this is non-obvious

The Jackson documentation describes `DeserializationProblemHandler` as the extension point for "handling unknown type IDs" — the natural assumption is that it has access to the parsing context including sibling fields. The signature doesn't include a `JsonParser`, but this only becomes apparent by reading the source code, not the API docs or any tutorial.
