## Symptom

A `@QuarkusTest` annotated with `@TestProfile` fails at deployment with 30+ "Unsatisfied dependency" CDI errors for standard persistence repositories (`CaseInstanceRepository`, `EventLogRepository`, etc.) that are correctly configured in `src/test/resources/application.properties` and work fine for all other test classes.

The JUnit class-loading error appears as:

```
TestEngine with ID 'junit-jupiter' encountered a critical issue during test discovery:
(1) [ERROR] ClassSelector [className = 'com.example.MyProfiledTest'] resolution failed
Caused by: jakarta.enterprise.inject.spi.DeploymentException: Found 32 deployment problems
```

## Context

`src/test/resources/application.properties`:
```properties
quarkus.arc.selected-alternatives=\
  io.casehub.persistence.memory.InMemoryCaseInstanceRepository,\
  io.casehub.persistence.memory.InMemoryEventLogRepository,\
  ...
```

Profile class:
```java
public static class Profile implements QuarkusTestProfile {
    @Override
    public Set<Class<?>> getEnabledAlternatives() {
        return Set.of(FailingWorkItemStore.class); // only the test-specific one
    }
}
```

The Javadoc for `getEnabledAlternatives()` says "alternatives to enable in addition to those configured via `quarkus.arc.selected-alternatives`" — but this is not what happens in practice.

## Root Cause

`QuarkusTestProfile.getEnabledAlternatives()` **replaces** (not appends to) the global `quarkus.arc.selected-alternatives` for the test context where the profile is active. Any alternative not listed in `getEnabledAlternatives()` is not activated, even if it is listed in `application.properties`.

The Javadoc is misleading — the actual Quarkus ARC behaviour is replacement, not union.

## Fix

Re-declare all globally required alternatives in the profile:

```java
public static class Profile implements QuarkusTestProfile {
    @Override
    public Set<Class<?>> getEnabledAlternatives() {
        // Must include ALL alternatives needed for deployment, not just the profile-specific ones.
        // getEnabledAlternatives() replaces quarkus.arc.selected-alternatives — does not append.
        return Set.of(
            FailingWorkItemStore.class,           // profile-specific
            InMemoryCaseInstanceRepository.class,  // globally required
            InMemoryEventLogRepository.class,
            MemorySubCaseGroupRepository.class,
            MemoryPlanItemStore.class,
            NoOpLedgerEntryRepository.class,
            NoOpReactiveLedgerEntryRepository.class
        );
    }
}
```

## Why this is non-obvious

The Javadoc explicitly says "in addition to those configured via `quarkus.arc.selected-alternatives`", which implies additive behaviour. The symptom (30+ CDI errors for standard repos) does not point toward the profile's `getEnabledAlternatives()` as the cause — it looks like a configuration problem with the persistence layer.
