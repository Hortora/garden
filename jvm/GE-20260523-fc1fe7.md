## Symptom

A `@QuarkusTest` test class injects a repository, calls `save()`, receives a `LedgerEntry`
(or any `@Entity`) back, and immediately fails with `IllegalAccessError` when reading a field:

```
java.lang.IllegalAccessError: class InMemoryLedgerEntryRepositoryTest cannot access field
io.casehub.ledger.runtime.model.LedgerEntry.sequenceNumber (io.casehub.ledger.runtime.model.LedgerEntry
is in module UNNAMED of loader 'app'; InMemoryLedgerEntryRepositoryTest is in module UNNAMED of loader
'QuarkusClassLoader@...')
```

The field is declared `public int sequenceNumber` in source. The error says the test class
cannot access it.

## Context

Occurred while writing `@QuarkusTest` tests for `InMemoryLedgerEntryRepository`. The test
class injected the repository, saved entries, and tried to assert `e.sequenceNumber == 1`.
The field is `public` in the `LedgerEntry` source — the assertion should compile and run
without issue.

## Root Cause

Quarkus Hibernate ORM bytecode enhancement instruments `@Entity` class fields at build
time to intercept reads and writes for dirty checking and lazy loading. The instrumentation
changes field access semantics in the enhanced bytecode: fields declared `public` in source
become `protected` in the enhanced class that the `@QuarkusTest` classloader loads. The
JVM enforces this — a class in a different classloader that is not a subclass of the entity
cannot access `protected` fields.

## Fix

Create a package-private subclass of the entity in the test package. Because it extends
the entity, the JVM grants it `protected` field access. Expose each needed field via an
accessor method:

```java
package io.casehub.ledger.memory;

/** Concrete subclass for in-memory tests — subclass access satisfies protected field rules. */
class MemoryTestEntry extends LedgerEntry {
    int getSequenceNumber() { return sequenceNumber; }
    UUID getId()            { return id; }
    Instant getOccurredAt() { return occurredAt; }
    // add others as needed
}
```

Then use `MemoryTestEntry` as the concrete type in tests and read fields via the accessors:

```java
MemoryTestEntry e = new MemoryTestEntry();
e.subjectId = subjectId;
e.entryType = LedgerEntryType.EVENT;
repo.save(e);
assertThat(e.getSequenceNumber()).isEqualTo(1);
```

The same pattern applies to `LedgerAttestation` and any other `@Entity` whose fields the
test class needs to read directly.

## Why This Is Non-Obvious

The fields are `public` in the source file — every Java developer's expectation is that
`public` means universally accessible. Nothing in the Quarkus or Hibernate documentation
warns that bytecode enhancement changes field visibility. The error message mentions
`QuarkusClassLoader` but does not explain why a `public` field is inaccessible. The
natural first response is to check for a typo or a module-info restriction — neither
applies. The fix (subclass for accessor access) is the Java workaround for `protected`
inheritance, not a pattern any developer would reach for when looking at a `public` field.
