## Use javap -verbose to read @NamedQuery annotations from JPA entity class files in compiled JARs

When you depend on a library that defines JPA `@NamedQuery` annotations on its
entities, and you need to call those queries by name in your own repository
code, you can read them directly from the compiled `.class` file — no source
access required.

### The technique

```bash
# 1. Extract the entity class from the JAR
cd /tmp
jar xf ~/.m2/repository/io/casehub/casehub-ledger/0.2-SNAPSHOT/casehub-ledger-0.2-SNAPSHOT.jar \
  io/casehub/ledger/runtime/model/LedgerAttestation.class

# 2. Dump the annotation metadata — @NamedQuery values appear verbatim
javap -verbose io/casehub/ledger/runtime/model/LedgerAttestation.class \
  | grep -A3 "NamedQuery"

# Output includes the full query strings:
#   name="LedgerAttestation.findByEntryIdAndCapabilityTag"
#   query="SELECT a FROM LedgerAttestation a WHERE a.ledgerEntryId = :entryId
#          AND a.capabilityTag = :capabilityTag ORDER BY a.occurredAt ASC"
```

### Context

Use this when:
- A library JAR declares JPA entities with `@NamedQuery` annotations
- You need to call those queries from a repository you implement
- The library source isn't checked out or isn't searchable locally
- Guessing query names leads to silent failures at test runtime (no query found exception)

### Why this is non-obvious

Most developers would: look in docs (often absent for SNAPSHOT libs), clone the
source repo, or guess the query names and discover mismatches at runtime. The
`javap -verbose` approach gives you the exact query strings in seconds — the
JVM constant pool stores annotation values as UTF-8 literals, and `javap`
exposes them in full.
