### Context
CDI observers and SPI implementations that use Panache Active Record entities (`Entity.findById()`, `Entity.count()`, etc.) cannot be unit-tested without `@QuarkusTest` — the static methods require an active Hibernate session. But `@QuarkusTest` is slow (full CDI container, database, migrations) for testing pure business logic.

### Technique
Define a package-private `EntityResolver` functional interface inside the CDI bean. Production uses an inner `PanacheEntityResolver` class. Tests inject a `TestEntityResolver` record stub.

```java
@ApplicationScoped
public class MyObserver implements SomeSPI {

    private EntityResolver entityResolver;

    @Inject
    public MyObserver(/* CDI deps */) {
        this.entityResolver = new PanacheEntityResolver();
    }

    // Package-private for testing
    void setEntityResolver(EntityResolver resolver) {
        this.entityResolver = resolver;
    }

    @Override
    public void onEvent(SomeEvent event) {
        MyEntity entity = entityResolver.findEntity(event.entityId());
        // business logic using entity...
    }

    interface EntityResolver {
        MyEntity findEntity(UUID id);
        long countByParent(UUID parentId);
    }

    private static class PanacheEntityResolver implements EntityResolver {
        @Override public MyEntity findEntity(UUID id) { return MyEntity.findById(id); }
        @Override public long countByParent(UUID parentId) {
            return MyEntity.count("parentId", parentId);
        }
    }
}
```

Test side — zero Quarkus overhead:

```java
class MyObserverTest {
    record TestEntityResolver(MyEntity entity, long count) implements MyObserver.EntityResolver {
        @Override public MyEntity findEntity(UUID id) { return entity; }
        @Override public long countByParent(UUID parentId) { return count; }
    }

    @BeforeEach
    void setUp() {
        observer = new MyObserver(mockCbrService);
        observer.setEntityResolver(new TestEntityResolver(testEntity, 2));
    }
}
```

### Why this is non-obvious
The natural instinct with Panache Active Record is to either (a) use `@QuarkusTest` for everything (slow, heavy) or (b) try to mock the static methods with Mockito's `mockStatic` (fragile, fights Panache's bytecode enhancement). The EntityResolver pattern sidesteps both: no static mocking, no container overhead, and the resolver interface documents exactly which data access the observer needs — making the dependency explicit rather than hidden behind static calls.

### Alternatives considered
- **`@QuarkusTest` for all observer tests** — works but 15-25s startup per test class for testing pure logic that runs in <1ms. Scales poorly with many observer classes.
- **Mockito `mockStatic(MyEntity.class)`** — unreliable with Panache because bytecode enhancement modifies the static methods at build time. The mock intercepts the pre-enhancement method, not the enhanced one.
