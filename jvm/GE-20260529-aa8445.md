## Symptom

`GreenMailExtension` has no `deliver(MimeMessage)` method. Calling it produces a compile error:

```
cannot find symbol
  symbol:   method deliver(jakarta.mail.internet.MimeMessage)
  location: variable GREEN_MAIL of type com.icegreen.greenmail.junit5.GreenMailExtension
```

## Context

When writing JUnit 5 IMAP connector tests, the obvious approach is to inject a test email directly into the Greenmail mailbox via `GreenMailExtension.deliver(msg)`. The `GreenMail` bare class (not the JUnit 5 extension) does have a `deliver()` method. The extension wraps GreenMail but does not expose this method.

## What Was Tried

1. Calling `GREEN_MAIL.deliver(msg)` — compile error
2. Calling `GREEN_MAIL.getGreenMail()` to get the underlying instance — `getGreenMail()` is `protected`, inaccessible from test code
3. Confirmed via `javap -classpath greenmail-junit5-2.1.3.jar com.icegreen.greenmail.junit5.GreenMailExtension`

## Root Cause

`GreenMailExtension` extends `GreenMailProxy`. `GreenMailProxy.getGreenMail()` is declared `protected` — not part of the public API exposed to test code. No `deliver()` method is added at the extension level. The `GreenMailOperations` interface (which the extension implements) does not include `deliver()`.

## Fix

Two alternatives depending on the need:

**For most tests (when header presence doesn't matter):** use SMTP delivery via `Transport.send()`:

```java
Properties props = new Properties();
props.put("mail.smtp.host", "localhost");
props.put("mail.smtp.port", String.valueOf(GREEN_MAIL.getSmtp().getPort()));
props.put("mail.smtp.auth", "true");
Session session = Session.getInstance(props);
try (Transport transport = session.getTransport("smtp")) {
    transport.connect("inbox@example.com", "password");
    transport.sendMessage(msg, new Address[]{ new InternetAddress("inbox@example.com") });
}
```

Requires Greenmail configured with both SMTP and IMAP (`ServerSetupTest.SMTP_IMAP` or port-0 equivalents).

**For tests requiring precise header control** (e.g. testing absent Message-ID): use `MailFolder.appendMessage()` — see companion entry GE-20260529-4691e8.

## Why This Is Non-Obvious

The `GreenMail` bare class (used in JUnit 4 and standalone) has `deliver()`. The JUnit 5 extension wraps it but silently drops this method from the public API. The Greenmail documentation doesn't mention this difference. The only way to discover it is `javap` on the jar.

**See also:** [GE-20260529-4691e8](GE-20260529-4691e8.md) — deliver MimeMessage directly to Greenmail IMAP mailbox via MailFolder.appendMessage()
