### Symptom

Maven Surefire reports test-*discovery* failure, not a test failure:

```
[ERROR] TestEngine with ID 'junit-jupiter' encountered a critical issue during test discovery:
(1) [ERROR] ClassSelector [className = 'io.example.MyQuarkusTest', classLoader = null] resolution failed
    Cause: org.junit.platform.commons.PreconditionViolationException: Could not load class
```

Every `@QuarkusTest` class shows this error. `BUILD FAILURE` with `Tests run: 0`. The class `.class` file is present in `target/test-classes/` — the class compiled fine.

### Root cause

Surefire tries to load the test class to discover its test methods. When Quarkus augmentation fires (which happens during class loading for `@QuarkusTest`), CDI fails to boot because a required `@Default`-qualified bean is unsatisfied. The `jakarta.enterprise.inject.UnsatisfiedResolutionException` is thrown during augmentation, not test execution — so Surefire catches it as a class-loading failure, not a test failure.

The actual CDI error is in the `.dumpstream` file:
```
Caused by: jakarta.enterprise.inject.UnsatisfiedResolutionException:
  Unsatisfied dependency for type io.casehub.platform.api.identity.CurrentPrincipal
  and qualifiers [@Default]
```

### Diagnosis

Check the dump file — it is printed somewhere in the Surefire output as:
```
[ERROR] Please refer to /path/to/surefire-reports for individual test results.
[ERROR] Please refer to dump files (if any exist) [date].dump, [date]-jvmRun[N].dump and [date].dumpstream.
```

Read the `.dumpstream` file and look for `Caused by: jakarta.enterprise.inject.*`.

### Fix

Find the CDI bean type in the exception, identify which module provides the `@DefaultBean` mock implementation, and add it as a `<scope>test</scope>` dependency.

**Example:** `CurrentPrincipal` needs `MockCurrentPrincipal @DefaultBean` which lives in `casehub-platform`. Adding:
```xml
<dependency>
  <groupId>io.casehub</groupId>
  <artifactId>casehub-platform</artifactId>
  <version>0.2-SNAPSHOT</version>
  <scope>test</scope>
</dependency>
```
…resolves all `@QuarkusTest` classes immediately.

### Why this is non-obvious

Three layers of indirection hide the actual cause:

1. **Symptom points to classloader** — "classLoader = null" and "PreconditionViolationException: Could not load class" looks like a module-path or bytecode issue, not a CDI misconfiguration.
2. **Error appears at *discovery*, not execution** — CDI boot errors normally manifest as test failures (the Quarkus app fails to start and all tests in the class fail). Here it fails even earlier — before JUnit can enumerate the test methods — because Quarkus augments at class-load time for `@QuarkusTest`.
3. **Real error is in the dump file, not the log** — the main Surefire output only shows the "PreconditionViolationException" wrapper. The root `UnsatisfiedResolutionException` with the bean type name is in `target/surefire-reports/*.dumpstream`, which is not shown by default.

### When this fires

This fires when a new SNAPSHOT dependency introduces a CDI bean that injects a type without a `@DefaultBean` on the test classpath — specifically when the dep was recently updated and the new bean uses a type previously unused by the module. Common culprit: updating a platform library that adds `@Inject CurrentPrincipal`, `@Inject TenantContext`, or similar identity beans.
