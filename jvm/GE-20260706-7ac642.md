# drools-reliability requires three factory initializations — only one is documented

## Pain (4)

`NullPointerException` in `KieServiceLoader.load()` with no useful message when creating a persisted KieSession. The NPE occurs on a null tag reference deep in the service loader, giving no clue about missing initialization.

## Non-obvious (4)

Only `StorageManagerFactory.get("h2mvstore")` is mentioned in documentation and README. The other two required initializations (`ReliableGlobalResolverFactory.get("core")` and `SimpleReliableObjectStoreFactory.get("core")`) are only visible in Drools' own `TestConfigurationUtils.prioritizeH2MVStoreServices()` test utility. No public documentation links the three together.

## Reuse (3)

Anyone using drools-reliability will hit this. The pattern applies to all drools-reliability backends (H2MVStore, Infinispan), not just H2MVStore.

## Problem

Using drools-reliability with H2MVStore backend for KieSession persistence results in `NullPointerException` in `KieServiceLoader.load()` on null tag when attempting to create a persisted KieSession.

## Root Cause

`StorageManagerFactory.get("h2mvstore")` alone is insufficient. You also need `ReliableGlobalResolverFactory.get("core")` and `SimpleReliableObjectStoreFactory.get("core")`. Without these, the KieService SPI loaders have null tags and fail at load time.

## Solution

Call all three factory initializations before creating sessions:

```java
ReliableGlobalResolverFactory.get("core");
SimpleReliableObjectStoreFactory.get("core");
StorageManagerFactory.get("h2mvstore");
```

Order matters — the core factories must be initialized before the storage manager factory.
