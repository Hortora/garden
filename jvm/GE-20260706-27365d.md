# drools-reliability PersistenceStrategy.FULL is non-functional in Drools 10.1.0 — only STORES_ONLY works

## Pain (3)

Wastes time trying `PersistenceStrategy.FULL` before discovering it's broken. The strategy is offered in the public API without deprecation or warning, so developers naturally try it first.

## Non-obvious (3)

Not mentioned in release notes, changelog, or public documentation. The only evidence is in the test source code, where all FULL-strategy tests are commented out with `// FULL fails with "ReliablePropagationList; no valid constructor"` and replaced with STORES_ONLY variants.

## Reuse (3)

Universal for drools-reliability users choosing a persistence strategy.

## Problem

When choosing between `STORES_ONLY` and `FULL` persistence strategies for drools-reliability, `PersistenceStrategy.FULL` is a valid enum value in `PersistedSessionOption` and appears functional from the API. However, using it results in serialization errors: `"ReliablePropagationList; no valid constructor"`.

## Root Cause

All FULL-strategy tests in `drools-reliability-tests` are commented out with `// FULL fails with "ReliablePropagationList; no valid constructor"` and replaced with STORES_ONLY variants. The FULL strategy is broken in Drools 10.1.0 but remains in the public API without any deprecation annotation or documentation note.

## Solution

Use `PersistenceStrategy.STORES_ONLY` instead of `FULL`:

```java
PersistedSessionOption.newSession()
    .withPersistenceStrategy(PersistenceStrategy.STORES_ONLY)
    // ...
```

Check newer Drools versions to see if FULL strategy tests are re-enabled.
