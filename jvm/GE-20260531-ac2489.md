## Symptom

`quarkus:build` (or `@QuarkusTest` startup) fails with:

```
IllegalStateException: Parameter of a RESOURCE_TEMPLATE method may not be annotated
with @ToolArg: MyResource#myMethod(java.lang.String, java.lang.Long)
```

No indication in the IDE — the annotation compiles fine; the error only fires at Quarkus augmentation time.

## Context

Adding an optional `since` query-style parameter to a `@ResourceTemplate` method using `@ToolArg` (the standard annotation for optional tool parameters).

```java
@ResourceTemplate(
    name = "channel-recent",
    uriTemplate = "casehub://channel/{agentId}/recent"
)
public ResourceResponse channelRecent(
    @ResourceTemplateArg(name = "agentId") String agentId,
    @ToolArg(description = "Since cursor", required = false) Long since  // ← WRONG
) { ... }
```

## Root cause

`@ToolArg` is exclusively for `@Tool`-annotated methods. `@ResourceTemplate` methods use a separate parameter model: URI template variables use `@ResourceTemplateArg`; there is no equivalent optional-parameter annotation for resource methods because MCP resources don't support query parameters — the URI template is the complete address.

## Fix

For URI-bound parameters: use `@ResourceTemplateArg(name = "paramName")` and add the variable to `uriTemplate`.

For parameters that can't be expressed in the URI: redesign. Either make the resource URI include the parameter (`casehub://channel/{agentId}/recent/{since}`) or drop the parameter and always return full state. The latter is often the right choice for read-only resources — cursored fetching belongs in the consuming client (plugin), not in the MCP resource.

## Why this is non-obvious

`@ToolArg(required = false)` works perfectly on `@Tool` methods for optional parameters. Developers naturally reach for the same annotation on resource methods when they want optional query-style inputs. The compile error is absent — only augmentation catches it — so the feedback loop is delayed. The annotation name `ToolArg` doesn't signal that it is tool-exclusive.
