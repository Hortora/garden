### What exists but isn't documented

`jakarta.enterprise.inject.Instance.handles()` (CDI 4.1) returns `Iterable<Instance.Handle<T>>`,
not a `Stream`. The CDI spec's `Handle` interface only exposes `get()`, `getBean()`, `destroy()`,
`close()` — no `stream()` method. To sort CDI beans by metadata (e.g. `@Priority`), you need
Arc's extension: `InjectableInstance.handlesStream()` returns `Stream<InstanceHandle<T>>`.

### Context

Quarkus applications that need to iterate CDI beans in a specific order — enricher pipelines,
ordered observers, sorted strategy lists.

### How to use it

Inject `InjectableInstance` instead of `Instance`, then call `handlesStream()`:

```java
import io.quarkus.arc.InjectableInstance;
import io.quarkus.arc.InjectableBean;
import io.quarkus.arc.InstanceHandle;

@Inject @Any InjectableInstance<MyBean> beans;

beans.handlesStream()
    .sorted(Comparator.comparingInt(h ->
        (h.getBean() instanceof InjectableBean<?> ib) ? ib.getPriority() : Integer.MAX_VALUE))
    .map(InstanceHandle::get)
    .forEach(bean -> bean.doWork());
```

### Why not in docs

`InjectableInstance` is an Arc implementation detail, not part of the CDI 4.x standard.
The Quarkus user guide for CDI shows `Instance<T>` injection only. `handlesStream()` is
discoverable via Arc's JavaDoc but not mentioned in any getting-started or ordering guide.
