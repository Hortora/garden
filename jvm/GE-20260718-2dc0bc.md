### Symptom
A WorkItem is created with a past deadline. The Quartz timer is scheduled
with `startAt(Date.from(pastInstant))`. The timer fires, the job runs
`workItemStore.get(workItemId)` — but the WorkItem isn't visible yet
because the creating transaction hasn't committed. The job's `ifPresent()`
lambda never executes. No error, no warning, no retry.

### Context
`WorkItemService.create()` is `@Transactional`. Inside that transaction it
persists the WorkItem and calls `timerService.scheduleExpiry(id, tenancyId,
expiresAt)`. Quartz schedules the job with `startAt(Date.from(expiresAt))`.
When `expiresAt` is in the past, Quartz fires the trigger immediately — in
a separate thread, outside the creating transaction. The job reads from the
database and finds nothing because the creating transaction is still open.

### Root cause
Quartz job scheduling is not transactional — `scheduler.scheduleJob()` takes
effect immediately regardless of the surrounding JPA/JTA transaction state.
For future fire times this is invisible (the transaction commits long before
the timer fires). For past fire times, the timer fires before the data it
operates on is committed.

### Fix
In tests, call the expiry service directly instead of relying on the Quartz
timer:

```java
@Inject ExpiryLifecycleService expiryService;

// After creating the task (transaction committed by REST call returning 201):
expiryService.expireItem(UUID.fromString(workItemId));

// Then assert the result synchronously — no Awaitility needed
given().when().get("/life-tasks/" + workItemId)
    .then().statusCode(200)
    .body("candidateGroups", hasItem("household-admin"));
```

### Why this is non-obvious
The race is invisible for future deadlines — which is every normal case.
It only manifests when testing with past deadlines, where the symptom
(candidateGroups unchanged) looks like the breach policy isn't working,
not like a transaction visibility issue. Awaitility polling masks the
root cause by suggesting "it just needs more time" when the real problem
is the job already ran and found nothing.
