### What exists but isn't documented

`GET /rest/items/{groupItemName}` on an Equipment Group item returns a `members`
array where each member item has its current `state` field populated. This is a
single HTTP call that gives you the fully assembled device state for a logical
device — equivalent to what Home Assistant returns in a single entity GET.

There is no dedicated semantic model endpoint like `/rest/semantics/equipment/{name}`.
The semantic REST API surface (`GET /rest/tags`) only manages tag definitions, not
item queries. The assembly capability is a side-effect of the Group item endpoint,
not a named feature.

### Discovery

Found via inspection of `ItemResource.java` in `openhab-core`:
`EnrichedItemDTOMapper.map(item, recursive, ...)` with `recursive` defaulting to
`true` — the recursive parameter causes member items to be expanded with their
current states. Not mentioned in the REST API documentation or the semantic model
documentation.

### How to use it

```http
GET /rest/items/LivingRoom_Thermostat
Authorization: Bearer <token>
```

Response includes a `members` array with each Point item's current `state`:

```json
{
  "name": "LivingRoom_Thermostat",
  "tags": ["Equipment", "HVAC"],
  "members": [
    { "name": "LivingRoom_Temperature", "type": "Number:Temperature", "state": "21.5 °C" },
    { "name": "LivingRoom_Setpoint",    "type": "Number:Temperature", "state": "22.0 °C" },
    { "name": "LivingRoom_Mode",        "type": "String",             "state": "heat" }
  ]
}
```

### Limitation

Only works if the semantic model is configured — the Equipment item must be a
Group with Point members tagged appropriately. Many OpenHAB installations use
Things/Channels/Items without the semantic model layer, in which case there is
no equivalent single-call assembly. For those, device state must be assembled
from N separate `GET /rest/items/{itemName}` calls.

### SSE events are still per-item

The SSE event stream (`GET /rest/events`) fires `ItemStateChangedEvent` per
individual item, not per Equipment group. The assembly endpoint is useful for
initial device discovery and state hydration — for real-time updates, a
per-item state cache still needs to be maintained.
