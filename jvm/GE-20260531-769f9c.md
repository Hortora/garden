### Symptom

Integration test seeds a trust score via `trustScoreRepository.updateGlobalTrustScore(actorId, 0.15)`, then asserts `trustGateService.currentScore(actorId)` returns the seeded value. Instead, `currentScore()` returns `Optional.empty()` — as if the actor has no trust history at all. No error is thrown; the UPDATE executes silently and returns with 0 rows affected.

### Context

Writing a `@QuarkusTest` for a component that calls `TrustGateService.currentScore(actorId)`, which returns the actor's global trust score. The actor is new (no existing `ActorTrustScore` rows). The test tries to seed a below-threshold score to verify the gate rejects it.

### Root cause

`ActorTrustScoreRepository.updateGlobalTrustScore(actorId, score)` issues a JPQL `UPDATE` that patches the `globalTrustScore` column (EigenTrust share) on an **existing** `GLOBAL` row. Two problems:

1. **It does not insert**: if no `ActorTrustScore` row exists for the actor, the UPDATE affects 0 rows — silent no-op.
2. **It patches the wrong field**: `TrustGateService.currentScore()` reads the `trustScore` column, not `globalTrustScore`. Even if the row existed, `updateGlobalTrustScore()` does not touch `trustScore`.

The method name implies it sets the actor's readable global trust score. It does not — it updates only the EigenTrust-computed share used internally by `TrustScoreJob`.

### Fix

Use `upsert()` to create the row and set `trustScore` directly:

```java
// Seed a global trust score of 0.15 for a test actor
trustScoreRepository.upsert(
    actorId,
    io.casehub.ledger.api.model.ActorTrustScore.ScoreType.GLOBAL,
    null,           // capabilityTag — null for global
    null,           // dimensionTag — null for global
    io.casehub.platform.api.identity.ActorType.AGENT,
    0.15,           // trustScore — this is what TrustGateService.currentScore() reads
    0,              // windowSize
    1,              // observations
    0.15,           // mean
    0.0,            // variance
    0,              // hits
    1,              // total
    java.time.Instant.now()
);
```

After `upsert()`, `trustGateService.currentScore(actorId)` returns `Optional.of(0.15)`.

### Why this is non-obvious

The method name `updateGlobalTrustScore` strongly implies it sets the global trust score that callers read. The distinction between `globalTrustScore` (EigenTrust share, write-only from job) and `trustScore` (readable gate field) is not reflected in the API naming. A developer seeding test data would naturally reach for the method that sounds like "set the global trust score" — and would see no error when it silently does nothing.
