### What
In a Quarkus test suite where multiple `@QuarkusTest` classes share the same CDI container (one JVM fork per module), use `await()` on case-specific state rather than a shared static list to avoid races with other test classes' cases firing the same CDI beans.

### Context
When multiple `@QuarkusTest` classes share a CDI container, `@Alternative @Priority(1) @ApplicationScoped` recording beans (e.g. classifiers, listeners) are active across all test classes simultaneously. A shared static `CopyOnWriteArrayList<PlannedAction> capturedActions` collects captures from ALL running cases — including cases started by other test classes in the same JVM.

An `await(!capturedActions.isEmpty())` passes when ANY case triggers the classifier, not when THIS test's case does. The subsequent assertion on `caseInstanceCache.get(caseId).getPendingActionGate()` then fails because the captured action belongs to a different case.

### The technique
```java
// Wrong — races with other test classes in the same Quarkus session
await().atMost(5, TimeUnit.SECONDS)
    .until(() -> !CapturingClassifier.capturedActions.isEmpty());

// Correct — awaits state specific to this test's caseId
await().atMost(5, TimeUnit.SECONDS)
    .until(() -> {
        final var inst = caseInstanceCache.get(caseId);
        return inst != null && inst.getPendingActionGate() != null;
    });

// Also correct — filter captured list by caseId when reading
PlannedAction action = CapturingClassifier.capturedActions.stream()
    .filter(a -> caseId.equals(a.caseId()))
    .findFirst()
    .orElseThrow();
```

### Why this is non-obvious
Tests pass when run in isolation (`-Dtest=SpecificTest`) and fail only when run as part of the full module (`mvn test`). The failure is non-deterministic — it depends on test ordering and timing of other test classes' async operations. The root cause looks like a state management bug in the handler, not a test isolation issue.

### Constraints
Only applies when multiple `@QuarkusTest` classes share the same JVM (same module, surefire `forkCount=1` per module). With `@QuarkusTestProfile` isolation or `forkCount=1perThread`, each class gets a fresh CDI container.
