### Symptom

```
io.smallrye.config.ConfigValidationException: Configuration validation failed:
  SRCFG00050: casehub.ledger.reactive.enabled in PropertiesConfigSource[...] does not map to any root
```

At startup — not augmentation — even though the property is declared as `@ConfigRoot(BUILD_TIME)` in the deployment module.

### Context

A Quarkus extension has two config roots over the same prefix (`casehub.ledger`):
- Runtime module: `@ConfigMapping(prefix = "casehub.ledger") @ConfigRoot(phase = RUN_TIME) LedgerConfig`
- Deployment module: `@ConfigMapping(prefix = "casehub.ledger") @ConfigRoot(phase = BUILD_TIME) LedgerBuildTimeConfig` with a `reactive.enabled` property

At augmentation, `LedgerBuildTimeConfig.reactive().enabled()` is read correctly. But at startup, `LedgerConfig` is re-validated against `application.properties`. Since `LedgerConfig` has no `reactive()` method, Quarkus rejects the `casehub.ledger.reactive.enabled` entry as unmapped — `SRCFG00050`.

### Root cause

`@ConfigRoot(BUILD_TIME)` in the deployment module does not exempt the property from the RUN_TIME config validation pass. The runtime `@ConfigMapping` owns the `casehub.ledger.*` namespace at runtime and validates all keys under it against its declared methods. The deployment-module `BUILD_TIME` declaration is invisible at runtime.

### Fix

Declare the property in the runtime `@ConfigMapping` as well, even if no runtime code uses it. Add a clear Javadoc warning that the field exists only for validation purposes:

```java
// In LedgerConfig (runtime module)
interface ReactiveConfig {
    /**
     * Declared here only to satisfy Quarkus runtime config validation (SRCFG00050).
     * The authoritative source is LedgerBuildTimeConfig (deployment module).
     * Do NOT read this at runtime to gate CDI behaviour.
     */
    @WithDefault("false")
    boolean enabled();
}
```

### Why this is non-obvious

The `@ConfigRoot(BUILD_TIME)` annotation implies the property is a build-time concern — a developer reasonably expects it to be invisible at runtime. Quarkus documentation does not clearly state that a `BUILD_TIME` property under a prefix owned by a `RUN_TIME` mapping will still be validated at runtime by the runtime mapping. The error message `does not map to any root` points at SmallRye Config internals, not at the BUILD_TIME/RUN_TIME split.
