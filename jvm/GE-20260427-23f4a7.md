### Symptom
`RuntimeException: Scope predicate invocation failed: wrong number of arguments: 2 expected: 1` when a no-ctx predicate is registered as a scope filter and scope evaluation fires.

### Context
A reactive rule engine evaluates scope predicates via reflection. The existing `invokePredicate()` always prepends `ctx` as the first argument:

```java
Object[] args = new Object[facts.length + 1];
args[0] = ctx;
System.arraycopy(facts, 0, args, 1, facts.length);
return (Boolean) m.invoke(pred, args);
```

When a user registers a no-ctx predicate `Predicate1<String>` with method `test(String)` (1 param), the invocation passes `[ctx, "value"]` (2 args) — one too many.

### Root cause
The invocation code assumes all predicates are ctx-first. Adding ctx-optional overloads (where no-ctx predicates are stored unwrapped) breaks this assumption. The reflection call doesn't check whether the predicate's method signature actually includes a ctx parameter.

### Fix
Check the predicate method's parameter count against both forms and pass args accordingly:

```java
Method m = findTestMethod(pred);
Object[] args;
if (m.getParameterCount() == facts.length + 1) {
    // ctx-first: (ctx, fact1, fact2, ...)
    args = new Object[facts.length + 1];
    args[0] = ctx;
    System.arraycopy(facts, 0, args, 1, facts.length);
} else if (m.getParameterCount() == facts.length) {
    // no-ctx: (fact1, fact2, ...)
    args = facts;
} else {
    throw new IllegalArgumentException("arity mismatch");
}
return (Boolean) m.invoke(pred, args);
```

### Why this is non-obvious
The error appears at runtime during scope evaluation, not at the call site where the predicate is registered. The stack trace points to reflection invocation, not to where the wrong predicate was stored. Without knowing that `invokePredicate` assumes ctx-first, the fix isn't obvious from the symptom.
