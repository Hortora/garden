## Technique

In Quarkus, JPA entity listeners can be full CDI beans. Annotate the listener
class with `@ApplicationScoped` and Quarkus's Hibernate-CDI bridge handles
instantiation and injection automatically — no extra configuration needed.

```java
// The listener — a plain CDI bean
@ApplicationScoped
public class MyEntityListener {

    @Inject
    SomeService service;   // CDI injection works

    @PrePersist
    public void prePersist(Object entity) {
        if (!(entity instanceof MyEntity e)) return;
        service.doSomething(e);
    }
}

// The entity — register via @EntityListeners as normal
@Entity
@EntityListeners(MyEntityListener.class)
public class MyEntity { ... }
```

## Context

Works for any JPA lifecycle callback (`@PrePersist`, `@PostPersist`, `@PreUpdate`,
`@PostUpdate`, `@PreRemove`, `@PostRemove`, `@PostLoad`). The listener is
inherited by subclasses when placed on an abstract base entity — useful for
cross-cutting concerns like audit timestamps, trace ID population, or encryption.

## Why this is non-obvious

In standard JPA, entity listeners are instantiated by the JPA provider with no
injection support. Developers either avoid injection entirely (static utilities,
`CDI.current()` lookups) or write boilerplate `BeanManager` bridge code.
Quarkus's Hibernate ORM extension silently wires CDI into the listener lifecycle —
`@ApplicationScoped` just works without any additional setup.
