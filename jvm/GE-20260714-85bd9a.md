### Symptom
`UnsupportedOperationException` at runtime when calling `sort()`, `add()`, or any mutating method on the list returned by `scrollResult.getResultList()`.

### Context
When scrolling Qdrant points via `scrollAsync()`, the result's `getResultList()` returns a protobuf-generated `UnmodifiableList`. The Java type system shows `List<RetrievedPoint>` — no compile-time hint that mutation is forbidden.

### What was tried (didn't work)
Calling `points.sort(comparator)` directly on the returned list to sort by a payload field before selecting the oldest N points for deletion.

### Root cause
Protocol Buffers generates `Collections.unmodifiableList()` wrappers for repeated fields. The Qdrant Java client exposes these directly without copying. The `List` interface has no way to signal immutability at compile time.

### Fix
Copy to a mutable list before any mutating operation:
```java
var points = new ArrayList<>(scrollResult.getResultList());
points.sort((a, b) -> Long.compare(
    b.getPayloadOrDefault("_stored_at", ValueFactory.value(0L)).getIntegerValue(),
    a.getPayloadOrDefault("_stored_at", ValueFactory.value(0L)).getIntegerValue()));
```

### Why this is non-obvious
The returned type is `List<RetrievedPoint>` — indistinguishable from a regular mutable `ArrayList` at compile time. The exception only surfaces at runtime when you actually mutate. Easy to miss in code review since the `sort()` call looks correct.
