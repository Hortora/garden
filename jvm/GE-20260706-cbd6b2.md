### Symptom

`quarkus:dev` fails with `UnsatisfiedResolutionException` for beans that are provided by the full platform stack at deployment time but absent in dev mode (e.g. `@CrossTenant` qualified stores, registry SPIs).

### Context

In a multi-repo platform architecture, app modules depend on SPI interfaces whose implementations live in other repos. Production deployment composes all modules together. Dev mode runs a single app module â€” the SPI implementations are absent.

The test classpath already has `TestBeanProvider` with `@Mock` for `@QuarkusTest`, but `@Mock` beans aren't active outside test mode.

### Fix

Create a `@IfBuildProfile("dev")` producer that provides no-op implementations:

```java
@IfBuildProfile("dev")
@ApplicationScoped
public class DevBeanProvider {

    @Produces
    @CrossTenant
    @ApplicationScoped
    public CrossTenantMessageStore crossTenantMessageStore() {
        return new CrossTenantMessageStore() {
            @Override public List<Message> scan(MessageQuery q) { return List.of(); }
            @Override public long count(MessageQuery q) { return 0; }
            // ... other methods returning empty/zero
        };
    }

    @Produces
    @ApplicationScoped
    public ProvisionerConfigRegistry provisionerConfigRegistry() {
        return new ProvisionerConfigRegistry() {
            @Override public Map<String, Object> configFor(String p, String a) { return Map.of(); }
            @Override public Set<String> declaredAgentIds(String p) { return Set.of(); }
        };
    }
}
```

`@IfBuildProfile("dev")` ensures these beans are only active in dev mode â€” production uses the real implementations, test mode uses `@Mock`/`@InjectMock`.

### Why this is non-obvious

The natural instinct is to use `@DefaultBean` or `@Alternative`, but both are profile-agnostic â€” they'd also activate in production if the real bean isn't present. `@IfBuildProfile` is the correct Quarkus mechanism for dev-only beans, but it's less commonly used than `@Alternative` and not well-documented for this use case.
