### Symptom

`CaseMemoryStore.store()` or `storeAll()` called from an `@ObservesAsync` CDI handler silently fails — no exception visible to the caller (if the caller catches and swallows), no facts stored. The `InMemoryMemoryStore` (and all adapters) calls `MemoryPermissions.assertTenant(input.tenantId(), principal)` which reads `CurrentPrincipal.tenancyId()`. In the async observer thread, there is no request scope — `CurrentPrincipal` is either unavailable or returns a default that doesn't match the input's tenantId.

```java
@ApplicationScoped
public class MyEmitter {
    @Inject Instance<CaseMemoryStore> store;

    void onEvent(@ObservesAsync SomeEvent event) {
        var s = store.get();
        try {
            s.storeAll(facts);  // silently fails — assertTenant throws SecurityException
        } catch (Exception e) {
            LOG.warn("swallowed");  // this fires, but the caller sees no problem
        } finally {
            store.destroy(s);
        }
    }
}
```

### Context

CDI `@ObservesAsync` handlers run on a managed executor thread, not the request thread. Quarkus does not propagate request-scoped beans (like `CurrentPrincipal`) to async observer threads by default. The `InMemoryMemoryStore` is `@ApplicationScoped` with `CurrentPrincipal` injected via constructor — but the principal's request-scoped state is empty/default in the async context.

All CaseMemoryStore adapters (InMemory, JPA, SQLite, Mem0) call `MemoryPermissions.assertTenant()` as a security guard. The tenantId is already in the `MemoryInput` — the assertion is a double-check against the calling principal's tenant, but in async context there is no calling principal.

### What was tried (didn't work)

1. **Using the NoOp store in async paths** — NoOp works (returns "" from store()) but defeats the purpose of testing memory emission.
2. **Setting FixedCurrentPrincipal.tenancyId in @BeforeEach** — only affects the test thread, not the async observer thread.

### Root cause

`MemoryPermissions.assertTenant()` reads `CurrentPrincipal.tenancyId()` which requires an active request scope. `@ObservesAsync` runs outside request scope on the managed executor. The tenantId is available in the `MemoryInput` but the adapter doesn't trust it — it asserts against the principal.

### Fix

Three options (none shipped yet — tracked as devtown#67):

1. **SmallRye Context Propagation** — configure `@ObservesAsync` to propagate `CurrentPrincipal` to async threads. Platform-level fix.
2. **Accept tenantId from MemoryInput** — adapters already have `input.tenantId()`. Skip assertTenant when no request scope is active. Platform-level fix.
3. **Test-scoped store** — `@Alternative` CaseMemoryStore in test sources that skips tenant assertion. Test-only workaround.

### Why this is non-obvious

The failure is completely silent. The emitter's catch-all logs a WARN, but since memory is enrichment (not critical path), the application continues normally. Facts are simply never stored. The integration test times out waiting for facts that will never arrive — the timeout is the only signal, and it could be mistaken for a slow async chain rather than a security assertion failure.
