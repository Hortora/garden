### Symptom

`@QuarkusTest` tests fail during discovery with:

```
ClassSelector [className = '...Test', classLoader = null] resolution failed
Caused by: java.lang.ClassNotFoundException: io.quarkus.vertx.http.runtime.HttpBuildTimeConfig
```

In `quarkus:dev` mode, augmentation fails with the same `ClassNotFoundException`. The error does not mention Quinoa — it appears as a generic classloader/augmentation failure.

### Context

Quarkus 3.19 migrated config classes to `@ConfigMapping`. Legacy classes (`HttpBuildTimeConfig`, `HttpConfiguration`, `GlobalDevServicesConfig`) were kept as deprecated shims. Quarkus 3.26 removed them entirely. Quinoa versions prior to ~2.6 reference `HttpBuildTimeConfig` directly — when paired with Quarkus 3.26+, the class is absent and augmentation fails.

### What was tried (didn't work)

- Suspected CDI bean resolution issues — added `TestBeanProvider` with mock beans
- Suspected config validation — tried `quarkus.config.mapping.validate-unknown=false`
- Checked if the error was caused by the test class itself (it wasn't)

### Root cause

Quinoa 2.5.3 internally references `io.quarkus.vertx.http.runtime.HttpBuildTimeConfig`, which was removed in Quarkus 3.26. The Quarkiverse team updated all extensions to use `VertxHttpBuildTimeConfig` — Quinoa 2.8.x has the fix.

### Fix

Upgrade Quinoa to 2.8.3 (or latest):

```xml
<dependency>
  <groupId>io.quarkiverse.quinoa</groupId>
  <artifactId>quarkus-quinoa</artifactId>
  <version>2.8.3</version>
</dependency>
```

### Why this is non-obvious

The `ClassNotFoundException` names a Quarkus internal class, not Quinoa. The `classLoader = null` in the JUnit error further obscures the cause. Nothing in Quinoa's release notes or README warns about the minimum Quarkus version. The symptom appears during test discovery or augmentation — not during the Quinoa-specific build phase — making it look like a test infrastructure problem.

### Alternatives considered

- Downgrading Quarkus to 3.25 — rejected; platform standardised on 3.32.2
- Shading/providing the removed class — rejected; the proper fix is upgrading Quinoa
