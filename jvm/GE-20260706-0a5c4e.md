# drools-reliability session ID counter resets to 0 after TestableStorageManager.restart() — causes silent recovery failures

## Pain (3)

Recovery silently creates a new session instead of recovering the persisted one. No error is thrown — the `PersistedSessionOption.fromSession(savedId)` call succeeds, but returns a fresh empty session instead of the persisted one. The failure is discovered only when the recovered session doesn't contain expected data.

## Non-obvious (3)

Counter management is an internal concern that happens automatically in a real JVM restart (class reloading). `TestableStorageManager.restart()` simulates the storage restart but doesn't trigger the counter refresh. Drools' own test base classes handle this automatically, but it's not documented as a requirement for direct usage.

## Reuse (3)

Affects all developers writing drools-reliability tests that simulate JVM restart using `TestableStorageManager.restart()`.

## Problem

After calling `TestableStorageManager.restart()` in a test, `PersistedSessionOption.fromSession(savedId)` creates a fresh session instead of recovering the persisted one. No error is thrown.

## Root Cause

The session ID counter in `ReliableRuntimeComponentFactoryImpl` resets to 0 on restart. When creating a "recovered" session, the new session gets ID 0 (or 1), which doesn't match the `savedId` from before restart. The storage lookup uses the wrong ID and silently falls back to creating a new empty session.

## Solution

Call `ReliableRuntimeComponentFactoryImpl.refreshCounterUsingStorage()` after `TestableStorageManager.restart()` to reload the counter from persisted state:

```java
StorageManager storageManager = StorageManagerFactory.get().getStorageManager();
((TestableStorageManager) storageManager).restart();

// Required after restart to refresh session ID counter
ReliableRuntimeComponentFactoryImpl.refreshCounterUsingStorage();

// Now recovery works correctly
KieSession recovered = kieBase.newKieSession(
    PersistedSessionOption.fromSession(savedId)
        .withPersistenceStrategy(PersistenceStrategy.STORES_ONLY)
);
```

The counter refresh is automatic in a real JVM restart (class reloading), but must be called explicitly when using `TestableStorageManager.restart()` for test simulation.
