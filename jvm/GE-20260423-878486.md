## Symptom

`@QuarkusTest` classes in a module with `quarkus-langchain4j-jlama` fail immediately at test startup ā€” compile and `mvn install -DskipTests` both succeed, but running tests throws:

```
java.lang.IllegalStateException: Unsupported value type: [ALL-UNNAMED]
  at io.quarkus.bootstrap.json.Json.appendValue(Json.java:541)
  at io.quarkus.bootstrap.app.ApplicationModelSerializer.writeJson(...)
  at io.quarkus.bootstrap.BootstrapAppModelFactory.resolveAppModelForWorkspace(...)
```

The error message points to Quarkus bootstrap internals with no mention of Jlama or the Vector API.

## Context

Occurs when adding `quarkus-langchain4j-jlama 0.26.1` as a dependency to a Quarkus 3.32.2 project and running `@QuarkusTest` tests via Maven Surefire. Triggered on Java 21 and Java 26. A `mvn clean` before running does not help ā€” the issue reproduces on the first clean build.

## Root Cause

Jlama uses Java's Vector API (Panama) for pure-Java SIMD inference. Its Quarkus deployment extension adds JVM module opens arguments of the form `--add-opens java.base/java.nio=ALL-UNNAMED` to the test JVM. Quarkus 3.32.2's `ApplicationModelSerializer` serialises the application model to a JSON cache at test bootstrap time, but `Json.appendValue()` does not handle the `[ALL-UNNAMED]` module name ā€” it is treated as an unknown type and throws `IllegalStateException`.

The extension adds these opens unconditionally regardless of Java version, including Java 26 where the Vector API is a standard feature (JEP 469, graduated Java 23).

## Fix

No confirmed workaround found as of 2026-04-23. Approaches to investigate:

1. **Downgrade to quarkus-langchain4j-jlama 0.22.x** ā€” earlier versions may not add the problematic JVM args
2. **Run via `quarkus:test` goal** instead of Surefire ā€” may bypass app model serialisation
3. **Add opens to Surefire `<argLine>` manually** and check if jlama's extension can be configured to suppress automatic arg injection
4. **File against quarkus-langchain4j-jlama** ā€” the extension should check whether the JVM opens are already satisfied before adding `ALL-UNNAMED` opens

## Why Non-Obvious

`mvn compile`, `mvn install -DskipTests`, and `mvn test-compile` all succeed cleanly. The failure only appears at test runtime during Quarkus bootstrap. The error message ā€” `Unsupported value type: [ALL-UNNAMED]` in `Json.appendValue` ā€” gives no indication that Jlama or the Vector API is involved. `[ALL-UNNAMED]` is a Java module system value that most developers would not connect to an LLM inference engine.
