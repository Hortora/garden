### What

`ConcurrentHashMap.computeIfPresent(key, fn)` removes the map entry atomically when the function returns `null`. The function executes under the map's internal per-bucket lock, so the condition check and the entry removal are a single atomic operation — no separate lock or CAS loop required.

### Context

Replacing the two-step "remove item from collection value, then remove map entry if empty" pattern that has a TOCTOU race window (see GE-20260522-98b286 for the gotcha this technique resolves):

```java
// WRONG — TOCTOU between isEmpty() and remove()
list.remove(item);
if (list.isEmpty()) map.remove(key, list);
```

### Technique

```java
// CORRECT — atomic collapse of remove + prune under map segment lock
map.computeIfPresent(key, (k, list) -> {
    list.remove(item);
    return list.isEmpty() ? null : list;
});
```

The lambda:
1. Runs exclusively (per-bucket lock held for the duration)
2. Mutates the list value
3. Returns `null` → entry is atomically removed
4. Returns the list → entry is atomically kept

No window exists between the empty-check and the removal for a concurrent thread to add to the list.

### Why This Is Non-Obvious

The Javadoc for `computeIfPresent` focuses on the update-in-place use case. The `null`-return semantic — that it atomically removes the entry — is documented but easy to miss. Most developers encountering the TOCTOU race would reach for a `synchronized` block on the map or a per-key `ReentrantLock`, which adds significantly more complexity and contention surface. The `computeIfPresent`/`null` pattern is zero-overhead (same cost as the existing remove path), requires no auxiliary lock, and expresses the intent clearly in four lines.

### Why This Fix

`synchronized(map)` would work but serialises all operations on all keys globally. A `ConcurrentHashMap<K, ReentrantLock>` of per-key locks works but adds allocation and lookup overhead. `computeIfPresent` already holds the per-bucket lock — returning `null` is the natural way to express "conditionally remove" within that lock without widening the critical section.
