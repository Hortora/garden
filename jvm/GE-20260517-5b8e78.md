### What it is
`MessageService`, `CommitmentService`, `ChannelService`, and `ChannelGateway` are all `@ApplicationScoped` CDI beans and fully injectable in any Quarkus application that depends on `casehub-qhorus`. The qhorus documentation exclusively describes the MCP tool surface (`QhorusMcpTools`) — the Java CDI API is undocumented.

### Where it's documented
Not documented in the qhorus deep-dive, README, or DESIGN.md. Only discoverable by reading the source:

```java
// MessageService.java
@ApplicationScoped
public class MessageService { ... }

// CommitmentService.java
@ApplicationScoped
public class CommitmentService { ... }

// ChannelService.java
@ApplicationScoped
public class ChannelService { ... }

// ChannelGateway.java
@ApplicationScoped
public class ChannelGateway { ... }
```

### Usage pattern
```java
// In any Quarkus @ApplicationScoped service:
@Inject ChannelService channelService;
@Inject MessageService messageService;
@Inject CommitmentService commitmentService;

// Create a channel:
channelService.create("my/channel", "desc", ChannelSemantic.APPEND, null, null, null, null, null, "QUERY,COMMAND");

// Send a COMMAND (auto-opens Commitment if correlationId is non-null):
Channel ch = channelService.findByName("my/channel").orElseThrow();
messageService.send(ch.id, "sender", MessageType.COMMAND, content, correlationId, null, null, target, ActorType.SYSTEM);

// Fulfill a Commitment:
commitmentService.fulfill(correlationId);
```

### Why this matters
Server-side application code should use CDI injection — not the MCP tool wrappers (`QhorusMcpTools`) which are designed for LLM/MCP clients. A developer reading only the qhorus deep-dive would either try to call the MCP REST endpoints from Java code, or look for a separate "Java SDK" that doesn't exist.
