### Symptom

Quarkus startup fails with multiple `SRCFG00050` errors for every property under a `@ConfigMapping` prefix — even when `enabled=false` is set correctly and the intention is simply to disable a feature.

### Context

A `@ConfigMapping(prefix = "casehub.rag.expansion")` interface (`ExpansionConfig`) is injected into beans gated by `@IfBuildProperty(name = "casehub.rag.expansion.enabled", stringValue = "true")`. Setting `enabled=false` in `application.properties` correctly disables the beans, but ALL properties under the prefix (`enabled`, `mode`, `prompt-template`) now trigger `SRCFG00050: does not map to any root`.

### Root cause

Quarkus registers `@ConfigMapping` interfaces only when they are reachable from an active CDI bean's injection graph. When `@IfBuildProperty` eliminates every bean that injects the `@ConfigMapping`, the mapping is never registered with SmallRye Config. The properties remain in `application.properties` but now have no registered root — SmallRye strict validation treats them as orphans and fails.

The key insight: `@IfBuildProperty` is transitive. Disabling a bean also disables its entire dependency graph at augmentation time, including `@ConfigMapping` registrations.

### Fix

Comment out or remove the properties from `application.properties` entirely — setting `enabled=false` is not enough when the mapping itself disappears:

```properties
# DISABLED — commenting out prevents SRCFG00050 when beans are inactive
#casehub.rag.expansion.enabled=true
#casehub.rag.expansion.mode=session
#casehub.rag.expansion.prompt-template=Question: %s
```

### Why this is non-obvious

The developer's mental model is "set `enabled=false` to turn off the feature." The properties still describe the feature's configuration — they just aren't active. The expectation is that an inactive feature's config is harmless. Instead, the mapping registration is coupled to bean activation, and SmallRye's strict validation makes orphan properties fatal. The error message (`does not map to any root`) gives no hint that the root cause is `@IfBuildProperty` eliminating the mapping's consumer.
