## Podman on macOS requires explicit DOCKER_HOST for Testcontainers

**ID:** GE-20260520-c0e5b4
**Stack:** Podman 5.x, macOS arm64, Testcontainers 2.x

### Symptom

Testcontainers (and any tool relying on a Docker socket) reports "no valid Docker
environment" and fails immediately, even though Podman is installed and running.

```
ERROR Could not find a valid Docker environment.
UnixSocketClientProviderStrategy: failed with exception InvalidConfigurationException
(Could not find unix domain socket). Root cause NoSuchFileException (/var/run/docker.sock)
```

### Context

Fresh Podman installation on macOS without running the `podman-mac-helper` installer.
Podman's default machine configuration does not create `/var/run/docker.sock`. The
system helper service must be explicitly installed to create that socket, but the
installer step is optional and easy to skip.

### What was tried (didn't work)

- Verifying `podman machine list` — machine shows as available
- Running `podman info` — Podman reports healthy
- Checking `/var/run/docker.sock` — does not exist

### Root cause

Podman on macOS forwards the Docker API through a machine-specific socket at a path
inside `/var/folders/.../T/podman/`. This path is printed by `podman machine start`
but is not the well-known `/var/run/docker.sock`. Testcontainers only knows about
the standard path unless told otherwise.

### Fix

**Step 1 — Start the Podman machine:**
```bash
podman machine start
```
The output prints the API socket path:
```
API forwarding listening on: /var/folders/8m/.../T/podman/podman-machine-default-api.sock
...
export DOCKER_HOST='unix:///var/folders/8m/.../T/podman/podman-machine-default-api.sock'
```

**Step 2 — Export the socket path before running tests:**
```bash
export DOCKER_HOST='unix:///var/folders/<path>/podman/podman-machine-default-api.sock'
```

**Step 3 — Run tests with the env var set:**
```bash
DOCKER_HOST=$DOCKER_HOST TESTCONTAINERS_RYUK_DISABLED=true mvn test -pl <module>
```

Alternatively, install `podman-mac-helper` to create `/var/run/docker.sock` permanently:
```bash
sudo /opt/homebrew/Cellar/podman/5.8.2/bin/podman-mac-helper install
podman machine stop && podman machine start
```

### Why this is non-obvious

The error message mentions Docker and `/var/run/docker.sock` — making it look like a
Docker installation problem rather than a Podman socket routing issue. `podman machine start`
does print the correct `export DOCKER_HOST=...` command, but this is easy to miss in the
startup output, and Testcontainers provides no hint that `DOCKER_HOST` is the fix.

### Why this fix

Setting `DOCKER_HOST` is the zero-install path — useful in CI or when root access is
unavailable for the helper installer. The `podman-mac-helper` approach is the permanent
solution for local development if root access is available.
