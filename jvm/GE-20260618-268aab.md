### Technique

For testing code that owns a subprocess and reads its stdout via a blocking `readLine()` loop, start `/bin/cat` as the subprocess and write controlled lines to its stdin. `cat` echoes stdin to stdout. Closing stdin triggers EOF, causing `readLine()` to return `null` and the loop to exit naturally with exit code 0.

```java
Process catProc = new ProcessBuilder("cat").start();
MySubprocessReader reader = new MySubprocessReader(catProc);

// Write controlled JSON lines to stdin
try (PrintWriter writer = new PrintWriter(catProc.getOutputStream())) {
    writer.println("{\"type\":\"assistant\",\"message\":{\"content\":[{\"type\":\"text\",\"text\":\"Hello\"}]}}");
    writer.println("{\"type\":\"result\",\"subtype\":\"success\",\"duration_ms\":0,\"is_error\":false,\"num_turns\":1}");
}
// Closing writer triggers EOF on cat's stdin → cat exits with code 0 → stdout closes
// The readLine() loop sees null and terminates naturally

List<AgentEvent> events = reader.stream()
        .collect().asList()
        .await().atMost(Duration.ofSeconds(5));

assertThat(events).hasSize(1);
assertThat(((AgentEvent.TextDelta) events.get(0)).text()).isEqualTo("Hello");
```

For testing `destroyForcibly()` concurrent with an active read:

```java
Process catProc = new ProcessBuilder("cat").start();  // cat blocks waiting for stdin
MySubprocessReader reader = new MySubprocessReader(catProc);

Thread streamThread = new Thread(() -> {
    try {
        reader.stream().collect().asList().await().atMost(Duration.ofSeconds(10));
    } catch (Exception e) {
        caught.set(e);  // AgentProcessException from IOException
    } finally {
        latch.countDown();
    }
});
streamThread.setDaemon(true);
streamThread.start();

Thread.sleep(50);  // let the thread block on readLine()
reader.destroyForcibly();  // closes stdoutReader → readLine() throws IOException

assertThat(latch.await(2, SECONDS)).isTrue();
assertThat(caught.get()).isInstanceOf(AgentProcessException.class);
```

### Why This Is Non-Obvious

Most developers mock at the interface level (e.g. inject a `MockAgentProvider`) or write a fake shell script as the subprocess. `/bin/cat` eliminates the mocking layer entirely — the real `BufferedReader`, the real `readLine()` blocking semantics, and the real pipe EOF mechanics all run, testing the actual production code path. It also tests `destroyForcibly()` + `closeQuietly(stdoutReader)` correctly triggering `IOException` in the blocked read thread, which interface-level mocks cannot cover.
