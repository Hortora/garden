### Symptom
Adding a `LOG.warn(...)` call to a service class fails to compile with "cannot find symbol: variable LOG" even though the companion reactive service has a Logger. Searching the class for `Logger` finds nothing.

### Context
In a codebase where a blocking service (`MessageService`) and its reactive mirror (`ReactiveMessageService`) coexist, developers often assume the Logger is inherited or available by convention. In the qhorus project, `ReactiveMessageService` had `private static final Logger LOG = Logger.getLogger(ReactiveMessageService.class)` at line 71. `MessageService` had none.

### Root cause
Java loggers are not inherited. Each class that needs logging must declare its own static Logger field. Quarkus provides JBoss Logging by default; there is no automatic injection or inheritance mechanism.

### Fix
Add the field at the top of the class body, before the first `@Inject`:

```java
import org.jboss.logging.Logger;

@ApplicationScoped
public class MessageService {

    private static final Logger LOG = Logger.getLogger(MessageService.class);

    @Inject
    ChannelService channelService;
    // ...
}
```

### Why this is non-obvious
When two sibling service classes exist — one with a Logger, one without — developers assume the one without still has access to the Logger (via CDI, inheritance, or a shared parent class). In Quarkus `@ApplicationScoped` beans there is no shared parent in scope; each CDI bean needs its own Logger declaration.
