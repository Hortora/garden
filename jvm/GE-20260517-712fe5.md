### What it does

Captures both sync (`@Observes`) and async (`@ObservesAsync`) CDI events fired during `@QuarkusTest` tests, with proper concurrency handling.

### Why non-obvious

Most developers reach for mocking (`@InjectMock`) or `ArgumentCaptor`, neither of which works for CDI event firing. The CDI container fires events to all registered observers — you need a real observer bean in the test container that records what it receives. The async case additionally requires waiting for off-thread delivery, which `CountDownLatch` handles correctly.

### Implementation

Create a **top-level** `@ApplicationScoped` class in test sources (not an inner static class — Jandex scanning for inner CDI beans is unreliable in Quarkus tests):

```java
@ApplicationScoped
public class MyEventCapture {

    private final List<MyEvent> syncEvents = new CopyOnWriteArrayList<>();
    private volatile MyEvent lastAsyncEvent;
    private volatile CountDownLatch asyncLatch = new CountDownLatch(1);

    void onSync(@Observes final MyEvent event) {
        syncEvents.add(event);
    }

    CompletionStage<Void> onAsync(@ObservesAsync final MyEvent event) {
        lastAsyncEvent = event;
        asyncLatch.countDown();
        return CompletableFuture.completedFuture(null);
    }

    public List<MyEvent> syncEvents() { return syncEvents; }
    public MyEvent lastAsyncEvent() { return lastAsyncEvent; }
    public CountDownLatch asyncLatch() { return asyncLatch; }

    public void reset() {
        syncEvents.clear();
        lastAsyncEvent = null;
        asyncLatch = new CountDownLatch(1);  // create new — can't reset a latched latch
    }
}
```

In the test:
```java
@Inject MyEventCapture capture;

@BeforeEach void setUp() { capture.reset(); }

@Test void asyncEventFired() throws Exception {
    // ... trigger the event ...
    boolean received = capture.asyncLatch().await(5, TimeUnit.SECONDS);
    assertThat(received).isTrue();
    assertThat(capture.lastAsyncEvent().entryId()).isEqualTo(expected);
}
```

### Key details

- `CopyOnWriteArrayList` for sync events — CDI observer delivery can be multi-threaded
- `volatile` on the async field — guarantees visibility across threads
- **Create a new `CountDownLatch` on reset** — you cannot reset a `CountDownLatch` that has already been counted down; `new CountDownLatch(1)` is the only way
- **Top-level class** — Quarkus/Jandex sometimes misses inner static CDI beans; always use a top-level class in test sources
