## Context

A risk classifier maps planned actions to gate decisions. When context is missing (null, empty, or insufficient to evaluate), the classifier must fail closed — return a `GateRequired` — rather than allow the action autonomously. The fail-closed path must construct a `GateRequired` with all required metadata: reason, reversible, candidateGroups, expiresIn, scope.

## Technique

All gate metadata in the fail-closed path must come from the domain action type's own attributes, not from defaults or constants:

```java
// ❌ BAD — hardcoded reversible=true regardless of action type's actual policy
private RiskDecision.GateRequired missingContext(AmlActionType type) {
    return new RiskDecision.GateRequired(
        "Risk assessment unavailable",
        true,                          // hardcoded — wrong for SAR_FILING, LAW_ENFORCEMENT_REFERRAL
        List.of(AmlGroups.AML_COMPLIANCE),  // hardcoded — wrong for MLRO-gated types
        null,
        "casehubio/aml/oversight");
}

// ✅ GOOD — all metadata derived from the type
private RiskDecision.GateRequired missingContext(AmlActionType type) {
    return new RiskDecision.GateRequired(
        "Risk assessment unavailable — human review required",
        type.reversible(),             // honours the type's actual reversibility policy
        type.candidateGroups(),        // routes to the type's designated reviewers
        type.expiresIn(),              // honours the type's SLA
        type.scope());                 // honours the type's scope
}
```

The domain enum is the single source of truth for gate policy. The fail-closed path is just a path through the same policy — not a separate policy.

## Why this is non-obvious

The fail-closed path is written once to "be safe" — the natural instinct is to pick reasonable defaults: `reversible=true` (so reviewers can undo), `candidateGroups=aml-compliance` (the general compliance group). But "reasonable defaults" diverge from the domain type's actual policy. A `SAR_FILING` that is `reversible=false` by design (SAR filing cannot be recalled) produces an incorrect audit trail if the fail-closed gate claims `reversible=true`. The reviewer sees a gate that says the action is undoable when it is not. The bug passes tests because the test checks that a gate fires, not that the gate's metadata matches the type's policy.

The type enum already encodes the correct policy for every action type. The fail-closed path should use it, not bypass it.
