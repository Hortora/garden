### Symptom

Attempting to subscribe to dynamically-named Vert.x event bus addresses (e.g. `casehub.context.changed.{panelName}`) using `@ConsumeEvent` per-name fails at design time — the annotation requires a string literal and is resolved at build time by Quarkus Arc.

### Context

Any design that publishes events to per-name addresses (e.g. panel-scoped change events, per-tenant channels, per-entity topics) and wants subscribers to filter by name.

### What was tried (didn't work)

Having separate `@ConsumeEvent("casehub.context.changed.extracted")` and `@ConsumeEvent("casehub.context.changed.inferences")` handlers for user-defined panel names declared at runtime — these are unknown at compile time so the handlers cannot be defined.

### Root cause

`@ConsumeEvent(value = "...")` is a compile-time Quarkus Arc annotation. The value must be a string literal or a constant expression. There is no runtime-dynamic equivalent.

### Fix

Use a **single `@ConsumeEvent` handler on the base address** and encode the specific sub-address as a **field in the event payload**. Subscribers filter on the field:

```java
// Event record carries changedPanel
public record CaseContextChangedEvent(CaseInstance instance, CaseContext snapshot, String changedPanel) {}

// Single handler on the base address
@ConsumeEvent("casehub.context.changed")
public Uni<Void> onContextChanged(CaseContextChangedEvent event) {
    String changedPanel = event.changedPanel();
    if ("episodic".equals(changedPanel)) return Uni.createFrom().voidItem(); // skip
    // listenPanel filtering:
    if (cct.getListenPanel() != null && !cct.getListenPanel().equals(changedPanel)) continue;
    // ...
}

// Panel-scoped addresses can still be published for external consumers
// (Claudony, monitoring) via eventBus.publish(panelChanged(panel), event)
// but the primary handler uses field-based filtering, not per-address subscription.
```

### Why this is non-obvious

The Vert.x event bus natively supports dynamic address subscription at runtime (`eventBus.consumer("dynamic.address", handler)`), and SmallRye documentation covers this for CDI-free code. Developers familiar with the reactive event bus assume `@ConsumeEvent` provides the same capability — it does not. The Quarkus Arc annotation model resolves consumers at build time and there is no equivalent `@ConsumeEvent(computed = true)` or runtime registration variant via annotation.
