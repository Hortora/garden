## Symptom

Compile error `cannot find symbol: method empty()` after copying the test construction
example from the `SlaBreachContext` Javadoc.

## Context

Writing a `@QuarkusTest` that constructs a `SlaBreachContext` directly — following the
official Javadoc guidance for unit tests.

## What Was Tried

```java
var ctx = new SlaBreachContext(BreachType.CLAIM_EXPIRED, task,
                               Path.root(), MapPreferences.empty());
```

Copied verbatim from `SlaBreachContext` Javadoc:
> "In unit tests, construct with `Path.root()` and `MapPreferences.empty()` — both are
> zero-dependency and require no mocking."

## Root Cause

`MapPreferences` has exactly one constructor — `new MapPreferences(Map<String, Object>)`.
No `empty()` static factory method exists. The Javadoc is wrong.

## Fix

```java
var ctx = new SlaBreachContext(BreachType.CLAIM_EXPIRED, task,
                               Path.root(), new MapPreferences(Map.of()));
```

## Why This Is Non-Obvious

The official API Javadoc explicitly names the non-existent method. There is no deprecation
notice, no compiler suggestion, no IDE quick-fix — just a compile error on code you copied
directly from the documentation. Nothing in the error message points back to the Javadoc
being wrong.
