## Technique

When an `@ApplicationScoped` infrastructure bean must call a method that exists only on
a concrete `@Alternative` implementation (not on the SPI interface), inject the concrete
type via `@Any Instance<ConcreteType>` and guard every call with `isUnsatisfied()`.

```java
@ApplicationScoped
public class LedgerLifecycleAdapter {

    @Inject @Any
    Instance<InMemoryLedgerEntryRepository> memoryLedger;

    void onGameStop(@Observes GameStopped event) {
        if (!memoryLedger.isUnsatisfied()) {
            memoryLedger.get().clear();   // clear() is not on LedgerEntryRepository SPI
        }
    }
}
```

In profiles where the alternative is selected (e.g. `%test`, `%mock`, `%emulated` —
all with `quarkus.arc.selected-alternatives=...InMemoryLedgerEntryRepository,...`),
`isUnsatisfied()` returns `false` and `get()` resolves the in-memory bean.

In profiles where the alternative is NOT selected (e.g. `%sc2` using JPA),
`isUnsatisfied()` returns `true` and the block is a no-op — no exception is thrown,
no conditional `@IfBuildProfile` annotation is needed, and production code never
imports or references the concrete memory type directly.

## Why this is non-obvious

The natural alternatives are:
1. Inject the SPI (`LedgerEntryRepository`) and cast — fails if JPA impl doesn't implement `clear()`
2. Add `clear()` to the SPI — forces all implementors to implement it
3. Use `@IfBuildProfile` on the adapter — couples the adapter to specific profile names
4. Null-check the field — wrong in CDI (CDI-managed fields are never null at runtime; null only
   when constructed with `new` in plain unit tests)

`@Any Instance<ConcreteType>` is the CDI-idiomatic way to make an injection point optional:
`@Any` bypasses qualifier matching (accepts the bean regardless of qualifiers), and
`Instance<T>.isUnsatisfied()` tests at runtime whether any bean is available —
returning `true` when the alternative is not selected for the current profile.

## Context where it was applied

`LedgerLifecycleAdapter` in quarkmind observes `GameStopped` and clears
`InMemoryLedgerEntryRepository` to prevent unbounded memory growth across game sessions.
`clear()` is not on `LedgerEntryRepository` SPI (casehub-ledger). Injecting the concrete
type via `@Any Instance<>` made the adapter a no-op in `%sc2` (JPA persistence, no memory
bean) without any profile-conditional annotation or build flag.
