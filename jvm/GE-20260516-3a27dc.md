## Symptom

Adding a Maven profile that selects tests by `<groups>report</groups>` produces:

```
[INFO] Tests run: 0, Failures: 0, Errors: 0, Skipped: 0
[INFO] BUILD SUCCESS
```

No error, no warning. The profile appears to activate, tests appear to pass, but none of the tagged tests were executed.

## Context

You add a profile to run tests with a specific JUnit 5 `@Tag`:

```xml
<profile>
    <id>report</id>
    <build>
        <plugins>
            <plugin>
                <artifactId>maven-surefire-plugin</artifactId>
                <configuration>
                    <groups>report</groups>
                    <excludedGroups/>
                </configuration>
            </plugin>
        </plugins>
    </build>
</profile>
```

The default surefire configuration (outside `<profiles>`) sets `<excludedGroups>benchmark,browser,report</excludedGroups>`. Without `combine.self="override"`, Maven merges the profile's `<configuration>` into the parent — `excludedGroups` from the parent wins. The `report` tag is excluded before `groups` can select it.

## What was tried

Setting `<excludedGroups/>` (empty) inside the profile config — this does not override the parent's `excludedGroups`; it merges and the parent value survives.

## Root cause

Maven's `<configuration>` element uses child-level merge semantics by default. Profile plugin configurations are merged with the parent plugin configuration, not replaced. The parent's `<excludedGroups>` wins because the profile's empty element is interpreted as "no override" rather than "set to empty".

## Fix

Add `combine.self="override"` to the profile's `<configuration>` element to force replacement:

```xml
<profile>
    <id>report</id>
    <build>
        <plugins>
            <plugin>
                <artifactId>maven-surefire-plugin</artifactId>
                <configuration combine.self="override">
                    <groups>report</groups>
                    <argLine>@{argLine}</argLine>
                    <systemPropertyVariables>
                        <java.util.logging.manager>org.jboss.logmanager.LogManager</java.util.logging.manager>
                        <maven.home>${maven.home}</maven.home>
                    </systemPropertyVariables>
                </configuration>
            </plugin>
        </plugins>
    </build>
</profile>
```

Note: when using `combine.self="override"`, include all required surefire fields (argLine, systemPropertyVariables) since the entire config is replaced. Copy from an existing working profile (e.g. `benchmark`).

## Why this is non-obvious

`Tests run: 0, BUILD SUCCESS` looks like the tests passed. There is no warning that `excludedGroups` from the parent is silently overriding the profile's `groups` selector. The `<excludedGroups/>` override attempt looks reasonable — setting it to empty should clear it — but Maven's merge semantics treat it as a no-op.
