## Context

A CDI `@ApplicationScoped` bean has `@Inject` constructor dependencies, a field-injected
dependency, and a `@PostConstruct` method that uses the field. You want to unit-test the bean's
core logic without a Quarkus container ‚Äî pure Mockito, fast, no augmentation.

The challenge: field injection happens after construction; the `@PostConstruct` method references
the field-injected dependency and will NPE if called with null.

## Technique

**Step 1 ‚Äî Move field-injected dependency to constructor parameter.**

All dependencies that the bean uses should be constructor-injected. This makes the dependency
explicit and removes the `@PostConstruct` fragility:

```java
@ApplicationScoped
public class ChannelAgentDispatcher {

    private final DebateAgentProvider provider;
    private final MessageService messageService;
    private final Iterable<ChannelAgentHandler> handlers;
    private final InstanceService instanceService;  // was field-injected

    @Inject  // CDI calls this ‚Äî @Any qualifier on Instance<T> parameter works here
    ChannelAgentDispatcher(DebateAgentProvider provider,
                           MessageService messageService,
                           @Any Instance<ChannelAgentHandler> handlers,
                           InstanceService instanceService) {
        // delegates to test constructor
        this(provider, messageService, (Iterable<ChannelAgentHandler>) handlers, instanceService);
    }

    // Package-private ‚Äî accessible from same-package test; invisible to CDI (not annotated)
    ChannelAgentDispatcher(DebateAgentProvider provider,
                           MessageService messageService,
                           Iterable<ChannelAgentHandler> handlers,
                           InstanceService instanceService) {
        this.provider = provider;
        this.messageService = messageService;
        this.handlers = handlers;
        this.instanceService = instanceService;
    }
```

**Step 2 ‚Äî Guard `@PostConstruct` against the null test path.**

Tests pass `null` for `instanceService` (since `@PostConstruct` is never invoked outside CDI).
Add a null guard so the method is safe on the test constructor path:

```java
    @PostConstruct
    void registerSenderInstance() {
        if (instanceService == null) return;  // test constructor path ‚Äî @PostConstruct never called by CDI
        instanceService.register(SENDER_ID, "description", List.of("capability"));
    }
```

**Step 3 ‚Äî Test uses the package-private constructor directly.**

```java
@ExtendWith(MockitoExtension.class)
class ChannelAgentDispatcherTest {

    @Mock DebateAgentProvider provider;
    @Mock MessageService messageService;

    ChannelAgentDispatcher dispatcher;

    @BeforeEach
    void setUp() {
        // Direct construction ‚Äî no CDI, no container, @PostConstruct not called
        dispatcher = new ChannelAgentDispatcher(
                provider, messageService, List.of(handler), null);  // null = instanceService
    }

    @Test
    void dispatches_to_matching_handler() { ... }
}
```

## Why this is non-obvious

Developers reach for `@InjectMock`, `@QuarkusTest`, or Weld SE to test CDI beans. The same-package
constructor trick bypasses CDI entirely: CDI `@Inject` annotations are just metadata ‚Äî they don't
prevent direct construction from within the package. The null guard pattern makes the CDI path
safe without complicating the test path.

## Constraints

- Test class must be in the same package as the production class (or a sub-package won't work ‚Äî
  package-private is package-scoped only)
- `@PostConstruct` must not be essential to the logic under test ‚Äî it is legitimately skipped
- Works for Quarkus 3.x CDI (Arc); standard Jakarta CDI should behave the same

## Alternatives considered

- `@InjectMock` with `@QuarkusTest` ‚Äî starts the full Quarkus container; slow, fragile under CDI
  augmentation issues
- Weld SE / CDI SE for unit testing ‚Äî more correct but heavyweight and requires additional test dep
- Reflection to set fields after construction ‚Äî verbose, brittle under refactoring
- Making all fields public or protected ‚Äî pollutes the production API
