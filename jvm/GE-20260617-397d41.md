## Symptom

Maven cannot resolve `io.quarkus:quarkus-rest-client-reactive` — `Could not find artifact` or dependency resolution failure. Or the artifact resolves at a very old version but is missing from the Quarkus 3.x BOM.

## Context

Adding the JAX-RS SSE client runtime for `ClientBuilder.newClient()` and `SseEventSource`:

```xml
<!-- WRONG — artifact does not exist in Quarkus 3.x -->
<dependency>
  <groupId>io.quarkus</groupId>
  <artifactId>quarkus-rest-client-reactive</artifactId>
  <scope>test</scope>
</dependency>
```

## Root cause

In the Quarkus 3.0 consolidation, the RESTEasy Reactive artifacts were renamed:
- Server: `quarkus-resteasy-reactive` → `quarkus-rest`
- Client: `quarkus-resteasy-reactive-client` → `quarkus-rest-client`

`quarkus-rest-client-reactive` was a name that existed during the transition period and is absent from the Quarkus 3.x BOM. Documentation and blog posts from the transition period mention both names.

The server-only artifact (`quarkus-rest`) does NOT include the JAX-RS client runtime — `ClientBuilder.newClient()` will throw at runtime with "No ClientBuilder implementation found" if only the server artifact is present.

## Fix

```xml
<!-- CORRECT — provides ClientBuilder + SseEventSource for Quarkus 3.x -->
<dependency>
  <groupId>io.quarkus</groupId>
  <artifactId>quarkus-rest-client</artifactId>
  <scope>test</scope>
</dependency>
```

No version needed — managed by the Quarkus BOM.

## Why this is non-obvious

The Quarkus docs and many tutorials still show the old name or mix old/new naming. The word "reactive" feels like it should be in the artifact ID for a Reactive REST client. The server is `quarkus-rest`; the client is `quarkus-rest-client` — and neither has `-reactive` in the name despite both using RESTEasy Reactive under the hood.
