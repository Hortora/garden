### What exists but isn't documented

`FaultPolicyEngine` in casehub-desiredstate-runtime injects `List<FaultPolicy>` (a plain Java List, not `@All Instance<FaultPolicy>`). Quarkus Arc does not auto-collect CDI beans into plain `List<>` injection points — it requires a `@Produces` method that wraps `Instance<FaultPolicy>`.

### How it was discovered

@QuarkusTest bootstrap failed with UnsatisfiedResolutionException for `List<FaultPolicy>` even though a `@DefaultBean @ApplicationScoped` FaultPolicy implementation was present. Adding the `@DefaultBean` satisfied `Instance<FaultPolicy>` lookups but not the plain `List<>` injection point.

### Fix

Add a CDI producer that bridges `Instance<FaultPolicy>` to `List<FaultPolicy>`:

```java
@ApplicationScoped
public class FaultPolicyListProducer {
    @Inject
    Instance<FaultPolicy> policies;

    @Produces
    @ApplicationScoped
    List<FaultPolicy> faultPolicies() {
        return policies.stream().toList();
    }
}
```

### Why this is non-obvious

CDI spec says `Instance<T>` is always satisfiable for any bean type. But `List<T>` is NOT automatically populated from discovered beans in Quarkus Arc — it's an explicit injection point that needs a producer. This differs from Spring's `List<T>` auto-collection behaviour, which many developers expect.
