## Context

The Graphiti OSS REST server (`getzep/graphiti`). The HTTP API is described in the README as
"poorly documented" (see getzep/graphiti#169).

## What exists

`DELETE /episode/{uuid}` exists in `server/graph_service/routers/ingest.py` and works:

```
@router.delete('/episode/{uuid}', status_code=status.HTTP_200_OK)
async def delete_episode(uuid: str, graphiti: ZepGraphitiDep):
    await graphiti.delete_episodic_node(uuid)
    return Result(message='Episode deleted', success=True)
```

## The partial deletion caveat

`delete_episodic_node()` deletes the `EpisodicNode` (the source episode record) but does NOT
cascade to `EntityEdge` or `EntityNode` records that were derived from that episode by LLM
extraction. Relationship facts extracted from the episode remain in the graph. This is a known
upstream issue (getzep/graphiti#1083 — entities without `MENTIONS` relationships are not cleaned
up on episode deletion).

**GDPR implication:** For Art.17 compliance requiring complete erasure of all data derived from
a stored episode, `DELETE /episode/{uuid}` alone is insufficient. Use `DELETE /group/{group_id}`
for full cascading deletion of all data for an entity.

## Why undocumented

The HTTP API docs don't mention this endpoint at all. Found only by reading
`server/graph_service/routers/ingest.py` source. The partial-deletion behaviour is only
described in the upstream bug report, not in any documentation.
