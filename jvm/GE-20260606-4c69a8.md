## What exists but isn't documented

`@Retention(RetentionPolicy.RUNTIME)` annotations on local enum classes declared inside
`@Test` methods are preserved in the compiled class file and readable via reflection at
runtime. This enables inline test vocabulary enums that carry annotation metadata without
requiring top-level test class declarations.

## Context

When building a registry that auto-discovers classes via `@SomeAnnotation`, tests often
need ad-hoc annotated types to exercise error paths or edge cases. Local enums (JEP 395,
Java 16+) satisfy this without polluting the test file's type namespace.

## Working example

```java
@Test
void register_missing_annotation_throws() {
    // Local enum — no @VocabularyMetadata
    enum NoMeta implements VocabularyTerm {
        TERM("t", "T", List.of());
        // ... accessor implementations
    }
    assertThatThrownBy(() -> registry.register(NoMeta.class))
        .isInstanceOf(IllegalArgumentException.class);
}

@Test
void register_annotated_local_enum_works() {
    @VocabularyMetadata(uri = "urn:test:source", name = "Source", version = "1.0")
    enum Source implements VocabularyTerm {
        ALPHA("alpha", "Alpha", List.of());
        // ...
    }
    // Annotation readable at runtime — register() reads it via reflection:
    // Source.class.getAnnotation(VocabularyMetadata.class).uri() → "urn:test:source"
    assertThatNoException().isThrownBy(() -> registry.register(Source.class));
}
```

## Why this works

JEP 395 (Java 16) legalised local classes in static contexts (including `static void` test
methods). Annotation retention applies to the class file based on the annotation's
`@Retention` policy, regardless of where the class is declared. The compiled local enum
class carries the annotation in its `.class` bytecode and the JVM exposes it via
`Class.getAnnotation()` at runtime.

## Discovery

Not mentioned in JEP 395 documentation or in any Java annotation reference. Discovered by
attempting to use a local enum as a test fixture for a registration method that required
`@VocabularyMetadata` — it worked without any workaround.
