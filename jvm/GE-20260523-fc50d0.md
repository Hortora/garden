### Symptom
An ACL check using capability tags and role entries works for registered internal senders (who have tags in the registry) but silently blocks all external/unregistered senders (A2A agents, human backends) because their tag lookup returns empty — even if a `role:agent` entry in the ACL should match them.

### Context
`AllowedWritersPolicy.isAllowedWriter(sender, allowedWriters, Supplier<List<String>>)` was designed for senders registered in the Qhorus instance registry (with real capability tags). When the same policy was applied to external A2A senders (unregistered, but with a known `ActorType`), capability/role ACL entries never matched them — their tag list was always empty.

### Root cause
The ACL check is generic: it compares ACL entries starting with `capability:` or `role:` against the sender's tag list. Registered senders have tags; external senders don't. The supplier is the only per-caller customisation point.

### Fix
Provide a **unified lazy supplier** that combines both sources:

```java
() -> {
    List<String> tags = new ArrayList<>(
        instanceService.findCapabilityTagsForInstance(sender));
    tags.add("role:" + dispatch.actorType().name().toLowerCase());
    return tags;
}
```

This gives:
- **Registered Qhorus instance** (AGENT with `capability:analysis`): `["capability:analysis", "role:agent"]` — capability and role ACL entries both work
- **Unregistered A2A agent** (`ActorType.AGENT`, not in registry): `["role:agent"]` — role entries work, capability entries block (correct: external agents have no attested capabilities)
- **Human sender** (`ActorType.HUMAN`): `["role:human"]`

The supplier is lazy — only called if the ACL string contains a `capability:` or `role:` entry. The registry lookup only fires when needed.

### Why this is non-obvious
The natural instinct is to provide different suppliers for different caller types (registered vs external), requiring the ACL method to know which caller type it's dealing with. The unified supplier is cleaner: it adds a synthetic `role:` tag derived from `ActorType` that works for all callers, keeping the ACL method completely generic. The synthesis step is transparent — registered senders just get one extra tag that role-ACL entries can also match.
