## @ConfigMapping interface methods without Javadoc cause a compile error — not a runtime warning

### Symptom

Build fails with a compile error when a `@ConfigMapping` interface has methods without Javadoc. The error message does not obviously identify "missing Javadoc" as the cause — it may reference the config documentation processor or annotation processing.

### What was tried (didn't work)

- Checking for missing annotations — annotations are correct
- Verifying the config property names — names are valid

### Root cause

Quarkus's config documentation processor runs at compile time and requires Javadoc on every method in a `@ConfigMapping` interface — including simple `boolean enabled()` accessors and group-returning methods. A missing Javadoc on any method causes an annotation processing compile error.

### Fix

Every method in a `@ConfigMapping` interface must have a Javadoc comment:

```java
@ConfigMapping(prefix = "casehub.work")
public interface WorkConfig {
    /** Whether the work module is enabled. */
    boolean enabled();

    /** SLA configuration. */
    SlaConfig sla();

    interface SlaConfig {
        /** Default deadline in hours. */
        int defaultDeadlineHours();
    }
}
```

### Why this is non-obvious

Javadoc is optional everywhere else in Java — there is no compiler enforcement. The error message from the annotation processor does not clearly state "add Javadoc to method X". Developers typically look for missing annotations or wrong configuration keys, not missing documentation.
