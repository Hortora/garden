## Symptom

A REST Assured test asserting `.body("$.findAll { it.domain == 'tools' }", not(empty()))` passes
even when the response body contains no entries matching the predicate — the feature under test
was absent.

## Context

Writing TDD tests for a new multi-domain filter on a JSON array endpoint. New tests added before
implementing the feature should fail (RED phase). Instead they all passed immediately, masking
the missing implementation.

## Root Cause

REST Assured's GPath evaluator returns `null` (not an empty `List`) when `$.findAll { }` finds
no matches in the response body. Hamcrest's `empty()` matcher expects a `Collection` — when
applied to `null` it does not match, so `not(empty())` evaluates to `true` on null. The
assertion silently passes.

## Fix

Use `hasSize(greaterThan(0))` or `hasSize(N)` to assert non-emptiness. These fail correctly
when the value is null (ClassCastException path is avoided because REST Assured passes null
directly to the matcher, and `hasSize` on null fails as expected).

```java
// Wrong — silently passes when findAll returns null (no matches)
.body("$.findAll { it.domain == 'tools' }", not(empty()))

// Correct — fails when null or empty
.body("$.findAll { it.domain == 'tools' }", hasSize(greaterThan(0)))

// Or assert an exact expected count
.body("$.findAll { it.domain == 'tools' }", hasSize(2))
```

The complement for "must be empty" is already safe:
```java
// This correctly handles null (existing documented pattern)
.body("$.findAll { it.domain == 'tools' }", anyOf(nullValue(), hasSize(0)))
```

## Why This Is Non-obvious

The asymmetry between the "must be empty" and "must be non-empty" assertions is invisible
until a test passes when it should fail. REST Assured documentation does not describe GPath
null return semantics for `findAll`. Hamcrest's `not(empty())` on null is technically a
misuse that silently produces an incorrect result rather than throwing — there is no runtime
error to alert you.

The broken test is indistinguishable from a passing test. The only signal is the TDD
red-green cycle failing silently.
