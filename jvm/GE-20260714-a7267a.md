## Quarkus ArC Instance<NamedStrategy> does not discover beans implementing transitive subtypes

### Symptom

`EngineStrategyResolver` injects `@Any Instance<NamedStrategy>` to discover all strategy beans. Beans that implement `NamedStrategy` transitively (e.g. `ContinuationPolicy implements ClaimSlaPolicy extends NamedStrategy`) are NOT found. The resolver logs `Available: []` for the strategy type, even though the bean is `@ApplicationScoped @Unremovable` with a Jandex index.

### Context

The bean is discoverable by CDI for direct typed injection (`@Inject ContinuationPolicy policy` works). Only `Instance<NamedStrategy>` fails to include it. Tested with:
- Bean in an external JAR with embedded Jandex index
- Bean local to the application module (same source tree)
- `quarkus.index-dependency` force-indexing the JAR
- `@Unremovable` annotation on the bean

None of these made the bean visible to `Instance<NamedStrategy>`.

### Root cause

Quarkus ArC's build-time bean type resolution for `Instance<T>` injection points appears not to include beans whose assignability to `T` requires traversing intermediate interfaces. The CDI spec says bean types include all supertypes and interfaces, but ArC's build-time pruning may not perform the full transitive check for `Instance<>` resolution.

### Fix

Inject the concrete bean type directly and register it programmatically at startup:

```java
@Unremovable
@ApplicationScoped
public class WorkCoreStrategyRegistrar {

    @Inject EngineStrategyResolver strategyResolver;
    @Inject ContinuationPolicy continuationPolicy;
    @Inject LeastLoadedStrategy leastLoadedStrategy;

    void onStartup(@Observes @Priority(1) StartupEvent ev) {
        strategyResolver.registerEntry(continuationPolicy, false);
        strategyResolver.registerEntry(leastLoadedStrategy, false);
    }
}
```

`@Priority(1)` ensures registration happens before other `StartupEvent` observers that use the resolver.

### Why this is non-obvious

CDI developers expect `Instance<T>` to find all beans assignable to `T`, including through interface inheritance chains. The bean IS a valid CDI bean (direct injection works), and it IS in the Jandex index — the failure is specifically in `Instance<T>` resolution for transitive supertypes. No error or warning is logged; the bean is silently absent from the `Instance` iteration.
