### Symptom
Tests asserting that recent positive attestations outweigh old negative ones
pass — but the score is 0.5 (tie), not > 0.5. The recency weighting is not
being exercised at all.

### Context
Any trust scoring system where decay is computed from `attestation.occurredAt`
(not `decision.occurredAt`). Test fixtures set the decision timestamp but leave
`attestation.occurredAt` null.

### Root cause
The decay algorithm uses `attestation.occurredAt` for the age calculation:

```java
final Instant attestationTime = attestation.occurredAt != null
    ? attestation.occurredAt : now;
final long ageInDays = Duration.between(attestationTime, now).toDays();
```

When `attestation.occurredAt` is null, age defaults to 0 for **every**
attestation — recent and old alike. All attestations get identical decay
weight. The "recent vs old" distinction the test is trying to verify never
fires.

The test passes because 0.5 satisfies neither `isGreaterThan(0.5)` nor
`isLessThan(0.5)` — but the assertion `isGreaterThan(0.5)` fails silently
only when both weights are equal. A test that expects a directional score
difference will give a false negative (fail) rather than a false positive.

### Fix
Set `attestation.occurredAt` in the fixture to match the intended age:

```java
final LedgerAttestation recentSound = attestation(recent.id, SOUND);
recentSound.occurredAt = now.minus(1, ChronoUnit.DAYS);   // ← required

final LedgerAttestation oldFlagged = attestation(old.id, FLAGGED);
oldFlagged.occurredAt = now.minus(180, ChronoUnit.DAYS);  // ← required
```

### Why this is non-obvious
Developers naturally set `decision.occurredAt` to represent when the decision
happened, then assume the decay algorithm uses that timestamp. Reading the
algorithm to find it uses `attestation.occurredAt` requires knowing to look —
the test appears to be testing recency correctly because the decisions have
different timestamps. The `@PrePersist` hook that sets `occurredAt = now` if
null only fires during persistence; in pure unit tests (no Quarkus context),
`@PrePersist` never runs and `occurredAt` stays null throughout.
