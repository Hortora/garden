### What exists but isn't documented
`BuildTimeEnabledProcessor` in `quarkus-arc-deployment` scans the full `CombinedIndexBuildItem` when evaluating `@IfBuildProperty` conditions. This combined index includes external JARs discovered via built-in Jandex index (`META-INF/jandex.idx`) or `quarkus.index-dependency`. There is no special handling for `@Decorator` classes — all bean types are treated uniformly.

### Why this matters
The Quarkus CDI reference documents `@IfBuildProperty` for general bean enablement but does not explicitly confirm it works for:
1. Beans from external indexed dependencies (vs application classes)
2. `@Decorator` annotated classes specifically

When debugging a decorator that wasn't intercepting from an external JAR, "Arc skips `@IfBuildProperty` for external JARs" was a plausible hypothesis that consumed investigation time. The actual root cause was unrelated (producer method beans — see GE-20260706-a4d5b0).

### How it works (from source)
The `ifBuildProperty` build step calls `getAnnotations()` which invokes `index.getAnnotationsWithRepeatable(annotationName, index)` on the `ComputingIndex` obtained from `CombinedIndexBuildItem`. For each annotated class, it creates a `BuildTimeConditionBuildItem(target, enabled)`. If disabled, an `AnnotationsTransformerBuildItem` adds `@Vetoed` to the class. If enabled, no transformation — the class proceeds through normal bean discovery.

### How to verify
Enable DEBUG logging for `BuildTimeEnabledProcessor`:
```properties
quarkus.log.category."io.quarkus.arc.deployment.BuildTimeEnabledProcessor".level=DEBUG
```
Look for "Enabling" or "Disabling" messages for your decorator class name. If "Enabling" appears, `@IfBuildProperty` is working — the problem is elsewhere.

### Why this is non-obvious
The Quarkus docs describe `@IfBuildProperty` in the context of general beans. When a decorator from an external JAR doesn't intercept, it's natural to suspect the build-time condition isn't being evaluated for that class. The docs don't confirm the combined index scope, and without reading the source, you can't rule it out.
