## Transitive hibernate-reactive-panache on classpath causes H2 test startup failure — disable reactive datasource in test config

### Symptom

`@QuarkusTest` suite fails to start entirely with a datasource configuration error. The module does not use reactive persistence directly. Removing an unrelated dependency fixes the startup failure.

### What was tried (didn't work)

- Removing explicit reactive dependencies from the module's pom — problem persists because the dependency is transitive
- Configuring the reactive datasource URL for H2 — H2 has no native reactive driver; this path has no fix

### Root cause

If any transitive dependency pulls `hibernate-reactive-panache` onto the classpath, Quarkus detects it and attempts to configure a reactive datasource — even when the module itself doesn't use reactive persistence. H2 has no native reactive driver, so startup fails with a datasource configuration error.

### Fix

Add this to `src/test/resources/application.properties`:

```properties
%test.quarkus.datasource.reactive=false
```

This disables reactive datasource configuration for the test profile only, without affecting production configuration.

### Why this is non-obvious

The module doesn't use reactive persistence — the failure appears unrelated to reactive at all. The root cause is a transitive classpath entry, not the module's own dependencies. The error message points to datasource configuration, not to the reactive extension, making the connection to `hibernate-reactive-panache` non-obvious.
