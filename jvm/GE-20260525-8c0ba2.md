### Context
A compiled evaluation engine (`ReteCompiledEngine`) is shared across multiple `UnitInstance`s. Each unit contributes DataStore adapters that subscribe to `PropagatingDataStore` instances via `ContextRouterAdapter`. The engine deduplicates subscriptions: one adapter per unique DataStore, regardless of how many units share it. On `disposeUnit()`, the unit must be removed from the Router without unsubscribing adapters that other live units still need.

The obvious approach — tracking which DataStores each unit subscribed to in a `Map<UnitInstance, List<DataStore>>` — adds a per-unit data structure and requires explicit registration at `createUnit()` time.

### The technique
Use a reference count (`AtomicInteger`, one per DataStore) alongside the existing adapter map. At `disposeUnit()`, recover the DataStore instances by re-applying the accessor functions to the unit's original CTX rather than looking them up from a stored list:

```java
// createUnit() — increment refcount for each DataStore
for (Map.Entry<Object, Integer> entry : sourceToSlot.entrySet()) {
    Object ds = ((Function1<CTX, DataSource<?>>) entry.getKey()).apply(ctx);
    if (ds instanceof PropagatingDataStore<?> pds) {
        if (!adapters.containsKey(pds)) {
            ContextRouterAdapter adapter = new ContextRouterAdapter<>(entry.getValue(), router);
            pds.subscribe(adapter);
            adapters.put(pds, adapter);
            adapterRefCounts.put(pds, new AtomicInteger(1));
        } else {
            adapterRefCounts.get(pds).incrementAndGet();
        }
    }
}

// disposeUnit() — re-apply accessors, decrement, unsubscribe on zero
CTX ctx = unit.getContext().context();
for (Object accessor : sourceToSlot.keySet()) {
    Object ds = ((Function1<CTX, DataSource<?>>) accessor).apply(ctx);
    if (ds instanceof PropagatingDataStore<?> pds) {
        AtomicInteger count = adapterRefCounts.get(pds);
        // count is null only if disposeUnit() called without a matching createUnit()
        if (count != null && count.decrementAndGet() == 0) {
            pds.unsubscribe(adapters.remove(pds));
            adapterRefCounts.remove(pds);
        }
    }
}
```

### Why re-application works
The `CTX` type is a Java record — all component fields are `final`. Re-applying an accessor (`CTX::persons`) to the same record instance always returns the same DataStore object as at `createUnit()` time. No additional bookkeeping is needed: the CTX itself is the lookup table.

### Why this is non-obvious
The instinct is to store "what this unit subscribed to" at creation time in a per-unit list. The re-application approach inverts this: instead of storing the result, re-derive it from the source (the CTX) at dispose time. This only works because Java records guarantee field finality — it would be unsafe with mutable CTX types. Most developers wouldn't think to use the record's immutability as a substitute for an explicit reverse mapping.
