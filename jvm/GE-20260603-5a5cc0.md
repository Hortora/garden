### Symptom
`PreferenceProvider.resolve(scope)` returns a `Preferences` instance where all `get(key)` calls return null (the default value from PreferenceKey). No error, no warning. All trust routing blend factors and quality floors silently fall back to defaults.

### Context
`casehub-platform-config` ships `ConfigFilePreferenceProvider @ApplicationScoped` which reads YAML files declared in `casehub.platform.config.files`. If the YAML file is absent from the classpath, resolve() returns empty preferences — not an error. Additionally, in `@QuarkusTest`, test resources override main resources. If the YAML file exists only in `src/main/resources/` but not in `src/test/resources/`, tests see empty preferences even when the main config is correct.

### Root cause
`ConfigFilePreferenceProvider` treats missing files as "no configuration for this scope" rather than "configuration error". This is arguably intentional (scopes without config are valid), but when the file should exist and doesn't, the failure is silent.

### Fix
1. Create the YAML file before adding the dependency — even an empty `entries: []` is sufficient
2. Copy the YAML file to `src/test/resources/` at the same path for `@QuarkusTest` environments

### Why this is non-obvious
The expectation is that a missing config file would produce a startup warning or a resolution error. Instead, all preferences resolve to their coded defaults, which may be functionally correct but semantically wrong (e.g., blend factor 0.0 instead of the intended 0.70).
