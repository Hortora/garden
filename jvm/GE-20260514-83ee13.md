### Symptom
`cannot find symbol: class DefaultBean` at compile time when using
`import jakarta.enterprise.inject.DefaultBean`.

### Context
Using Quarkus ARC's `@DefaultBean` annotation to mark SPI fallback implementations
that should yield to consumer-provided beans automatically — the platform CDI pattern
for "use this unless something better is wired in."

### What was tried
`import jakarta.enterprise.inject.DefaultBean` — mirrors `jakarta.enterprise.inject.Alternative`,
which IS the correct package for `@Alternative`. Natural assumption: both live in the same
jakarta.enterprise.inject package.

### Root cause
`@DefaultBean` is NOT a standard CDI 4.x annotation. It is a Quarkus Arc extension annotation
defined in `io.quarkus.arc.DefaultBean`. The CDI spec has no equivalent — Arc invented it.
`jakarta.enterprise.inject.DefaultBean` does not exist.

### Fix
```java
import io.quarkus.arc.DefaultBean;  // NOT jakarta.enterprise.inject.DefaultBean
```

No additional dependency is needed. `io.quarkus.arc:arc` is always on compile scope in any
Quarkus project — it is a transitive dependency of Quarkus core and cannot be absent. Confirmed
via `mvn dependency:list -pl <module>` showing `io.quarkus.arc:arc:jar:3.32.2:compile`.

### Why this is non-obvious
`@Alternative` — the annotation `@DefaultBean` replaces semantically — lives in
`jakarta.enterprise.inject`. The expectation is that its counterpart does too.
`@DefaultBean` does not appear in any CDI 4.x spec document; it only surfaces by reading
Arc source code or knowing to search inside `quarkus-arc-*.jar`.
