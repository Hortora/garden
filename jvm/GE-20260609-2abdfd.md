# Qdrant Java client 1.9.x lacks Query API — prefetch + fusion requires client ≥1.10.0

## Symptom

`QueryPoints`, `PrefetchQuery`, `Fusion`, and `Rrf` classes do not exist — compilation fails with "cannot resolve symbol" errors. IDE shows these types as unavailable in `io.qdrant.client.grpc.Points`.

## Context

Implementing hybrid search with prefetch + RRF fusion using the Qdrant Java gRPC client. The Qdrant documentation describes the Query API as a core feature for combining multiple retrieval strategies.

## What was tried

- Verified imports — correct package path for other Points types that do exist
- Checked if the classes were in a different package — not found anywhere in client 1.9.1
- Searched Qdrant docs for version requirements — not clearly documented

## Root cause

The Query API (QueryPoints, prefetch, fusion) was introduced in **Qdrant server 1.10** and the **Java client 1.10.0**. Client 1.9.x does not have these protobuf types at all because they were added to the gRPC spec in a later version.

The Qdrant server and Java client version numbers track each other for API features — breaking or additive changes to the gRPC interface require corresponding client updates.

## Fix

Bump `io.qdrant:client` to **≥1.10.0**. Recommend using the latest stable client (currently 1.18.1) to get all Query API features and bug fixes:

```xml
<dependency>
    <groupId>io.qdrant</groupId>
    <artifactId>client</artifactId>
    <version>1.18.1</version>
</dependency>
```

The client version must match or exceed the Qdrant server feature version you're targeting.

## Why non-obvious

No compile-time warning or runtime error explains the version requirement — the classes simply don't exist in older clients. The Qdrant documentation shows the Query API as a feature of Qdrant without noting:

1. Which Qdrant server version introduced it
2. Which Java client version added support
3. That older clients can't use it at all

Most developers discover this only when imports fail, then have to cross-reference server release notes and client changelogs to find the version boundary.
