# quarkus.security.auth.enabled-in-dev-mode=false disables @RolesAllowed enforcement in dev — allows OIDC service to run without a real OIDC server

### Context
Running a Quarkus service with `quarkus-oidc` and `@RolesAllowed` annotations in dev mode
without a real OIDC server (no DevServices, no Keycloak). All guarded endpoints return 401
for every request.

### Technique
Set in `application.properties`:
```properties
%dev.quarkus.security.auth.enabled-in-dev-mode=false
%dev.quarkus.oidc.enabled=false
%dev.quarkus.keycloak.devservices.enabled=false
```

`quarkus.security.auth.enabled-in-dev-mode=false` activates `DevModeDisabledAuthorizationController`
(`@Alternative @Priority @Singleton` in `quarkus-security-runtime-spi`), which returns `false`
from `isAuthorizationEnabled()`. Both `EagerSecurityHandler` (REST path — checks at line 53
before processing the request) and `StandardSecurityCheckInterceptor` (CDI path — checks via
`controller.isAuthorizationEnabled()`) bail out early. `@RolesAllowed` is not enforced.

`quarkus.oidc.enabled=false` is also required — without it, OIDC intercepts requests before
the authorization check fires, returning 401 for missing tokens even with auth disabled.

### Why this is non-obvious
The natural dev-mode approaches are: (a) spin up Keycloak DevServices (60–90s cold start,
heavyweight), or (b) configure a real OIDC server. `quarkus.security.auth.enabled-in-dev-mode`
is not mentioned in the main OIDC documentation; it appears only in `quarkus-security-runtime-spi`
source as the activation mechanism for `DevModeDisabledAuthorizationController`. The interceptor
mechanism is two-level — disabling OIDC alone is insufficient because the authorization
interceptor fires independently of the authentication mechanism.

### Why this fix
Over disabling DevServices alone: `quarkus.keycloak.devservices.enabled=false` without the
auth flag still returns 401 — the security framework enforces `@RolesAllowed` against anonymous
identity regardless of whether an auth mechanism is registered. The auth flag is the only clean
way to bypass the enforcement layer itself.
