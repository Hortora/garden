## Symptom

`QhorusMcpTools.sendMessage(channel, sender, "handoff", content, correlationId, null, null, null, null)` throws `IllegalArgumentException` at runtime. No message about which parameter is invalid.

## Context

Writing integration tests for A2A delegation scenarios. All other message types (QUERY, COMMAND, RESPONSE, STATUS, etc.) accept `null` for the `target` argument without error. Only HANDOFF validates and rejects it.

## Root cause

HANDOFF semantics require a recipient — the `target` field encodes where the obligation is being delegated (e.g. `"role:specialist"`). `MessageService.send()` enforces non-null target for HANDOFF at the server-side safety net level. Other message types have no such constraint.

## Fix

Pass a non-null `target` string when sending HANDOFF messages:

```java
// FAILS — null target
tools.sendMessage(channel, "agent", "handoff", "delegating", correlationId,
        null, null, null, null);

// WORKS — provide the delegate target
tools.sendMessage(channel, "agent", "handoff", "delegating to specialist", correlationId,
        null, null, "role:specialist", null);
```

The `target` parameter is the 8th positional argument in `sendMessage`.

## Why this is non-obvious

Every other message type silently accepts `null` for target — it's not part of the method signature's obvious semantics. The error message does not name which parameter failed, making it appear to be a content or channel validation error.
