### Context

Calibrating building construction times (in game loops) from SC2 replay data. You need exact `T_real = completionLoop - startLoop` per building type.

### What developers typically try

Applying the same approach as unit training calibration: pair GAME_EVENTS build commands (by abilLink or selection) with tracker completion events. This requires matching command events to construction events across two event streams, with the same noise sources as training calibration (missed commands, selection gaps, simultaneous builds).

The paired-stream range-bounded modal approach (see `GE-20260522-a575c3`) is needed for unit training because the completion event (`UnitBorn`) doesn't record when construction started — only when it finished. The start time comes from a command event in a different stream.

### The technique

Building construction is captured with exact loop precision in **both** tracker event streams:

- `ID_UNIT_INIT` event: fires when construction begins. `event.getLoop()` = exact start loop.
- `ID_UNIT_DONE` event: fires when construction completes. `event.getLoop()` = exact end loop.

Both events carry the same unit tag (`tagIndex` + `tagRecycle`). Match them by tag:

```java
// On ID_UNIT_INIT:
String tag = makeTag(event.getUnitTagIndex(), event.getUnitTagRecycle());
initByTag.put(tag, rawEvent.getLoop());  // int

// On ID_UNIT_DONE:
Integer tagIndex   = rawEvent.get("unitTagIndex");
Integer tagRecycle = rawEvent.get("unitTagRecycle");
String tag = makeTag(tagIndex, tagRecycle);
Integer initLoop = initByTag.remove(tag);
int tReal = rawEvent.getLoop() - initLoop;
```

`T_real` is exact — no range-bounding or modal required unless you want to aggregate across replays for confidence. For buildings, every pair produces the identical value (SC2 stores times as exact integers).

### Why non-obvious

The GAME_EVENTS approach is the first thing you reach for because training calibration uses it. Unit training doesn't have a paired tracker event for start time — only for completion. Buildings do. `UnitInit` is the building-specific event that signals construction start; there is no equivalent for units. Nothing in the scelight documentation surfaces this asymmetry.
