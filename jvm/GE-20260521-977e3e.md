## What it is

In a Quarkus extension deployment module, a `@BuildStep` method that returns
`void` and takes no `BuildProducer<?>` parameters is treated as a graph orphan
by the Quarkus build framework and may be elided — it silently does not run.

This means a side-effect-only `@BuildStep` (e.g. one that logs a validation
warning) will appear to work in some build configurations but not others,
depending on whether Quarkus decides it has downstream consumers.

## How to use it

Annotate the void step with `@Produce(ArtifactResultBuildItem.class)` from
`io.quarkus.deployment.annotations`. This declares the step as a contributor
to artifact building — a phase Quarkus always executes — guaranteeing the step
runs regardless of what downstream steps consume.

```java
import io.quarkus.deployment.annotations.BuildStep;
import io.quarkus.deployment.annotations.Produce;
import io.quarkus.deployment.pkg.builditem.ArtifactResultBuildItem;

@BuildStep
@Produce(ArtifactResultBuildItem.class)
void validateFlywayMigrationLocation(FlywayBuildTimeConfig config) {
    // Guaranteed to run — anchored to artifact build phase.
    // Without @Produce, this step may be silently skipped.
    boolean hasLedgerPath = config.datasources().values().stream()...;
    if (!hasLedgerPath) {
        LOG.warn("...");
    }
}
```

## Where this is not documented

The Quarkus extension guide covers `@BuildStep` methods that produce and consume
build items via method signatures. The mechanism for forcing a void step into the
graph is not mentioned. `@Produce` appears only in internal Quarkus source code
(`FlywayProcessor`, `HibernateOrmProcessor`) without commentary on why it is used
on void methods. Discoverable only by reading extension source.

## Constraints

- `ArtifactResultBuildItem` must be imported from `io.quarkus.deployment.pkg.builditem`
  (not `io.quarkus.deployment.builditem` — the latter does not exist).
- The `@Produce` annotation is a declaration only — the method does not need to
  actually produce an `ArtifactResultBuildItem` instance.
