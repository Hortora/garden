## @QuarkusTest classes named *IT.java silently report 0 tests — maven-failsafe collects them instead of surefire

### Symptom

`Tests run: 0, Failures: 0, Errors: 0, Skipped: 0` — no failure signal, no tests executed. The `@QuarkusTest` class exists and is syntactically correct but is never run.

### What was tried (didn't work)

- Verifying the class has `@QuarkusTest` annotation — it does
- Checking for missing test methods — they exist

### Root cause

`*IT.java` files are collected by `maven-failsafe-plugin` (integration test phase), not `maven-surefire-plugin` (test phase). When a `@QuarkusTest` class is named `WorkItemServiceIT.java`, failsafe picks it up and runs it in isolation — reporting 0 tests with no failure signal because failsafe expects a packaged artifact and `@QuarkusTest` requires the embedded Quarkus dev container.

### Fix

All `@QuarkusTest` classes must end in `Test.java`. Reserve the `IT.java` suffix strictly for `@QuarkusIntegrationTest` classes that test a packaged artifact:

```
// Wrong — collected by failsafe, reports 0 tests
WorkItemServiceIT.java

// Right — collected by surefire, runs normally
WorkItemServiceTest.java
```

### Why this is non-obvious

"IT" naturally reads as "Integration Test" and many developers use it for any test that spins up an application container. The 0-tests result with no error is indistinguishable from a misconfigured test runner. Nothing in the output points to the naming convention as the cause.
