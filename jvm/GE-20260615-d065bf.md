### Symptom

Bean A's `@PostConstruct` sets a shared global (e.g. `Path.setDefaultParser()`). Bean B's `@PostConstruct` calls code that depends on that global. Works in some environments, fails silently or intermittently in others because the order in which Quarkus initialises `@Startup` beans is not defined.

### Context

Any Quarkus application with two or more `@Startup @ApplicationScoped` beans where one bean's `@PostConstruct` produces a side effect (global state, static field, registry population) that another bean's `@PostConstruct` consumes.

### Root cause

CDI specifies no initialization ordering between two `@Startup @ApplicationScoped` beans. The order is implementation-defined and can vary between test and production environments. `@Startup` only guarantees that all such beans initialise before the first non-startup request — not relative to each other.

### Fix

Each `@Startup` bean reads `@ConfigProperty` directly rather than depending on another bean's `@PostConstruct` side effect. Pass the config value explicitly to helpers that need it.

```java
// BAD — depends on PathParserConfigurator.@PostConstruct having set the default parser
Path parser = Path.getDefaultParser(); // may be null or default at this point

// GOOD — read the same config key directly; CDI injects @ConfigProperty fields
// before @PostConstruct, regardless of @Startup ordering between beans
@ConfigProperty(name = "casehub.platform.path.separator", defaultValue = "/")
String pathSeparator;

@PostConstruct
void load() {
    PathParser parser = PathParser.of(pathSeparator); // explicit, no cross-bean dependency
    // ...
}
```

`@ConfigProperty` fields are always injected before `@PostConstruct` runs — that guarantee is per-bean, not cross-bean, and is reliable.

### Why this is non-obvious

`@Startup` implies "at startup" — developers assume multiple `@Startup` beans initialise in a predictable order (e.g. dependency graph or declaration order). The CDI spec guarantee is only that they initialise before the first non-startup request, not relative to each other. The failure is intermittent and environment-dependent, making it hard to reproduce reliably.
