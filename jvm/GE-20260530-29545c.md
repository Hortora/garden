## Symptom

`@QuarkusTest` fails to load the test class at all:

```
java.lang.TypeNotPresentException: Type io.quarkus.deployment.dev.devservices.GlobalDevServicesConfig$Enabled not present
    at io.quarkiverse.wiremock.devservice.WireMockServerProcessor
```

Test discovery reports `ClassSelector resolution failed` — 0 tests run, build fails.

## Context

Adding the Quarkiverse WireMock extension (`io.quarkiverse.wiremock:quarkus-wiremock:1.4.1`,
`quarkus-wiremock-deployment:1.4.1`, `quarkus-wiremock-test:1.4.1`) to a `@QuarkusTest`
module running on Quarkus 3.32.2. The extension's `WireMockServerProcessor` deployment
step references `GlobalDevServicesConfig$Enabled`, an internal Quarkus class that was
removed or relocated in 3.32.x.

## What was tried (didn't work)

- Using `@ConnectWireMock` from `quarkus-wiremock-test:1.4.1` — triggers the broken processor during augmentation
- Searching Quarkiverse WireMock documentation for a Quarkus version compatibility matrix — none exists

## Root cause

`WireMockServerProcessor` (in `quarkus-wiremock-deployment`) is annotated with
`@BuildStep(onlyIf = GlobalDevServicesConfig.Enabled.class)`. The class
`io.quarkus.deployment.dev.devservices.GlobalDevServicesConfig$Enabled` no longer
exists in Quarkus 3.32.2's deployment module. The JVM throws `TypeNotPresentException`
when loading the annotation proxy — before any test code runs.

## Fix

Drop the Quarkiverse extension entirely. Use raw WireMock via `QuarkusTestResourceLifecycleManager`
(see GE-20260526-286ac7):

```xml
<dependency>
    <groupId>org.wiremock</groupId>
    <artifactId>wiremock</artifactId>
    <version>3.13.0</version>
    <scope>test</scope>
</dependency>
```

```java
public class MyWireMockResource implements QuarkusTestResourceLifecycleManager {

    static volatile WireMockServer INSTANCE;
    private WireMockServer server;

    @Override
    public Map<String, String> start() {
        server = new WireMockServer(WireMockConfiguration.options().dynamicPort());
        server.start();
        INSTANCE = server;
        // inject the port into whatever REST client config key you need
        return Map.of("quarkus.rest-client.my-client.url",
                      "http://localhost:" + server.port());
    }

    @Override
    public void stop() {
        if (server != null) server.stop();
    }
}
```

In the test class:

```java
@QuarkusTest
@QuarkusTestResource(MyWireMockResource.class)
class MyTest {
    @BeforeEach
    void reset() { MyWireMockResource.INSTANCE.resetAll(); }
}
```

## Why this is non-obvious

The exception names an internal Quarkus class (`GlobalDevServicesConfig$Enabled`), not the
WireMock extension. Nothing in the error output mentions `quarkus-wiremock` or version
incompatibility. A developer hitting this will search for `GlobalDevServicesConfig$Enabled`,
find nothing useful, and spend significant time before realising the source is the
Quarkiverse WireMock DevService processor.

The `QuarkusTestResourceLifecycleManager` fix is also non-obvious in its own right —
see GE-20260526-286ac7 for why `@BeforeAll`/`@BeforeEach` doesn't work as an alternative.
