### What happened
`String.matches(regex)` was used to classify actor IDs in a security-sensitive resolver. The regex `[\\w-]+:[\\w-]+@[\\w.]+` was intended to match versioned persona IDs (e.g. `claude:analyst@v1`) but **not** plain email addresses (e.g. `alice@example.com`). It worked correctly — but only because `matches()` anchors the full string. A developer familiar with regex from Python or JavaScript would expect substring matching and might not realise why the email correctly returns false.

### Why this is non-obvious
In Python, `re.search(pattern, string)` finds the pattern anywhere in the string. In JavaScript, `regex.test(string)` also searches for a substring match by default. Java's `String.matches(regex)` behaves differently: it is equivalent to `Pattern.matches(regex, str)`, which checks whether the **entire** string matches the pattern — implicit `^...$`. There is no substring search unless you add `.*` at both ends.

A developer who writes a classifier expecting `"alice@example.com".matches("[\\w-]+:[\\w-]+@[\\w.]+")` to return `true` (finding the `@[\\w.]+` portion) will be silently wrong — the method returns `false` because the full string has no colon before the `@`.

### Root cause
`String.matches(regex)` is documented as equivalent to `Pattern.matches(regex, str)`, which anchors. The Javadoc is accurate but the method name `matches` does not signal "full-string match" to developers whose regex intuition comes from other languages.

### Fix
Use `matches()` knowingly as a full-string check. If substring search is needed, use `Pattern.compile(regex).matcher(str).find()` instead.

For classification logic where full-string anchoring is the correct semantics (e.g. type resolvers, format validators), `matches()` is correct as-is — just document the anchor assumption explicitly in the regex comment.

```java
// Full-string match — anchored implicitly. "alice@example.com" does NOT match.
if (actorId.matches("[\\w-]+:[\\w-]+@[\\w.]+")) return ActorType.AGENT;

// Substring search — use find() instead of matches():
Pattern.compile("[\\w-]+:[\\w-]+@[\\w.]+").matcher(actorId).find()
```
