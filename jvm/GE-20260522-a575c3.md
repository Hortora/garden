## Context

Calibrating timing constants (e.g. unit training duration in game loops) from a simulation against ground-truth replay data. You have two event streams: (A) commands with timestamps and (B) completions with timestamps. You want to find T such that `completion_loop - command_loop = T` for correctly paired events.

## What developers typically try

FIFO matching: sort commands by loop, sort completions by loop, pair i-th command with i-th completion. Simple but breaks when commands are missing (selection-tracking gaps in AbilityMapping) or when events are out of sync (multiple buildings training simultaneously, queued units).

Closest-previous-command: for each completion, find the most recent command before it. Fails for queued training where the matching command may be much earlier than the nearest command.

## The technique

Range-bounded modal algorithm:

1. Collect all command loops `C = [c0, c1, ...]` and completion loops `B = [b0, b1, ...]`
2. For each (bi, cj) pair where `bi - cj` falls in the expected range `[T_min, T_max]` (derived from a rough estimate ± slack), increment a frequency counter for that diff value
3. The modal (most frequent) diff value is T_real

```java
Map<Integer, Integer> freq = new TreeMap<>();
for (int born : borns) {
    for (int cmd : cmds) {
        int diff = born - cmd;
        if (diff >= tMin && diff <= tMax) {
            freq.merge(diff, 1, Integer::sum);
        }
    }
}
int tReal = freq.entrySet().stream()
    .max(Map.Entry.comparingByValue())
    .orElseThrow().getKey();
```

The modal value is robust because:
- Non-queued correct pairs cluster at exactly T_real (high frequency)
- Queued pairs give T_real + k×T_other (fall outside range or are non-modal)
- Missed commands produce random large diffs (non-modal)
- Cross-unit-type false pairs produce random diffs (non-modal within the unit-type window)

Require modal count ≥ 2 as a confidence check. Works across multiple replays by aggregating all pairs before taking the mode.

## Why this is non-obvious

The O(n×m) all-pairs approach seems wasteful compared to FIFO O(n log n), but for typical SC2 calibration datasets (tens of commands, hundreds of born events per unit type) it is fast in practice. The key insight is that non-matching pairs produce uniformly distributed diffs (noise floor) while matching pairs cluster at T_real — the range constraint eliminates most noise before counting.
