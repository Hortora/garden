### Symptom

Every `@QuarkusTest` fails at startup:

```
org.hibernate.query.SemanticException: Operand of - is of type 'java.lang.Object'
which is not a numeric type (it is not an instance of 'java.lang.Number',
'java.time.Temporal', or 'java.time.TemporalAmount')
```

Triggered by `io.quarkus.scheduler.common.runtime.StatusEmitterInvoker`. Secondary failure: `Already a default codec registered for class StageActivatedEvent` on restart attempts.

### Context

The Quartz scheduler runs a status query during Quarkus startup. The query uses a numeric operation on a column whose type Hibernate resolves as `java.lang.Object` rather than a numeric type. This is specific to the H2 test database â€” the column type inference differs from PostgreSQL.

Unit tests (non-`@QuarkusTest`) are unaffected.

### Root cause

Likely a Hibernate 6.x type inference regression for H2 `MODE=PostgreSQL`. A column used in arithmetic by the scheduler query is not typed correctly in the H2 schema.

### Fix

Tracked in devtown#83. Workaround: write unit tests with Mockito mocks instead of `@QuarkusTest` for new code until the scheduler query is fixed.

### Why this is non-obvious

The error appears in a framework-internal class (`StatusEmitterInvoker`) with no obvious connection to application code. The secondary codec collision error (`StageActivatedEvent`) further obscures the root cause â€” it looks like a CDI issue when it's actually a Hibernate query failure on first attempt triggering cascading failures.
