### Technique

When testing that a child entity inherits its `tenantId` from the parent entity (not from
the caller's principal), use a **two-assertion write-invariant test**:

1. Create the parent entity as the default test tenant.
2. Switch the principal to `"admin-tenant"` with `setCrossTenantAdmin(true)`.
3. Create the child entity (the write operation under test).
4. Reset to the default test tenant; GET the child → assert 200 (child is readable by the
   parent's actual tenant — proves child.tenantId == parent.tenantId).
5. Switch to `"admin-tenant"` **without** the cross-tenant bypass; GET the child → assert 404
   (child is NOT readable by the admin's own tenant — proves child.tenantId ≠ admin.tenantId).

```java
@Test
void site_inherits_trial_tenantId_not_principal_tenantId() {
    // Create trial under default test tenant
    String trialLoc = given()
        .contentType("application/json")
        .body("{\"protocolId\":\"TEST\",\"phase\":\"PHASE_I\",\"sponsor\":\"T\",\"targetEnrollment\":5}")
        .when().post("/trials").then().statusCode(201).extract().header("Location");
    UUID trialId = UUID.fromString(trialLoc.substring(trialLoc.lastIndexOf('/') + 1));

    // Cross-tenant admin creates a child entity on the default tenant's trial
    principal.setTenancyId("admin-tenant");
    principal.setCrossTenantAdmin(true);
    String siteLoc = given()
        .contentType("application/json")
        .body("{\"investigatorId\":\"pi-iso\"}")
        .when().post("/trials/{id}/sites", trialId).then().statusCode(201).extract().header("Location");
    UUID siteId = UUID.fromString(siteLoc.substring(siteLoc.lastIndexOf('/') + 1));

    // Assertion 1: default tenant can read it → site.tenantId = trial.tenantId (correct)
    principal.reset();
    given().when().get("/trials/{t}/sites/{s}", trialId, siteId).then().statusCode(200);

    // Assertion 2: admin's own tenant (no bypass) cannot read it
    //   → proves site.tenantId ≠ "admin-tenant" (the invariant holds)
    principal.setTenancyId("admin-tenant");
    given().when().get("/trials/{t}/sites/{s}", trialId, siteId).then().statusCode(404);
}
```

The `@AfterEach` must call `principal.reset()` to prevent state leakage to other tests in
the class.

### Why This Is Non-Obvious

Most developers write only a **positive** isolation test: "wrong tenant cannot read entity."
That test does not verify what tenant the entity was stamped with — it only proves the
isolation mechanism works at read time.

The write-invariant test is the structural proof that the **creation path** stamps the
correct tenant. Without the negative assertion (step 5), a test could pass even if
`site.tenantId = "admin-tenant"` — because a cross-tenant admin can still read it. The
second GET, made with the admin's tenant **without bypass**, fails precisely when the stamp
is wrong, making the invariant machine-checkable.

The test also naturally requires the read-path isolation to be in place (the resource must
use `findByIdForTenant`, not bare `findById`) — so it catches both the write stamp and the
read guard simultaneously.

### Design note

In `@QuarkusTest` with RestAssured, the principal is `@ApplicationScoped` (singleton).
Mutating it between RestAssured calls within one test is safe because each `.when()` call
is a synchronous HTTP round-trip; the mutation is complete before the request is dispatched.
`@AfterEach` reset is essential — without it, a principal left in an admin state will
silently bypass isolation guards in subsequent tests.
