## Context
When building a registry or routing layer that needs all CDI beans of a type
(e.g. routing to the right `Connector` by id), the natural CDI approach is
`Instance<T>`. But `Instance<T>` is a CDI interface with many abstract methods
— difficult to stub in plain JUnit tests without a full container.

## What developers normally do
Inject `Instance<T>`, iterate with `.stream()`, or use `@InjectMock` inside a
`@QuarkusTest` — both require a CDI container.

## The technique
Inject `@All List<T>` instead. ArC populates it at startup with all discovered
beans. In tests, pass `List.of(...)` directly to the constructor — no CDI
context, no mocking of `Instance<T>`, no `@QuarkusTest` needed.

```java
// Production: ArC populates this automatically
ConnectorService(@All List<Connector> connectors) {
    this.registry = connectors.stream()
        .collect(Collectors.toMap(Connector::id, Function.identity()));
}

// Test: plain JUnit, no CDI
new ConnectorService(List.of(new SlackConnector(), new TeamsConnector()));
```

## Why this is non-obvious
`Instance<T>` is the CDI standard; `@All` is ArC-specific and not prominently
documented for this use case. The standard guidance says to use `@QuarkusTest`
for CDI testing. `@All List<T>` sidesteps the container requirement entirely
for patterns where the bean set is static at startup.

## Constraints
Requires `io.quarkus.arc.All`. Only works when the bean set is consumed at
construction time (static registry). Not suitable if beans are added/removed
at runtime.
