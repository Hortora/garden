### Symptom
Following the virtual-thread migration cookbook (`engine/docs/guides/virtual-thread-migration.md`) — "delete all Reactive* files, keep blocking" — breaks neocortex at compile time. The blocking classes reference deleted reactive delegates.

### Context
The cookbook pattern works for platform, ledger, eidos, and qhorus because their blocking implementations are self-contained (own the real logic) and reactive classes are thin wrappers. Neocortex is architecturally reversed.

### Root cause
In neocortex, the reactive implementations are the *primary* code — they contain the actual Qdrant gRPC calls, Mem0/Graphiti REST client invocations, and CBR decorator chain logic. The blocking classes are thin wrappers that delegate via `.await().indefinitely()`.

Affected modules:
- `memory-qdrant` — `ReactiveQdrantCbrCaseMemoryStore` is real; `QdrantCbrCaseMemoryStore` delegates
- `memory-mem0` — `ReactiveMem0CaseMemoryStore` is real; `Mem0CaseMemoryStore` delegates
- `memory-graphiti` — `ReactiveGraphitiCaseMemoryStore` is real; `GraphitiCaseMemoryStore` delegates
- `memory/runtime` — 4 reactive CBR decorators are real; blocking bridges delegate
- `corpus` — 3 reactive corpus bridges contain the logic

### Fix
Do NOT follow "delete reactive, keep blocking" for neocortex. Instead:
1. Convert each reactive implementation to blocking in-place (rename, change `Uni<T>` → `T`)
2. For REST clients: create blocking client interfaces (drop `Uni<>` from return types)
3. For Qdrant: switch from `ListenableFuture` wrapping to sync API
4. Delete the thin blocking wrapper (it was just the delegation shell)

### Why this is non-obvious
The cookbook is correct for 4 out of 5 repos. A developer applying it mechanically would delete 65 reactive files, see the build succeed for the API/SPI layer, then discover the blocking implementations are broken only at the next module's compile step.

### Alternatives considered
- Tried the standard "delete reactive" approach first — had to `git restore` the entire repo after discovering the reversed architecture
