## Symptom

```
error: cannot find symbol
  symbol: class DefaultBean
  location: package jakarta.enterprise.inject
```

The error gives no hint that the annotation exists — just that it doesn't exist
in the package you reached for.

## Context

When writing a `@DefaultBean` CDI bean in a Quarkus extension or application.
`@DefaultBean` marks a bean as the fallback when no higher-priority bean is
present — the Quarkus Arc equivalent of Spring's `@ConditionalOnMissingBean`.

## Root cause

`@DefaultBean` is a Quarkus Arc-specific annotation. It is not part of the
Jakarta CDI spec, so it doesn't live in `jakarta.enterprise.inject`. It lives
in `io.quarkus.arc`.

## Fix

```java
// Wrong — jakarta CDI doesn't have this
import jakarta.enterprise.inject.DefaultBean;

// Right — Quarkus Arc annotation
import io.quarkus.arc.DefaultBean;
```

## Why this is non-obvious

If you know CDI, `jakarta.enterprise.inject` is the natural package to reach for.
The compile error only says the class doesn't exist there — not that it exists
somewhere else. Easy to waste time searching for a CDI spec addition that never
happened.
