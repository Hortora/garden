### Symptom
JUnit 5 `@Nested` inner class annotated with `@QuarkusTest` fails at startup:
```
RuntimeException: Internal error. The test class should have been loaded
with a QuarkusClassLoader
```

### Context
You want to group related test methods using JUnit 5 `@Nested` classes,
each with its own `@QuarkusTest` and optionally `@TestProfile`. This is
standard JUnit 5 practice and works fine in non-Quarkus tests.

### Root cause
Quarkus test framework uses a custom `QuarkusClassLoader` to load the
test class. This classloader is not propagated to `@Nested` inner classes
— JUnit discovers them via the standard classloader, but Quarkus expects
its own.

### Fix
Use separate top-level test classes instead of `@Nested`:

```java
// Instead of:
class MyTest {
    @QuarkusTest @Nested class EmptyConfig { ... }
    @QuarkusTest @TestProfile(P.class) @Nested class WithConfig { ... }
}

// Use:
@QuarkusTest
class MyEmptyConfigTest { ... }

@QuarkusTest @TestProfile(P.class)
class MyWithConfigTest { ... }
```

### Why this is non-obvious
`@Nested` is a core JUnit 5 feature that works perfectly in all other
contexts. The error message mentions `QuarkusClassLoader` but does not
reference `@Nested` — making it difficult to connect the two. Quarkus
documentation does not warn against using `@Nested` with `@QuarkusTest`.
