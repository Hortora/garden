Quarkus test application.properties replaces selected-alternatives entirely — reactive in-memory repos need explicit activation

### Symptom

`quarkus:build` (production CDI validation) fails with `Unsatisfied dependency for type ReactiveCaseInstanceRepository` and similar for `ReactiveEventLogRepository`, `ReactiveSubCaseGroupRepository`. Tests pass because `engine-testing` provides `@Priority(1)` wrappers. The build step validates against main `application.properties` only.

### Context

A dual-stack engine (blocking + reactive SPIs) uses in-memory `@Alternative` implementations for both. Main `application.properties` lists only the blocking repos in `selected-alternatives`. Test `application.properties` lists different alternatives (ledger JPA repos, platform memory stores). `@Priority(1)` Test* wrappers from engine-testing auto-activate in test mode but don't exist on the production classpath.

### Root cause

Quarkus MicroProfile Config resolves `quarkus.arc.selected-alternatives` as a single string property. When test `application.properties` declares the same key, it **replaces** the main value entirely — it does not merge or append. The main properties' blocking repos are no longer activated in test mode, but that's fine because Test* wrappers cover them.

However, `quarkus:build` (invoked by `mvn verify`) validates CDI against the **main** `application.properties`. If the main properties only activate blocking repos, the reactive SPIs are unsatisfied. The reactive in-memory repos (`InMemoryReactiveCaseInstanceRepository`, `InMemoryReactiveEventLogRepository`, etc.) are `@Alternative` without `@Priority` — they need explicit `selected-alternatives` activation.

### Fix

Add all reactive in-memory repo alternatives to the **main** `application.properties`:

```properties
quarkus.arc.selected-alternatives=\
  io.casehub.persistence.memory.MemoryPlanItemStore,\
  io.casehub.persistence.memory.MemoryReactivePlanItemStore,\
  io.casehub.persistence.memory.InMemoryCaseInstanceRepository,\
  io.casehub.persistence.memory.InMemoryReactiveCaseInstanceRepository,\
  io.casehub.persistence.memory.InMemoryEventLogRepository,\
  io.casehub.persistence.memory.InMemoryReactiveEventLogRepository,\
  io.casehub.persistence.memory.InMemoryCaseMetaModelRepository,\
  io.casehub.persistence.memory.InMemoryReactiveCaseMetaModelRepository,\
  io.casehub.persistence.memory.InMemorySubCaseGroupRepository,\
  io.casehub.persistence.memory.InMemoryReactiveSubCaseGroupRepository
```

### Why this is non-obvious

Tests pass — the `@Priority(1)` Test* wrappers handle everything in test mode. The failure only appears during `quarkus:build`, which runs CDI validation against the main properties. The error messages name reactive SPI interfaces that the developer never directly uses, making it unclear which concrete beans to activate.
