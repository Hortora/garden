## Symptom

ARC reports `"Unsatisfied dependency for type com.example.SomeBean"` at augmentation time. `javap -p -verbose` on the `.class` file inside the JAR confirms `@ApplicationScoped` is present as a `RuntimeVisibleAnnotation`. The class is on the classpath.

## Context

A third-party or internal library JAR has `@ApplicationScoped` (or other CDI scope) beans but was not built with the `jandex-maven-plugin` and ships no `META-INF/jandex.idx`. Quarkus ARC relies on Jandex for efficient bean discovery — without an index it does not scan the JAR by default.

## Root cause

Quarkus ARC uses Jandex as its primary bean discovery mechanism. JARs without a Jandex index and without `beans.xml` are treated as non-bean-archives and skipped entirely. The CDI annotations in bytecode are present but ARC never reads them.

## Fix

Add an explicit index-dependency entry in `application.properties`:

```properties
quarkus.index-dependency.mylib.group-id=com.example
quarkus.index-dependency.mylib.artifact-id=mylib
```

Quarkus will then build an in-memory Jandex index for that JAR at augmentation time, discovering all CDI beans.

If the JAR should only be indexed in production (not in `@QuarkusTest`), use the `%prod.` profile prefix:

```properties
%prod.quarkus.index-dependency.mylib.group-id=com.example
%prod.quarkus.index-dependency.mylib.artifact-id=mylib
```

And add a separate test-scope entry to `src/test/resources/application.properties` if needed there.

## Verifying the JAR

```bash
python3 -c "
import zipfile
jar = 'path/to/mylib.jar'
with zipfile.ZipFile(jar) as z:
    has_jandex = any('jandex' in n for n in z.namelist())
    has_beans = any('beans.xml' in n for n in z.namelist())
    print(f'jandex={has_jandex}, beans.xml={has_beans}')
"
```

## Why this is non-obvious

The error "Unsatisfied dependency" implies the class doesn't exist or isn't a CDI bean — it says nothing about indexing. `javap` shows the annotation is genuinely in the bytecode. There is no Quarkus warning at augmentation time that a JAR was skipped due to missing Jandex.
