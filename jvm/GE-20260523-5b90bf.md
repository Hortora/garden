### Symptom
Running `mvn test -Dtest="ClassA+ClassB"` fails with:
> No tests matching pattern "ClassA+ClassB" were executed!

The error message says no tests matched — not that the separator is wrong.

### Context
Running multiple specific test classes in a single Maven invocation using `-Dtest=`.

### Root Cause
Maven Surefire uses `,` (comma) as the separator for multiple test class names. `+` is not a valid separator and is treated as part of the pattern literal, which matches nothing.

### Fix
Use commas:
```bash
mvn test -Dtest="ClassA,ClassB,ClassC"
```

### Why This Is Non-Obvious
The error "No tests matching pattern" points at the pattern being wrong, not the separator syntax. A developer debugging this naturally tries adjusting class names or paths rather than the separator character. The `+` is a natural guess for "and" — it's used in many other CLI tools.
