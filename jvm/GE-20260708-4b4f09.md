### Symptom
Running `quarkus:dev` on a fresh checkout of casehub-devtown fails with a sequence of five errors. Fixing one reveals the next. Total wall-clock: ~45 minutes of iterative restart cycles.

### Context
Multi-module Quarkus application (casehub-devtown) with: two named Flyway datasources scanning overlapping classpath locations, five CDI beans implementing the same `CurrentPrincipal` interface across foundation JARs, H2 file-mode JDBC URL with incompatible flags, and Quinoa frontend with `index.html` outside the build directory.

### What was tried
Each error was diagnosed and fixed individually. The cascade nature only became apparent in retrospect.

### Root cause
Five independent configuration gaps, each of which would be a straightforward fix in isolation. The cascade pattern is the gotcha: Quarkus dev mode fails fast on the first error, so errors 2-5 are invisible until error 1 is fixed.

### Fix

#### 1. H2 AUTO_SERVER incompatibility
`AUTO_SERVER=TRUE && DB_CLOSE_ON_EXIT=FALSE` is unsupported in the bundled H2 version. Use `%dev` profile with in-memory H2:
```properties
%dev.quarkus.datasource.jdbc.url=jdbc:h2:mem:work;MODE=PostgreSQL
%dev.quarkus.datasource.qhorus.jdbc.url=jdbc:h2:mem:qhorus;MODE=PostgreSQL
```

#### 2. Flyway V2000 duplicate
`quarkus.flyway.qhorus.locations` scans `db/qhorus/migration` (qhorus V2000) and `db/engine-ledger/migration` (engine-ledger V2000). Drop engine-ledger in dev:
```properties
%dev.quarkus.flyway.qhorus.locations=classpath:db/qhorus/migration,classpath:db/ledger/migration,classpath:db/devtown/migration
```

#### 3. Five competing CurrentPrincipal beans
Excluding `SecurityIdentityCurrentPrincipal` (OIDC) reveals four others. Exclude all five globally and add a `DevCurrentPrincipal`:
```properties
quarkus.arc.exclude-types=...,io.casehub.platform.oidc.SecurityIdentityCurrentPrincipal,io.casehub.work.runtime.service.TenantScopedPrincipal,io.casehub.persistence.memory.DefaultTestPrincipal,io.casehub.qhorus.runtime.identity.QhorusInboundCurrentPrincipal,io.casehub.platform.mock.MockCurrentPrincipal
```

#### 4. WorkAdapterPlanItemEntity not indexed
Add the missing package to Hibernate config:
```properties
quarkus.hibernate-orm.packages=io.casehub.work.runtime,io.casehub.workadapter
```

#### 5. Quinoa index.html not served
`quarkus.quinoa.build-dir=dist` only copies files inside `dist/`. Copy `index.html` into `dist/` during esbuild build and reference `app.js` (not `dist/app.js`).

### Why this is non-obvious
Each individual fix is trivial. The cascade is the problem: you cannot diagnose issue 3 until issue 2 is fixed, cannot see issue 4 until issue 3 is resolved. A developer seeing "Flyway duplicate migration" has no reason to suspect four more unrelated blockers await.

### Alternatives considered
- Using `%dev` profile overrides for Flyway and H2 only — fails because CDI and Hibernate issues are build-time, not profile-scoped
- Using `@IfBuildProfile("dev")` on DevCurrentPrincipal — doesn't deactivate competing beans from foundation JARs
