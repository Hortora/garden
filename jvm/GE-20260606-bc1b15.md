## Symptom

`@WithDefault("${sys:user.home}/.drafthouse/reviews")` on a `Path` return type in a `@ConfigMapping` sub-interface resolves to `user.home/.drafthouse/reviews` (literal string) rather than the actual home directory path. No error is thrown — the wrong path is silently used.

## Context

Occurs when configuring a default filesystem path in a SmallRye Config `@ConfigMapping` interface, with the intent to reference the JVM `user.home` system property. The `${sys:...}` prefix looks like a system-property handler prefix (common in other config frameworks), but SmallRye Config uses a different syntax.

## What was tried

```java
interface Storage {
    @WithDefault("${sys:user.home}/.drafthouse/reviews")
    Path root();
}
```

Expected: resolves to `/Users/alice/.drafthouse/reviews`.
Actual: resolves to `user.home/.drafthouse/reviews` (literal — `sys` property absent, fallback `user.home` treated as literal string).

## Root cause

In SmallRye Config's expression syntax, the colon in `${key:default}` is the **fallback-value separator**:

> Expand config property named `key`; if not found, use `default` as a literal string.

So `${sys:user.home}` means: "find config property named `sys`; if absent, produce the literal string `user.home`."

It does **not** invoke any system-property handler. There is no `sys:` handler prefix in SmallRye Config's expression evaluator by default. The double-colon syntax `${sys::user.home}` would invoke a named handler called `sys`, but handler availability is version-specific and not guaranteed.

The System Properties `ConfigSource` is registered at ordinal 400 (highest default). System properties are accessed directly via property name with no prefix: `${user.home}`.

## Fix

Use the bare property name — SmallRye resolves it from `SysPropConfigSource` (ordinal 400):

```java
interface Storage {
    @WithDefault("${user.home}/.drafthouse/reviews")
    Path root();
}
```

The `${user.home}` expression correctly resolves to the JVM `user.home` system property.

## Why this is non-obvious

The `${sys:property}` prefix is a common convention in other config frameworks (Spring's `${...}`, Typesafe Config's system property references). SmallRye's `:` as a fallback separator looks identical to a namespace separator. A developer who sees `${sys:user.home}` in documentation for another framework will naturally try the same pattern in SmallRye and get wrong results with no diagnostic output.
