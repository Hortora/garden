### Symptom
After refactoring a `@DefaultBean` class from inheritance to composition (removing `extends ParentClass`), tests fail at runtime with `UnsatisfiedResolutionException` for the parent class type — even though the parent class is `@Alternative @ApplicationScoped` and present on the classpath. No compile error.

### Context
When `ChildRepo extends ParentRepo`, CDI treats `ChildRepo` as a bean of both `ChildRepo` type AND `ParentRepo` type. If `ChildRepo` is `@DefaultBean` and `ParentRepo` is `@Alternative`, the child implicitly satisfies injection points for both types without requiring `ParentRepo` to be activated via `quarkus.arc.selected-alternatives`.

When the inheritance is removed (composition refactor), `ChildRepo` no longer satisfies `ParentRepo` injection points. The `@Alternative` parent needs explicit activation — but nothing in the refactor signals this.

### What was tried (didn't work)
Removing `extends JpaLedgerEntryRepository` from `CaseLedgerEntryRepository` and injecting `LedgerEntryRepository` (the SPI interface) in dependent services. Build succeeded. All 78 tests failed at runtime with empty query results — the `LedgerEntryRepository` injection point was unsatisfied because `JpaLedgerEntryRepository` (`@Alternative`) was never activated.

### Root cause
CDI type resolution includes all supertypes. A `@DefaultBean` child that extends an `@Alternative` parent acts as a stealth activation — the child provides the parent type without the parent needing `selected-alternatives`. Removing the inheritance removes this implicit activation silently.

### Fix
Add the `@Alternative` parent to `quarkus.arc.selected-alternatives` in the module's test `application.properties`:
```properties
quarkus.arc.selected-alternatives=\
  io.casehub.ledger.runtime.repository.jpa.JpaLedgerEntryRepository
```

### Why this is non-obvious
The refactor is purely structural — composition over inheritance, a textbook improvement. Nothing in the IDE, compiler, or Quarkus build warns that removing `extends` will break injection points for a type the child never explicitly declared it satisfies.
