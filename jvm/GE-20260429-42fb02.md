### Symptom
Trust score tests fail: `expected 1.0 but was 0.5` for an actor with multiple
completed decisions and no negative attestations. Increasing the number of
clean decisions does not change the score.

### Context
Any Bayesian Beta trust scoring system using Beta(α, β) with a uniform prior
Beta(1, 1). Decisions with no attestations are evaluated and the test expects
"no bad news = high score."

### Root cause
The Beta(1, 1) prior encodes maximum uncertainty — it says "I know nothing."
The score is α/(α+β). With no attestations: α=1.0, β=1.0 → score = 0.5.

Unattested decisions contribute **nothing** to the score. Only attestations
(positive or negative) move α or β. A clean decision record with zero
attestations leaves the score at the prior: 0.5, regardless of how many
decisions exist.

This is the correct Bayesian interpretation: "no evidence" means "maximum
uncertainty", not "no evidence of wrongdoing = trustworthy."

### Fix
Align test expectations with the algorithm:

```java
// WRONG — intuitive but incorrect for Bayesian Beta
assertThat(score.trustScore()).isCloseTo(1.0, within(0.01));

// CORRECT — no attestations → prior → neutral
assertThat(score.trustScore()).isCloseTo(0.5, within(0.01));
```

For a single SOUND attestation (confidence=0.9, age≈0):
- α = 1.0 + 0.9 = 1.9, β = 1.0 → score = 1.9/2.9 ≈ 0.655

For a single FLAGGED attestation (confidence=0.9, age≈0):
- α = 1.0, β = 1.0 + 0.9 = 1.9 → score = 1.0/2.9 ≈ 0.345

### Why this is non-obvious
Developers reason from a "presumption of innocence" model: no evidence of
wrongdoing implies trustworthiness. The Bayesian model instead reasons from
evidence: no evidence implies maximum uncertainty. The two models give opposite
results for empty attestation histories. Eight tests in two test classes had
this wrong simultaneously, including tests written by the same team that
implemented the algorithm.
