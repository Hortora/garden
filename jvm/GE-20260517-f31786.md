### Symptom

Type mismatch when bridging `Event.fireAsync()` to a Mutiny `Uni` pipeline:

```java
// Fails — CompletionStage<Event<MyEvent>>, not CompletionStage<VerificationResult>
return Uni.createFrom().completionStage(
    () -> suspectEvent.fireAsync(new MyEvent(...)))
    // compiler error: cannot convert Uni<Event<MyEvent>> to Uni<VerificationResult>
```

### Context

Quarkus reactive service using CDI `Event<T>` needs to fire an async event and return a `Uni<SomeType>` from the same pipeline.

### Root cause

`Event.fireAsync()` is declared in the CDI spec as returning `CompletionStage<Event<T>>` — it returns the event object itself wrapped in a completion stage, not `CompletionStage<Void>`. Most usage examples ignore the return value entirely, hiding the actual type.

### Fix

Use `.replaceWith()` to discard the `Event<T>` wrapper and substitute the intended result:

```java
return Uni.createFrom().completionStage(
    () -> suspectEvent.fireAsync(new MyEvent(entryId, actorId, keyRef)))
    .replaceWith(VerificationResult.SUSPECT);
```

### Why this is non-obvious

CDI spec says "returns a CompletionStage" without emphasising the `<Event<T>>` type parameter. Developers writing async event firing naturally assume `CompletionStage<Void>` because they're triggering a side effect and discarding the event payload. The full generic type is only visible in the CDI API Javadoc — not in any tutorials or common usage examples.
