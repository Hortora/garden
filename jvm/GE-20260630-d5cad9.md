### Symptom
Maven build fails with "Could not find artifact io.quarkiverse.quinoa:quarkus-quinoa:2.5.2" when following the casehub-pages Quinoa convention template.

### Context
The casehub-pages convention doc (casehubio/casehub-pages#26) specifies `@casehubio` package versions (0.2.0) but does not pin the quarkus-quinoa extension version. Plans and specs that reference a version may use one that was never published.

### Root cause
Quarkiverse extensions are published independently of Quarkus and casehub-pages. The convention template assumes a version that does not exist in Maven Central. The actual latest at time of discovery was 2.8.3.

### Fix
Always check Maven Central for the actual latest quarkus-quinoa version before adding the dependency. Do not assume version numbers from convention docs or plans.

```xml
<!-- Check https://central.sonatype.com/artifact/io.quarkiverse.quinoa/quarkus-quinoa -->
<dependency>
    <groupId>io.quarkiverse.quinoa</groupId>
    <artifactId>quarkus-quinoa</artifactId>
    <version>2.8.3</version> <!-- verify latest -->
</dependency>
```

### Why this is non-obvious
The convention doc is authoritative for casehub-pages adoption. Developers trust it and don't expect to hit a non-existent version on their first build attempt.
