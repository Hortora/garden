### Symptom
Every endpoint returns 403. Always. Regardless of authentication status, role, or `@RolesAllowed` annotation. The application is completely non-functional.

### Context
When adding OIDC + `@RolesAllowed` to a Quarkus application, a natural instinct is to add a catch-all deny policy so missing annotations fail closed:
```properties
quarkus.http.auth.permission.default.paths=/*
quarkus.http.auth.permission.default.policy=deny
```

### What was tried (didn't work)
The above configuration. Also tried adding `/q/*` health exemptions — the deny policy still blocks all application endpoints.

### Root cause
`quarkus.http.auth.permission.*` policies operate at the **Vert.x HTTP filter layer** — before the JAX-RS container processes the request and before CDI security interceptors run. A `deny` policy on `/*` rejects all requests at that filter, regardless of authentication status, role, or any `@RolesAllowed` annotation. `@RolesAllowed` is never evaluated — it is unreachable dead code.

Additionally, `quarkus.security.auth.enabled-in-dev-mode=false` (which suppresses `@RolesAllowed` evaluation in dev mode) does **not** suppress HTTP permission policies — those are at the `io.quarkus.vertx.http.runtime.security` layer, not the CDI authorization controller layer. Dev mode is also completely broken.

### Fix
Do not use `quarkus.http.auth.permission.*.policy=deny` as a catch-all. Use the JAX-RS-layer mechanism instead:
```properties
quarkus.security.deny-unannotated-members=true
```
This operates inside the JAX-RS container after authentication. `@RolesAllowed` annotations remain fully functional. It is suppressed in dev mode by `auth.enabled-in-dev-mode=false` (same CDI authorization controller layer).

### Why this is non-obvious
The two security layers (Vert.x HTTP filter vs CDI/JAX-RS interceptor) are not clearly distinguished in the Quarkus documentation. Both use `quarkus.` config prefixes. The HTTP permission approach looks like a standard deny-by-default pattern and produces no startup warning — the 403s only appear at runtime.
