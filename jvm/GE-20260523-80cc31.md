### What's Surprising

Mutiny's `flatMap` catches any synchronous exception thrown inside the supplier function and converts it to a failed `Uni`. This means you can use a normal Java `throw` for early-exit error conditions inside a `flatMap` chain — including exceptions thrown inside `Optional.ifPresent()` — and the failure propagates correctly as a reactive failure.

### Context

This comes up when adding guard conditions to a reactive service method. The temptation is to use `Uni.createFrom().failure(...)` for the early-exit path, but a plain `throw` works identically and is often simpler.

### Example

```java
public Uni<DispatchResult> dispatch(final MessageDispatch dispatch) {
    return Panache.withTransaction("qhorus", () ->
        channelStore.find(dispatch.channelId())
            .flatMap(chOpt -> {
                // Throwing from inside Optional.ifPresent() — works correctly.
                // Mutiny's flatMap catches the synchronous exception and converts
                // it to Uni.createFrom().failure(ex).
                chOpt.ifPresent(ch -> {
                    if (ch.paused) {
                        throw new IllegalStateException(
                            "Channel '" + ch.name + "' is paused");
                    }
                });
                // ... rest of the chain
                return messageStore.put(message);
            }));
}
```

### Why This Is Non-Obvious

The natural Mutiny idiom for conditional failure is `Uni.createFrom().failure(ex)`, not a plain Java `throw`. Developers unfamiliar with Mutiny's exception-capture semantics often avoid `throw` inside reactive chains because they expect it might bypass the reactive pipeline or produce an unhandled exception rather than a properly wrapped failure. The Mutiny documentation mentions this behavior in general terms but the pattern of throwing from a nested non-Mutiny call (`Optional.ifPresent()`, `Collection.forEach()`, etc.) inside the supplier is not documented explicitly.

### Why This Works

Mutiny's `flatMap` operator wraps the entire supplier function invocation in a try-catch. Any `Throwable` that escapes the supplier (including ones thrown inside `Optional.ifPresent()`, `Collection.forEach()`, or similar constructs) is caught and turned into `Uni.createFrom().failure(throwable)`. The resulting `Uni` propagates the failure downstream via `onFailure` handlers as normal.
