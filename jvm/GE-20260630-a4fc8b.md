FileCursorStore cursor path defaults to java.io.tmpdir â€” undiscoverable without bytecode inspection

### What exists

`casehub-rag`'s `FileCursorStore` persists ingestion cursors as files at:

```
${java.io.tmpdir}/casehub-ingestion-cursors/<gardenId>.cursor
```

The default path is declared via `@WithDefault("${java.io.tmpdir}/casehub-ingestion-cursors")` on the `cursorDir` property of `IngestionConfig` (SmallRye ConfigMapping).

Each cursor file contains a JSON map of `{filepath: lastModifiedTimestamp}` representing which files have been ingested and when.

### How it was discovered

Extracted string constants from the compiled `FileCursorStore.class` and `IngestionConfig.class` bytecodes in the casehub-rag JAR using Python `zipfile` module. The default path string `${java.io.tmpdir}/casehub-ingestion-cursors` was found in `IngestionConfig.class`.

### Why this matters

On macOS, `java.io.tmpdir` resolves to `/var/folders/xx/xxxxxxxxx/T/` â€” not `/tmp/` as most developers expect. The cursor files are:

- Not in the project directory
- Not in `~/.hortora/` or any application-specific config directory
- Not findable via `find /tmp` or `ls /tmp`
- Not mentioned in any configuration documentation or property file comments

Without knowing the exact path, there is no way to inspect, reset, or troubleshoot ingestion cursor state. The `casehub.rag.ingestion.cursor-dir` config property exists to override it but is not documented in any README or application.properties template.

### Key details

| Property | Value |
|----------|-------|
| Config property | `casehub.rag.ingestion.cursor-dir` |
| Default | `${java.io.tmpdir}/casehub-ingestion-cursors` |
| macOS resolved | `/var/folders/xx/.../T/casehub-ingestion-cursors/` |
| Linux resolved | `/tmp/casehub-ingestion-cursors/` |
| File pattern | `<gardenId>.cursor` (e.g. `garden.cursor`) |
| File format | JSON map: `{"path": epochMillis, ...}` |

### Finding the cursor

```bash
# macOS â€” java.io.tmpdir is NOT /tmp/
find /var/folders -name 'casehub-ingestion-cursors' 2>/dev/null

# Linux
ls /tmp/casehub-ingestion-cursors/

# Or ask Java directly
java -XshowSettings:all 2>&1 | grep java.io.tmpdir
```
