## Behaviour

`GreenMailExtension` extends `GreenMailProxy`. In `greenmail-junit5 2.1.3`, `GreenMailProxy.getGreenMail()` is declared `protected`:

```bash
$ javap -classpath greenmail-junit5-2.1.3.jar com.icegreen.greenmail.junit5.GreenMailExtension
...
protected com.icegreen.greenmail.util.GreenMail getGreenMail();
```

Attempting to call it from test code:

```java
GreenMail mail = GREEN_MAIL.getGreenMail(); // compile error: getGreenMail() has protected access
```

## Why This Matters

`GreenMail` (the bare class) has a `deliver(MimeMessage)` method and other useful internal APIs. The extension does not expose them. Since `getGreenMail()` is protected, you cannot retrieve the underlying instance to call them.

**Workaround:** In `@QuarkusTestResource` implementations that construct `GreenMail` directly (not via the extension), you hold the instance yourself as a static field — giving you full access to `deliver()`, `getManagers()`, etc.

## Why It's Undocumented

The Greenmail documentation and README make no mention of this access restriction. The Greenmail JUnit 5 guide shows the extension as a drop-in for the JUnit 4 Rule and implies equivalent functionality. The only way to discover the `protected` modifier is via `javap` on the jar.
