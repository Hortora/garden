## Symptom

Fixed abilLink constants used to identify training commands in SC2 replays silently find 0 commands when run against replays from a different SC2 version. No error is raised.

## Context

SC2 replay GAME_EVENTS contain `Cmd` events with an `abilLink` integer identifying which ability was used. Any code that maps `abilLink → intent` (e.g. `abilLink=175 → Nexus train Probe`) uses constants that were discovered from replays of a specific patch. Cross-dataset analysis that mixes replay sources from different SC2 versions will silently fail to match commands.

## What was tried

Built a training-time calibration test using AI Arena replays (2023+, build ~67188) and IEM10 JSON replays (2016, build 39948). Used the same abilLink constants (NEXUS=175, GATEWAY=172, ROBOTICS=173, STARGATE=174) for both sources. The IEM10 source returned 0 training commands across 30 games despite the replays clearly containing Nexus training activity.

## Root cause

SC2 patches change abilLink IDs. IEM10 2016 replays use abilLink=167 for Nexus training (Probe) where AI Arena 2023+ replays use abilLink=175. The binary format is identical but the ability ID numbering changed between patches. A dump of IEM10 game events shows abilLink=167 appearing ~29 times per game for a Protoss player — the exact count expected for Probe training commands.

## Fix

Maintain patch-version-aware abilLink tables if cross-era analysis is needed. For single-patch analysis, confirm the abilLink values by dumping all unique `(abilLink, abilCmdIndex, count)` tuples for the target player and cross-referencing with expected unit-born counts:

```python
from collections import Counter
counts = Counter()
for e in data['gameEvents']:
    if e.get('evtTypeName') != 'Cmd': continue
    if e.get('userid', {}).get('userId') != user_id: continue
    abil = e.get('abil')
    if abil:
        counts[(abil.get('abilLink'), abil.get('abilCmdIndex'))] += 1
# Compare top entries against known unit-born counts from tracker events
```

## Why this is non-obvious

The SC2 replay binary format is stable and the protocol decoder (scelight) handles all versions. The abilLink *numbers* themselves appear to be stable constants once you discover them for a given codebase — they're not versioned in the format, only in the game data. Nothing in the replay file signals that ability IDs have changed across patches.
