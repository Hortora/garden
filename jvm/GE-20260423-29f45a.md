## Context

When a CDI event publisher wants to skip expensive pre-computation (e.g. a database pre-read for delta calculation) when no observers are registered for a given payload type, it needs to know at startup whether any observers exist — without paying a per-call cost.

## Technique

Use `BeanManager.resolveObserverMethods(payload)` in a `@PostConstruct` method to detect observer presence once and cache the result:

```java
@ApplicationScoped
public class TrustScoreRoutingPublisher {

    @Inject BeanManager beanManager;

    private boolean hasDeltaObservers;
    private boolean hasFullObservers;

    @PostConstruct
    void detectObservers() {
        // resolveObserverMethods covers both @Observes and @ObservesAsync
        hasDeltaObservers = !beanManager
            .resolveObserverMethods(new TrustScoreDeltaPayload(List.of()))
            .isEmpty();
        hasFullObservers = !beanManager
            .resolveObserverMethods(new TrustScoreFullPayload(List.of()))
            .isEmpty();
    }

    public boolean needsPreviousSnapshot() {
        return hasDeltaObservers;  // skip DB pre-read if no delta observers
    }
}
```

`resolveObserverMethods` takes an instance of the event payload type and returns the full set of observer methods for that type — including both `@Observes` and `@ObservesAsync` observers. An empty set means no observers are registered.

## Why this is non-obvious

Most CDI documentation focuses on `Event.fire()` / `Event.fireAsync()`. `BeanManager.resolveObserverMethods()` is a lower-level API used primarily by CDI extension authors. Using it from application code for observer-presence detection at startup is unusual — the more obvious approach would be to try firing and catching exceptions, or to always do the pre-computation "just in case".

### Alternatives considered

- Always compute delta (pre-read DB every run): wastes resources when no delta consumers exist
- Use `Event.select().hasObservers()`: this method does not exist on `Event<T>` in CDI 4.x
- Check at fire-time via try/catch: races and adds per-call overhead
