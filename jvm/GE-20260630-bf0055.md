### Symptom
Quarkus startup fails with a confusing error about missing Node.js when `quarkus.quinoa.package-manager-install=true` is set but no node version is specified.

### Context
Quinoa 2.8.x introduced `package-manager-install` which downloads and caches Node.js locally in `.quinoa/`. The feature requires `quarkus.quinoa.package-manager-install.node-version` to be set explicitly.

### Root cause
The Quinoa extension does not default to a Node.js version — it requires explicit configuration. This property is not mentioned in the casehub-pages convention template or the Quinoa getting-started docs.

### Fix
Add the node version property to `application.properties`:

```properties
quarkus.quinoa.package-manager-install=true
quarkus.quinoa.package-manager-install.node-version=22.11.0
```

Quinoa caches the downloaded Node in `.quinoa/` — add this to `.gitignore`.

### Why this is non-obvious
The property name is not discoverable from the error message. The convention template does not mention it because it was written for an earlier Quinoa version.
