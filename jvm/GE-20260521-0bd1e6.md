## Symptom

CDI deployment failure or `UnsatisfiedResolutionException` in a consumer app. All injected dependencies of the gated service are activated, but the service itself is not injectable. Adding the class to `quarkus.arc.selected-alternatives` fixes it.

## Context

A Quarkus extension adds a service bean with both `@Alternative` and `@IfBuildProperty`:

```java
@Alternative
@IfBuildProperty(name = "casehub.qhorus.reactive.enabled", stringValue = "true")
@ApplicationScoped
public class QhorusDashboardService {
    @Inject ReactiveChannelService channelService; // ← this gets activated
    ...
}
```

The consumer app sets the build property, expects the bean to be injectable — and gets a deployment failure because `QhorusDashboardService` itself is not in the CDI context.

## Root cause

`@IfBuildProperty` controls whether the bean class is included in the CDI context at augmentation time. `@Alternative` controls whether an included bean is *selected* (active) — and without `@Priority`, it defaults to deselected unless listed in `quarkus.arc.selected-alternatives`.

These two mechanisms compose independently: `@IfBuildProperty` gates **inclusion**; `@Alternative` gates **selection**. A bean can be included (build property met) but still deselected (`@Alternative` without `@Priority`). When the bean is deselected, Arc does not inject it anywhere — but its own injected dependencies are still resolved at augmentation, causing the mismatch.

## Fix

Remove `@Alternative` entirely when there is no interface or superclass to override. `@IfBuildProperty` alone correctly controls activation:

```java
// Before — @Alternative forces manual selection
@Alternative
@IfBuildProperty(name = "casehub.qhorus.reactive.enabled", stringValue = "true")
@ApplicationScoped
public class QhorusDashboardService { ... }

// After — @IfBuildProperty handles activation; no manual selection needed
@IfBuildProperty(name = "casehub.qhorus.reactive.enabled", stringValue = "true")
@ApplicationScoped
public class QhorusDashboardService { ... }
```

If `@Alternative` is legitimately needed (the bean overrides another bean's type), add `@Priority(1)` to activate it globally — consistent with how Quarkus's own reactive alternatives work (e.g. `ReactiveChannelService`, `ReactiveMessageService` in casehub-qhorus).

## Why this is non-obvious

`@IfBuildProperty` looks like it fully controls the bean's lifecycle. The interaction with `@Alternative`'s deselection mechanism is not documented in Quarkus — you only discover it when the build property is set, the service's dependencies are all activated, but the service itself isn't injectable. The error message points to an unsatisfied dependency, not the `@Alternative` deselection.
