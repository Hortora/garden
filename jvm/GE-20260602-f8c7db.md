### Symptom
`mvn dependency:get -Dartifact=com.github.spring-ai-community:claude-agent-sdk-java:1.0.0`
fails with artifact not found, even though the library exists on GitHub.

### Context
Attempting to add `com.github.spring-ai-community:claude-agent-sdk-java` as a Maven
dependency to a Java project.

### Root cause
The `spring-ai-community/claude-agent-sdk-java` project is hosted on GitHub but the
artifact is not published to Maven Central. The Maven coordinate exists in documentation
and GitHub READMEs but the artifact is absent from the central repository.

### Fix
Use JitPack to resolve the artifact. Add the JitPack repository and use the GitHub
coordinates:

```xml
<repositories>
    <repository>
        <id>jitpack.io</id>
        <url>https://jitpack.io</url>
    </repository>
</repositories>
<dependency>
    <groupId>com.github.spring-ai-community</groupId>
    <artifactId>claude-agent-sdk-java</artifactId>
    <version>1.0.0</version>
</dependency>
```

Verify availability first: `https://jitpack.io/#spring-ai-community/claude-agent-sdk-java`

### Why this is non-obvious
The artifact coordinate is widely referenced in blog posts and the project README as
though it were on Maven Central. The GitHub presence and 1.0.0 version tag implies
a published release. Maven's error message (`Could not find artifact`) does not
distinguish "coordinate wrong" from "not on Central" — both look identical.
