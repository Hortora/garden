### What exists
Some Xenova ONNX model exports on HuggingFace are silently gated — the download URL returns HTTP 401 with no error page, no "request access" button, and no explanation. This happens even when the source model (e.g. `naver/splade-cocondenser-ensembledistil`) is public and ungated.

### How to discover it
Only discoverable by attempting the download. `curl -sI <URL>` returns `HTTP/2 401` with `content-length: 29`. The HuggingFace web UI for the Xenova repo may show files but clicking download also fails silently.

### Workaround
The `onnx-models/` namespace on HuggingFace has publicly accessible ONNX exports of the same model architectures without authentication requirements. For example, `onnx-models/Splade_PP_en_v1-onnx` provides a working SPLADE ONNX export where `Xenova/splade-cocondenser-ensembledistil` returns 401.

### Why this is non-obvious
The gating is on the derived export, not the source model. A developer checking that `naver/splade-cocondenser-ensembledistil` is public and ungated would reasonably expect the Xenova ONNX export to also be public. There is no documentation explaining which Xenova exports are gated or why.
