### Context
A Quarkus application ships multiple provider implementations of an SPI (e.g. `DeviceProvider`) in one Docker image. Each provider should be independently enableable via config (`provider.enabled=true/false`). Disabled providers must be invisible to `Instance<DeviceProvider>` iteration — not just "present but returning empty results."

### What a skilled developer would normally do
Guard in `@PostConstruct`: check `enabled`, skip initialization if false. Make every method return a no-op result (`discover()` → empty list, `dispatch()` → FAILED, `status()` → DISCONNECTED). This is the "inert-bean" pattern — the bean exists but does nothing.

Problems with inert beans:
- The bean IS discovered by `Instance<DeviceProvider>` — callers see a broken provider
- `@PostConstruct` still runs (and may fail if config properties are missing)
- Every method needs guard logic
- SmallRye Config still validates all `@ConfigMapping` properties at startup (see GE-20260626-e0ecaf)

### Technique
Use `@LookupIfProperty` from Quarkus ARC:

```java
@ApplicationScoped
@LookupIfProperty(name = "casehub.iot.homeassistant.enabled", stringValue = "true")
public class HomeAssistantProvider implements DeviceProvider { ... }
```

When `enabled` is absent or `false`:
- The bean is never instantiated
- `@PostConstruct` never runs
- `Instance<DeviceProvider>` iteration skips it entirely
- No guard logic needed in any method

Combined with `Optional<String>` on all `@ConfigMapping` properties (GE-20260626-e0ecaf), this allows disabled providers to exist on the classpath without causing config validation failures or appearing in the provider registry.

### Why this is non-obvious
`@LookupIfProperty` is less well-known than `@IfBuildProperty` (which is build-time only and prevents runtime reconfiguration). The naming — "lookup if" vs "if build" — doesn't clearly convey the difference. Many developers reach for `@IfBuildProperty` first, discover it prevents runtime toggling in Docker images, and then fall back to the inert-bean pattern instead of finding `@LookupIfProperty`.
