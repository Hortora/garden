## Symptom

A test using `@TestProfile` with `getEnabledAlternatives()` returning `Set.of(MyTestBean.class)` causes `@Alternative` beans selected via `quarkus.arc.selected-alternatives` in `application.properties` to be silently deactivated. For example, `MemoryPlanItemStore` stops being active and the application starts with `JpaPlanItemStore` instead — typically causing a startup failure (`HumanTaskRecoveryService` queries `WorkAdapterPlanItemEntity` which isn't mapped to Hibernate in the test context) or a wrong bean injection with no diagnostic pointing at the profile.

## Root Cause

`getEnabledAlternatives()` in `QuarkusTestProfile` **replaces** the effective set of selected alternatives for that profile — it does not ADD to the `quarkus.arc.selected-alternatives` value in `application.properties`. Any `@Alternative` bean that relies on the config property for activation is deactivated if not explicitly listed in `getEnabledAlternatives()`.

## Fix

**Option A:** Include ALL required `@Alternative` beans in `getEnabledAlternatives()` — not just the test-specific one:

```java
@Override
public Set<Class<?>> getEnabledAlternatives() {
    return Set.of(
        MyTestBean.class,
        MemoryPlanItemStore.class,
        MemorySubCaseGroupRepository.class,
        JpaLedgerEntryRepository.class
        // every @Alternative from application.properties goes here
    );
}
```

**Option B:** Replace the `@TestProfile` + `@Alternative` inner class with `@InjectMock` on the SPI interface. This avoids the profile entirely and leaves the standard `selected-alternatives` config intact:

```java
@QuarkusTest
class MyTest {
    @InjectMock MySpi spi;

    @BeforeEach
    void setup() {
        when(spi.evaluate(any())).thenReturn(testResult);
    }
}
```

## Why This Is Non-Obvious

The name `getEnabledAlternatives()` implies additive behaviour — "enable these alternatives (in addition to the defaults)". The actual semantics are replacement — enabling these alternatives IS the complete set. The `application.properties` config is ignored in favour of the profile's set, with no log message indicating that previously-selected alternatives have been dropped.
