### Symptom
`@ConfigProperty(name = "prefix") Map<String, String> myMap` throws
`NoSuchElementException` at startup when no `prefix.*` keys are configured —
even though the expectation is an empty map.

### Context
Quarkus / SmallRye Config collects all properties matching a prefix into a
`Map<String, String>`. If NO properties match the prefix, it throws instead of
returning an empty map.

```java
// Throws NoSuchElementException at startup if no matching properties
@ConfigProperty(name = "casehub.platform.preferences.defaults")
Map<String, String> defaults;
```

### Fix
Wrap in `Optional<Map<String, String>>`:

```java
@ConfigProperty(name = "casehub.platform.preferences.defaults")
Optional<Map<String, String>> defaults;

// Usage
defaults.orElse(Map.of()).forEach((k, v) -> ...);
```

`Optional` resolves to `Optional.empty()` cleanly when no keys match — no exception.

### Why this is non-obvious
Scalar `@ConfigProperty` supports `defaultValue = ""` as a fallback. There is no
equivalent `defaultValue` for `Map<String, String>` — it is not documented, and the
failure mode (exception vs empty map) is counterintuitive given that an absent prefix
logically means zero entries.
