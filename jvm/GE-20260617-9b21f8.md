### Symptom
Quarkus fails to start with:
```
SRCFG00040: The config property foo.bar is defined as the empty String ("")
which the following Converter considered to be null: io.smallrye.config.Converters$BuiltInConverter
```
This fires even though you have `@ConfigProperty(name = "foo.bar", defaultValue = "")` in the source.

### Context
A property with an empty value in `application.properties` (`foo.bar=`) combined with a non-`Optional<String>` `@ConfigProperty` field.

### Root cause
SmallRye Config's built-in `StringConverter` converts `""` (empty string) to `null`. The `@ConfigProperty` `defaultValue = ""` attribute is used as a fallback when the property is **absent**, but when the property is explicitly present with an empty value, the converter runs first and produces `null`. Injecting `null` into a non-Optional field violates the non-null contract and causes the startup failure.

### Fix
Use `Optional<String>` for fields that legitimately have no value:
```java
@ConfigProperty(name = "foo.bar")
Optional<String> fooBar;
```
SmallRye maps absent or empty-string properties to `Optional.empty()` correctly.
Alternatively, don't set the property in `application.properties` at all and rely on `defaultValue = ""`.

### Why this is non-obvious
The `defaultValue = ""` annotation makes it look like the empty case is handled. The failure happens not from absence but from presence-with-empty-value — two different code paths in SmallRye that have different semantics. The error message references "Converter" which doesn't immediately point to the `Optional<String>` fix.
