## Symptom

`@QuarkusTest` startup fails with an OIDC connectivity error — "OIDC server is not available at…" — after adding `casehub-platform-oidc` (or any `quarkus-oidc` dependency) even though `%dev.quarkus.oidc.enabled=false` is set in `application.properties`.

## Context

`%dev.` prefix properties apply only to the Quarkus `dev` profile (i.e. `quarkus dev` / `mvn quarkus:dev`). `@QuarkusTest` runs in the `%test` profile. The dev profile is never activated during `mvn test`.

## Root cause

`%dev.quarkus.oidc.enabled=false` is never read in a `@QuarkusTest` context. With `quarkus.oidc.application-type=service` active (no profile prefix, applies to all profiles), the OIDC extension starts during test startup and attempts OIDC discovery — which fails because no Keycloak or auth server is running.

## Fix

Add a test-profile OIDC block to `src/test/resources/application.properties` (no `%test.` prefix needed — this file is test-only):

```properties
# Prevents OIDC from attempting discovery in @QuarkusTest
# jwks-path is lazy-loaded and never actually fetched when @TestSecurity is used
quarkus.oidc.auth-server-url=http://localhost:8180/realms/test
quarkus.oidc.discovery-enabled=false
quarkus.oidc.jwks-path=protocol/openid-connect/certs
quarkus.keycloak.devservices.enabled=false
```

`@TestSecurity` (from `quarkus-test-security`) bypasses actual token validation entirely — the OIDC extension only needs its config to be structurally valid, not connected to a running server.

## Why this is non-obvious

The intent "disable OIDC in non-production contexts" maps naturally to the dev profile — that's where interactive development happens. The `%test` profile is a separate activation that requires separate properties. It's easy to assume `%dev.` covers all non-production runs including tests.

**See also:** GE-20260521-f50602 — `discovery-disabled` requires `jwks-path` (structural config validation detail); GE-20260601-08a351 — `quarkus-oidc-client` on the classpath triggers Keycloak DevServices even with static token auth.
