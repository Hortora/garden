### Symptom

Enricher pipeline sorted by `@Priority` always produces the same arbitrary order regardless of the
annotation value. No error. Adding `System.out.println(e.getClass())` reveals the class is
`MyBean$$QuarkusProxy`, not `MyBean`.

### Context

Any Quarkus application that iterates a CDI `Instance<T>` and tries to sort the beans by
`@Priority` using reflection on the bean class — e.g. an enricher pipeline, interceptor chain,
or ordered CDI observer list.

### What was tried (didn't work)

```java
// WRONG — e is a proxy; getAnnotation returns null even when @Priority(20) is on the real class
for (T enricher : enrichers) {
    Priority p = enricher.getClass().getAnnotation(Priority.class);
    int priority = p != null ? p.value() : 100; // always 100
}
```

### Root cause

ArC wraps CDI beans in proxy classes (e.g. `AgentSignatureEnricher$$QuarkusProxy`).
`e.getClass()` returns the proxy class. The proxy class does not carry annotations from the
original class — `proxyClass.getAnnotation(Priority.class)` returns `null` silently.

### Fix

Use `InjectableInstance` (Arc-specific) and access priority via CDI bean metadata, not reflection
on the proxy class:

```java
import io.quarkus.arc.InjectableBean;
import io.quarkus.arc.InjectableInstance;

@Inject @Any InjectableInstance<LedgerEntryEnricher> enrichers;

enrichers.handlesStream()
    .sorted(Comparator.comparingInt(h ->
        (h.getBean() instanceof InjectableBean<?> ib) ? ib.getPriority() : Integer.MAX_VALUE))
    .map(InstanceHandle::get)
    .forEach(e -> e.enrich(entry));
```

`h.getBean()` returns Arc's `InjectableBean<T>` (which extends CDI's `Bean<T>`) and exposes
`getPriority()` directly from CDI bean metadata — never touches the proxy class.

### Why this is non-obvious

`e.getClass().getAnnotation()` works for non-proxied objects. CDI proxies are transparent to most
callers. The JVM reflection API never mentions that it sees proxy subclasses instead of the
original class. No error is thrown — the annotation is simply absent, leading to silent wrong ordering.
