### The technique

`LedgerEntry.supplements` is declared as `new ArrayList<>()` — initialised in the constructor, not by JPA. `attach(LedgerSupplement)` populates it in-memory immediately. `compliance()` filters by `ComplianceSupplement.class::isInstance` from that list.

All of this is purely in-memory. No persistence unit, no `@QuarkusTest` CDI container, and no `@ApplicationScoped` bean is needed to verify supplement content.

```java
// Plain Mockito test — no @QuarkusTest, no H2, no CDI:
@ExtendWith(MockitoExtension.class)
class RegulatorySubmissionLedgerWriterTest {
    @Mock LedgerEntryRepository ledgerEntryRepository;
    @Mock Clock clock;
    @InjectMocks RegulatorySubmissionLedgerWriter writer;

    @Test
    void grade5_writes_c1i_planRef() {
        when(clock.instant()).thenReturn(Instant.parse("2026-06-15T12:00:00Z"));
        when(ledgerEntryRepository.findLatestBySubjectId(any(), any())).thenReturn(Optional.empty());

        AdverseEvent ae = new AdverseEvent();
        ae.grade = CtcaeGrade.GRADE_5;
        ae.id = UUID.randomUUID();
        ae.enrollmentId = UUID.randomUUID();

        writer.writeEntry(ae);

        verify(ledgerEntryRepository).save(
            argThat(entry ->
                ((RegulatorySubmissionLedgerEntry) entry).compliance()
                    .map(c -> c.planRef)
                    .orElse("")
                    .contains("(c)(1)(i)")),
            eq("default"));
    }
}
```

### Why this is non-obvious

The natural assumption is that JPA relationships (`@OneToMany supplements`) require a JPA session to function. Most developers would reach for `@QuarkusTest` with H2 to test supplement attachment — adding 15–25 seconds of container startup per test run. The in-memory list is not documented; you'd only discover it by reading the `LedgerEntry` source.
