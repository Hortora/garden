### What's surprising

`StartRaw` is **optional** in `ResponseGameInfo` (parsed with `orElse(nothing())`). But once included, ALL five sub-fields are **mandatory** ‚Äî each uses `orElseThrow(required(...))`. Including `StartRaw` with only the pathing grid (the one field SC2BotAgent actually reads) crashes with an opaque "required" error for the other four.

### Symptom

```
java.lang.IllegalArgumentException: map size is required
```

Or similar for `pathing grid`, `terrain height`, `placement grid`, or `playable area` ‚Äî whichever is missing first in the constructor's validation order.

### Root cause

`StartRaw.from()` in ocraft-s2client-protocol 0.4.21:

```java
mapSize       = ... .orElseThrow(required("map size"));
pathingGrid   = ... .orElseThrow(required("pathing grid"));
terrainHeight = ... .orElseThrow(required("terrain height"));
placementGrid = ... .orElseThrow(required("placement grid"));
playableArea  = ... .orElseThrow(required("playable area"));
```

All five are hard-required. `startLocations` (the 6th field) is a list that defaults to empty ‚Äî only the five above crash.

### Fix

Provide all five fields when including StartRaw. For stubs (where only pathingGrid is functionally needed):

| Field | Stub value | Notes |
|-------|-----------|-------|
| `mapSize` | `Size2dI(64, 64)` | Match actual map dimensions |
| `pathingGrid` | Real 1bpp data from `TerrainGrid.toPathingGrid()` | Functional ‚Äî SC2BotAgent reads this |
| `terrainHeight` | 1√ó1 8bpp stub (1 byte) | Validation-only |
| `placementGrid` | 8√ó1 1bpp stub (1 byte) | Validation-only ‚Äî see GE-20260609-f0d149 for the 1√ó1 1bpp edge case |
| `playableArea` | `RectangleI((0,0), (64,64))` | Validation-only |

The safe alternative: omit `StartRaw` entirely (it's optional). `FakeSC2Server` does this ‚Äî `SC2BotAgent.onGameStart()` uses `getStartRaw().ifPresent(...)` and silently skips terrain extraction when absent.

### Why this is non-obvious

The optional/required boundary is invisible from the proto definition. `ResponseGameInfo` ‚Üí `StartRaw` is optional. But within `StartRaw`, all 5 fields are mandatory. A developer who adds StartRaw for terrain (the common case) gets a crash about `map size` ‚Äî a field they didn't know existed. The error message says "required" but doesn't explain that the requirement is an ocraft parsing constraint, not a proto schema constraint.
