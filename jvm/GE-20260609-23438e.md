### Symptom

Need to override annotation values at runtime from Quarkus config, but upstream reads them directly with no interception point.

### Context

Upstream's declarative flow reads annotation values and passes them to builders. The `AgentConfigurator` callback only covers inner `AgentBuilder`, not workflow builders. Options: (A) bypass the declarative path entirely (reimplementing parsing), (B) annotation transformation (doesn't work — Jandex transformations don't affect runtime reflection), (C) wrap existing SPIs.

### Fix

Wrap the upstream SPI at the builder factory level. For `WorkflowAgentsBuilder`:

1. Register a custom implementation via `AgenticServices.setWorkflowAgentsBuilder()`
2. The wrapper delegates all builder creation to the real implementation
3. For specific builders (e.g. `loopBuilder()`), wrap the returned builder in a decorator
4. The decorator delegates all fluent methods + returns `this` (chain integrity)
5. On `build()`, apply the config override before delegating

For `A2AService` (no public setter): same pattern but register via reflection on the provider field.

### Why this is non-obvious

Three approaches were considered. Annotation transformation (Jandex) seems natural for Quarkus but doesn't work — upstream reads annotations via Java reflection at runtime, not Jandex. Bypassing the declarative path works but reimplements upstream logic. SPI wrapping is the lightest touch — preserves the declarative flow, intercepts only what's needed, and is cleanly removable when upstream adds proper SPIs.
