### What happened

Adding `casehub-testing` as a test dependency and indexing it via
`quarkus.index-dependency` caused Quarkus augmentation to fail with CDI deployment
errors. `casehub-testing` transitively pulls `casehub-persistence-memory`, whose
`InMemoryCaseInstanceRepository`, `InMemoryCaseMetaModelRepository`, and
`InMemoryEventLogRepository` are abstract classes annotated with `@ApplicationScoped`.
Quarkus Arc attempted to create proxy subclasses for them and failed.

### Root cause

Jandex indexes every class in a jar, including abstract ones. Quarkus Arc's bean
discovery considers any class with a CDI scope annotation (`@ApplicationScoped`,
`@RequestScoped`, etc.) as a bean candidate — abstract or not. Arc then tries to
generate a CDI proxy (which must be instantiable) and fails at augmentation time
because abstract classes cannot be directly instantiated.

The classes in `casehub-persistence-memory` are abstract base classes intended to be
extended by concrete test implementations (e.g. `TestCaseInstanceRepository`). They
carry `@ApplicationScoped` so that subclasses automatically inherit the scope without
re-declaring it. This is a valid CDI pattern, but breaks when the abstract parent
is indexed independently.

### Fix

Exclude the abstract parent classes explicitly in `quarkus.arc.exclude-types`:

```properties
%test.quarkus.arc.exclude-types=\
  io.casehub.persistence.memory.InMemoryCaseInstanceRepository,\
  io.casehub.persistence.memory.InMemoryCaseMetaModelRepository,\
  io.casehub.persistence.memory.InMemoryEventLogRepository
```

The concrete subclasses (`TestCaseInstanceRepository` etc.) remain active and inherit
the scope from the excluded abstract parents — exclusion is not inherited.

### Why this is non-obvious

Developers expect abstract classes to be automatically filtered from CDI bean
discovery — you can't instantiate an abstract class, so it shouldn't be a bean.
But Arc's discovery is annotation-driven, not instantiability-driven. Any class
with a scope annotation is a candidate unless explicitly excluded.
