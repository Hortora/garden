### What it is

A pattern for writing an audit/side-effect record when a primary entity is saved, without
coupling the audit write to the parent transaction or violating JPA's `@PrePersist` constraints.

### The technique

Fire an async CDI event from the write path (enricher, service, or listener). Observe it in
a separate `@ApplicationScoped` bean with `@ObservesAsync`. Persist the audit entry in a
dedicated method annotated `@Transactional(REQUIRES_NEW)`.

```java
// Write path (enricher, service, @EntityListeners)
event.fireAsync(new AuditEvent(actorId, did, status, ...));

// Separate @ApplicationScoped observer bean
void onAudit(@ObservesAsync AuditEvent e) {
    persistAuditEntry(e.actorId(), e.did(), e.status());
}

@Transactional(Transactional.TxType.REQUIRES_NEW)
void persistAuditEntry(String actorId, String did, Status status) {
    AuditEntry entry = new AuditEntry();
    // ... populate ...
    entityManager.persist(entry);
    // commits in its own transaction when method returns
}
```

### Properties of this pattern

- **Independent commit**: the audit entry commits in its own transaction. If the parent write
  rolls back (e.g. ENFORCE mode validation throws), the audit entry can still be written (or
  skipped separately).
- **Failure isolation**: if the audit write fails, the exception is caught and logged by the
  Quarkus async event infrastructure — the parent transaction is unaffected.
- **No `@PrePersist` unsafe-persist**: avoids calling `em.persist()` inside a JPA lifecycle
  callback, which violates the JPA spec (see GE-20260530-c05d12).
- **Thread safety**: `@ObservesAsync` dispatches on a different thread from the CDI managed
  executor; `@ApplicationScoped` CDI beans are thread-safe proxies.

### Why a skilled developer wouldn't reach for this

The instinct is to call `auditRepo.save(entry)` synchronously in the write path, wrapping it
in its own `REQUIRES_NEW` helper. That still violates the `@PrePersist` constraint and
tightly couples audit failure to the write path. The async event indirection seems like
overhead but is precisely the right amount of decoupling: the event is the contract; the
observer is independently replaceable.
