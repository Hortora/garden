### Symptom
Setting `quarkus.quinoa.just-build=true` in `application.properties` to disable the frontend
build when the `-Pui` profile is not active. Build still downloads Node.js and runs `npm install`
and `npm run build` ‚Äî just doesn't serve the output at runtime.

### Context
A Quarkus module with an optional casehub-pages frontend gated behind a `-Pui` Maven profile.
The requirement: `mvn clean install -Pdemo` must pass without Node.js installed.

### What was tried (didn't work)
- `quarkus.quinoa.just-build=true` ‚Äî still runs the full npm pipeline
- `quarkus.quinoa.dev-server=false` ‚Äî only affects dev mode hot-reload, not the build
- `%ui.quarkus.quinoa.just-build=false` with profile-specific overrides ‚Äî properties are
  evaluated after Quinoa is already on the classpath

### Root cause
`just-build` controls whether Quinoa *serves* the built frontend at runtime ‚Äî it does NOT
control whether the npm install and build steps execute. When `quarkus-quinoa` is on the
classpath, it always processes `src/main/webui/`.

### Fix
Put the `quarkus-quinoa` dependency inside the Maven profile:

```xml
<profile>
  <id>ui</id>
  <dependencies>
    <dependency>
      <groupId>io.quarkiverse.quinoa</groupId>
      <artifactId>quarkus-quinoa</artifactId>
      <version>2.8.3</version>
    </dependency>
  </dependencies>
</profile>
```

Quinoa properties in `application.properties` (`quarkus.quinoa.build-dir`, etc.) are inert
when the extension is not on the classpath ‚Äî Quarkus logs warnings about unrecognised keys
but continues normally.

### Why this is non-obvious
The property name `just-build` implies a minimal mode ‚Äî "just build, nothing else." A developer
trying to conditionally disable Quinoa would naturally reach for this property. The actual
semantics ("build AND don't serve") are the opposite of what the name suggests for this use case.

### Alternatives considered
- `quarkus.quinoa.just-build=true` ‚Äî still runs npm, just doesn't serve (wrong semantics)
- `quarkus.quinoa.dev-server=false` / `managed=false` ‚Äî affects dev mode only, not install phase
- Profile-specific property overrides (`%ui.quarkus.quinoa.*`) ‚Äî Quinoa must already be on classpath to read these
