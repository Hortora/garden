### Symptom
Cannot find `KieHelper` class anywhere on the Drools 10 classpath. Import
`org.kie.internal.utils.KieHelper` fails with "class not found". No deprecation warning in
migration guides — the class is simply absent.

### Context
Migrating from Drools 7/8/9 to Drools 10. `KieHelper` was a widely-used convenience class for
building a KieBase from DRL strings or resources without the full
KieFileSystem → KieBuilder → KieContainer pipeline. Common in tests and simple applications.

### What was tried (didn't work)
- Searched all Drools 10.1.0 jars (drools-compiler, drools-core, drools-model-compiler,
  drools-model-codegen, and every other module) — `KieHelper` is not present in any of them
- Searched drools-base, drools-kiesession — not there either

### Root cause
`KieHelper` was removed entirely from Drools 10. Not deprecated — deleted. No replacement
utility was provided. The full `KieFileSystem → KieBuilder → KieContainer → KieBase` chain is
the only supported path for programmatic KieBase construction.

### Fix
Use the standard pipeline:

```java
KieServices ks = KieServices.Factory.get();
KieFileSystem kfs = ks.newKieFileSystem();
kfs.write("src/main/resources/rules.drl", drlContent);
KieBuilder kb = ks.newKieBuilder(kfs)
        .buildAll(ExecutableModelProject.class);
Results results = kb.getResults();
if (results.hasMessages(Message.Level.ERROR)) {
    throw new IllegalStateException("DRL compilation failed: " + results.getMessages());
}
KieModule module = kb.getKieModule();
KieBaseConfiguration kbc = ks.newKieBaseConfiguration();
kbc.setOption(EventProcessingOption.STREAM); // if CEP needed
KieBase kieBase = ks.newKieContainer(module.getReleaseId()).newKieBase(kbc);
```

### Why this is non-obvious
`KieHelper` appears in many Drools examples, blog posts, and Stack Overflow answers. Migration
guides don't mention its removal. A developer migrating from Drools 9 would search for the
class, find nothing, and not know the replacement pattern without reading the Drools 10 source.
