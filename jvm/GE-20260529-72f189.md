## Symptom

```
java.lang.IllegalStateException: Could not start mail server imap:127.0.0.1:3143,
try to set server startup timeout > 2000 via ServerSetup.setServerStartupTimeout(timeoutInMs)
```

One test class passes, the next one fails at startup. The error message points at a timeout but the actual cause is a port conflict.

## Context

`ServerSetupTest.SMTP_IMAP` is a Greenmail constant that pre-assigns SMTP to port 3025 and IMAP to port 3143. When both a `@QuarkusTestResource` (which starts Greenmail in `start()`) and a `@RegisterExtension static final GreenMailExtension` (which starts Greenmail in `@BeforeAll`) use the same constant, they both try to bind the same ports. Maven Surefire runs all tests in one JVM by default, so one class's Greenmail is still holding the port when the next class starts.

## What Was Tried

The obvious read of the error ‚Äî "increase startup timeout" ‚Äî is a red herring. The port is occupied, not slow.

## Root Cause

Both the `@QuarkusTestResource` lifecycle manager and the JUnit 5 `@RegisterExtension` static extension use `ServerSetupTest.SMTP_IMAP`, which hardcodes ports 3025 and 3143. The QuarkusTest resource starts Greenmail before Quarkus, and the unit test extension starts its own Greenmail independently. They collide.

## Fix

Use OS-assigned random ports (port `0`) for the unit test `@RegisterExtension` ‚Äî keep the QuarkusTestResource on fixed ports (so it can inject them into MP Config before Quarkus starts):

```java
// Unit test ‚Äî random ports, no conflicts
@RegisterExtension
static final GreenMailExtension GREEN_MAIL = new GreenMailExtension(new ServerSetup[]{
        new ServerSetup(0, "localhost", ServerSetup.PROTOCOL_SMTP),
        new ServerSetup(0, "localhost", ServerSetup.PROTOCOL_IMAP)})
        .withConfiguration(GreenMailConfiguration.aConfig()
                .withUser("inbox@example.com", "password"))
        .withPerMethodLifecycle(false);
```

The actual assigned ports are retrieved at runtime: `GREEN_MAIL.getImap().getPort()`, `GREEN_MAIL.getSmtp().getPort()`.

The `@QuarkusTestResource` must continue to use `ServerSetupTest.IMAP` (or `SMTP_IMAP`) because it needs to inject the port into MP Config before Quarkus starts, and MP Config injection happens before the JUnit extension can assign a random port.

## Why This Is Non-Obvious

The error message says "startup timeout" and suggests increasing it. The real cause is a port conflict ‚Äî a completely different problem with a completely different fix. Nothing in the Greenmail documentation warns that `ServerSetupTest.*` constants conflict when two instances run in the same JVM.
