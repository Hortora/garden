### Symptom

Adding a dev-profile-only CDI bean (e.g., a `CurrentPrincipal` for dev mode) via `%dev.quarkus.arc.selected-alternatives=com.example.DevBean` causes startup failures — all beans from the non-profiled `quarkus.arc.selected-alternatives` list disappear.

### Context

Quarkus profile-specific properties (`%dev.`, `%test.`, `%prod.`) completely **replace** their non-profiled counterparts. If the base `application.properties` has:

```properties
quarkus.arc.selected-alternatives=\
  com.example.JpaRepo,\
  com.example.JpaPlanItemStore,\
  com.example.JpaSubCaseGroupRepo
```

Adding `%dev.quarkus.arc.selected-alternatives=com.example.DevBean` drops all three JPA alternatives in dev mode — CDI startup fails with unsatisfied dependency errors for every bean that was in the original list.

### Root cause

Profile-specific properties in Quarkus are full replacements, not merges. This is by design (SmallRye Config behaviour), but the failure mode is silent — no warning that the non-profiled value was dropped.

### Fix

Use bean-level annotations instead of `selected-alternatives`:

```java
@ApplicationScoped
@Alternative
@Priority(150)  // beats OidcCurrentPrincipal @Priority(100)
@IfBuildProfile("dev")
public class DemoCurrentPrincipal implements CurrentPrincipal {
    @Override public String tenancyId() { return "demo-tenant"; }
    @Override public String actorId()   { return "demo-user"; }
}
```

**Requirements:**
1. The bean must live in a **Jandex-indexed JAR** (e.g., the application's own `runtime/` module). ArC discovers it automatically — no `selected-alternatives` entry needed.
2. `@IfBuildProfile("dev")` excludes the bean entirely from non-dev builds — no ambiguity in production.
3. `@Priority(150)` resolves over competing `@Alternative` beans (e.g., `OidcCurrentPrincipal @Priority(100)`) via standard ArC priority resolution.

This is the same mechanism by which `OidcCurrentPrincipal @Alternative @Priority(100)` beats `MockCurrentPrincipal @DefaultBean` in production — without being listed in `selected-alternatives`.

### Why this is non-obvious

The natural instinct is `%dev.quarkus.arc.selected-alternatives=MyDevBean` — it looks like the standard profile-override pattern. The replacement behaviour is documented for SmallRye Config but not prominently flagged in the Quarkus CDI guide. The failure (dropped JPA alternatives) manifests as unrelated `UnsatisfiedResolutionException` errors that don't mention `selected-alternatives` at all.

### Why this fix

`@IfBuildProfile + @Priority` solves the problem at the bean level — no configuration property needed. The alternative (duplicating the full `selected-alternatives` list in every profile override) is fragile and drifts as new alternatives are added.

### Alternatives considered

- **Duplicate the full list in `%dev.`** — works but fragile; every new `@Alternative` bean must be added to every profile-specific override. Drifts silently.
- **Use `@UnlessBuildProfile("prod")` instead of `@IfBuildProfile("dev")`** — activates in both dev and test profiles, which may not be desired. Test profile has its own `FixedCurrentPrincipal @Priority(200)`.
