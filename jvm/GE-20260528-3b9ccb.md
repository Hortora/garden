## Symptom

`IEM10CommandExtractor` produces zero training intents with no error. All `TrainIntent`
objects have building tags that never match any unit in the current selection.

## Context

In quarkmind, unit and building tags from IEM10 SC2EGSet JSON tracker events are
formatted as `"j-{index}-{recycle}"` — the `j-` prefix is the IEM10 JSON format
marker (set in `IEM10JsonSimulatedGame.makeTag()`, which is `private static`).

The binary SC2Replay parser (`ReplaySimulatedGame`) uses a different tag format:
`"r-{index}-{recycle}"`, produced by `Sc2ReplayShared.makeTag()` (package-private).

## Root cause

`Sc2ReplayShared.makeTag()` and `IEM10JsonSimulatedGame.makeTag()` are both in the
same package (`io.quarkmind.sc2.mock`), so `Sc2ReplayShared.makeTag()` is accessible
from `IEM10CommandExtractor`. When called, it silently produces `r-` prefix tags.

The selection-state tracker in `IEM10CommandExtractor` also builds tags from packed
`addUnitTags` integers in `SelectionDelta` game events using the same decode formula.
If those tags are built with `Sc2ReplayShared.makeTag()`, they also get `r-` prefix —
and they still match each other, so the selection state looks correct internally.
The failure only manifests when these tags are passed to `EmulatedGame` or looked up
against tracker-event buildings, which use `j-` prefix. Zero intents matched, zero error.

## Fix

Do not call `Sc2ReplayShared.makeTag()` in any class operating on IEM10 JSON game
events. Inline the construction directly:

```java
// In IEM10CommandExtractor.applySelectionDelta():
long packed  = tagNode.asLong();
int  index   = (int) (packed >> 18);
int  recycle = (int) (packed & 0x3FFFF);
String tag   = "j-" + index + "-" + recycle;  // IEM10 tracker tag format
```

## Why this is non-obvious

Both methods are in the same package, have the same signature, and their only
difference is the string prefix. The IDE will suggest `Sc2ReplayShared.makeTag()`
as the natural reuse — it is package-visible, semantically named, and collocated.
The resulting `r-` tags are structurally valid strings and produce no runtime
error; the breakage is a data-level mismatch that silently returns empty results.
