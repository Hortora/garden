## What it is

In SC2EGSet pre-processed JSON replays, `Cmd` events in the `gameEvents` array can
be classified as **training commands** (self-targeted ability activations) versus
**movement/attack commands** (point- or unit-targeted) by inspecting the `data` field:

```json
// Training command (train Probe, train Stalker, research, etc.)
{ "evtTypeName": "Cmd", "abil": { "abilLink": 167, "abilCmdIndex": 0 },
  "data": { "None": null }, ... }

// Movement command (Smart/Move/Attack)
{ "evtTypeName": "Cmd", "abil": { "abilLink": 45, "abilCmdIndex": 0 },
  "data": { "TargetPoint": { "x": 82.5, "y": 66.25 } }, ... }

// Unit-targeted command
{ "evtTypeName": "Cmd", "abil": { "abilLink": 45, "abilCmdIndex": 0 },
  "data": { "TargetUnit": { ... } }, ... }
```

## How to use it

To filter only training/self-targeted commands in Java:

```java
private static boolean isTrainingCommand(JsonNode event) {
    JsonNode data = event.path("data");
    if (data.isMissingNode()) return true;           // no data field = self-targeted
    return data.has("None") && data.size() == 1;    // data: {None: null}
}
```

## How it was discovered

Discovered by inspecting the raw JSON structure of SC2EGSet game events from the
IEM10 2016 Taipei dataset. Cross-validated by comparing frequency counts of
`data: {None: null}` events against expected training command counts from tracker
event unit birth counts using narrow-window modal analysis.

No SC2EGSet documentation, GitHub README, or paper describes this field structure.
The SC2EGSet Python preprocessing scripts are the authoritative source, but the
specific `None`/`TargetPoint`/`TargetUnit` discriminated-union encoding is not
explained in any user-facing documentation.

## Why this matters

Without this filter, extracting training commands from SC2EGSet JSON requires
selection-state tracking and abilLink allowlisting — which still cannot distinguish
a Chrono Boost (no-target Nexus ability) from a Probe training command without the
abilLink table. The `data.None` check cleanly separates all self-targeted commands
from movement commands before the abilLink dispatch, allowing a correct combined
filter: `isTrainingCommand(e) && dispatch(abilLink, abilCmdIndex) != null`.

Note: Chrono Boost and Research commands also satisfy `data: {None: null}` — the
abilLink dispatch (returning null for unrecognised links) correctly handles those.
