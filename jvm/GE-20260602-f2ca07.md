### Technique
When an LLM judge must score rendered text without knowledge of the source data (blind scoring), pass only the `String` content as the `UserMessage` — not the eval case record that contains the source descriptor. This structurally prevents the descriptor from leaking into the payload regardless of future refactoring.

```java
// Wrong: passes the full eval case; descriptor fields visible if serialized
UserMessage.from(mapper.writeValueAsString(evalCase))

// Right: passes only the rendered text
UserMessage.from(rendered.content())
```

Verify with a test that captures the user message and asserts the descriptor sentinel is absent:
```java
req.messages().forEach(m -> {
    if (m instanceof UserMessage um)
        capturedPayload.set(um.singleText());
});
assertThat(capturedPayload.get())
    .isEqualTo("You are a careful engineer.")
    .doesNotContain("agentId")
    .doesNotContain("tenancyId");
```

### Why this is non-obvious
A judge method typically receives the full eval case for context — slot, descriptor, goal context — and developers naturally pass it to the LLM for reference. The blind scoring constraint (judge must not see the descriptor) is easy to violate accidentally if the method signature includes the full case. Restricting the payload to a `String` is a stronger guarantee than hoping developers remember the constraint across refactors.

The `doesNotContain` assertion turns the constraint into a compile-tracked invariant: if someone later modifies the method to include the descriptor "for convenience," the test fails immediately.
