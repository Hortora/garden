### What it is
`DefaultExpressionEngineRegistry.evaluate(ExpressionEvaluator evaluator, CaseContext ctx)` short-circuits to `true` when `evaluator` is `null`. A `ContextChangeTrigger` constructed with a null evaluator (from `contextChange: {}` in YAML or `new ContextChangeTrigger((ExpressionEvaluator) null)` in Java) therefore fires unconditionally on every `CONTEXT_CHANGED` event — effectively a start-trigger with no filter condition.

### Bytecode evidence
```
// DefaultExpressionEngineRegistry.evaluate():
0: aload_1        // push evaluator param
1: ifnonnull 6    // if non-null, jump to real evaluation
4: iconst_1       // push true
5: ireturn        // return true — null evaluator always evaluates to true
6: ...            // real evaluation path
```

Verified by decompiling `casehub-engine-0.2-SNAPSHOT.jar`.

### How to use it
In YAML — use `contextChange: {}` (empty map, no `filter:` key) under the `on:` block:

```yaml
on:
  contextChange: {}
```

In Java — cast null to `ExpressionEvaluator` to select the correct constructor overload:

```java
.on(new ContextChangeTrigger((ExpressionEvaluator) null))
```

The trigger will fire on every `CONTEXT_CHANGED`, including the one `CaseStartedEventHandler` publishes at case start — making it an effective "fire on case start" trigger without a dedicated `StartTrigger` type.

### Why this is not in the docs
There is no documented null-filter binding in the engine YAML reference. The pattern is only discoverable by reading the mapper source or decompiling the JAR. The engine's `JQExpressionEngine.evaluate()` has the same null-short-circuit, providing defence in depth.

### Caveat
The null-filter binding fires on **every** subsequent `CONTEXT_CHANGED`, not just the first. Combine with a `when:` guard to prevent re-firing (see GE-20260608-1a56c3).
