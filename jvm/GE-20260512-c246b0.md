## Test Quarkus CDI SPI implementations with @Alternative static inner classes — Mockito cannot be injected as CDI beans

### What it achieves

Verifies that CDI SPI wiring works correctly in a `@QuarkusTest` context, with full control over what the test double records and returns.

### Context

Quarkus CDI resolves beans at augmentation time (build time). Mockito mocks and anonymous classes cannot be registered as CDI alternatives at test runtime — ARC doesn't know about them during augmentation. Attempting to inject a Mockito mock as a CDI alternative fails silently or selects the wrong bean.

### The technique

Define SPI test doubles as `@Alternative @ApplicationScoped` **static inner classes** within the test class. Use static fields to record invocations. Reset in `@BeforeEach`.

**Critical:** static inner classes do NOT auto-activate just because they are `@Alternative`. You must explicitly list them in `quarkus.arc.selected-alternatives` in the test profile using the outer class `$` inner class FQN:

```properties
# In the test profile section of application.properties:
%my-test-profile.quarkus.arc.selected-alternatives=\
  io.example.JpaRepository,\
  io.example.WorkerProvisionerTest$RecordingProvisioner
```

Note: `@Priority(N)` alone is NOT sufficient for inner class activation in Quarkus Arc — the explicit `selected-alternatives` entry is always required.

```java
@QuarkusTest
@TestProfile(WorkerProvisionerTest.Profile.class)
class WorkerProvisionerTest {

    public static class Profile implements QuarkusTestProfile {
        @Override public String getConfigProfile() { return "my-test-profile"; }
    }

    @Alternative @ApplicationScoped
    static class RecordingProvisioner implements WorkerProvisioner {
        static List<ProvisionContext> calls = new ArrayList<>();

        @Override
        public Worker provision(ProvisionContext ctx) {
            calls.add(ctx);
            return Worker.noop();
        }
    }

    @BeforeEach
    void reset() { RecordingProvisioner.calls.clear(); }

    @Test
    void verifyProvisionerCalled() {
        // trigger work that should call the provisioner
        assertThat(RecordingProvisioner.calls).hasSize(1);
    }
}
```

### Why this is non-obvious

Mockito is the default testing reflex for Java. The CDI augmentation constraint isn't documented prominently — it only surfaces as a wrong-bean-injected failure or a silent no-invocation result. The static inner class pattern feels unusual but is the correct CDI-aware approach.

The additional non-obvious constraint: unlike top-level `@Alternative` classes in `src/test/java/`, static inner classes inside a `@QuarkusTest` class are indexed by Quarkus but are NOT automatically activated as alternatives — they still need explicit `selected-alternatives` registration. This contradicts the intuition that "if Quarkus can see it, it will use it." See GE-20260417-c59817 for the full `selected-alternatives` mechanism.

**Update 2026-05-13:** Verified on Quarkus 3.32.2 — `@Priority(1)` on the inner class alone does not activate it. Only `quarkus.arc.selected-alternatives` with the `OuterClass$InnerClass` FQN works reliably.
