### Symptom

Compilation error: `getDefinition() in MyApp cannot override getDefinition() in io.casehub.api.engine.YamlCaseHub — overridden method is final`.

### Context

Any `YamlCaseHub` subclass that overrides `getDefinition()` to augment the YAML-loaded `CaseDefinition` with in-process workers breaks on the new SNAPSHOT.

### Root cause

`YamlCaseHub.getDefinition()` was made `final` in the engine-api SNAPSHOT. A new protected hook `augment(CaseDefinition definition)` was added — called once internally by `getDefinition()` after YAML parsing.

### Fix

Replace the `getDefinition()` override with `augment()`:

```java
// Before (broken)
@Override
public CaseDefinition getDefinition() {
    CaseDefinition def = super.getDefinition();
    def.getWorkers().addAll(myWorkers());
    return def;
}

// After (works)
@Override
protected void augment(CaseDefinition definition) {
    definition.getWorkers().addAll(myWorkers());
}
```

The `augment()` hook receives the already-parsed definition. No need for double-checked locking or volatile fields — `getDefinition()` handles caching internally.

### Why this is non-obvious

No changelog, no deprecation notice, no compile-time hint beyond the error. The `augment()` method name is only discoverable by reading the decompiled `YamlCaseHub` bytecode from the SNAPSHOT jar.
