## Symptom

After a casehub-engine upgrade, the first `@QuarkusTest` class (e.g. `SmokeTest`) fails
at discovery time with:

```
PreconditionViolationException: Could not load class with name: io.casehub.claudony.SmokeTest
Caused by: RuntimeException: io.quarkus.builder.BuildException: Build failure: Build failed due to errors
  [error]: Build step io.quarkus.arc.deployment.ArcProcessor#validate threw an exception:
  DeploymentException: UnsatisfiedResolutionException: Unsatisfied dependency for type CaseHubRuntime
```

The failing bean (`QhorusMessageSignalBridge`) is not named at the `SmokeTest` level — only
at the end of a four-level exception chain.

## Context

Triggered when a casehub-engine upgrade introduces a new `@ApplicationScoped` bean that
injects `CaseHubRuntime` in its constructor, while the test profile excludes `CaseHubRuntimeImpl`
(the only implementation). Any engine upgrade can introduce such beans without warning.

Specific instance: engine#349 introduced `QhorusMessageSignalBridge`
(`io.casehub.engine.internal.bridge`), which bridges Qhorus `MessageReceivedEvent`s into
CaseHub signals. The bean is a dead observer when Qhorus is absent — but CDI still validates
its constructor injection at startup.

## Root Cause

`QhorusMessageSignalBridge` is `@ApplicationScoped` and injects `CaseHubRuntime` via
constructor injection. Test profiles that exclude `CaseHubRuntimeImpl` leave this injection
point unsatisfied. CDI deployment fails, Quarkus cannot build the test application, and
JUnit reports the first test class as unloadable — not the failing bean.

## Fix

Add the new bean to `%test.quarkus.arc.exclude-types` in `src/test/resources/application.properties`:

```properties
%test.quarkus.arc.exclude-types=...,\
  io.casehub.engine.internal.bridge.QhorusMessageSignalBridge
```

The failing bean must be found by tracing the exception chain:
`PreconditionViolationException → RuntimeException → BuildException → DeploymentException → UnsatisfiedResolutionException`
where the last exception finally names the type. With `-Dquarkus.log.level=DEBUG`, the ARC
processor logs the specific bean and injection point.

## Why This Is Non-Obvious

The error surfaces at the test class level ("SmokeTest cannot be loaded"), not the CDI bean
level. Engine upgrades are expected to add features — a new bridge bean registering as a
dead observer appears benign. There is no compile-time warning: the new bean compiles
fine and only fails at Quarkus CDI augmentation, which runs after all source files compile
successfully.
