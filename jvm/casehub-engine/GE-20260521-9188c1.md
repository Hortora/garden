## Symptom

All bindings with `on: { contextChange: {} }` fire on every `CONTEXT_CHANGED` event, even when their `when:` conditions are false. This causes every capability binding to schedule provisioning on every tick (generating "No workers defined" warnings for all bindings), and `humanTask` bindings to create duplicate WorkItems.

## Context

Any CaseHub harness using YAML case definitions where bindings have `when:` conditions. The engine fires ALL contextChange bindings unconditionally on each tick, regardless of the `when:` expression. Only scheduled triggers evaluate `when:` at runtime (via `SchedulerService`).

## Root Cause

`CaseContextChangedEventHandler.rules()` evaluates `cct.getFilter()` (the trigger's inner `filter` field) to decide which bindings are eligible. For `on: { contextChange: {} }`, `getFilter()` returns `null` — and `expressionEngineRegistry.evaluate(null, snapshot)` returns `true` by design ("always matches").

The YAML `when:` field is mapped to `Binding.when` (a `JQExpressionEvaluator`) via `CaseDefinitionYamlMapper.convertBinding()`, but `CaseContextChangedEventHandler` never calls `binding.getWhen()`. Only the trigger's own filter is checked — a separate concept.

`DefaultPlanningStrategy.select()` and `PlanningStrategyLoopControl.select()` also pass all eligible bindings through without checking `binding.getWhen()`.

## Why This Is Non-Obvious

Unit tests (`PrReviewBindingConditionTest`) test `when:` conditions directly by calling `((LambdaExpressionEvaluator) b.getWhen()).test(ctx)` — this works correctly and all tests pass, creating false confidence that `when:` gates execution at runtime. The same `when:` expression in YAML is correctly parsed into `Binding.when`, so the YAML is valid and the model looks correct. The distinction between the trigger's `filter` and the binding's `when` is subtle and undocumented.

### Why this fix

The `when:` condition IS used by the Scheduler for scheduled triggers — this pattern works correctly in that context, reinforcing the expectation that it works everywhere.
