## Symptom

Adding `casehub-engine` to a project and starting Quarkus fails with:

```
jakarta.enterprise.inject.spi.DeploymentException: Found 7 deployment problems:
[1] Unsatisfied dependency for type io.casehub.engine.internal.jq.JQEvaluator and qualifiers [@Default]
    - injection target: parameter 'jqEvaluator' of io.casehub.blackboard.subcase.SubCaseCompletionService constructor
[2] Unsatisfied dependency for type io.casehub.engine.internal.jq.JQEvaluator and qualifiers [@Default]
    - injection target: io.casehub.engine.internal.engine.handler.CaseContextChangedEventHandler#jqEvaluator
...
```

All 7 problems reference `JQEvaluator` or related engine CDI beans. The engine's
`CaseContextChangedEventHandler`, `WorkItemLifecycleAdapter`, `WorkOrchestrator`,
`SubCaseCompletionService`, and others all inject `JQEvaluator`, which has no
available CDI bean in the deployment context.

## Context

Any Quarkus project that adds `casehub-engine` (or `casehub-engine-work-adapter`,
`casehub-engine-blackboard`) to its pom.xml WITHOUT also adding `casehub-platform`
and `casehub-platform-expression`.

## Root Cause

`casehub-engine-common` provides `io.casehub.engine.internal.jq.JQEvaluator`
(`@ApplicationScoped`), which requires `io.casehub.engine.internal.config.ConfigManager`
and `io.casehub.engine.internal.config.SecretManager` from the same module.

These beans are NOT discoverable by Quarkus CDI unless `casehub-engine-common` is
**explicitly indexed** via `quarkus.index-dependency`. The engine's `CaseContextChangedEventHandler`
and others are in `casehub-engine-blackboard` (auto-indexed via `beans.xml`), but their
`JQEvaluator` injection point depends on a bean in `casehub-engine-common`, which lacks
auto-discovery.

Additionally, engine beans inject `@DefaultBean` mocks (`MockPreferenceProvider`,
`MockCurrentPrincipal`) from `casehub-platform`. Without `casehub-platform` on the
classpath, these injection points are also unsatisfied.

## Fix

Add these three dependencies to `pom.xml`:

```xml
<!-- Required by casehub-engine for @DefaultBean mocks (MockPreferenceProvider etc.) -->
<dependency>
  <groupId>io.casehub</groupId>
  <artifactId>casehub-platform</artifactId>
</dependency>
<!-- Required by casehub-engine for JQEvaluator (expression evaluation) -->
<dependency>
  <groupId>io.casehub</groupId>
  <artifactId>casehub-platform-expression</artifactId>
</dependency>
<!-- Test fixture alternatives for CurrentPrincipal etc. -->
<dependency>
  <groupId>io.casehub</groupId>
  <artifactId>casehub-platform-testing</artifactId>
  <scope>test</scope>
</dependency>
```

Also add `casehub-engine-common` to `quarkus.index-dependency` in `application.properties`:

```properties
quarkus.index-dependency.engine-common.group-id=io.casehub
quarkus.index-dependency.engine-common.artifact-id=casehub-engine-common
```

Reference: devtown uses these exact same three platform artifacts to make engine CDI work.

## Why This Is Non-Obvious

`casehub-platform` is not listed as a required Maven dependency of `casehub-engine`.
The CDI error mentions `JQEvaluator` — not `casehub-platform`. A developer
adding `casehub-engine` would look at engine documentation (none exists for this
requirement) or at the engine pom.xml (which shows only indirect platform-api
dependency). The fix is only discoverable by examining a working consumer (devtown)
and diffing its pom against the failing project.
