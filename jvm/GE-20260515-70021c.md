## Always scope Maven commands to a specific module with -pl

**ID:** GE-20260515-70021c
**Stack:** Maven (all versions)
**What it achieves:** Fast, targeted builds that isolate failures to the module under test without rebuilding unrelated siblings.
**Context:** Multi-module Maven reactors where running from the root rebuilds everything.

### The technique

Use `-pl <module>` to target a specific submodule:

```bash
# Targeted — builds only casehub-work-ledger
mvn test -pl work/ledger --batch-mode

# With enforced timeouts via wrapper scripts
scripts/mvn-test work/ledger      # 90s
scripts/mvn-install work/ledger   # 60s
scripts/mvn-compile work/ledger   # 45s
```

`mvn test` at the reactor root rebuilds all submodules sequentially, taking 5–10× longer and burying the failing module's output.

### Why this is non-obvious

Developers coming from single-module projects run Maven from the root by habit — it works, so the habit persists. In a multi-module reactor the root build succeeds from a correctness standpoint but is the wrong default for iteration speed. The `-pl` flag is documented but not emphasised as the primary development workflow pattern for multi-module work.

### When to use it

Any time you're iterating on a single module. Reserve root-level builds for integration-test runs or final pre-commit verification across the whole reactor.
