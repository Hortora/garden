### Symptom
`AmbiguousResolutionException` at Quarkus startup when two `@Produces` methods carry the same qualifier annotation with different `value()` attributes — e.g. `@Inference("splade")` and `@Inference("reranker")`.

### Context
Creating CDI producer methods for two different ONNX inference models, each identified by a qualifier annotation (`@Inference`) with a `@Nonbinding String value()`. The expectation is that `@Inference("splade")` and `@Inference("reranker")` are distinct qualifiers that route to different producers.

### What was tried
Two separate `@Produces` methods in the same class, each annotated with a different qualifier value:
```java
@Produces @Inference("splade") InferenceModel splade() { ... }
@Produces @Inference("reranker") InferenceModel reranker() { ... }
```

### Root cause
`@Nonbinding` on a qualifier's `value()` attribute means CDI ignores that attribute for bean resolution. All `@Inference` qualifiers are equivalent regardless of their value — `@Inference("splade")` and `@Inference("reranker")` resolve to the same bean type. Two producers for the same type + qualifier = ambiguity.

### Fix
Use a single `@Produces` method that accepts `InjectionPoint` and dispatches by reading the qualifier value at runtime:

```java
@Produces @Dependent @Inference("")
InferenceModel produce(InjectionPoint ip) {
    String name = ip.getAnnotated().getAnnotation(Inference.class).value();
    return switch (name) {
        case "splade" -> createSpladeModel();
        case "reranker" -> createRerankerModel();
        default -> throw new IllegalStateException("Unknown model: " + name);
    };
}
```

This is exactly the pattern used by `InferenceModelProducer` in `casehub-inference-quarkus`.

### Why this is non-obvious
The `@Nonbinding` annotation lives on the qualifier definition (in a different module), not on the producer. The error message (`AmbiguousResolutionException`) doesn't mention `@Nonbinding`. Developers unfamiliar with the CDI spec's treatment of `@Nonbinding` will try to debug the ambiguity by examining their producers — which appear distinct because the qualifier values are different.
