### What happened

`quarkus.arc.exclude-types` (and the build-time `ExcludedTypeBuildItem`) was used to
exclude `A2AResource` and `ReactiveA2AResource` from CDI to resolve a duplicate-endpoint
conflict. Both resources still registered their `@Path` routes, causing Quarkus to fail
at startup with duplicate route errors.

### Root cause

Quarkus uses two independent scanners: one for CDI beans (Arc) and one for JAX-RS
endpoints (RESTEasy Reactive). `quarkus.arc.exclude-types` removes a class from the CDI
container only. The JAX-RS scanner reads `@Path` annotations from all classes on the
Jandex index regardless of CDI exclusion — the class is still visible as a REST endpoint.

### Fix

Gate the resource class at the source using Quarkus build-time conditionals:

```java
// Reactive variant — only active when reactive datasource configured
@IfBuildProperty(name = "quarkus.datasource.qhorus.reactive", stringValue = "true")
@Path("/a2a")
public class ReactiveA2AResource { ... }

// Blocking variant — active by default (reactive flag absent or false)
@UnlessBuildProperty(name = "quarkus.datasource.qhorus.reactive", stringValue = "true",
    enableIfMissing = true)
@Path("/a2a")
public class A2AResource { ... }
```

Alternatively, exclude the class from Jandex entirely (prevents both CDI and JAX-RS
registration) using `quarkus.index-dependency` negation or by removing the class from
the classpath altogether.

### Why this is non-obvious

CDI exclusion sounds like it should prevent a class from being "seen" by the framework.
But in Quarkus, CDI and JAX-RS are registered through separate pipelines. Most users
discover this by noticing that excluded beans still serve HTTP traffic — which is alarming.

### Why this fix

`@IfBuildProperty`/`@UnlessBuildProperty` on the class itself is the correct Quarkus
pattern for build-time conditional registration of both CDI beans and JAX-RS resources.
It uses the same mechanism Quarkus extensions use internally, survives ahead-of-time
compilation, and keeps the intent visible at the class declaration site.

### Alternatives considered

- `quarkus.arc.exclude-types` — removes from CDI only; JAX-RS routes still register. Does not fix the problem.
- Removing from Jandex index entirely — works but is coarser; also removes the class from type-safe CDI injection lookups that may be needed elsewhere.
