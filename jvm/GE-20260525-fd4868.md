## What changed

In LangChain4j 1.x, `ChatLanguageModel` was renamed to `ChatModel`. Code that compiled against 0.x fails immediately with `cannot find symbol: class ChatLanguageModel`.

The import path is now `dev.langchain4j.model.chat.ChatModel`.

## Extension point also changed

In 0.x, `ChatLanguageModel` had abstract methods that implementations overrode directly (`generate()`). In 1.x, **all** `ChatModel` methods are `default`. The single extension point is `doChat(ChatRequest)` — override it in anonymous classes or implementations. Lambda stubs fail because `ChatModel` is not a functional interface.

```java
// 0.x — lambda worked (was a functional interface)
ChatLanguageModel stub = messages -> Response.from(AiMessage.from("ok"));

// 1.x — must use anonymous class, override doChat
ChatModel stub = new ChatModel() {
    @Override
    public ChatResponse doChat(ChatRequest request) {
        return ChatResponse.builder().aiMessage(AiMessage.from("ok")).build();
    }
};
```

## Convenience entry points

`ChatModel` retains convenience defaults: `chat(String)` builds a `ChatRequest` from the string and calls `doChat()`. Callers invoking `llm.chat(prompt)` continue to work — only the implementation class and override point changed.

## Why non-obvious

The rename is not surfaced in compiler errors — `ChatLanguageModel` simply doesn't exist in 1.x, so the error looks like a missing dependency rather than a rename. Searching for `ChatLanguageModel` in the new codebase returns nothing. The migration guide exists but isn't the first place developers look when their code stops compiling.
