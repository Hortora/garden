## Symptom

A REST endpoint returns JSON where a field contains `Path[value=legal/billing, segments=[legal, billing]]` instead of the expected `legal/billing` string.

## Context

When a Java record type (such as a domain value object) is included in a `Map<String, Object>` or similar generic response structure, Jackson/JSON-B calls `toString()` on the value rather than accessing its string representation directly. For compact, single-field records, `toString()` returns the full record representation including class name and component names.

## What was tried

Including the record field directly in a response map:

```java
return Response.ok(Map.of("path", definition.path)).build();
// Produces: {"path": "Path[value=legal/billing, segments=[legal, billing]]"}
```

## Root cause

`Map<String, Object>` serialization calls `toString()` on values of unknown type. Java's auto-generated record `toString()` returns `ClassName[field1=..., field2=...]`. For records with a single value component, callers expect the "bare" string but get the structured representation.

## Fix

Always extract the string value explicitly when building REST responses:

```java
// ✅
return Response.ok(Map.of("path", definition.path.value())).build();
```

Or use a typed response record/DTO that declares the field as `String` so serialization handles the conversion at a known boundary.

## Why this is non-obvious

The field appears to hold a simple string value in the code — `path.value()` is what you store and display. Calling `toString()` on any record produces structured output, not just the primary component. The failure only appears at runtime by inspecting actual JSON output, not at compile time.
