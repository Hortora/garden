## What exists

`io.casehub.qhorus.runtime.channel.ChannelService` (blocking) exposes:

```java
Optional<Channel> findByName(String name)    // exact name lookup, more efficient than prefix
Optional<Channel> findById(UUID id)          // UUID primary key lookup
```

Both are in the blocking `ChannelService`; `ReactiveChannelService` mirrors them.

## Why it matters

The casehub-qhorus platform deep-dive only documents `findByNamePrefix(String prefix)`. For exact channel name lookup (e.g. idempotent `openChannel()`) or channel-by-UUID lookup (e.g. delivery webhook validation), `findByName` and `findById` are more efficient and semantically correct than `findByNamePrefix("")` scanning all channels.
