### Symptom

`ArithmeticException: long overflow` when executing a JPQL query parameterized with
`Instant.MAX` against a `TIMESTAMP WITH TIME ZONE` column.

### Context

Using `Instant.MAX` as a JPQL parameter for bulk DELETE operations (e.g. test cleanup
via `removeExpired(Instant.MAX)` to clear all rows). The query itself is valid — the
overflow happens inside Hibernate's type conversion, not in user code.

### What was tried (didn't work)

Passing `Instant.MAX` directly as a JPQL parameter:

```java
@BeforeEach
void cleanUp() {
    store.removeExpired(Instant.MAX).await().indefinitely();
}
```

Throws `ArithmeticException` before the query reaches PostgreSQL.

### Root cause

Hibernate ORM converts `java.time.Instant` to `java.sql.Timestamp` internally when
binding JPQL parameters. `Instant.MAX` represents year +1000000000 — its epoch
millisecond value overflows `long` range when `Timestamp.from(Instant)` calls
`instant.toEpochMilli()`. The conversion is inside Hibernate's type system, not
exposed to user code.

PostgreSQL's `TIMESTAMP WITH TIME ZONE` supports years up to 294276 — well within
range. But Hibernate's intermediate `java.sql.Timestamp` conversion truncates the
usable range to `Long.MAX_VALUE` milliseconds (year ~292278994), and `Instant.MAX`
exceeds even that.

### Fix

Use a far-future constant instead of `Instant.MAX`:

```java
private static final Instant FAR_FUTURE = Instant.parse("9999-12-31T23:59:59Z");

@BeforeEach
void cleanUp() {
    store.removeExpired(FAR_FUTURE).await().indefinitely();
}
```

Year 9999 is well within both PostgreSQL's and `java.sql.Timestamp`'s range.

### Why this is non-obvious

`Instant.MAX` is the natural Java way to express "the latest possible time." It works
fine with `Instant.isBefore()`, `Instant.isAfter()`, and all `java.time` comparisons.
The overflow only surfaces when Hibernate converts to `java.sql.Timestamp` — a layer
below the JPA API surface. The error message (`ArithmeticException: long overflow`)
doesn't mention Hibernate, Timestamp, or the conversion.
