## Symptom

After sending a HANDOFF message for a given `correlationId`, querying `CommitmentService.findByCorrelationId(correlationId)` returns a commitment in state `OPEN`, not `DELEGATED`. Code that assumes DELEGATED state after a HANDOFF will read the wrong state and make incorrect decisions.

## Context

Implementing A2A task state derivation: `GET /a2a/tasks/{id}` queries the `CommitmentStore` to determine whether to report `"submitted"`, `"working"`, `"completed"`, or `"failed"`. After a HANDOFF, the intuitive expectation is that `findByCorrelationId` returns the parent commitment in state `DELEGATED`.

## Root cause

When a HANDOFF message is processed, the parent commitment transitions to `DELEGATED` (terminal), **and** a new child commitment is created for the delegate with the same `correlationId` in state `OPEN`. `CommitmentService.findByCorrelationId` returns the most recent active (non-terminal) commitment for that ID — which is the child OPEN, not the parent DELEGATED.

## Fix

Do not rely on `fromCommitmentState(DELEGATED)` to detect delegation at the HTTP layer. Instead, guard against `OPEN` state explicitly and fall through to message-history derivation:

```java
// WRONG — simple null check; OPEN child commitment after HANDOFF yields "submitted"
String state = (commitment != null)
        ? A2ATaskState.fromCommitmentState(commitment.state)
        : A2ATaskState.fromMessageHistory(messages);

// CORRECT — OPEN commitment deferred to message history, which finds the HANDOFF
String state = (commitment != null && commitment.state != CommitmentState.OPEN)
        ? A2ATaskState.fromCommitmentState(commitment.state)
        : A2ATaskState.fromMessageHistory(messages);
```

`fromMessageHistory` correctly maps the HANDOFF message to `"working"` via the last-message heuristic.

## Why this is non-obvious

`CommitmentState.DELEGATED` is listed as terminal in `CommitmentState.isTerminal()`. It is natural to assume `findByCorrelationId` would return this terminal parent. The child commitment creation is an implementation detail of the commitment lifecycle and is not surfaced in the API signature.
