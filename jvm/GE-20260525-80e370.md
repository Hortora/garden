## Symptom

Compile error: `cannot find symbol: method text()` when calling `.text()` on a `ChatMessage`.

## Context

Processing messages from a `ChatRequest` in a `doChat(ChatRequest)` override, trying to extract the text content to inspect or capture it in a test.

```java
// Fails — ChatMessage has no text() method
String content = request.messages().get(0).text();  // ← compile error
```

## Root Cause

`ChatMessage` is an interface with no `text()` accessor. Text content is accessible on the concrete `UserMessage` type only, via `singleText()` — not `text()`.

## Fix

Cast to `UserMessage` and call `singleText()`:

```java
// Correct
request.messages().stream()
    .filter(m -> m instanceof UserMessage)
    .map(m -> ((UserMessage) m).singleText())
    .collect(Collectors.joining());
```

## Why non-obvious

The method name change from `text()` to `singleText()` is a 1.x change. The `UserMessage` constructor accepts a plain `String`, so it reads naturally as if `.text()` should return it — but it doesn't. The method name `singleText()` signals that `UserMessage` can contain multiple `Content` objects and this returns the first text content.
