## Symptom

`IllegalArgumentException: EVENT messages must not carry content — use STATUS for
content-bearing observe-channel broadcasts.` thrown from `MessageDispatch.Builder.build()`.

Tests that dispatch EVENT messages with `.content("...")` fail at build time, not at
dispatch time.

## Context

Protocol PP-20260508-90428f established that EVENT messages carry no content — they are
pure signals. The `MessageObserverDispatcher` forces `content=null` on delivery (covered
by GE-20260607-d051f2). The builder-side enforcement was added later: `Builder.build()`
now rejects EVENT messages that carry `.content(...)`.

Tests written before this enforcement used patterns like:
```java
messageService.dispatch(MessageDispatch.builder()
    .channelId(observe.id)
    .sender("ledger-audit")
    .type(EVENT)
    .content("{\"entry\":\"audit-record\"}")  // ← now throws
    .actorType(ActorType.SYSTEM)
    .build());
```

## Root cause

`MessageDispatch.Builder.build()` validates that EVENT messages have null content.
The validation is at builder level, so the exception occurs before any CDI or channel
logic runs.

## Fix

Remove `.content(...)` from EVENT dispatches. If the message needs to carry content,
use `STATUS` type instead — that is the correct type for content-bearing observe-channel
broadcasts.

## Why this is non-obvious

The tests passed before the builder validation was added. The failure appears as a test
regression with no code change in the test itself — only the qhorus dependency version
changed. The error message is clear, but the root cause (a SNAPSHOT dependency quietly
tightened its validation) is not visible without inspecting the dependency diff.
