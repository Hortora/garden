### Symptom
Tests pass in isolation, fail in suite — tenant-scoped store queries return empty `Optional` or empty `List`, not errors. No CDI exception, no `NullPointerException`. The failure is completely silent.

### Context
A `MutableCurrentPrincipal` test bean (`@ApplicationScoped @Alternative @Priority(100)`) provides mutable `tenancyId` for tenancy isolation tests. Tenancy tests call `setTenancyId(TENANT_B)` and reset via `@BeforeEach` within their own class. But `@ApplicationScoped` beans survive across `@QuarkusTest` classes — after the last test method of a tenancy test class, the bean retains `TENANT_B`. Subsequent test classes expecting `DEFAULT_TENANT_ID` silently get wrong query results.

### What was tried (didn't work)
- Adding `@AfterEach` to tenancy test classes — fragile, easy to miss for new tests
- Making each non-tenancy test inject and reset the principal — doesn't scale

### Root cause
`@ApplicationScoped` CDI beans persist across `@QuarkusTest` classes within the same Quarkus test application lifecycle. JUnit test class ordering is non-deterministic. A tenancy test class that sets `tenancyId = TENANT_B` in its last test method leaves the bean in that state for all subsequent classes.

### Fix
Register a `QuarkusTestBeforeEachCallback` via ServiceLoader that resets the principal before every test method across all test classes:

```java
public class MutablePrincipalResetCallback implements QuarkusTestBeforeEachCallback {
    @Override
    public void beforeEach(QuarkusTestMethodContext context) {
        var container = Arc.container();
        if (container != null) {
            var handle = container.instance(MutableCurrentPrincipal.class);
            if (handle.isAvailable()) {
                handle.get().reset();
            }
        }
    }
}
```

Register in `META-INF/services/io.quarkus.test.junit.callback.QuarkusTestBeforeEachCallback`.

### Why this is non-obvious
The failure is silent — tenant-scoped queries filter by `tenancyId` and return empty results when the ID doesn't match. No exception is thrown. The symptoms (empty Optional, empty List) look like missing test data, not wrong tenant context. You only discover the real cause when test ordering changes or a new tenancy test class is added.

### Why this fix
A ServiceLoader callback is infrastructure-level — it fires before every test method in every `@QuarkusTest` class without requiring any per-class wiring. Individual `@BeforeEach` resets are fragile (miss one class, silently break another). The callback makes the problem structurally impossible.

### Alternatives considered
- Per-class `@BeforeEach` reset: fragile, doesn't scale
- `@RequestScoped` principal: would need request context activation in every test, breaks `@ApplicationScoped` store injection patterns
- JUnit `@Order` on test classes: masks the problem, doesn't fix it
