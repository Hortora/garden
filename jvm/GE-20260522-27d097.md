## What exists but isn't documented

SC2 stores unit training times as integer game loops internally. The widely used community formula `trainTime = seconds × 22.4` (where 22.4 is the Faster-speed game-loop rate) produces floating-point values that are systematically wrong — and not in a consistent direction.

**Actual values (calibrated from 29 AI Arena replays, ~500 Probe observations, AI vs AI Protoss games, build ~67188):**

| Unit | Formula result | Actual loop count |
|------|---------------|------------------|
| Probe (12s) | 268.8 | **272** |
| Zealot (28s) | 627.2 | **618** |
| Stalker (31s) | 694.4 | **698** |
| Immortal (40s) | 896.0 | 896 (exact) |
| Observer (22s) | 492.8 | uncalibrated |

The discrepancy is non-trivial (up to 9.2 loops for Zealot) and non-uniform — some values round up from the formula, some round down. You cannot derive the correct integer by simply applying `ceil()` or `round()`.

## Why this matters

Using float values in a floor-division formula for completion tick calculation causes off-by-one timing errors at boundary loop offsets. For Probe at offset 17:
- Float formula: `floor((17 + 268.8) / 22) = floor(12.99) = 12` (wrong)
- Integer formula: `(17 + 272) / 22 = floor(13.14) = 13` (correct)

This produces `firstUnitDivergenceTick = 86` instead of 150+ in replay validation — a systematic 1-tick-early completion that looks like a formula bug but is actually a wrong constant.

## How to discover correct values

Empirically calibrate from replay data using paired GAME_EVENTS training commands and tracker UnitBorn events. The range-bounded modal algorithm is robust to queued units and missed commands (see GE-20260522-a575c3).

## Why this isn't in the docs

SC2 exposes training times in the in-game UI and in community wikis as "X seconds at Faster speed." The loop count is an internal engine value not surfaced in any official API or documentation. The 22.4 loops/second value is accurate for the display rate but the training duration stored in game data is a different (integer) quantity derived from the game's internal timing engine.
