# Test @RequestScoped CDI beans directly with @InjectMock + manual Arc request context ŌĆö no HTTP layer needed

### Context
Testing a `@RequestScoped` CDI bean that injects other `@RequestScoped` beans
(e.g. `SecurityIdentity`, `JsonWebToken`). The default approach is to go through
HTTP ŌĆö set up a test REST resource, use `@TestSecurity`, call via RestAssured.
This adds significant scaffolding and requires OIDC-specific test infrastructure.

### Technique
Use `@InjectMock` from `quarkus-junit-mockito` to replace the injected beans with
Mockito mocks, and manually activate the request context in `@BeforeEach`:

```java
@QuarkusTest
class OidcCurrentPrincipalTest {

    @Inject CurrentPrincipal principal;       // @RequestScoped proxy

    @InjectMock SecurityIdentity identity;    // replaces real @RequestScoped bean
    @InjectMock JsonWebToken jwt;             // replaces real @RequestScoped bean

    @BeforeEach
    void activateRequestContext() {
        Arc.container().requestContext().activate();
    }

    @AfterEach
    void terminateRequestContext() {
        Arc.container().requestContext().terminate();
    }

    @Test
    void authenticated_actorId_returns_principal_name() {
        when(identity.isAnonymous()).thenReturn(false);
        when(identity.getPrincipal()).thenReturn(() -> "alice");

        assertEquals("alice", principal.actorId());
    }
}
```

`@InjectMock` creates an application-scoped Mockito mock that replaces the CDI
bean regardless of the original bean's scope. `Arc.container().requestContext().activate()`
gives the `@RequestScoped` target bean a context to live in when methods are called.

### Why this is non-obvious
`@TestSecurity` only works in the HTTP request context (PP-20260513-7c227e) ŌĆö it
has no effect when calling CDI beans directly without RestAssured. The natural
alternative (HTTP test via RestAssured) requires a test REST resource, `@TestSecurity`,
and for JWT claims specifically `quarkus-test-security-oidc`. The `Arc.container()
.requestContext().activate()` pattern avoids all of this and is significantly simpler,
but most Quarkus testing docs focus on the HTTP path.

### Caveats
- `@InjectMock` creates an *application-scoped* mock regardless of original bean scope
- `Arc.container().requestContext().terminate()` must be called in `@AfterEach` to
  prevent scope leaking between tests
- For generic methods with type parameters (e.g. `JsonWebToken.claim(String)`), use
  `doReturn(Optional.of(value)).when(mock).method(arg)` instead of
  `when(mock.method(arg)).thenReturn(value)` to avoid unchecked cast warnings
