## The Technique

When assembling a response from N independent backends that run concurrently, define a typed visitor/accumulator interface rather than a shared mutable response object. Contributors call typed accumulator methods (`acc.trustScore()`, `acc.workItem()`, `acc.commitment()`) instead of returning objects to be merged. The accumulator implementation is thread-safe by construction.

```java
// Interface (stdlib types only — no domain objects)
public interface ActorStateAccumulator {
    void trustScore(Double score);               // null = unscored
    void workItem(UUID id, String title, ...);
    void commitment(UUID id, UUID channelId, ...);
}

// Thread-safe implementation
class ActorStateAccumulatorImpl implements ActorStateAccumulator {
    private final AtomicReference<Double> trustScore = new AtomicReference<>(null);
    private final CopyOnWriteArrayList<WorkItemSummary> workItems = new CopyOnWriteArrayList<>();
    private final ConcurrentHashMap<String, String> warnings = new ConcurrentHashMap<>();
    // ... typed methods populate fields ...
}

// Aggregator runs contributors concurrently
contributors.forEach(c -> CompletableFuture.runAsync(() -> {
    try {
        c.contribute(actorId, accumulator);
        accumulator.markSucceeded(c.sourceName());
    } catch (Exception e) {
        accumulator.markFailed(c.sourceName(), ...);
    }
}, managedExecutor));
CompletableFuture.allOf(futures).join();
return accumulator.build();
```

## Why Non-obvious

The natural approach is to make each contributor return a partial `Map<String, Object>` or a typed partial response, then merge them. This requires either synchronization on the merge step or collecting all futures before touching shared state. The visitor approach is cleaner: contributors never see each other's data; the accumulator encapsulates all thread-safety; and the interface becomes the contract for what data each backend can contribute — no `instanceof` checks, no map key strings.

## Why This Fix

The typed interface also enforces null contracts at the API level (`trustScore(Double)` accepting null is explicit; `trustScore(double)` would prevent it). The accumulator's `build()` method runs on a single thread after all contributors finish, making response assembly simple and safe.

## Constraints

Works best when contributors are truly independent (different backends, different transaction contexts). If contributors must share data with each other, the pattern breaks down.
