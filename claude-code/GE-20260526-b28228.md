## Symptom
`work-start` and `handover` resolve the wrong workspace path, pointing to a parent project's workspace instead of the active session's.

## Context
Any Claude Code session in a repository that has ancestor CLAUDE.md files (e.g. a project repo whose parent directory also has a CLAUDE.md).

## Root cause
Claude Code loads all CLAUDE.md files from the repo root upward — global, parent project, and project-level. When session skills scan CLAUDE.md for a `**Workspace:**` declaration, they find the parent project's declaration first. It silently returns the wrong path with no error.

## Fix
Never derive workspace path from CLAUDE.md content. Always use:
```bash
git rev-parse --show-toplevel
```
from the session's CWD. That is always the correct workspace root.

## Why non-obvious
CLAUDE.md feels like the authoritative source of project configuration. The multi-file loading behaviour is not surfaced in the UI — you see only the merged result, not which file contributed which value. Skills that worked correctly in single-repo setups silently break in multi-repo setups.
