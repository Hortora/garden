## Project-local .claude/settings.json creates an isolated Claude Code agent context with pre-approved commands

**Stack:** Claude Code
**What it does:** Placing a `.claude/settings.json` in any directory creates a fully isolated Claude Code agent context when `claude` is invoked from that directory. Pre-approved commands and `acceptEdits` mode mean the agent runs without any permission prompts — completely independent of any parent Claude Code session.

### What it looks like

```json
// ~/.hortora/garden/.claude/settings.json
{
  "defaultMode": "acceptEdits",
  "permissions": {
    "allow": [
      "Bash(git show *)",
      "Bash(git add *)",
      "Bash(git commit *)",
      "Bash(python3 */dedupe_scanner.py *)"
    ],
    "deny": []
  }
}
```

When `claude --print "task"` or `claude "task"` is invoked with `cwd` set to `~/.hortora/garden/`, Claude Code reads this settings file and operates within those boundaries — no prompts, no interaction with the settings of any other session running on the machine.

### Pattern: background autonomous agent

```bash
# garden-agent.sh — runs as background agent, no terminal, no prompts
cd ~/.hortora/garden
nohup claude --print "Run the dedup sweep" >> garden-agent.log 2>&1 &
```

Combined with a git post-commit hook, this creates a fully autonomous background agent that fires on every commit without blocking the user or consuming tokens from the parent session.

### Why non-obvious

There is no documentation describing that directory-local `.claude/settings.json` creates an isolated context. The Claude Code docs describe global and project-level settings, but the implication that each `cwd` gets its own independent permission scope — enabling true multi-agent isolation on a single machine — is not stated anywhere.

### Why this fix

The alternative (running the agent in the same session) blocks R&D work and consumes context window. The local settings.json approach gives the background agent exactly the permissions it needs, nothing more, with zero interaction with the foreground session.
