### Symptom

You reduce a CLAUDE.md from 587 to 528 lines (10% reduction) and assume the
context window cost dropped proportionally. The file is still 51 KB — unchanged.

### Root cause

Line count is a proxy for byte count only when lines are uniform length. CLAUDE.md
files have two zones: short structural lines (headings, table rows) and long prose
lines (gotcha bullets, build notes). Removing 60 short lines while leaving dense
prose untouched barely moves the byte count.

Claude Code's 40 KB warning threshold is a **byte check**, not a line check.

### Fix

Always measure both:

```bash
wc -l CLAUDE.md   # lines — useful for structure review
wc -c CLAUDE.md   # bytes — the number that actually matters for context
```

Do the line review for readability; do the byte audit for actual context window cost.
The byte count is what Claude checks at session start.
