## torch.onnx.export renamed use_external_data_format to external_data — silent TypeError

### Symptom

`torch.onnx.export()` raises `TypeError: export() got an unexpected keyword argument 'use_external_data_format'` on PyTorch 2.12+.

### Context

When exporting large ONNX models (>2GB, e.g. BGE-M3 at 550M parameters), the weights exceed protobuf's 2GB serialization limit. The export must use external data format, which produces `model.onnx` (graph, ~3MB) + `model.onnx.data` (weights, ~2.2GB).

### Root cause

PyTorch 2.12 renamed the `use_external_data_format` parameter to `external_data` in `torch.onnx.export()`. No deprecation warning was added — the old name simply raises a TypeError. The rename is not documented in the PyTorch 2.12 migration guide.

### Fix

```python
# Before (PyTorch < 2.12)
torch.onnx.export(model, args, path, use_external_data_format=True)

# After (PyTorch 2.12+)
torch.onnx.export(model, args, path, external_data=True)
```

To discover the current parameter names:
```python
import inspect
print(list(inspect.signature(torch.onnx.export).parameters.keys()))
```

### Why this is non-obvious

- No deprecation warning — the old parameter name fails with a generic TypeError, not a "deprecated, use X instead" message
- The rename is not in the PyTorch 2.12 migration guide or changelog
- Most ONNX export tutorials and Stack Overflow answers still use `use_external_data_format`
- The error message names the rejected parameter but doesn't suggest the replacement

### Alternatives considered

- Checking PyTorch source code diff between versions — works but slow
- Using `inspect.signature()` — the approach that actually worked, showing the full parameter list including `external_data`
