## Technique

To test an `except ImportError` path in a script run via `subprocess.run()`, create a fake module file that raises `ImportError` at module level, prepend its directory to `PYTHONPATH`, and run the subprocess with that environment. Python's import machinery loads the fake module first, hits the `ImportError`, and the script's `except ImportError` clause fires cleanly.

```python
import os, subprocess, sys
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as fake_dir:
    (Path(fake_dir) / 'yaml.py').write_text(
        "raise ImportError('yaml not available in test')\n"
    )
    env = os.environ.copy()
    env['PYTHONPATH'] = f"{fake_dir}:{env.get('PYTHONPATH', '')}"
    result = subprocess.run(
        [sys.executable, 'script_under_test.py', ...],
        capture_output=True, text=True, env=env
    )
```

## Why this is non-obvious

The standard approach (`unittest.mock.patch('yaml', None)` or patching `sys.modules`) only affects the current process — it has no effect on child processes spawned by `subprocess.run()`. Most developers reach for `mock.patch` first and discover it doesn't cross subprocess boundaries. The PYTHONPATH shadowing approach requires knowing that Python resolves imports from `PYTHONPATH` entries before installed packages.
