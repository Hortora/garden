## What
`validate_examples.py` uses the pattern `{[^}]*}` to detect "template markers" in code blocks — content like `{placeholder}`, `{your-value}`, `<your-key>` that indicates an example isn't real code. Any fenced JSON block whose entire content matches this pattern is silently skipped without being validated.

## The trap
Standard invalid JSON like `{"key": "value",}` (trailing comma) or `{"bad": json}` triggers the template check and is silently skipped — the validator never reaches the JSON parsing step and emits no WARNING.

## Correct approach for testing the WARNING path
Use array-format JSON that cannot match `{[^}]*}`:
```python
# This is silently skipped (matches template pattern):
content = '```json\n{"key": "value",}\n```\n'

# This reaches the JSON validator and produces a WARNING:
content = '```json\n[1, 2, 3,]\n```\n'
```

## Why undocumented
The template-skip logic is an implementation detail not described in any README or docstring. The only indicator is a comment in the source. Without reading the code, a developer writing tests would use object-format JSON and be confused when no WARNING is produced.
