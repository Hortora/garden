## LinkedHashMap as a zero-dependency LRU cache

### Technique

`LinkedHashMap` has a constructor that enables access-order mode (`accessOrder=true`).
In this mode the map reorders entries on every `get()` and `put()` — most-recently-used
entries move to the tail. Overriding `removeEldestEntry()` makes it evict the
least-recently-used entry (head) when the map exceeds a size bound:

```java
private final Map<String, float[]> cache;

EmbeddingCache(final int maxSize) {
    this.cache = new LinkedHashMap<>(maxSize, 0.75f, true /* accessOrder */) {
        @Override
        protected boolean removeEldestEntry(final Map.Entry<String, float[]> eldest) {
            return size() > maxSize;
        }
    };
}

public float[] getOrCompute(final String key, final Provider provider) {
    synchronized (cache) {
        final float[] cached = cache.get(key);
        if (cached != null) return cached;
    }
    // Compute outside the lock — avoid holding it during slow I/O
    final float[] computed = provider.compute(key);
    synchronized (cache) {
        cache.putIfAbsent(key, computed);
        return cache.get(key);
    }
}
```

`removeEldestEntry` is called automatically by `put()` after insertion — no explicit
eviction logic needed.

### Why this is non-obvious

Most developers reach for Caffeine or Guava for in-process caching. Both require build
dependencies and add configuration surface area. `LinkedHashMap` with `accessOrder=true`
is in `java.util` since Java 1.4 and covers the most common case — bounded LRU — with
four lines of anonymous-class override.

The `accessOrder` parameter in the `LinkedHashMap(int, float, boolean)` constructor is
the non-obvious entry point. Javadoc describes it but developers rarely reach the
three-argument constructor when thinking "I need a cache."

### When NOT to use this

- **TTL expiry** — `LinkedHashMap` has no time-based eviction. Use Caffeine.
- **Concurrent high-throughput** — `synchronized` becomes a bottleneck under heavy
  contention. Use `ConcurrentHashMap` with `computeIfAbsent` + a size-bounded wrapper, or Caffeine.
- **Statistics / monitoring** — Caffeine and Guava provide hit/miss metrics. `LinkedHashMap` doesn't.
