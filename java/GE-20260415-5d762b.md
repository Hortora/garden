## Symptom

A class stores a `boolean[][]` field with `this.walkable = Arrays.copyOf(walkable, walkable.length)` in its constructor. The field appears safely copied but callers that mutate the original array after construction see changes reflected in the stored grid. The object is not immutable despite using `Arrays.copyOf`.

## Context

`Arrays.copyOf` on an array of arrays (or array of objects) copies the outer array to a new reference but copies the inner element references — not the inner arrays themselves. The inner `boolean[]` rows are shared between the original and the copy.

## What was tried (didn't work)

`Arrays.copyOf(walkable, walkable.length)` — produces a shallow copy only.

## Root cause

Java arrays are reference types. `boolean[][]` is an array of `boolean[]` references. `Arrays.copyOf` creates a new outer array with the same inner-array references, not deep copies.

## Fix

Perform a row-by-row copy in the constructor:

```java
this.walkable = new boolean[width][height];
for (int x = 0; x < width; x++) {
    this.walkable[x] = Arrays.copyOf(walkable[x], height);
}
```

`Arrays.copyOf` on a `boolean[]` (1D) IS a true copy of primitive values. Only the outer loop is needed to make the copy deep.

## Why this is non-obvious

`Arrays.copyOf` is widely used for defensive copies. The name implies completeness. The distinction between "copy of array of primitives" (deep) vs "copy of array of arrays" (shallow) is easy to overlook, especially when the method signature accepts `T[]` generically and the 2D case is not highlighted in the Javadoc.
