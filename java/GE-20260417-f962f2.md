## Symptom

Porting Python URL path splitting to Java produces an off-by-one in the
path segments array. What was `parts[0]` in Python becomes `parts[1]` in
Java, and `parts.length >= 2` in Python becomes `parts.length >= 3`.

## Context

Porting code that parses a URL path into segments — for example, extracting
`user` and `gist-id` from `https://gist.github.com/user/abc123.js`.

Python:
```python
parts = parsed.path.strip('/').split('/')
# "/user/abc123.js".strip('/') → "user/abc123.js"
# .split('/') → ["user", "abc123.js"]
# parts[0] = "user", parts[1] = "abc123.js"
```

Java:
```java
String[] parts = uri.getPath().split("/");
// "/user/abc123.js".split("/") → ["", "user", "abc123.js"]
// parts[0] = "", parts[1] = "user", parts[2] = "abc123.js"
```

Java's `String.split("/")` splits on every `/`, including the one at the
start of the path, producing a leading empty string `""` in `parts[0]`.

## What was tried

Using the same index offsets as the Python code — result is wrong silently
(a blank "user" field is treated as a valid value).

## Root cause

Python's `str.strip('/')` removes all leading and trailing slashes before
splitting. Java's `URI.getPath()` returns the raw path string with the
leading `/` intact. `String.split("/")` produces a leading empty string
for any string that begins with the delimiter.

## Fix

Strip the leading slash before splitting:

```java
String path = uri.getPath().replaceAll("^/+", "").replaceAll("/+$", "");
String[] parts = path.split("/");
// "user/abc123.js".split("/") → ["user", "abc123.js"]
// parts[0] = "user", parts[1] = "abc123.js"
```

Or equivalently: `uri.getPath().strip().replaceFirst("^/+", "")`.

The Python-equivalent in a single line:
```java
// Python: parts = parsed.path.strip('/').split('/')
String[] parts = uri.getPath().replaceAll("^/+|/+$", "").split("/");
```

## Why this is non-obvious

Python developers porting path-splitting code don't expect the leading-slash
behaviour because Python's idiom (`strip('/').split('/')`) handles it
transparently. Java's `split()` on a leading-delimiter string is documented,
but the contrast with Python doesn't register until you see wrong output in
a test.
