## Symptom

A code generator that synthesises delegation methods for interface implementations produces classes that fail to compile with: `attempting to assign weaker access privileges; was public`.

## Context

Any code generator (annotation processor, JavaParser-based, AST transformer) that synthesises concrete method bodies implementing an interface. The synthesised methods omit `public` because the source AST (the interface) doesn't have `public` on its method declarations — interface methods are implicitly public.

## Root cause

In Java, interface methods are implicitly `public`. When a code generator reads the interface's method signatures using a parser/reflection API, the `public` modifier is absent from the method node. Naively copying that signature into the implementing class produces a method with package-private access, which is a weaker access level than the interface contract — a compile error.

## Fix

When synthesising delegation methods that implement an interface, always add `public` explicitly:

```java
// Reading from source interface — method has no explicit 'public'
MethodDeclaration srcMethod = sourceInterface.getMethods().get(0);
// srcMethod.getModifiers() → [] (empty — implicitly public in interface)

// WRONG: copy modifiers directly
String methodSrc = srcMethod.getModifiers() + " " + srcMethod.getType() + " ...";
// → "void result(A a)" — package-private in the class, compile error

// CORRECT: always add public for interface implementations
String methodSrc = "public " + srcMethod.getType() + " " + srcMethod.getNameAsString() + ...;
```

In JavaParser, check the source type: if `sourceType instanceof ClassOrInterfaceDeclaration coid && coid.isInterface()`, add `public` modifier.

## Why this is non-obvious

When reading interface methods from source AST or reflection, the absence of `public` in the method node is technically correct — Java omits it. But this creates a trap for any code generator that copies method signatures: the correct behaviour in an interface context becomes incorrect in a class context. The `public` must be synthesised, not copied.
