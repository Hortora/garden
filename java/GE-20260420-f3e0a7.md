## Use `jar tf` to verify a class exists in an installed Maven artifact before attempting a compile

**ID:** GE-20260420-f3e0a7
**Stack:** Maven, JDK (jar command, all versions)
**Context:** A dependency updates its API and removes or renames a class. Compilation
fails with "cannot find symbol" or similar. Before reading changelogs, grepping
source trees, or attempting a recompile, you can instantly verify whether the class
still exists in the installed jar.

### The technique

```bash
jar tf ~/.m2/repository/<group>/<artifact>/<version>/<artifact>-<version>.jar \
  | grep ClassName
```

**Examples:**

```bash
# Check if LedgerHashChain still exists in quarkus-ledger
jar tf ~/.m2/repository/io/quarkiverse/ledger/quarkus-ledger/1.0.0-SNAPSHOT/quarkus-ledger-1.0.0-SNAPSHOT.jar \
  | grep LedgerHashChain

# Check if ObservabilitySupplement still exists
jar tf ~/.m2/repository/io/quarkiverse/ledger/quarkus-ledger/1.0.0-SNAPSHOT/quarkus-ledger-1.0.0-SNAPSHOT.jar \
  | grep -i "observability\|supplement"

# List all classes in a package
jar tf ~/.m2/repository/com/example/my-lib/1.0/my-lib-1.0.jar \
  | grep "^io/example/mypackage/"
```

**Output:** Each matching line is a `.class` path inside the jar, e.g.
`io/quarkiverse/ledger/runtime/service/LedgerHashChain.class`.
No output = class does not exist in this artifact version.

### Why non-obvious
When compilation fails with "cannot find symbol", the instinct is to check
`pom.xml` for a missing or misconfigured dependency, not to inspect the jar's
contents directly. `jar tf` is the `ls` of jar files — fast, no extraction
needed, works on any jar in the local repo. Most developers reach for IDE
"navigate to class" or `mvn dependency:resolve` before thinking of this.

### Alternatives considered
- **`mvn dependency:resolve` + reading changelogs** — slower, doesn't confirm
  what's actually in the installed artifact (changelogs may be wrong or absent
  for local snapshots)
- **Unzipping the jar** — works but leaves files on disk; `jar tf` lists without
  extracting
- **IDE "Navigate to class"** — only works if the IDE has indexed the dependency;
  fails for locally-installed snapshots not yet indexed
