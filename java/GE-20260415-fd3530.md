## Context

A class has 2–4 switch statements that all switch on the same key (e.g. an event suffix string) and each returns a different field from a conceptually-related tuple. Adding a new case means updating N switch statements — easy to forget one.

## Technique

Replace N parallel switches with a single `Map<Key, String[]>` where the array index represents each field. Named integer constants make the array indices readable:

```java
// BEFORE — 3 parallel switches over the same key, must stay in sync:
private String deriveCommandType(String eventType) {
    return switch (suffix(eventType)) {
        case "created" -> "CreateWorkItem";
        case "completed" -> "CompleteWorkItem";
        // ... 10 more cases
        default -> null;
    };
}
private String deriveEventType(String eventType) {
    return switch (suffix(eventType)) {
        case "created" -> "WorkItemCreated";
        case "completed" -> "WorkItemCompleted";
        // ... 10 more cases
        default -> null;
    };
}
private String deriveActorRole(String eventType) {
    return switch (suffix(eventType)) {
        case "created" -> "Initiator";
        case "completed" -> "Resolver";
        // ... 10 more cases
        default -> "Assignee";
    };
}

// AFTER — single map, one place to add new cases:
private static final Map<String, String[]> EVENT_META = Map.ofEntries(
    Map.entry("created",   new String[] { "CreateWorkItem",   "WorkItemCreated",   "Initiator" }),
    Map.entry("completed", new String[] { "CompleteWorkItem", "WorkItemCompleted", "Resolver"  }),
    // ... remaining cases
);

private static final int META_COMMAND = 0;
private static final int META_EVENT   = 1;
private static final int META_ROLE    = 2;

private String deriveCommandType(String eventType) {
    String[] m = EVENT_META.get(suffix(eventType));
    return m != null ? m[META_COMMAND] : null;
}
private String deriveEventType(String eventType) {
    String[] m = EVENT_META.get(suffix(eventType));
    return m != null ? m[META_EVENT] : null;
}
private String deriveActorRole(String eventType) {
    String[] m = EVENT_META.get(suffix(eventType));
    return m != null ? m[META_ROLE] : "default";
}
```

`Map.ofEntries` supports up to any number of entries (unlike `Map.of` which caps at 10).

## Why This Is Non-Obvious

The instinct is to refactor switch statements into polymorphism or enums. But when the variants are string-keyed (e.g. event suffix from a CDI event type string) and the tuples are fixed-size, a lookup map with index constants is simpler and more maintainable — one row per variant, impossible to add a case to one switch and forget another.

The named int constants (`META_COMMAND = 0`) preserve readability without requiring a record or class per entry.
