## What it does

`Jsoup.parse(xml, "", Parser.xmlParser())` parses XML content preserving element case, namespace declarations, and processing instructions. Without it, jsoup's default HTML parser lowercases all element names — `<loc>` becomes `<loc>` (fine), but `<urlset>` becomes `<urlset>` (also fine in this case), and namespace-prefixed elements like `<image:loc>` lose their prefix silently.

## Where it's needed

Sitemap XML parsing (`sitemap.xml`, `sitemap-index.xml`), RSS/Atom feeds, and any XML that uses namespace-prefixed elements or is case-sensitive.

```java
// WRONG — HTML parser lowercases tags, breaks namespace prefixes
Document doc = Jsoup.parse(sitemapXml, baseUrl);

// RIGHT — XML parser preserves case and namespaces
Document doc = Jsoup.parse(sitemapXml, "", Parser.xmlParser());

// Then select normally:
Elements locs = doc.select("url > loc");           // sitemap URLs
Elements items = doc.select("item > link");        // RSS items
Elements entries = doc.select("entry > link[href]"); // Atom entries
```

## Why it's undocumented

The jsoup Javadoc for `parse(String, String)` and `parse(String, String, Parser)` both exist, but the XML parser variant is only mentioned in the class-level Javadoc for `Parser` — not in the main `Jsoup` class Javadoc where developers look first. The HTML parser silently produces a usable (but subtly wrong) result for well-formed XML, so the bug only surfaces for namespace-prefixed content or strict element-name matching.
