## Symptom

`java.util.zip.ZipInputStream` throws `java.util.zip.ZipException: invalid compression method` when reading a ZIP entry. The error message gives no indication that BZip2 is involved.

## Context

Occurs when opening a ZIP file where entries use BZip2 (method 12) compression rather than standard DEFLATE (method 8). The ZIP specification allows BZip2, but `java.util.zip.ZipInputStream` only supports STORED (0) and DEFLATE (8). The outer file still opens as a valid `.zip` (magic bytes match), so nothing signals a problem until `getNextEntry()` is called.

Encountered reading the IEM10 SC2EGSet outer ZIP (`2016_IEM_10_Taipei.zip`) — its single outer entry (`_data.zip`) is BZip2-compressed; the inner `_data.zip` itself uses standard DEFLATE and works fine with standard `ZipInputStream`.

## What was tried (didn't work)

- Reading with `java.util.zip.ZipInputStream` — throws at `getNextEntry()` on the BZip2 entry
- Checking file extension or magic bytes — the outer file is a valid `.zip`; nothing surface-level signals BZip2

## Root cause

Java's built-in `ZipInputStream` implements only STORED and DEFLATE. BZip2 (compression method 12) is part of the ZIP specification but is rare and unsupported by the stdlib.

## Fix

Use `org.apache.commons.compress.archivers.zip.ZipArchiveInputStream` for the outer ZIP. Inner ZIPs using DEFLATE can continue using standard `java.util.zip.ZipInputStream`.

```java
// pom.xml
<dependency>
    <groupId>org.apache.commons</groupId>
    <artifactId>commons-compress</artifactId>
    <version>1.27.1</version>
</dependency>

// reading the outer BZip2-compressed ZIP
try (ZipArchiveInputStream outer = new ZipArchiveInputStream(
        Files.newInputStream(outerZip))) {
    ZipArchiveEntry outerEntry;
    while ((outerEntry = outer.getNextEntry()) != null) {
        if (outerEntry.getName().endsWith("_data.zip")) {
            byte[] innerBytes = outer.readAllBytes();
            try (ZipInputStream inner = new ZipInputStream(
                    new ByteArrayInputStream(innerBytes))) {
                ZipEntry innerEntry;
                while ((innerEntry = inner.getNextEntry()) != null) {
                    // process inner entries normally
                }
            }
        }
    }
}
```

## Why this is non-obvious

`ZipInputStream` is the canonical Java ZIP reader and works for virtually all ZIP files encountered in practice. BZip2 in ZIP is permitted by the spec but rare, the error message doesn't name BZip2, and the outer file opens normally — nothing signals a problem until entry reading fails.

### Why this fix

`ZipArchiveInputStream` from commons-compress is a drop-in replacement with BZip2 support. Pre-extracting with a native `unzip` subprocess works but breaks "no temp files" requirements and ties code to the OS. Commons Compress adds one small dependency and is well-maintained.
