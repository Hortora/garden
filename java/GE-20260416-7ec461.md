## Maven `-am -Dtest=ClassName` propagates test filter to all upstream modules ‚Äî 'No tests matching pattern' on unrelated modules

**ID:** GE-20260416-7ec461
**Stack:** Maven 3.9+
**Symptom:** Running `mvn test -pl mymodule -am -Dtest=MySpecificTest` fails with `No tests matching pattern 'MySpecificTest' were executed!` in an upstream module (e.g., `api`) that has no such test class. The build fails before reaching the target module.
**Context:** Multi-module Maven project. You want to run one test class in a submodule while ensuring its upstream dependencies are built first (`-am`).

### What was tried (didn't work)
- `mvn test -pl casehub-persistence-hibernate -am -Dtest=JpaCaseMetaModelRepositoryTest` ‚Äî surefire propagates `-Dtest` to `api`, `engine`, etc., which don't have that test class ‚Üí fails immediately

### Root cause
`-Dtest=` is a surefire system property that applies globally to all reactor modules in the build, not just the target module specified by `-pl`. When `-am` causes upstream modules to run their `test` phase, surefire looks for the named test class in each module and fails if not found (unless `surefire.failIfNoSpecifiedTests=false` is set).

### Fix
Install upstream modules without tests, then run the target module separately:

```bash
# One-time install of upstream deps (10-15s)
mvn install -DskipTests -q

# Then run the specific test in isolation
mvn test -pl casehub-persistence-hibernate -Dtest=JpaCaseMetaModelRepositoryTest

# Or to also rebuild upstream (not install):
mvn install -DskipTests -am -pl casehub-persistence-hibernate -q && \
  mvn test -pl casehub-persistence-hibernate -Dtest=JpaCaseMetaModelRepositoryTest
```

Alternatively, suppress the filter failure in upstream modules:
```bash
mvn test -pl casehub-persistence-hibernate -am \
  -Dtest=JpaCaseMetaModelRepositoryTest \
  -Dsurefire.failIfNoSpecifiedTests=false
```
(This runs all tests in upstream modules, which may be slow or require Docker.)

### Why this is non-obvious
`-am` (also make) is understood as "build the prerequisites" ‚Äî it's natural to assume `-Dtest=` applies only to the explicit `-pl` target. The error `No tests matching pattern` on `api` is confusing because you didn't ask to test `api` at all.
