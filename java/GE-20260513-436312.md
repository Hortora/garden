## Symptom

Running `mvn test -q 2>&1 | tail -10` returns exit code 0 even when tests fail. The truncated output contains `[ERROR]` lines, but `$?` is 0.

## Context

Any Maven command piped into `tail` (or any other command). Occurs whenever you want to suppress noise but still see the last N lines of output.

## Root Cause

In bash, the exit code of a pipeline is the exit code of the **last** command — `tail`, not `mvn`. `tail` exits 0 as long as it can read its input. Maven's non-zero exit is silently discarded.

## Fix

Use a semicolon instead of a pipe for exit-code transparency:

```bash
mvn test -q; echo "EXIT:$?"
```

Or check for the failure marker in output:

```bash
mvn test -q 2>&1 | tee /tmp/mvn-out.txt; grep -c "\[ERROR\]" /tmp/mvn-out.txt
```

Or use `pipefail` if you need the pipe form:

```bash
set -o pipefail
mvn test -q 2>&1 | tail -10
```

## Why This Is Non-Obvious

The pipe form reads naturally ("run Maven quietly and show me the last 10 lines"). The exit-code masking is a shell behaviour that doesn't announce itself — CI scripts written this way will silently pass on test failures.
