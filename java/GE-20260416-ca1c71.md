## Symptom

Test count after adding `*IT.java` tests is lower than expected. `mvn test` passes but the new tests never appear in the output — no failure, no skip message, no warning.

## Context

Any Maven Java project where a test class is named with the `*IT` suffix (e.g. `EmulatedTerrainIT.java`). The developer expects `mvn test` to run it alongside other tests.

## Root Cause

Maven's naming convention routes files matching `*IT.java` and `IT*.java` to the **Failsafe plugin**, not Surefire. `mvn test` only invokes Surefire. Failsafe runs during the `integration-test` phase, triggered by `mvn verify` or later lifecycle goals.

The silence is complete: Surefire doesn't report the class as skipped — it simply never considers it. The only observable symptom is a count discrepancy between expected and actual tests.

## Fix

Three options depending on intent:

**Option A — Run under surefire (rename the class):**
```bash
# Rename EmulatedTerrainIT.java → EmulatedTerrainTest.java
# Surefire picks up *Test.java by default
```

**Option B — Run explicitly by name:**
```bash
mvn test -Dtest=EmulatedTerrainIT
# Forces surefire to run the named class regardless of suffix
```

**Option C — Run under failsafe (intended for integration tests):**
```bash
mvn verify
# Runs both surefire (unit) and failsafe (integration) phases
```

## Why This Is Non-Obvious

The failure mode is complete silence. There is no error, no skip annotation, no `[INFO] Tests skipped: 1` line — the class is simply absent from surefire's classpath scan. The only way to detect it is by noticing the total count is lower than expected. Developers who add `*IT.java` expecting `mvn test` to run it will see green builds with invisible test gaps.
