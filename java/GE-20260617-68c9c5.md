## Symptom

`NullPointerException` thrown from `Set.contains()` even though no null elements were added to the set. Stack trace points to an innocuous-looking contains check.

## Context

```java
static final Set<String> TERMINAL_STATES = Set.of("completed", "failed", "cancelled");

AtomicReference<String> stateRef = new AtomicReference<>();  // default null
// ... lambda exits early without setting stateRef ...

if (TERMINAL_STATES.contains(stateRef.get())) {  // NPE here
```

The `AtomicReference` has a null default. The lambda exits a code path without setting the reference. `stateRef.get()` returns null. `TERMINAL_STATES.contains(null)` throws `NullPointerException`.

## Root cause

`Set.of()` (and `List.of()`, `Map.of()`) immutable collections explicitly prohibit null elements AND null arguments to `contains()`, `get()`, etc. The Javadoc states this, but developers often assume `contains()` returns false for null (as `HashSet`, `ArrayList`, and `HashMap` do).

```java
new HashSet<>(Set.of("a")).contains(null)  // → false (no NPE)
Set.of("a").contains(null)                 // → NullPointerException
```

## Fix

Initialize `AtomicReference` with a safe non-null default:

```java
// Before:
AtomicReference<String> stateRef = new AtomicReference<>();

// After:
AtomicReference<String> stateRef = new AtomicReference<>("submitted");
```

Or null-guard the check:

```java
String state = stateRef.get();
if (state != null && TERMINAL_STATES.contains(state)) { ... }
```

## Why this is non-obvious

`HashSet.contains(null)` silently returns false, which is the intuitive behavior. Java's immutable `Set.of()` collections behave differently — not for null membership (they can't contain null) but for the `contains()` query itself. This asymmetry exists because the immutable implementation uses hash-based lookup that doesn't special-case null the way `HashMap`/`HashSet` does.

The NPE only surfaces on a code path that was silently omitting the `set()` call — making the actual root cause (missing initialization) invisible until the null finally reaches the `contains()` call.
