## Symptom

`mvn install` fails immediately at reactor resolution — before any code compiles:

```
[ERROR] The projects in the reactor contain a cyclic reference:
Edge between 'Vertex{label='io.example:module-b:1.0-SNAPSHOT'}' and
'Vertex{label='io.example:module-a:1.0-SNAPSHOT'}' introduces a cycle:
io.example:module-a → io.example:module-b → io.example:module-a
```

## Context

You want to use a module's implementations in another module's tests — e.g. `module-b` implements SPIs defined in `module-a`, and you want `module-a` tests to activate `module-b` implementations. The natural move is to add `module-b` as `<scope>test</scope>` in `module-a/pom.xml`.

This creates the cycle: `module-a → module-b → module-a`. Maven rejects **all cycles** regardless of scope at reactor resolution time — before any module is built.

## What was tried

- `<scope>test</scope>` — rejected; Maven evaluates cycles before scopes
- `<optional>true</optional>` — same; optional doesn't affect cycle detection

## Root cause

Maven's reactor resolves the full dependency graph before building any module. Scope is a compile/runtime concern, not a graph concern. A test-scope dep that creates a cycle is still a cycle.

## Fix

Copy the needed classes into the dependent module's `src/test/java/` directly. If the classes are simple (in-memory implementations, test doubles), this is lower friction than extracting a new module.

For a structural fix: extract the shared interfaces into a separate leaf module that neither side depends on circularly. Both modules can then depend on the leaf; the test consumer adds the implementation module as test-scope with no cycle.

```
Before (cycle):        After (no cycle):
module-a → module-b    api ← module-a
module-b → module-a    api ← module-b
                       module-a --[test]--> module-b  ✓
```

## Why this is non-obvious

`test` scope intuitively means "only needed for tests — not a real dependency". Developers expect Maven to be smarter about this. The error message names the cycle edge but gives no hint that scope is irrelevant to cycle detection or what the fix is.
