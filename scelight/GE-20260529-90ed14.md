### What this is

`SelectionDeltaEvent.getDelta()` always returns a non-null `Delta` object. The replay parser
infrastructure populates the delta field for every `SelectionDeltaEvent` — the binary protocol
always carries a delta payload for these events.

### Where you'd write the null guard

```java
var delta = event.getDelta();
if (delta == null) {
    selection.clear();
    return;
}
```

This guard is unreachable. The Scelight parser calls `get(F_DELTA)` on a struct that is
always initialised before delivering the event.

### Why this is non-obvious

The return type permits null (Scelight's generic `get(field)` accessor pattern casts
from Object). The always-populated guarantee is only visible by reading the Scelight
source — specifically `getDelta_()` which shows `get(F_DELTA)` on a parser-populated struct.
The API contract doesn't express it.
