### What goes wrong

Constructing a synthetic `SelectionDeltaEvent` for unit tests passes `0` (or any value < 16561) as `baseBuild`. This triggers an old-format compatibility block in the constructor that path-navigates `{"delta", "removeMask", Field.NAME_PARENT}` and overwrites the synthetic removeMask with the value at the "parent" key.

- If removeMask is a `Pair`: the navigation tries to cast Pair to Map → `ClassCastException`.
- If removeMask is a `Map`: navigation returns null for the "parent" key → the removeMask is silently replaced with `new Pair<>("Mask", null)`. All non-trivial variants (ZeroIndices, OneIndices, etc.) then behave identically to a no-op Mask with null payload — the selection is never modified.

### When it occurs

Any time `new SelectionDeltaEvent(struct, id, name, loop, userId, baseBuild)` is called with `baseBuild < 16561` and a synthetic `removeMask` in the delta struct.

### What was tried

1. Passed `Pair` as the removeMask value → `ClassCastException: Pair cannot be cast to Map`
2. Passed a one-entry `LinkedHashMap` as the removeMask value → no exception, but all non-trivial variants silently behaved as carry-forward

### Root cause

`SelectionDeltaEvent` constructor (source at `feature/standalone-modules` fork):

```java
if ( baseBuild < 16561 ) {
    getDelta_().put( Delta.F_REMOVE_MASK,
        new Pair<>( "Mask", get( P_OLD_REMOVE_MASK ) ) );
}
```

`P_OLD_REMOVE_MASK = { "delta", Delta.F_REMOVE_MASK, Field.NAME_PARENT }` is a path navigation through the struct. With a synthetic struct this path resolves to null, replacing whatever removeMask you set with `Pair<>("Mask", null)`.

### Fix

Pass `baseBuild >= 16561` when constructing synthetic events:

```java
// Pass baseBuild=16561 to skip the old-format compatibility block
return new SelectionDeltaEvent(struct, 0, "SelectionDeltaEvent", 0, userId, 16561);
```

### Why this is non-obvious

`baseBuild` looks like metadata for version-conditional parsing, not an active trigger for struct mutation. The compatibility block is not in `getDelta()` or `getRemoveMask()` — it runs unconditionally in the constructor, silently changing the struct before any method call. The symptom (all variant tests behave as carry-forward) points directly away from the cause.
